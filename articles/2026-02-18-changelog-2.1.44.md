# Claude Code v2.1.44: 安定性向上のバグ修正リリース

## 概要

Claude Code v2.1.44 は、安定性向上に焦点を当てたバグ修正リリースです。深くネストされたディレクトリパスで発生していた `ENAMETOOLONG` エラーの修正と、認証トークンのリフレッシュ処理に関するエラーの修正が含まれています。

## 変更点の詳細

### バグ修正

#### 深くネストされたディレクトリパスの `ENAMETOOLONG` エラー修正

深くネストされたディレクトリ構造を持つプロジェクトで作業する際に、ファイルパスの長さが OS の制限を超えてしまう場合がありました。この問題は `ENAMETOOLONG` エラー（ファイル名が長すぎることを示す POSIX エラー）として現れ、操作が失敗する原因となっていました。

v2.1.44 では、この問題に対処するための修正が加えられ、深くネストされたディレクトリパスを持つプロジェクトでも安定して動作するようになりました。

**影響を受ける可能性があったケース:**
- モノレポ（monorepo）など、ネストが深いプロジェクト構造
- 長いディレクトリ名を多数含むパス
- Linux / macOS でパス長が上限（通常 4096 バイト）に近いケース

#### 認証リフレッシュエラーの修正

認証トークンの更新（リフレッシュ）処理において発生していたエラーが修正されました。長時間の作業セッションや、トークンの有効期限切れが発生するタイミングで、認証が適切に更新されないケースがあったと考えられます。この修正により、認証の継続性が改善され、作業の中断リスクが低減されます。

## 技術的なポイント

- **`ENAMETOOLONG` エラー**: POSIX 標準で定義されたエラーコード（`ENAMETOOLONG`）で、ファイルパスやファイル名が OS の許容する最大長を超えた場合に発生します。Linux では `NAME_MAX`（通常 255 バイト）や `PATH_MAX`（通常 4096 バイト）が制限値として設定されています
- **認証リフレッシュ**: Claude Code は Anthropic の API や Claude.ai との連携に認証トークンを使用しており、トークンの有効期限が切れた際に自動的に更新する仕組みを持っています。今回の修正はこのリフレッシュ処理の信頼性を向上させるものです
- **小規模修正リリース**: v2.1.44 は機能追加を含まない純粋なバグ修正リリースです。より大きな機能拡張は後続バージョンで提供されています

## まとめ

v2.1.44 は 2 件のバグを修正した安定性重視のリリースです。深くネストされたディレクトリを扱う大規模プロジェクトや、長時間のセッションで認証の問題を経験したユーザーにとって、アップデートの効果が実感できる修正内容となっています。これらの修正により、Claude Code の日常的な利用における信頼性がさらに向上しています。

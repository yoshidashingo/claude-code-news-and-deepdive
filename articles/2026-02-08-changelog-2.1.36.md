# Claude Code Changelog v2.1.36 - Fast Mode の導入とその設計思想

## 概要

Claude Code v2.1.36 の Changelog には、Opus 4.6 向けの **Fast Mode** が新たに追加されたことが記載されています。このバージョンは単一機能にフォーカスしたリリースであり、レスポンス速度を重視するユーザー向けに新しい動作モードを提供します。

## 変更点の詳細

### Fast Mode for Opus 4.6

Changelog に記載された v2.1.36 の変更点は「Fast mode is now available for Opus 4.6」の 1 点です。シンプルな記述ですが、この背後には Claude Code の利用体験を大きく変える可能性を持つ機能が含まれています。

Fast Mode は、Opus 4.6 モデルの API 設定をレイテンシ優先に切り替えることで、**モデルの品質を一切変えずに**レスポンス速度を向上させる仕組みです。つまり、別のモデルに切り替わるわけではなく、同じ Opus 4.6 がより高速に応答するようになります。

#### 利用方法

Fast Mode の切り替えは非常にシンプルです。

- CLI または VS Code Extension で `/fast` コマンドを実行
- 設定ファイルに `"fastMode": true` を追記

有効化されると、プロンプト横に `↯` アイコンが表示され、Fast Mode が動作中であることを視覚的に確認できます。状態はセッションをまたいで維持されるため、毎回設定し直す必要はありません。

#### コストと料金

Fast Mode は速度と引き換えに、標準モードよりも高いトークン単価が設定されています。

| モード | Input (MTok) | Output (MTok) |
|--------|-------------|---------------|
| Fast Mode（200K トークン以下） | $30 | $150 |
| Fast Mode（200K トークン超） | $60 | $225 |

なお、**2026年2月16日 23:59（PT）まで全プランで 50% 割引**が適用されています。Fast Mode の使用料は Extra Usage として課金され、サブスクリプションの含有使用量には含まれません。

#### 利用要件と制限

- Pro / Max / Team / Enterprise プランまたは Claude Console ユーザーが対象
- Extra Usage の有効化が必須
- Amazon Bedrock、Google Vertex AI、Microsoft Azure Foundry では利用不可
- Teams / Enterprise では管理者による事前の有効化が必要

## 技術的なポイント

- **「モードの切り替え」という設計アプローチ**: Fast Mode は新しいモデルではなく、既存モデルの API 設定を変更するという設計です。これにより、ユーザーは品質と速度のトレードオフを明示的にコントロールできます
- **Effort Level との違い**: Fast Mode はレイテンシ（応答遅延）を最適化し、Effort Level は思考時間を調整します。両者は独立したパラメータであるため、組み合わせることでタスクの性質に応じた最適な設定が可能です
- **レート制限の分離設計**: Fast Mode には標準モードとは別のレート制限が設けられています。制限に達すると自動的に標準モードにフォールバックし、クールダウン後に自動復帰する仕組みは、ユーザー体験の連続性を保つ設計上の工夫です
- **セッション途中の切り替えコストに注意**: 会話途中で Fast Mode を有効にすると、既存のコンテキスト全体にキャッシュなしの Fast Mode 料金が適用されます。コスト効率を考慮すると、セッション開始時に有効にするのが推奨されます
- **1M トークン拡張コンテキストウィンドウとの互換性**: Fast Mode は拡張コンテキストウィンドウでも利用可能で、大規模なコードベースでの作業でも速度向上の恩恵を受けられます

## まとめ

v2.1.36 の Changelog は 1 行の変更点のみですが、その内容は Claude Code の利用体験に直接影響する重要な機能追加です。Fast Mode は、ライブデバッグや素早いイテレーションなど、レスポンス速度が作業効率に直結する場面で真価を発揮します。

一方で、コストが標準モードより高いため、長時間の自律的タスクやバッチ処理には標準モードが適しています。現在 50% 割引キャンペーンが 2月16日まで実施されているため、自分のワークフローに Fast Mode が合うかどうかを低コストで試す良い機会です。

今後のバージョンでは、この Research Preview の段階にある Fast Mode がどのように進化していくか注目されます。

# Claude Code Changelog v2.1.37 - Fast Mode 有効化タイミングの修正と v2.1.36 からの流れ

## 概要

Claude Code v2.1.37 の Changelog には、`/extra-usage` を有効にした直後に `/fast` コマンドがすぐに利用できない問題の修正が記載されています。v2.1.36 で Fast Mode が導入された直後のバグ修正リリースであり、新機能の安定化に向けた素早い対応となっています。

## 変更点の詳細

### /fast が /extra-usage 有効化直後に利用できない問題の修正

Changelog に記載された v2.1.37 の変更点は「Fixed an issue where /fast was not immediately available after enabling /extra-usage」の 1 点です。

この問題は、以下のようなワークフローで発生していました:

1. ユーザーが月間の利用上限に到達する
2. `/extra-usage` コマンドで追加利用枠を有効化する
3. `/fast` コマンドで Fast Mode に切り替えようとする
4. **Fast Mode がすぐに利用可能にならない**

v2.1.37 ではこの問題が修正され、`/extra-usage` を有効にした直後から `/fast` コマンドが正しく動作するようになりました。

### v2.1.36 からの連続リリースとしての位置づけ

v2.1.36 と v2.1.37 の Changelog を並べてみると、明確な流れが見えてきます。

| バージョン | 変更内容 | 種別 |
|-----------|---------|------|
| v2.1.36 | Fast mode is now available for Opus 4.6 | 新機能 |
| v2.1.37 | Fixed an issue where /fast was not immediately available after enabling /extra-usage | バグ修正 |

v2.1.36 で Fast Mode という新機能が追加され、v2.1.37 でその機能に関連するエッジケースが修正されるという、典型的な「機能追加 → 安定化」のパターンです。2 つのバージョンが同日（2026-02-08）にリリースされていることからも、問題の発見から修正までが非常に迅速であったことがうかがえます。

### GitHub Release との情報の比較

Changelog の記述は 1 行と簡潔ですが、GitHub Release のリリースノートでも同じ 1 点のバグ修正のみが記載されています。今回のリリースでは新機能や Breaking Changes は含まれておらず、純粋なバグ修正リリースです。

## 技術的なポイント

- **機能フラグの状態同期の重要性**: `/extra-usage` と `/fast` は独立した機能ですが、`/fast` の利用可否が `/extra-usage` の状態に依存するケースがあります。こうした機能間の暗黙的な依存関係における状態同期は、見落としやすいバグの温床です
- **Fast Mode の利用前提条件**: Fast Mode は Extra Usage として課金されるため、`/extra-usage` の有効化が事実上の前提条件になります。この依存関係が今回のバグの背景にあると考えられます
- **Changelog の粒度と運用**: v2.1.36・v2.1.37 ともに 1 行のみの Changelog ですが、各バージョンが単一の目的（機能追加またはバグ修正）に集中していることで、変更の追跡が容易になっています。小さなリリースを頻繁に行うという Claude Code チームの開発方針が反映されています
- **迅速なバグ修正サイクル**: 新機能のリリースと同日にバグ修正版が出ていることは、ユーザーからのフィードバックを素早く取り込む開発体制が整っていることを示しています

## まとめ

v2.1.37 の Changelog は 1 行のバグ修正のみですが、v2.1.36 で追加された Fast Mode をスムーズに利用するために不可欠な修正です。`/extra-usage` で追加利用枠を有効化してから `/fast` を使うというワークフローは、Fast Mode を利用するユーザーにとって一般的な操作手順であり、その導線上の障害を取り除いた実用的な改善と言えます。

v2.1.36 での Fast Mode 導入から v2.1.37 でのバグ修正という流れは、Claude Code チームの「素早くリリースし、素早く修正する」という開発スタイルを体現しています。Fast Mode はまだ Research Preview の段階にあるため、今後も同様の改善が続くことが予想されます。利用中に問題を見つけた場合は、積極的にフィードバックを送ることで、機能の安定化に貢献できるでしょう。

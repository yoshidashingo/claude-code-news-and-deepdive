# Claude Code v2.1.37 - /fast コマンドの利用可能タイミングに関するバグ修正

## 概要

Claude Code v2.1.37 がリリースされました。本バージョンは、`/extra-usage` を有効にした直後に `/fast` コマンドがすぐに利用できない問題を修正するメンテナンスリリースです。直前の v2.1.36 で Fast モードが Opus 4.6 に対応したばかりであり、その機能をよりスムーズに利用できるようにするための重要なバグ修正となっています。

## バグ修正

### /fast コマンドが /extra-usage 有効化後に即座に利用できない問題の修正

**修正内容**: `/extra-usage` を有効にした後、`/fast` コマンドがすぐに利用可能にならない問題が修正されました。

**問題の詳細**:

Claude Code では、利用量の上限に達した場合に `/extra-usage`（追加利用枠）を有効化することで、引き続きサービスを利用できます。一方、`/fast` コマンドは、同じ Claude Opus 4.6 モデルをより高速な出力で利用できるモードを切り替えるためのコマンドです。

v2.1.37 以前のバージョンでは、`/extra-usage` を有効化した直後に `/fast` コマンドを実行しても、すぐには Fast モードに切り替えることができませんでした。これにより、追加利用枠を有効にしたユーザーが Fast モードを使いたい場合に、不必要な待ち時間やセッションの再起動が必要になるケースがありました。

本修正により、`/extra-usage` の有効化と `/fast` コマンドの利用可能状態が正しく同期されるようになり、追加利用枠を有効にした直後から Fast モードを利用できるようになりました。

**影響を受けるユーザー**: 利用量の上限に達した後、`/extra-usage` で追加利用枠を有効にし、さらに `/fast` モードを併用したいユーザー全般が対象です。

## 背景: /fast と /extra-usage について

### /fast コマンド

`/fast` は、Claude Code の Fast モードを切り替えるコマンドです。Fast モードでは、モデル自体は変わらず同じ Claude Opus 4.6 が使用されますが、より高速な出力が得られます。直前のリリース v2.1.36 で Opus 4.6 での Fast モード対応が追加されたばかりであり、本リリースはその機能の安定性を高めるものです。

### /extra-usage コマンド

`/extra-usage` は、通常の利用枠を超えた後に追加の利用枠を有効化するためのコマンドです。Claude Code のサブスクリプションプランにおいて、月間の利用上限に達した場合でも、このコマンドを使うことで引き続き作業を継続できます。

## 技術的なポイント

- **状態管理の同期問題**: `/extra-usage` の有効化と `/fast` コマンドの利用可能フラグが適切に同期されていなかったことが根本原因です。機能フラグの状態管理において、複数のコマンド間の依存関係を正しくハンドリングすることの重要性を示す事例です
- **v2.1.36 との関連性**: v2.1.36 で Fast モードが Opus 4.6 に対応した直後のバグ修正であり、新機能追加に伴うエッジケースへの迅速な対応となっています
- **ユーザー体験の改善**: 技術的には小さな修正ですが、`/extra-usage` と `/fast` を組み合わせて使用するワークフローにおいて、不必要な中断を排除する実用的な改善です
- **コミュニティの反応**: リリースに対して 37 件のリアクション（22 人）が寄せられており、この問題が多くのユーザーに影響していたことがうかがえます

## まとめ

Claude Code v2.1.37 は、v2.1.36 で導入された Opus 4.6 対応の Fast モードをより安定して利用できるようにするためのバグ修正リリースです。`/extra-usage` を有効にした直後でも `/fast` コマンドがすぐに利用可能になるよう改善されました。

Fast モードと追加利用枠を併用するワークフローを活用しているユーザーは、v2.1.37 へのアップデートをおすすめします。Claude Code のアップデートは以下のコマンドで実行できます:

```bash
claude update
```

今後も Fast モードの改善や安定性向上が続くことが期待されます。

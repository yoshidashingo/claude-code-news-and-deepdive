# Claude Code v2.1.44 リリース: パス処理と認証のバグ修正

## 概要

Claude Code v2.1.44 は、深くネストされたディレクトリパスによる `ENAMETOOLONG` エラーと、認証トークンのリフレッシュ処理に関するエラーを修正したメンテナンスリリースです。いずれも実際の利用環境で発生し得る問題への対処であり、安定性の向上に寄与します。

## 変更点の詳細

### バグ修正

#### `ENAMETOOLONG` エラーの修正

深くネストされたディレクトリパスを扱う際に発生していた `ENAMETOOLONG` エラーが修正されました。

`ENAMETOOLONG` は、ファイルパスやファイル名の長さがオペレーティングシステムの上限を超えたときに発生する POSIX 標準のエラーコードです。Linux では PATH_MAX（通常 4096 バイト）、macOS でも同様の制限があります。モノレポや深いディレクトリ構造を持つプロジェクト、または長い名前のディレクトリが多数ネストされている環境では、この制限に達しやすくなります。

今回の修正により、そうした環境でも Claude Code が正常に動作するようになりました。大規模なコードベースや複雑なディレクトリ構造を持つプロジェクトで作業している開発者にとって、恩恵が大きい修正です。

#### 認証リフレッシュエラーの修正

認証トークンのリフレッシュ処理で発生していたエラーが修正されました。

Claude Code は Anthropic の API にアクセスするために認証トークンを使用しており、一定時間が経過するとトークンのリフレッシュが必要になります。このリフレッシュ処理が失敗すると、予期しないエラーや認証切れによるセッション中断が発生する可能性がありました。今回の修正でリフレッシュ処理の信頼性が向上し、長時間の作業セッションでもより安定して利用できるようになります。

## 技術的なポイント

- **`ENAMETOOLONG` エラー**: POSIX 標準のエラーコード（errno 36）。ファイルパスの長さがカーネルの上限（Linux では通常 PATH_MAX = 4096 バイト）を超えた場合に発生する。モノレポや深いネスト構造を持つプロジェクトで顕在化しやすい
- **認証トークンのリフレッシュ**: OAuth や JWT などの認証方式では、セキュリティのためトークンに有効期限が設定されている。有効期限切れ前に新しいトークンを取得するリフレッシュ処理の安定性が今回向上した
- **v2.1.44 は純粋なバグ修正リリース**: 新機能の追加はなく、既存の動作の安定性・信頼性向上に特化したリリース

## まとめ

v2.1.44 は地味ながらも実用上重要な2つのバグを修正したリリースです。深いディレクトリ構造のプロジェクトで `ENAMETOOLONG` エラーに悩まされていた開発者や、長時間セッション中に認証エラーを経験していた開発者は、このアップデートで改善が期待できます。最新バージョンへのアップデートは `npm update -g @anthropic-ai/claude-code` で行えます。

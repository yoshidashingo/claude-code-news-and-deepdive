# Claude Code v2.1.69 リリース — 大規模バグ修正とメモリ最適化、Voice STT 20言語対応

## 概要

Claude Code v2.1.69 は、新機能の追加から大量のバグ修正・パフォーマンス改善まで非常に広範囲にわたるリリースです。Voice STT の対応言語が 10 言語から 20 言語へ倍増し、`/claude-api` スキルの追加や `/reload-plugins` コマンドなど開発者体験を向上させる機能が多数加わりました。また、長時間セッションでのメモリリーク修正やセキュリティ改善も含まれており、安定性・信頼性が大幅に向上しています。

## 新機能

### `/claude-api` スキル

Claude API や Anthropic SDK を使ったアプリケーション開発を支援する新スキル `/claude-api` が追加されました。`anthropic` や `@anthropic-ai/sdk` を使ったコードを書く際に自動でトリガーされ、API の使い方やベストプラクティスをサポートします。

### Voice STT の対応言語が 20 言語に拡大

音声入力（Speech-to-Text）の対応言語が 10 言語追加され、合計 20 言語になりました。新たに追加されたのは以下の言語です。

- ロシア語、ポーランド語、トルコ語、オランダ語、ウクライナ語
- ギリシャ語、チェコ語、デンマーク語、スウェーデン語、ノルウェー語

### Remote Control のセッション名カスタマイズ

`/remote-control` および `claude remote-control` コマンドにオプションの `name` 引数が追加されました。`claude.ai/code` 上のセッション一覧で識別しやすい名前を付けられます。

```bash
/remote-control My Project
claude remote-control --name "My Project"
```

### `/reload-plugins` コマンド

プラグインの変更を Claude Code の再起動なしに反映できる `/reload-plugins` コマンドが追加されました。プラグイン開発時の反復作業が効率化されます。

### その他の新機能

- **数字キーパッドのサポート**: インタビュー形式の質問でテンキーから選択肢を入力できるようになりました（従来は QWERTY 上部の数字列のみ）
- **エフォートレベルの表示**: ロゴとスピナーに現在のエフォート設定（例: "with low effort"）が表示されるようになりました
- **ターミナルタイトルにエージェント名を表示**: `claude --agent` 使用時、ターミナルのタイトルバーにエージェント名が表示されます
- **Ctrl+U で bash モード終了**: `!` プロンプトで Ctrl+U を押すことで bash モードを終了できます（従来の `Escape`/`Backspace` と同様）
- **`InstructionsLoaded` フックイベント**: CLAUDE.md や `.claude/rules/*.md` ファイルがコンテキストに読み込まれた際に発火する新フックイベントが追加されました
- **`sandbox.enableWeakerNetworkIsolation` 設定** (macOS のみ): カスタム MITM プロキシ使用時に `gh`、`gcloud`、`terraform` などの Go プログラムが TLS 証明書を検証できるようになります
- **`includeGitInstructions` 設定**: システムプロンプトから組み込みの commit・PR ワークフロー指示を除去できます（`CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS` 環境変数でも制御可能）
- **`${CLAUDE_SKILL_DIR}` 変数**: スキルが SKILL.md 内で自身のディレクトリを参照できる変数が追加されました
- **`pluginTrustMessage` 設定**: managed settings でプラグインの信頼確認ダイアログに組織固有のコンテキストを追加できます
- **`oauth.authServerMetadataUrl` 設定**: MCP サーバーがカスタム OAuth メタデータ探索 URL を指定できます
- **`git-subdir` プラグインソースタイプ**: git リポジトリ内のサブディレクトリをプラグインソースとして指定できます

## 改善点

### パフォーマンス・メモリ

- Yoga WASM のプリロードを遅延させることでベースラインメモリを約 **16MB** 削減
- スピナーのアニメーションループをシェルから分離し、CPU・レンダリング負荷を低減
- `--worktree` 起動時の git サブプロセスを削減し、起動速度を改善
- macOS での managed settings 解決時の設定ファイル再読み込みを削減
- MCP の `-p` 起動を並列化により高速化
- ファイル存在確認でファイル内容の読み込みを回避（6箇所）し、ファイル操作パフォーマンスを改善
- SDK・CCR セッションのメモリフットプリントを削減
- 大規模セッション再開時のメモリ使用量を削減
- マルチエージェントタスクのサブエージェント最終レポートを簡潔化してトークン使用量を削減
- macOS の React Compiler による UI レンダリングパフォーマンス向上

### UX・操作性

- MCP ツールが PDF、Office ドキュメント、音声ファイルを返す際、base64 をそのまま会話に挿入する代わりに、正しい拡張子でディスクに保存するように改善
- LSP ツールのレンダリングとメモリコンテキスト構築でファイル全体を読み込まないよう改善
- `/resume` ピッカーが最初のプロンプトではなく最新のプロンプトを表示するように変更
- Sonnet 4.5 の Pro/Max/Team Premium ユーザーが自動的に Sonnet 4.6 へ移行
- コンパクション後の再開で不要なプリアンブルの再表示を廃止
- 例示コマンド候補を Haiku 呼び出しではなく決定論的に生成するよう変更
- claude.ai MCP コネクタの接続失敗時に通知を表示（従来はサイレントに消えていた）

## バグ修正

### セキュリティ関連

- **ネストされたスキル探索の脆弱性修正**: `node_modules` などの gitignore 対象ディレクトリからスキルが読み込まれる問題を修正
- **MCP 信頼ダイアログの修正**: 初回実行時にすべての `.mcp.json` サーバーがサイレントに有効化されていた問題を修正。サーバーごとの承認ダイアログが正常に表示されるようになりました
- **シンボリックリンクバイパスの修正**: `acceptEdits` モードでシンボリックリンク経由で作業ディレクトリ外にファイルを書き込めた問題を修正
- **サンドボックスの非許可ドメイン処理**: `allowManagedDomainsOnly` 有効時、非許可ドメインがバイパスを促すプロンプトを表示せず自動的にブロックされるよう修正

### メモリリーク

長時間セッションや大規模プロジェクトで問題となっていた複数のメモリリークが修正されました。

- SDK/CCR セッションで会話メッセージが不要に保持されていた問題
- React Compiler `memoCache` に古いメッセージ配列が蓄積していた問題
- REPL レンダリングスコープが 1000 ターンで約 35MB 蓄積していた問題
- インプロセステーマメイトで親の会話履歴が `/clear` や自動コンパクト後も GC されなかった問題
- フックイベントがインタラクティブモードで無制限に蓄積していた問題

### 操作・UI 系

- `claude remote-control` が npm インストール時に "--sdk-url" オプションエラーでクラッシュする問題を修正
- `--model claude-opus-4-0` / `--model claude-opus-4-1` が廃止済みの Opus バージョンに解決されていた問題を修正
- macOS キーチェーンの破損問題（複数 OAuth MCP サーバー使用時）を修正
- `.credentials.json` が `subscriptionType` を失う問題（"Claude Pro"/"Claude Max" の代わりに "Claude API" と表示）を修正
- Linux でサンドボックス Bash コマンド後にゴースト dotfile（`.bashrc`、`HEAD` 等）が未追跡ファイルとして出現する問題を修正
- Ghostty over SSH で Shift+Enter が `[27;2;13~` を出力してしまう問題を修正
- Ctrl+S（スタッシュ）がメッセージ送信時にクリアされていた問題を修正
- ctrl+O（トランスクリプトトグル）が長いセッションで数秒フリーズする問題を修正
- プランモードのフィードバック入力で複数行テキスト入力が機能しなかった問題を修正（バックスラッシュ+Enter および Shift+Enter で改行挿入が可能に）
- `/stats` コマンドがタイムスタンプ不正なエントリーを含むトランスクリプトでクラッシュする問題を修正
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS` がコンパクション時に無視されていた問題を修正
- Windows での worktree ファイルコピーおよびグローバル `.claude` フォルダ検出の問題を修正
- `cd <outside-dir> && <cmd>` のパーミッションプロンプトが正しくコマンドを表示するよう修正
- Escape キーが下書きテキストのある入力ボックスで実行中のターンを中断しなかった問題を修正
- `/config` の検索ボックスにフォーカスがある時 Tab/矢印キーで Settings タブが切り替わらなかった問題を修正

### VSCode 拡張機能

- RTL テキスト（アラビア語、ヘブライ語、ペルシャ語）がチャットパネルで逆向きにレンダリングされていたリグレッションを修正（v2.1.63 から）
- パーミッションモードピッカーが managed settings の `permissions.disableBypassPermissionsMode` を尊重するよう修正
- コンパクション表示が折りたたみ可能な "Compacted chat" カードとして改善

## 技術的なポイント

- **メモリ管理の大幅強化**: 複数のメモリリークと不要なメモリ保持が同時に修正され、長時間セッションでの安定性が大幅に向上しました。特に 35MB 相当の REPL スコープリークの修正は注目に値します
- **セキュリティ強化**: gitignored ディレクトリからのスキル読み込みや MCP サーバーのサイレント承認など、複数のセキュリティ問題が修正されました。企業環境での利用時は特に重要です
- **`InstructionsLoaded` フックイベント**: CLAUDE.md の読み込みを検知して動的に処理を追加できるため、複雑なプラグイン・スキル開発に活用できます
- **`includeGitInstructions` 設定**: 独自の Git ワークフロー指示を持つチームは、組み込みの指示との競合を避けるためにこの設定を活用できます
- **`sandbox.enableWeakerNetworkIsolation`**: カスタム MITM プロキシを使う企業環境では、`gh` や `gcloud` などのインフラツールが正常動作するようになります
- **Sonnet 4.5 から Sonnet 4.6 への自動移行**: Pro/Max/Team Premium ユーザーは自動的に最新モデルへ移行されます。既存のワークフローやコストへの影響を確認することをお勧めします

## まとめ

v2.1.69 は新機能よりもバグ修正・安定性向上に重点を置いたリリースです。特にメモリリーク修正の多さは印象的で、長時間・大規模なセッションを行う開発者にとって大きな恩恵があります。セキュリティ面でも MCP サーバーの信頼ダイアログ修正やスキル探索の脆弱性修正など重要な改善が含まれており、早めのアップデートが推奨されます。Voice STT の 20 言語対応により、日本語以外のチームメンバーも音声入力機能を活用しやすくなりました。今後も安定性とパフォーマンスの向上が続くことが期待されます。

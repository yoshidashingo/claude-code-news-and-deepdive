# Claude Code v2.1.63 リリース — HTTP hooks、新スラッシュコマンド、大規模なメモリリーク修正

## 概要

Claude Code v2.1.63 では、HTTP hooks の追加や `/simplify`・`/batch` といった新しいスラッシュコマンドの導入など、開発ワークフローを強化する機能が複数追加されました。また、長時間セッションで問題となっていた多数のメモリリークが集中的に修正されており、安定性・パフォーマンスの大幅な改善が期待できるリリースです。

## 新機能

### `/simplify` と `/batch` スラッシュコマンドの追加

新たに 2 つのバンドル済みスラッシュコマンドが追加されました。

- **`/simplify`**: 変更されたコードの再利用性・品質・効率性をレビューし、問題を修正するコマンドです。
- **`/batch`**: 複数の処理をまとめて実行するためのコマンドです。

これらはビルトインのスキルとして提供され、インストール不要ですぐに利用できます。

### HTTP hooks のサポート

これまでのシェルコマンドを実行する hooks に加え、**HTTP hooks** が新たにサポートされました。HTTP hooks を使用すると、JSON を指定の URL に POST し、JSON レスポンスを受け取ることができます。

外部 API や webhook サービスとの連携がよりシンプルになります。例えば、ツールの実行前後に外部サービスへ通知したり、承認フローを組み込んだりといったユースケースに活用できます。

### `/model` コマンドの改善

`/model` コマンドのスラッシュコマンドメニューに、現在アクティブなモデルが表示されるようになりました。複数のモデルを使い分ける際に、現在の設定を素早く確認できます。

### MCP OAuth 認証での URL 手動入力フォールバック

MCP（Model Context Protocol）の OAuth 認証において、localhost へのリダイレクトが自動的に機能しない場合に、コールバック URL を手動でペーストして認証を完了できるフォールバック手段が追加されました。ネットワーク環境によってリダイレクトがうまく機能しないケースへの対応です。

### `/copy` コマンドの「常にレスポンス全体をコピー」オプション

`/copy` コマンドのピッカーに「Always copy full response（常にレスポンス全体をコピー）」オプションが追加されました。このオプションを選択すると、以降の `/copy` 実行時にコードブロックのピッカーをスキップし、レスポンス全体を直接コピーします。

### VSCode: セッションの名前変更・削除アクション

VSCode 拡張のセッションリストに、セッションの名前変更と削除のアクションが追加されました。複数のセッションを管理しやすくなります。

## 改善点

### git worktree 間でのプロジェクト設定・自動メモリの共有

同一リポジトリの git worktree 間で、プロジェクト設定と auto memory が共有されるようになりました。worktree を使ったブランチ並行作業の際に、設定の二重管理が不要になります。

### `ENABLE_CLAUDEAI_MCP_SERVERS=false` 環境変数の追加

claude.ai の MCP サーバーを利用可能にする機能をオプトアウトするための環境変数 `ENABLE_CLAUDEAI_MCP_SERVERS=false` が追加されました。セキュリティポリシーや組織のガイドラインに応じて制御できます。

### サブエージェントを使う長時間セッションのメモリ使用量改善

コンテキストコンパクション（圧縮）の際に、サブエージェントの重いプログレスメッセージのペイロードを削除することで、長時間セッションにおけるメモリ使用量が改善されました。

## バグ修正

### UI 修正

- ローカルスラッシュコマンド（`/cost` など）の出力が、システムメッセージではなくユーザー送信メッセージとして表示されていた問題を修正しました。

### メモリリークの集中修正

本リリースでは、多数のメモリリークが修正されています。

- **ブリッジポーリングループ**でのリスナーリーク修正
- **MCP OAuth フローのクリーンアップ**でのリスナーリーク修正
- **hooks 設定メニュー操作時**のメモリリーク修正
- **自動承認時のインタラクティブ権限ハンドラー**でのリスナーリーク修正
- **WebSocket トランスポート再接続時**のリスナーリーク修正
- **git ルート検出キャッシュ**のメモリリーク修正（長時間セッションで無制限に増大していた問題）
- **JSON パースキャッシュ**のメモリリーク修正（長時間セッションで無制限に増大していた問題）
- **長時間稼働するチームメイト**が会話コンパクション後も全メッセージを AppState に保持し続けていたメモリリークを修正
- **MCP サーバーのフェッチキャッシュ**が切断時にクリアされず、頻繁に再接続するサーバーでメモリ使用量が増大していた問題を修正
- **bash コマンドプレフィックスキャッシュ**のメモリリーク修正
- **MCP ツール／リソースキャッシュ**のサーバー再接続時のリーク修正
- **IDE ホスト IP 検出キャッシュ**がポートをまたいで結果を誤って共有していた問題を修正

### その他のバグ修正

- **ファイルカウントキャッシュ**が glob の ignore パターンを無視していた問題を修正
- **VSCode**: リモートセッションが会話履歴に表示されない問題を修正
- **REPL ブリッジのレースコンディション**修正: 初回接続フラッシュ中に、履歴メッセージと新規メッセージが混在してメッセージの順序がおかしくなる問題を修正
- **`/clear` コマンド**がキャッシュ済みスキルをリセットせず、新しい会話に古いスキルの内容が残り続けていた問題を修正

## 技術的なポイント

- **HTTP hooks** の追加により、hooks の統合先がシェルスクリプトに限定されなくなりました。外部サービスや内部 API との連携がよりクリーンに実装できます。
- メモリリークの修正が 10 件以上まとめて行われています。長時間セッションや MCP サーバーを多用する環境では、アップデートによる安定性の向上が特に期待できます。
- git worktree サポートの改善により、複数ブランチを並行して扱うワークフロー（feature ブランチ開発、レビュー対応など）での使い勝手が向上します。
- `ENABLE_CLAUDEAI_MCP_SERVERS=false` は、企業環境やセキュリティ要件の厳しい環境での利用時に有用な設定です。
- `/clear` でスキルキャッシュがリセットされるようになったため、スキルの内容を変更した後に古い動作が残り続ける問題が解消されます。

## まとめ

v2.1.63 は、新機能の追加と安定性改善を兼ね備えたバランスの良いリリースです。特に大量のメモリリーク修正は、長時間使用するユーザーや MCP サーバーとの連携を多用するユーザーにとって大きな恩恵があります。HTTP hooks の追加は、既存の CI/CD パイプラインや外部サービスとの連携シナリオを大幅に広げる可能性があります。また、`/simplify` や `/batch` といった新しいスラッシュコマンドにより、反復的な作業の自動化がさらに進められることが期待されます。

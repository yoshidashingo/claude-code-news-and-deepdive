# Claude Code v2.1.73 リリース — modelOverrides 設定追加と多数のバグ修正

## 概要

Claude Code v2.1.73 では、カスタムプロバイダーのモデル ID をマッピングできる `modelOverrides` 設定の追加や、SSL 証明書エラー時の案内強化といった新機能が導入されました。また、複雑な bash コマンドによるフリーズや、スキルファイル大量変更時のデッドロックなど、深刻なバグが複数修正されています。さらに Bedrock・Vertex・Microsoft Foundry でのデフォルト Opus モデルが Opus 4.6 に更新されました。

## 新機能

### `modelOverrides` 設定によるカスタムモデル ID マッピング

モデルピッカーのエントリをカスタムプロバイダーのモデル ID にマッピングできる `modelOverrides` 設定が追加されました。特に AWS Bedrock の推論プロファイル ARN（Inference Profile ARN）を指定する際に有用です。

```json
{
  "modelOverrides": {
    "claude-opus-4-6": "arn:aws:bedrock:us-east-1::foundation-model/anthropic.claude-opus-4-6"
  }
}
```

Bedrock のクロスリージョン推論プロファイルなど、標準モデル ID とは異なる識別子を使う環境で柔軟な設定が可能になります。

### SSL 証明書エラー時の案内強化

企業プロキシ環境などで OAuth ログインや接続チェックが SSL 証明書エラーで失敗した場合に、具体的な対処方法が案内されるようになりました。`NODE_EXTRA_CA_CERTS` 環境変数でカスタム CA 証明書を追加する方法などが表示され、問題の解決が容易になります。

## 改善点

### Up キーによる割り込み後の操作改善

Claude の応答を割り込んだ後に Up キーを押すと、割り込んだプロンプトの復元と会話の巻き戻しが1ステップで行われるようになりました。これまで複数操作が必要だった箇所がスムーズになります。

### その他の改善

- **IDE 検出の高速化**: 起動時の IDE 検出処理が高速化されました
- **macOS クリップボード画像貼り付けのパフォーマンス向上**: macOS での画像ペースト操作が改善されました
- **`/effort` コマンドのリアルタイム対応**: Claude が応答中でも `/effort` を実行できるようになり、`/model` コマンドと同様の挙動になりました
- **Voice mode の接続リトライ強化**: プッシュ・トゥ・トークを素早く再押下した際の一時的な接続障害を自動リトライするようになりました
- **Remote Control のスポーンモード選択プロンプト改善**: コンテキスト情報が充実し、選択がより分かりやすくなりました
- **デフォルト Opus モデルの更新**: Bedrock・Vertex・Microsoft Foundry でのデフォルト Opus モデルが Opus 4.1 から **Opus 4.6** に変更されました

## バグ修正

### 深刻なフリーズ・デッドロックの修正

- **複雑な bash コマンドでのフリーズ修正**: パーミッションプロンプトが複雑な bash コマンドに対してトリガーされた際に発生していた 100% CPU ループおよびフリーズが解消されました
- **スキルファイル大量変更時のデッドロック修正**: `.claude/skills/` ディレクトリに多数のスキルファイルがある状態で `git pull` などを実行した際に Claude Code がフリーズするデッドロックが修正されました

### マルチセッション・サブエージェント関連

- **複数セッションでの Bash ツール出力消失を修正**: 同一プロジェクトディレクトリで複数の Claude Code セッションを並行実行した際に Bash ツールの出力が失われる問題が修正されました
- **サブエージェントのモデルサイレントダウングレードを修正**: `model: opus`/`sonnet`/`haiku` を指定したサブエージェントが Bedrock・Vertex・Microsoft Foundry で古いモデルバージョンにサイレントダウングレードされる問題が修正されました
- **サブエージェント終了時のバックグラウンドプロセス未クリーンアップを修正**: サブエージェントが起動したバックグラウンド bash プロセスがエージェント終了後も残存する問題が修正されました

### コマンド・フック関連

- **`/resume` に現在のセッションが表示される問題を修正**: セッションピッカーに現在のセッション自体が表示されていた問題が修正されました
- **`/ide` の `onInstall is not defined` クラッシュを修正**: 拡張機能の自動インストール時にクラッシュしていた問題が解消されました
- **`/loop` の Bedrock/Vertex/Foundry・テレメトリ無効時の非対応を修正**: 特定環境やテレメトリ無効化時に `/loop` が使用できない問題が修正されました
- **SessionStart フックの二重発火を修正**: `--resume` や `--continue` でセッションを再開した際に SessionStart フックが2回発火する問題が修正されました
- **JSON 出力フックのノーオペメッセージ注入を修正**: JSON 出力フックが毎ターン no-op の system-reminder メッセージをモデルのコンテキストに注入していた問題が修正されました

### プラットフォーム固有の修正

- **Linux sandbox の `rg not found` エラーを修正**: ネイティブビルドで ripgrep（rg）が見つからないためサンドボックスが起動しない問題が修正されました
- **Amazon Linux 2 などでのネイティブモジュール読み込み失敗を修正**: glibc 2.26 系のシステムで Linux ネイティブモジュールが読み込めない問題が修正されました
- **Windows `/heapdump` の `EEXIST` エラーを修正**: Desktop フォルダが既に存在する場合に `/heapdump` が失敗する問題が修正されました

### その他のバグ修正

- **Voice mode のセッション破損を修正**: 低速な接続と新しい録音が重なった際のセッション破損が修正されました
- **Remote Control 経由の画像受信エラーを修正**: `"media_type: Field required"` API エラーが発生していた問題が修正されました
- **VSCode: プロキシ環境や Bedrock/Vertex での HTTP 400 エラーを修正**: Claude 4.5 モデルを使用するユーザーに発生していたエラーが解消されました

## 非推奨化

### `/output-style` コマンドの非推奨化

`/output-style` コマンドが非推奨となりました。今後は `/config` を使用してください。出力スタイルはセッション開始時に固定されるようになり、プロンプトキャッシングの効率が向上します。

## 技術的なポイント

- **`modelOverrides` 設定**: Bedrock の推論プロファイルや Vertex のカスタムエンドポイントなど、標準外のモデル ID を使用する環境で必須の設定となります。設定ファイル（`~/.claude/settings.json` または `.claude/settings.json`）で管理可能です
- **デッドロック修正の重要性**: `.claude/skills/` ディレクトリに多数のスキルファイルを持つチームや、CI/CD パイプラインで Claude Code を使用している場合は特に恩恵があります
- **Opus 4.6 へのデフォルト変更**: Bedrock・Vertex・Foundry ユーザーは設定変更なしに最新の Opus 4.6 モデルが使われるようになります。既存の動作に影響する可能性があるため、動作確認を推奨します
- **SessionStart フックの二重発火修正**: フック経由で初期化処理を行っている場合、この修正で副作用が解消される可能性があります
- **`/output-style` 非推奨化**: プロンプトキャッシングを最大限活用するため、出力スタイルの変更はセッション開始前に `/config` で行う設計に移行しています

## まとめ

v2.1.73 は新機能よりもバグ修正と安定性改善に重点を置いたリリースです。特に複雑な bash コマンドによるフリーズやスキルファイル起因のデッドロックといった深刻な問題が解消されており、チームでの利用や自動化環境での信頼性が向上します。また、`modelOverrides` 設定の追加により、Bedrock などのカスタムプロバイダー環境でのモデル管理がより柔軟になりました。Bedrock・Vertex・Foundry ユーザーはデフォルト Opus モデルが 4.6 に更新された点にも注目が必要です。

# Claude Code v2.1.75 リリース — Opus 4.6 の 1M コンテキストウィンドウ対応、/color コマンド追加、メモリタイムスタンプ機能など

## 概要

Claude Code v2.1.75 は 2026 年 3 月 12 日にリリースされました。最大の目玉は **Opus 4.6 への 1M コンテキストウィンドウのデフォルト開放**で、Max・Team・Enterprise プランのユーザーが追加料金なしで利用できるようになりました。さらに、セッションテーマをカスタマイズできる `/color` コマンドの追加、メモリエントリへのタイムスタンプ付与による推論精度向上など、ユーザー体験を改善する変更が盛り込まれています。音声モードやトークン推定など複数のバグ修正も含まれます。

## 新機能

### Opus 4.6 の 1M コンテキストウィンドウがデフォルト有効化

Opus 4.6 モデルの 1M（100 万トークン）コンテキストウィンドウが、**Max・Team・Enterprise プランで追加料金なしにデフォルト有効**になりました。これまでは追加の利用枠が必要でしたが、本リリースより標準で利用できます。

大規模なコードベースのレビューや長大なドキュメントを扱うタスクにおいて、コンテキスト制限を意識せずに作業できるようになります。

### `/color` コマンドによるセッションテーマのカスタマイズ

```
/color
```

セッションの配色テーマを変更できる `/color` コマンドが追加されました。すべてのユーザー層で利用可能です。ターミナルの見た目を好みに合わせてカスタマイズしたいユーザーに向けた機能です。

### メモリエントリへのタイムスタンプ付与

Claude が記憶するメモリエントリにタイムスタンプが付与されるようになりました。これにより、Claude がコンテキストを推論する際に**新しい情報と古い情報を区別できる**ようになります。

長期間にわたって同じプロジェクトを扱う場合や、過去の指示と最新の指示が矛盾する場合に、より正確な判断が期待できます。

## バグ修正

### 音声モード（Voice Mode）の複数の問題を修正

新規インストール直後に音声モードが正常に有効化されず、トグルを繰り返す必要があった問題が修正されました。音声機能を使用しているユーザーはよりスムーズに利用できるようになります。

### モデル表示名のリフレッシュ問題

`/model` コマンドや `Option+P` ショートカットでモデルを切り替えた後、Claude Code のヘッダー部分に表示されるモデル名が更新されない問題が修正されました。切り替え後は即座に正しいモデル名が反映されます。

### アタッチメントメッセージ計算時のクラッシュ

アタッチメントメッセージの計算が `undefined` を返した場合にセッションがクラッシュする問題が修正されました。セッションの安定性が向上します。

### Bash ツールにおける感嘆符のエスケープ問題

Bash ツールでパイプを含むコマンドを実行する際、感嘆符（`!`）が意図せず変換される問題が修正されました。

```bash
# 以下のようなクエリが正しく動作するようになりました
jq 'select(.x != .y)'
```

`jq` などのツールで `!=` 演算子を含む表現が正しく処理されます。

### トークン推定の過大報告

thinking ブロックおよび `tool_use` ブロックのトークンカウントが過大に報告されていた問題が修正されました。これにより、コンテキストコンパクションが早期にトリガーされる誤動作が解消されます。長い会話セッションでの動作が改善されます。

### `/resume` コマンドでのセッション名保持

フォークまたは継続された会話を `/resume` で再開した際に、カスタムセッション名が失われる問題が修正されました。セッション名が正しく引き継がれるようになります。

## 技術的なポイント

- **1M コンテキストウィンドウのデフォルト開放**は、大規模コードベースを扱うエンタープライズユーザーにとって実用的な恩恵が大きい変更です。以前は別途設定や追加費用が必要だったため、利用ハードルが大幅に下がります
- **メモリタイムスタンプ**は、長期プロジェクトでの指示の一貫性向上に貢献します。特に異なる時期に与えた指示が競合する場面での推論精度改善が期待されます
- **Bash ツールの感嘆符修正**は一見小さな修正ですが、`jq` や `awk` などのデータ処理ツールを多用する開発者にとっては重要な修正です。シェルの履歴展開（`!` による展開）との干渉が原因と考えられます
- **トークン推定の修正**はコンテキスト管理の信頼性に直結します。過大なトークンカウントにより不必要なコンパクションが起きていた場合、長い会話での応答品質向上が見込めます

## まとめ

v2.1.75 は、Opus 4.6 の 1M コンテキストウィンドウを追加コストなしで開放するという実用性の高い変更を中心に、音声モードやセッション安定性にかかわる複数のバグ修正が含まれるリリースです。トークン推定の修正によりコンテキスト管理の精度も向上し、長時間セッションでの信頼性が増しています。メモリタイムスタンプ機能は今後の長期プロジェクト支援における推論改善の布石として注目です。

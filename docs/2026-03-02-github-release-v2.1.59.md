# Claude Code v2.1.59 リリース — 自動メモリ機能と /copy コマンドの追加

## 概要

Claude Code v2.1.59 では、セッションをまたいで有用なコンテキストを自動的に保存する **auto-memory 機能**と、コードブロックを選択してコピーできる **`/copy` コマンド**が新たに追加されました。また、複合 bash コマンドのプリフィックス提案の精度向上やマルチエージェントセッションのメモリ使用量改善など、実用性を高める複数の改善も含まれています。

## 新機能

### 自動メモリ機能（Auto-memory）

Claude が会話を通じて得た有用なコンテキスト（プロジェクトの構成、ユーザーの好み、繰り返し参照される情報など）を、自動的に **auto-memory** へ保存するようになりました。これにより、セッションをまたいでも重要な情報が引き継がれ、毎回同じ説明をする手間が省けます。

保存されたメモリの確認・編集・削除は `/memory` コマンドで管理できます。

```
/memory
```

どのような情報が保存されているかを定期的に確認し、不要なものを削除することで、Claude の動作をより適切にコントロールできます。

### `/copy` コマンド

レスポンス中にコードブロックが存在する場合に `/copy` コマンドを実行すると、**インタラクティブなピッカー**が表示されます。個別のコードブロックだけを選択してコピーするか、レスポンス全体をコピーするかを選べます。

```
/copy
```

複数のコードブロックが含まれる長いレスポンスから、特定のスニペットだけをすばやく取り出したい場面で役立ちます。

## 改善点

### 複合 bash コマンドの "always allow" プリフィックス提案の向上

`&&` でつないだ複合コマンド（例: `cd /tmp && git fetch && git push`）に対して "always allow" を設定する際、これまではコマンド全体を1つのプリフィックスとして扱っていました。今回の改善により、**各サブコマンドごとに個別のスマートなプリフィックスが計算される**ようになりました。

これにより、意図せず広すぎるパーミッションが付与されるリスクが減り、より細かい粒度でコマンドの許可設定が可能になります。

### 短いタスクリストの順序改善

TodoWrite ツールなどが生成する短いタスクリストの並び順が改善され、より論理的・直感的な順序でタスクが表示されるようになりました。

### マルチエージェントセッションのメモリ使用量改善

複数のサブエージェントを並列で使用するマルチエージェントセッションにおいて、完了したサブエージェントのタスク状態をメモリから解放するようになりました。長時間・大規模なエージェント処理を行う際のメモリ効率が向上します。

## バグ修正

### MCP OAuth トークンリフレッシュの競合状態を修正

複数の Claude Code インスタンスを同時に起動している環境で、MCP（Model Context Protocol）の OAuth トークンリフレッシュ処理が競合し、認証エラーが発生するケースがありました。今回の修正でこの競合状態が解消され、複数インスタンスの同時使用が安定して動作するようになりました。

### 削除されたワーキングディレクトリでのエラーメッセージ改善

シェルコマンドを実行しようとした際に、カレントのワーキングディレクトリが既に削除されていた場合、以前は不明瞭なエラーメッセージしか表示されませんでした。今回の修正により、**ディレクトリが存在しないことを明示する明確なエラーメッセージ**が表示されるようになりました。原因の特定と対処が容易になります。

## 技術的なポイント

- **auto-memory はプロジェクト単位で機能します**。`/memory` コマンドで内容を定期的に見直し、プロジェクトに関係ない情報が蓄積していないか確認することを推奨します。
- **`/copy` コマンドはコードブロックが存在する場合のみピッカーを表示します**。コードブロックがない場合はレスポンス全体のコピーとして動作します。
- **複合 bash コマンドのプリフィックス提案改善**は、CI/CD 環境や自動化スクリプトで Claude Code を利用する際のセキュリティ設定の精度向上に直結します。
- **MCP OAuth 競合状態の修正**は、複数のターミナルや tmux セッションで Claude Code を並行利用しているユーザーに特に恩恵があります。
- **マルチエージェントのメモリ解放**により、長期実行のエージェントワークフローでのリソース消費が抑制されます。

## まとめ

v2.1.59 は auto-memory による**セッション横断のコンテキスト保持**と、`/copy` コマンドによる**インタラクティブなコードブロック選択**という、日常的なワークフローを改善する2つの新機能が目玉です。加えて、複合 bash コマンドの許可設定の精度向上やマルチエージェントセッションの安定性改善など、実際の開発現場での使い勝手を高める修正も多数含まれています。auto-memory 機能は今後もさらに洗練されることが期待され、Claude Code がプロジェクトコンテキストをより賢く扱えるようになる方向性が示されています。

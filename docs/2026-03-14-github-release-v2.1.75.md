# Claude Code v2.1.75 リリース — 1M コンテキスト対応・/color コマンド追加・多数のバグ修正

## 概要

2026年3月13日、Claude Code v2.1.75 がリリースされました。本バージョンでは Opus 4.6 の 1M トークンコンテキストウィンドウ対応、プロンプトバーの色をカスタマイズできる `/color` コマンドの追加、メモリファイルへのタイムスタンプ付与など、UX 向上に関わる新機能が追加されています。また、音声入力・モデル切り替え・bash ツール・セッション管理にわたる広範なバグ修正も含まれています。

## 新機能

### Opus 4.6 の 1M コンテキストウィンドウ対応

Max・Team・Enterprise プランのユーザーを対象に、Opus 4.6 モデルで 1M（100万）トークンのコンテキストウィンドウが利用可能になりました。これまでは追加利用料が必要でしたが、今回のアップデートによりプラン内で標準的に使用できます。大規模なコードベースや長期セッションにおける文脈保持がより強力になります。

### `/color` コマンドの追加

全ユーザー向けに `/color` コマンドが追加されました。セッションごとにプロンプトバーの色を設定できるため、複数のセッションを同時に操作している場合の視覚的な識別に役立ちます。

### セッション名のプロンプトバー表示

`/rename` で設定したカスタムセッション名がプロンプトバーに表示されるようになりました。長期にわたる作業や複数プロジェクトの並行管理がより直感的に行えます。

### メモリファイルへのタイムスタンプ付与

メモリファイルに最終更新タイムスタンプが追加されました。Claude がメモリの新旧を判断しやすくなり、古い情報よりも新しい情報を優先して活用できるようになります。

## バグ修正

### 音声入力

- フレッシュインストール時に音声モードが起動しない問題を修正
- 音声入力における `language` 設定が正しく反映されない問題を修正

### モデル切り替えと UI

- `/model` コマンドや `Option+P` でモデルを切り替えた際、Claude Code のヘッダーに表示されるモデル名が更新されない問題を修正
- 添付メッセージが `undefined` を返した場合にセッションがクラッシュする問題を修正

### bash ツール

- パイプを含むコマンドで `!` 文字が不正に変換される問題を修正
- 複雑な bash コマンドシナリオにおける処理を改善

### プラグイン管理

- 管理者によって無効化されたプラグインが `/plugin` の「インストール済み」タブに表示されなくなりました
- `thinking` ブロックおよび `tool_use` ブロックのトークン推定が過大に計上されていた問題を修正

### セッション管理

- フォークまたは継続されたセッションを `/resume` で再開するとセッション名が失われる問題を修正
- Config タブを訪問後、`/status` ダイアログが Escape キーで閉じられない問題を修正
- プランの承認・却下時における入力処理を修正

### 表示とパフォーマンス

- ダークテーマでの引用ブロック（blockquote）の可読性を向上（斜体テキストと左バーを追加）
- 古くなった worktree のクリーンアップ処理を改善
- Remote Control セッションのタイトルを最初のプロンプトから自動生成するよう改善

### 設定

- `--plugin-dir` フラグを単一パスのみ受け付けるよう変更。複数のディレクトリを指定する場合はフラグを繰り返して使用してください（例: `--plugin-dir /path/a --plugin-dir /path/b`）

### VSCode 統合

- VSCode ファイルピッカーで gitignore パターンにカンマが含まれる場合の処理を修正

## 技術的なポイント

- **1M コンテキストの実用化**: Opus 4.6 の大規模コンテキストがプラン内で標準利用可能になったことで、大規模リポジトリの解析や長期セッションにおける文脈維持が現実的な選択肢となります
- **`--plugin-dir` の破壊的変更**: 複数パスをカンマ区切りや空白区切りで指定していた場合は、フラグの繰り返しに変更が必要です
- **bash `!` 問題の修正**: 履歴展開を伴うコマンドが意図せず変換されるケースが解消され、シェルスクリプトとの親和性が向上します
- **トークン推定の精度向上**: `thinking` ブロックや `tool_use` ブロックの過大計上が修正されたことで、コスト管理や上限管理の精度が改善されます
- **メモリのタイムスタンプ活用**: 自動メモリシステムを使っている場合、Claude がメモリの鮮度を考慮した応答を返しやすくなります

## まとめ

v2.1.75 は新機能と広範なバグ修正を含む充実したリリースです。特に Opus 4.6 の 1M コンテキストウィンドウが追加費用なしで利用可能になった点は、大規模な開発作業を行うユーザーにとって大きなメリットです。また `/color` コマンドによるセッションの視覚的識別や、セッション名の永続化など、日常的なワークフローを快適にする改善も含まれています。`--plugin-dir` の仕様変更については、プラグインを複数ディレクトリから読み込んでいる環境では設定の見直しが必要です。

# Claude Code v2.1.63 リリース — HTTP フック追加・大規模メモリリーク修正・新スラッシュコマンド

## 概要

Claude Code v2.1.63 では、HTTP フックのサポートや `/simplify`・`/batch` といった新しいスラッシュコマンドの追加など、開発ワークフローを拡張する機能が複数導入されました。また、長時間セッションでの無制限なキャッシュ成長を含む多数のメモリリークが修正されており、安定性・パフォーマンス面での改善も大きなポイントです。VSCode 向けの機能拡張も含まれ、全体的に完成度が高まったリリースとなっています。

## 新機能

### `/simplify` および `/batch` スラッシュコマンドの追加

バンドル済みのスラッシュコマンドとして `/simplify` と `/batch` が新たに追加されました。これにより、ユーザーはコマンドを定義・設定しなくてもすぐに利用できます。

### HTTP フックのサポート

これまでフック機能はシェルコマンドの実行のみをサポートしていましたが、今回のリリースで **HTTP フック**が追加されました。JSON を任意の URL へ POST し、JSON レスポンスを受け取るフックを設定できます。外部サービスや Webhook エンドポイントと連携したワークフロー自動化が容易になります。

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "http",
            "url": "https://example.com/hooks/pre-bash",
            "timeout": 5000
          }
        ]
      }
    ]
  }
}
```

### `/copy` コマンドの「常に完全な応答をコピー」オプション

`/copy` ピッカーに「Always copy full response（常に完全な応答をコピー）」オプションが追加されました。このオプションを選択すると、以降の `/copy` コマンドはコードブロック選択ダイアログをスキップし、応答全体を直接クリップボードにコピーするようになります。

### `/model` コマンドの改善

`/model` コマンドのスラッシュコマンドメニューに、現在アクティブなモデルが表示されるようになりました。どのモデルが選択されているかを一目で確認できるため、複数モデルを切り替えながら使用する際の利便性が向上しています。

### MCP OAuth 認証の手動 URL 貼り付けフォールバック

MCP（Model Context Protocol）の OAuth 認証フローにおいて、自動的な localhost リダイレクトが失敗した場合に備え、コールバック URL を手動で貼り付けて認証を完了できるフォールバック機能が追加されました。ネットワーク制約のある環境でも認証が通りやすくなります。

### Git ワークツリー間でのプロジェクト設定・自動メモリの共有

同一リポジトリの複数の Git ワークツリー間で、プロジェクト設定と自動メモリが共有されるようになりました。ワークツリーごとに設定を繰り返す手間が省けます。

### VSCode: セッションの名前変更・削除機能

VSCode 拡張のセッションリストに、セッションの名前変更（rename）と削除（remove）アクションが追加されました。長期的なプロジェクト管理においてセッション管理が格段に使いやすくなります。

### `ENABLE_CLAUDEAI_MCP_SERVERS=false` 環境変数のサポート

`ENABLE_CLAUDEAI_MCP_SERVERS=false` を環境変数に設定することで、claude.ai の MCP サーバーを利用可能にする動作をオプトアウトできるようになりました。セキュリティポリシーや環境制限に応じて制御できます。

## 改善点

- `/model` コマンドに現在アクティブなモデルを表示するよう改善
- サブエージェントを使用した長時間セッションでのコンテキスト圧縮時に、重いプログレスメッセージペイロードをストリップすることでメモリ使用量を改善

## バグ修正

### メモリリーク・リスナーリークの大規模修正

今回のリリースでは、長時間セッションやリソース再接続時に問題となるメモリリーク・リスナーリークが集中的に修正されました。

- **ブリッジポーリングループのリスナーリーク**修正
- **MCP OAuth フロークリーンアップのリスナーリーク**修正
- **フック設定メニューナビゲーション時のメモリリーク**修正
- **インタラクティブパーミッションハンドラの自動承認時のリスナーリーク**修正
- **WebSocket トランスポート再接続時のリスナーリーク**修正
- **Git ルート検出キャッシュの無制限成長**修正 — 長時間セッションで際限なく増加する問題
- **JSON 解析キャッシュの無制限成長**修正 — 同様に長時間セッションで肥大化していた問題
- **MCP サーバーフェッチキャッシュの未クリア**修正 — 頻繁に再接続するサーバーで特に顕著だった問題
- **長時間実行中の Teammate がメッセージを保持し続けていた問題**修正 — 会話圧縮後も AppState に全メッセージが残っていた

### その他のバグ修正

- **ローカルスラッシュコマンドの表示バグ修正**: `/cost` などのシステムメッセージがユーザー送信メッセージとして UI に表示されていた問題を修正
- **ファイルカウントキャッシュのバグ修正**: glob の無視パターンが反映されていなかった問題を修正
- **Bash コマンドプレフィックスキャッシュのメモリリーク**修正
- **IDE ホスト IP 検出キャッシュのバグ修正**: 複数ポート間でキャッシュ結果が誤って共有されていた問題を修正
- **REPL ブリッジのレース条件修正**: 初回接続フラッシュ中に新しいメッセージが履歴メッセージと交差してメッセージ順序が乱れる問題を修正
- **VSCode リモートセッション表示バグ修正**: リモートセッションが会話履歴に表示されない問題を修正
- **`/clear` コマンドのスキルキャッシュリセット漏れ修正**: クリア後も古いスキルコンテンツが残り続ける問題を修正

## 技術的なポイント

- **HTTP フックの導入**により、CI/CD や外部 Webhook とのインテグレーションがシェルスクリプト不要で実現できるようになりました。セキュリティ要件が厳しい環境でも、シェルコマンド実行を避けながらフックを活用できます。
- **メモリリーク修正の集中投下**は、長時間のペアプログラミングや大規模なエージェントセッションを運用している開発者にとって大きな恩恵です。特に Git ルート検出キャッシュと JSON 解析キャッシュの無制限成長は、長時間セッションのクラッシュ原因となり得るものでした。
- **Git ワークツリーのサポート改善**は、機能ブランチごとにワークツリーを切り替えながら開発するワークフローと Claude Code を組み合わせる際の摩擦を大幅に軽減します。
- **`ENABLE_CLAUDEAI_MCP_SERVERS=false`** 環境変数は、エンタープライズ環境や社内セキュリティポリシーで外部サービスへの接続を制限したい場合に有用です。
- `/simplify` および `/batch` のバンドルにより、ユーザーが自前でコマンドを定義しなくても標準的なユースケースに対応できます。

## まとめ

v2.1.63 は機能追加・バグ修正・安定性改善が高いバランスで盛り込まれたリリースです。特に HTTP フックの追加とメモリリーク修正の大規模な対応が目立ちます。長時間のエージェントセッションを多用するユーザーにとっては、メモリ安定性の向上が実感しやすいアップデートとなるでしょう。HTTP フックは今後の自動化・インテグレーション用途の拡張が期待されます。VSCode ユーザーはセッション管理の利便性向上も享受できます。速やかなアップデートを推奨します。

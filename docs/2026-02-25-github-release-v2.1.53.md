# Claude Code v2.1.53 リリース: Windows・Linux の安定性改善と UI 修正

## 概要

Claude Code v2.1.53 が2026年2月25日にリリースされました。このバージョンでは新機能の追加はなく、UI のちらつき修正や Windows・Linux における複数のクラッシュ修正を中心とした安定性向上がメインの内容です。特に Windows ユーザーや Remote Control 機能を利用しているユーザーにとって重要な改善が含まれています。

## 改善点

### UI のちらつき修正

ユーザーがメッセージを送信した直後に、入力内容が一瞬消えてから再表示されるという UI のちらつき問題が修正されました。送信後にメッセージがレンダリングされる前の短い間に入力欄が空白になる現象で、体験上の違和感の原因となっていましたが、本バージョンで解消されました。

### バルクエージェント終了（Ctrl+F）の改善

`Ctrl+F` によるバルクエージェントキル（一括終了）の動作が改善されました。従来はエージェント1件ごとに通知が発行されていたため、多数のエージェントを同時終了した際に大量の通知が発生していましたが、本バージョンから集約された単一の通知にまとめられるようになりました。また、コマンドキューが適切にクリアされるよう修正されており、終了後の状態管理が改善されています。

### Remote Control のグレースフルシャットダウン修正

Remote Control（リモートコントロール）機能を使用している際、グレースフルシャットダウン（段階的な終了処理）が完了した後もセッションが残留する問題が修正されました。ネットワーク切断処理を並列化することで、セッションの確実なクリーンアップが実現されています。

## バグ修正

### `--worktree` オプションが初回起動時に無視される問題

`--worktree` オプションを指定して起動した際、最初の起動時にそのオプションが無視されることがあった問題が修正されました。ワークツリーを活用した開発フローを採用しているユーザーに影響していた不具合です。

### Windows でのパニック修正（"switch on corrupted value"）

Windows 環境において、「switch on corrupted value」というメッセージとともにパニック（プロセスの異常終了）が発生することがあった問題が修正されました。内部の値が破損した状態でのスイッチ処理に起因するもので、予期しないクラッシュの原因となっていました。

### Windows での多数プロセス起動時のクラッシュ修正

Windows 環境で多数のプロセスを同時に起動した際にクラッシュが発生することがあった問題が修正されました。複数のエージェントを並行して動作させるような高負荷なユースケースでの安定性が向上しています。

### WebAssembly インタープリタのクラッシュ修正（Linux x64 / Windows x64）

Linux x64 および Windows x64 環境において、WebAssembly（Wasm）インタープリタがクラッシュする問題が修正されました。Claude Code の内部実行基盤として WebAssembly が活用されており、これによる安定性の問題が解消されています。

### Windows ARM64 での2分後クラッシュ修正

Windows ARM64 環境において、起動からおよそ2分後にクラッシュが発生することがあった問題が修正されました。ARM64 アーキテクチャ固有の問題で、Surface Pro X などの ARM ベースの Windows デバイスを使用しているユーザーに影響していた不具合です。

## 技術的なポイント

- **プラットフォーム固有の安定性強化**: Windows（x64・ARM64）と Linux x64 に対して集中的なクラッシュ修正が行われており、クロスプラットフォーム対応の品質向上が図られています
- **WebAssembly 実行基盤の修正**: Claude Code が WebAssembly インタープリタを内部で利用していることが明示的に示されており、x64 アーキテクチャ向けの修正が実施されました
- **通知の集約**: バルクエージェント終了時の通知をエージェントごとではなく1件に集約することで、UI の応答性や体験が改善されています
- **並列処理によるシャットダウン最適化**: Remote Control のセッション終了処理を並列化することで、クリーンアップの確実性と速度が改善されました
- **`--worktree` の信頼性向上**: 初回起動時のオプション処理に関するバグが修正され、ワークツリーを活用した開発フローの信頼性が向上しました

## まとめ

v2.1.53 は新機能を含まないバグ修正リリースですが、特に Windows および Linux x64 環境でのクラッシュ修正が複数含まれており、安定性の観点から重要なアップデートです。Windows ARM64 ユーザーや Remote Control 機能を使っているチームにとっては、アップデートを強く推奨します。また、UI のちらつきやバルクエージェント終了の通知問題など、日常的な操作体験に関わる改善も含まれており、全ユーザーにとってアップデートの価値があります。

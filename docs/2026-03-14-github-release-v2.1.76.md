# Claude Code v2.1.76 リリース — MCP Elicitation サポートと大量のバグ修正

## 概要

Claude Code v2.1.76 では、MCP サーバーがタスク実行中にユーザーへの構造化入力を要求できる **MCP Elicitation（情報収集）サポート**が追加されました。また、`--worktree` の改善、`/effort` スラッシュコマンド、`PostCompact` フックなど多数の新機能が加わっています。加えて、会話コンパクション後のツールスキーマ消失、プランモードの再承認問題、Remote Control セッションの不安定性など、開発体験に影響する重要なバグ修正が多数含まれています。

## 新機能

### MCP Elicitation サポート

MCP（Model Context Protocol）サーバーがタスク実行中にインタラクティブなダイアログを通じて構造化入力をリクエストできるようになりました。フォームフィールドまたはブラウザ URL の形式で入力を受け付けます。

さらに、レスポンスをサーバーに返す前に介入・オーバーライドできる **`Elicitation` / `ElicitationResult` フック**も追加されました。MCP サーバーを自作している開発者にとって、ユーザーとのインタラクションをカスタマイズできる強力な機能です。

### CLI フラグ: `-n` / `--name`

セッション起動時に表示名を設定できる `-n` / `--name <name>` フラグが追加されました。

```bash
claude --name "my-feature-branch-session"
```

複数のセッションを並行して管理する際に、セッションを識別しやすくなります。

### `worktree.sparsePaths` 設定

`claude --worktree` を大規模モノレポで使用する際に、git sparse-checkout を活用して必要なディレクトリのみをチェックアウトできる `worktree.sparsePaths` 設定が追加されました。不要なファイルのチェックアウトを省くことで、起動時間とディスク使用量を削減できます。

### `PostCompact` フック

会話コンパクション完了後に発火する `PostCompact` フックが追加されました。コンパクション後の後処理を自動化したい場合に活用できます。

### `/effort` スラッシュコマンド

`/effort` コマンドでモデルの effort（思考リソースの割り当て）レベルをセッション中に動的に変更できるようになりました。

### セッション品質サーベイ

エンタープライズ向けに、セッション品質のフィードバックサーベイが追加されました。管理者は `feedbackSurveyRate` 設定でサンプリングレートを調整できます。

## 改善点

### `--worktree` 起動パフォーマンスの向上

git refs を直接読み取るように変更し、リモートブランチがローカルに存在する場合は冗長な `git fetch` をスキップするようになりました。大規模リポジトリでのワークツリー起動が高速化されます。

### バックグラウンドエージェントの改善

バックグラウンドエージェントを強制終了した場合でも、それまでの部分的な実行結果が会話コンテキストに保持されるようになりました。中断後も成果を活用できます。

### モデルフォールバック通知の改善

モデルフォールバックの通知が verbose モードに関わらず常に表示されるようになり、人間が読みやすいモデル名で表示されるようになりました。

### ダークターミナルテーマでの blockquote 可読性向上

ブロッククォートのスタイルが「薄い文字」から「左バー付きイタリック」に変更され、ダークテーマでの視認性が改善されました。

### ワークツリー自動クリーンアップ

並列実行が中断された際に残されたワークツリーが自動的にクリーンアップされるようになりました。

### Remote Control セッションタイトルの改善

セッションタイトルが "Interactive session" ではなく、最初のプロンプトの内容から自動生成されるようになりました。

### `/voice` の改善

- 有効化時にディクテーション言語を表示するようになりました
- `language` 設定が音声入力で未対応の場合に警告を表示するようになりました

### `--plugin-dir` の変更

`--plugin-dir` は1フラグにつき1パスのみ受け付けるように変更されました。複数のディレクトリを指定する場合はフラグを繰り返し使用します。

```bash
claude --plugin-dir /path/to/plugins1 --plugin-dir /path/to/plugins2
```

## バグ修正

### コンパクション後のツールスキーマ消失（重要）

`ToolSearch` で読み込まれた deferred ツール（遅延ロードツール）が、会話コンパクション後にスキーマ情報を失い、配列・数値パラメータがタイプエラーで拒否される問題が修正されました。ツールの動作が中断なく継続されるようになります。

### プランモードの再承認問題

プランが既に承認済みであるにもかかわらず、再度承認を求める問題が修正されました。

### スラッシュコマンド関連

- スラッシュコマンドが "Unknown skill" と表示される問題を修正
- ソフトヒドゥンコマンドの正確な名前を入力した際に見つからない問題を修正

### 音声モード関連

- パーミッションダイアログやプランエディタが開いている間、voice モードがキー入力を取りこぼす問題を修正
- npm でインストールした場合、Windows で `/voice` が動作しない問題を修正
- `/voice` が起動時に言語設定のサポート可否を警告するよう改善

### コンテキスト・モデル関連

- 1M コンテキストセッションで `model:` frontmatter 付きスキルを呼び出した際に "Context limit reached" が誤表示される問題を修正
- 非標準モデル文字列使用時の "adaptive thinking is not supported on this model" エラーを修正

### Bash パーミッション関連

- 引用符付き引数に `#` が含まれる場合に `Bash(cmd:*)` パーミッションルールがマッチしない問題を修正
- パイプ・複合コマンドで "don't ask again" がコマンド全文を表示する問題を修正

### コンパクション・自動回復

- 自動コンパクションが連続失敗後も無限にリトライし続ける問題を修正。サーキットブレーカーにより3回の失敗後に停止するようになりました

### MCP・LSP 関連

- MCP 再接続成功後もスピナーが表示され続ける問題を修正
- LSP Manager がマーケットプレイスの照合前に初期化された場合、LSP プラグインがサーバーを登録しない問題を修正

### ターミナル・SSH 関連

- tmux over SSH 環境でのクリップボードコピーを修正。直接ターミナル書き込みと tmux クリップボード統合の両方を試みるようになりました

### UI/UX 関連

- `/export` がフルパスではなくファイル名のみを成功メッセージに表示する問題を修正
- テキスト選択後に新しいメッセージへ自動スクロールされない問題を修正
- ログイン方法選択画面で Escape キーが機能しない問題を修正

### Remote Control 関連

- サーバーがアイドル環境を回収した際にセッションがサイレントに終了する問題を修正
- 急速なメッセージが1件ずつキューイングされる代わりにバッチ処理されるよう改善
- JWT リフレッシュ後に古いワークアイテムが再配信される問題を修正
- WebSocket が長時間切断された後にブリッジセッションが回復しない問題を修正

### VSCode 統合

- gitignore パターンにカンマが含まれる場合、@-mention ファイルピッカーでファイルタイプ全体が除外される問題を修正

## 技術的なポイント

- **MCP Elicitation フック**は、MCP サーバー開発者がユーザーインタラクションをカスタマイズする新しい拡張ポイントです。`Elicitation` と `ElicitationResult` の2フックを使ってリクエスト・レスポンス両方をインターセプトできます
- **`worktree.sparsePaths`** は git sparse-checkout ベースの機能です。大規模モノレポで `--worktree` を使用している場合は積極的に活用することで起動速度を大幅に改善できます
- **サーキットブレーカーによるコンパクション制御**は安定性の観点で重要な改善です。3回の失敗後に停止することで無限ループを防ぎます
- **deferred ツールのスキーマ保持修正**は、`ToolSearch` を利用したツールチェーンの信頼性に直結する重要な修正です
- **`--plugin-dir` の変更は破壊的変更に近い仕様変更**です。複数ディレクトリを指定していたスクリプトやエイリアスがある場合は更新が必要です

## まとめ

v2.1.76 は MCP エコシステムの拡張、worktree の使い勝手向上、Remote Control の安定性改善など、幅広い領域にわたる大型リリースです。特に MCP Elicitation サポートは、サーバー側からユーザーへの構造化入力要求を可能にする重要な機能追加であり、より複雑なインタラクティブワークフローの構築に活用できます。また、コンパクション後のツールスキーマ消失やプランモードの再承認といった開発体験を損なうバグも修正されており、日常的な利用の信頼性が高まっています。`--plugin-dir` の仕様変更については、既存のスクリプトやエイリアスを確認・更新することをお勧めします。

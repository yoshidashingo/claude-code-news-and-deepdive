# Claude Code v2.1.71 リリース — /loop コマンドとCronスケジューリング、多数のバグ修正

## 概要

Claude Code v2.1.71 では、プロンプトやスラッシュコマンドを定期実行できる `/loop` コマンドとCronスケジューリング機能が新たに追加されました。また、音声モードのキーバインド改善や Bash 自動承認リストの拡充など、開発者の利便性を高める機能強化が行われています。加えて、長時間セッションでのキー入力フリーズや起動時のUIフリーズなど、実用上の問題となっていた多数のバグが修正されています。

## 新機能

### `/loop` コマンドとCronスケジューリング

セッション内でプロンプトやスラッシュコマンドを定期実行できる `/loop` コマンドが追加されました。

```
/loop 5m check the deploy
```

上記のように実行すると、5分ごとに「check the deploy」というプロンプトが繰り返し実行されます。デプロイ状況の監視やCI/CDパイプラインの定期チェックなど、反復的なタスクの自動化に役立ちます。合わせてCronスケジューリングツールも追加され、セッション内での定期実行をより柔軟に制御できるようになりました。

### 音声アクティベーションキーのカスタマイズ

`keybindings.json` に `voice:pushToTalk` キーバインドが追加され、音声アクティベーションキーをカスタマイズできるようになりました。デフォルトはスペースキーですが、`meta+k` のようなモディファイアキーとの組み合わせも設定可能です。これにより、テキスト入力中の誤った音声アクティベーションを防ぎやすくなります。

```json
{
  "voice:pushToTalk": "meta+k"
}
```

### Bash 自動承認リストの拡充

以下のコマンドが Bash の自動承認リストに追加されました。これらは一般的なテキスト処理・計算用コマンドで、毎回の承認確認が不要になります。

- `fmt` — テキストのフォーマット
- `comm` — ソート済みファイルの行比較
- `cmp` — ファイルのバイト単位比較
- `numfmt` — 数値のフォーマット変換
- `expr` — 式の評価
- `test` — 条件テスト
- `printf` — フォーマット出力
- `getconf` — システム設定値の取得
- `seq` — 連番の生成
- `tsort` — トポロジカルソート
- `pr` — テキストファイルのページ分割

## 改善点

### 起動時間の短縮

ネイティブ画像プロセッサーのロードを初回使用時まで遅延するよう変更され、起動時間が改善されました。特に画像処理を使わないユースケースでは効果が顕著です。

### ブリッジセッション再接続の高速化

ラップトップのスリープ復帰後のブリッジセッション再接続が、従来の最大10分待機から数秒以内に完了するよう改善されました。リモートセッションを多用する開発者にとって大きな改善です。

### `/plugin uninstall` の挙動改善

`/plugin uninstall` がプロジェクトスコープのプラグインを `.claude/settings.json` ではなく `.claude/settings.local.json` に書き込むよう変更されました。これにより、プラグインのアンインストールがチームメンバーのリポジトリ設定に影響を与えなくなります。

### プラグインMCPサーバーの重複排除改善

プラグインが提供する MCP サーバーで、手動設定済みのサーバーと同一（同じコマンド/URL）のものは自動的にスキップされるようになりました。重複接続やツールセットの重複を防ぎ、`/plugin` メニューで抑制状況を確認できます。

### `/debug` コマンドのオンデマンド切り替え

`/debug` コマンドがセッション中にデバッグログのオン/オフを切り替えられるよう更新されました。デバッグログはデフォルトで書き込まれなくなったため、必要な時だけ有効化できます。

## バグ修正

### 重大なフリーズ・ハング問題の修正

- **長時間セッションでのキー入力フリーズ**: stdin がフリーズしてキーストロークが処理されなくなる問題を修正。プロセス自体は生きているのに入力を受け付けなくなるという厄介なバグでした。
- **音声モード有効時の起動フリーズ（5〜8秒）**: システムのスリープ復帰後に CoreAudio の初期化がメインスレッドをブロックしていた問題を修正。
- **claude.ai Proxy コネクターによる起動UIフリーズ**: 多数のプロキシコネクターが同時に期限切れ OAuth トークンを更新する際に発生していたフリーズを修正。
- **`--print` オプションのハング**: チームエージェントが設定されている場合に `--print` が永久にハングする問題を修正。長命な `in_process_teammate` タスクを終了ループが待ち続けていたことが原因です。

### `/fork` コマンドの修正

`/fork` でフォークしたコンバセーションが同一のプランファイルを共有していたため、一方のフォークでの編集が他方を上書きしてしまう問題を修正。各フォークが独立したプランファイルを持つようになりました。

### Read ツールの画像処理修正

画像処理に失敗した際に過大なサイズの画像がコンテキストに入り込む問題を修正。画像の多いセッションで後続のターンが壊れてしまう問題が解消されました。

### Bash 関連の修正

- heredoc コミットメッセージを含む複合Bashコマンドへの誤ったパーミッションプロンプトを修正。
- Windows で mingw スタイルのパスを使用した際に `cd <cwd> && git ...` で不要なプロンプトが表示される問題を修正。
- コマンドが非ゼロステータスで終了した際の Bash ツールエラーメッセージの重複出力を修正。

### プラグイン・MCP関連の修正

- 複数の Claude Code インスタンスを実行した際にプラグインインストールが失われる問題を修正。
- OAuth トークン更新後に claude.ai コネクターが再接続できなくなる問題を修正。
- MCP コネクターの起動通知が各コネクターごとに重複して表示される問題を修正（以前接続済みのコネクターのみに絞られました）。
- バックグラウンドエージェント完了通知に出力ファイルパスが含まれていなかった問題を修正。コンテキスト圧縮後に親エージェントが結果を回復しやすくなりました。
- `/plugin marketplace update` がブランチ/タグ参照にピン留めされている際にマージコンフリクトで失敗する問題を修正。
- `/plugin marketplace add owner/repo@ref` の `@` 区切り文字のパースが正しく機能していなかった問題を修正（以前は `#` のみが動作していました）。

### その他のUI・UX修正

- Chrome 拡張機能の自動検出が「未インストール」状態で永久にスタックする問題を修正。
- `/permissions` Workspace タブで末尾スラッシュの有無が異なる同一ディレクトリが重複して表示される問題を修正。
- `ToolSearch` 呼び出し後に REPL で「❯ Tool loaded.」が毎回表示される問題を修正。
- 未認証の組織登録済み claude.ai コネクターへの起動通知ノイズを削除。

## 技術的なポイント

- **`/loop` とCron**: セッション内自動化の新しいプリミティブとして、定期的な監視・チェックタスクをエージェントに委任できるようになりました。
- **`voice:pushToTalk` のカスタマイズ**: キーバインドのカスタマイズ性が音声機能にまで広がり、ハードウェアキーボードでの快適な音声入力を実現します。
- **プラグイン設定の分離**: `.claude/settings.local.json` を活用することで、個人設定とチーム共有設定を明確に分離する設計思想が強化されています。
- **MCP サーバー重複排除**: 複数のプラグインが同一のMCPサーバーを提供するケースへの対応で、大規模なプラグイン環境での安定性が向上します。
- **起動パフォーマンス**: 画像プロセッサーの遅延ロードにより、画像処理を必要としないワークフローでは起動時間が短縮されます。

## まとめ

v2.1.71 は `/loop` コマンドとCronスケジューリングという新しい自動化機能を中心に、音声機能の改善、パフォーマンス向上、そして多数の安定性バグ修正を含む充実したリリースです。特に長時間セッションでのフリーズ問題や起動時間の改善は、日常的に Claude Code を使用している開発者に直接メリットをもたらします。定期実行機能の追加により、CI/CDモニタリングやデプロイ状況確認などの反復タスクをより効率的に自動化できるようになり、Claude Code のエージェント活用の幅がさらに広がっています。

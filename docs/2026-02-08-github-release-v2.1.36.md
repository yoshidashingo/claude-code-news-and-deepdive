# Claude Code v2.1.36 - Opus 4.6 で Fast Mode が利用可能に

## 概要

Claude Code v2.1.36 がリリースされました。本リリースの主な変更点は、Opus 4.6 モデルに対する **Fast Mode** のサポートです。Fast Mode を有効にすることで、同じ Opus 4.6 モデルの品質を維持しながら、より高速なレスポンスを得ることができます。

## 新機能

### Fast Mode for Opus 4.6

Fast Mode は、Opus 4.6 のレスポンス速度を向上させる新機能です。重要なポイントとして、Fast Mode は**別のモデルではありません**。同じ Opus 4.6 モデルを使用しつつ、API の設定をレイテンシ優先に切り替えることで、品質や機能はそのままに高速な応答を実現します。

#### 有効化の方法

Fast Mode は以下の方法で切り替えることができます。

- CLI で `/fast` コマンドを入力して Tab を押す
- VS Code Extension でも `/fast` コマンドが利用可能
- 設定ファイルで `"fastMode": true` を指定

Fast Mode が有効になると、プロンプトの横に `↯` アイコンが表示されます。再度 `/fast` を実行すると無効になります。なお、Fast Mode の有効/無効状態はセッションをまたいで維持されます。

#### 料金体系

Fast Mode は標準の Opus 4.6 よりも高い料金が設定されています。

| モード | Input (MTok) | Output (MTok) |
|--------|-------------|---------------|
| Fast Mode（200K トークン以下） | $30 | $150 |
| Fast Mode（200K トークン超） | $60 | $225 |

**2026年2月16日 23:59（PT）まで、全プランで 50% 割引**が適用されます。Fast Mode の使用料金は Extra Usage（追加使用量）として課金され、サブスクリプションの含有使用量には含まれません。

#### 利用要件

Fast Mode を利用するには、以下の条件を満たす必要があります。

- **サブスクリプションプラン**（Pro / Max / Team / Enterprise）または Claude Console のユーザーであること
- **Extra Usage が有効**になっていること
- **サードパーティクラウドプロバイダーでは利用不可**（Amazon Bedrock、Google Vertex AI、Microsoft Azure Foundry は非対応）
- **Teams / Enterprise の場合**: 管理者が事前に Fast Mode を有効化する必要がある

#### レート制限の挙動

Fast Mode には標準の Opus 4.6 とは別のレート制限が設けられています。Fast Mode のレート制限に達した場合は、自動的に標準の Opus 4.6 にフォールバックし、`↯` アイコンがグレーに変わります。クールダウン期間が終了すると、自動的に Fast Mode が再有効化されます。

## 技術的なポイント

- **モデルは同一**: Fast Mode は Opus 4.6 の API 設定を変更したもので、モデル自体は変わりません。出力品質や機能に違いはなく、純粋にレイテンシとコストのトレードオフです
- **セッション途中の切り替えに注意**: 会話の途中で Fast Mode を有効にすると、それまでのコンテキスト全体に対して Fast Mode のキャッシュされていない入力トークン料金が適用されます。コスト効率を考えると、セッション開始時に有効にするのが望ましいです
- **1M トークンの拡張コンテキストウィンドウにも対応**: Fast Mode は拡張コンテキストウィンドウと互換性があります
- **Effort Level との組み合わせが可能**: Fast Mode（レイテンシ最適化）と Effort Level の低減（思考時間の短縮）は異なるアプローチのため、両方を組み合わせることで、単純なタスクに対して最大限の速度を実現できます
- **Research Preview の段階**: Fast Mode は現在 Research Preview であり、機能・料金・提供状況は今後変更される可能性があります

## まとめ

Claude Code v2.1.36 は、Opus 4.6 に Fast Mode を追加するフォーカスされたリリースです。ライブデバッグや素早いコードイテレーションなど、レスポンス速度が重要なインタラクティブな作業に特に有効です。一方で、長時間の自律的タスクやコスト重視のワークロードでは、標準モードを使い続けるのが合理的でしょう。

2月16日まで 50% 割引が適用されるため、まずはこの期間中に試してみて、自分のワークフローに合うかどうかを評価してみることをお勧めします。

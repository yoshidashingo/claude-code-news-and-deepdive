# Claude Code v2.1.74 リリース — コンテキスト最適化提案、メモリリーク修正、RTL文字対応など多数の改善

## 概要

Claude Code v2.1.74 では、`/context` コマンドへの実用的な最適化提案機能の追加と、Node.js/npm コードパスにおけるストリーミング API のメモリリーク修正が主要な変更点です。加えて、MCP OAuth 認証、音声モード、LSP サーバー、Windows Terminal での RTL 文字表示など、多岐にわたるバグ修正が含まれています。

## 新機能

### `/context` コマンドへの実用的なサジェスト機能

`/context` コマンドに、コンテキスト使用量の最適化を促すアクション可能なサジェスト機能が追加されました。コンテキストを大量消費しているツール、メモリの肥大化、および容量警告を検出し、具体的な最適化ヒントを提示します。

長いセッションでコンテキスト上限に近づいた際に、どのツールやメモリが原因かを把握しやすくなり、効率的なコンテキスト管理が可能になります。

### `autoMemoryDirectory` 設定の追加

オートメモリの保存先ディレクトリをカスタマイズするための `autoMemoryDirectory` 設定が追加されました。これにより、複数プロジェクト間での記憶領域の整理や、チーム共有ディレクトリへの配置が柔軟に行えるようになります。

```json
{
  "autoMemoryDirectory": "/path/to/custom/memory/dir"
}
```

## 改善点

### `--plugin-dir` の優先順位変更

`--plugin-dir` で指定したローカル開発用プラグインが、同名のマーケットプレイス製プラグインより優先されるようになりました（managed settings で force-enable されているプラグインを除く）。プラグイン開発者がマーケットプレイス版と並行してローカルの開発版をテストしやすくなります。

### VSCode 統合ターミナルのスクロール改善

VSCode 拡張の統合ターミナルにおいて、ターミナルの動作を考慮したスクロールアクセラレーションが実装され、マウスホイールによるスクロールの応答性が向上しました。

## バグ修正

### メモリリークの修正（Node.js/npm コードパス）

ストリーミング API レスポンスのバッファが、ジェネレーターが早期に終了した際に解放されないメモリリークが修正されました。このバグにより Node.js/npm コードパスで RSS（Resident Set Size）が際限なく増加する問題が発生していましたが、本バージョンで解消されています。長時間のセッションや繰り返し実行するワークフローでの安定性が向上します。

### managed policy の `ask` ルールがバイパスされる問題の修正

managed policy で設定された `ask` ルールが、ユーザーの `allow` ルールやスキルの `allowed-tools` によって上書き・バイパスされてしまう問題が修正されました。組織のポリシー管理において、意図しないツール実行の許可が防止されるようになります。

### エージェントのフルモデル ID が無視される問題の修正

エージェントの frontmatter `model:` フィールドや `--agents` JSON 設定において、`claude-opus-4-5` のようなフルモデル ID が無言で無視されていた問題が修正されました。エージェントでも `--model` フラグと同じモデル値が利用できるようになります。

```yaml
---
model: claude-opus-4-5
---
```

### MCP OAuth 認証のハング修正

コールバックポートがすでに使用中の場合に MCP OAuth 認証がハングする問題が修正されました。また、OAuth サーバーがエラーを HTTP 200 で返す場合（Slack など）にリフレッシュトークン期限切れ後の再認証プロンプトが表示されなかった問題も合わせて修正されています。

### macOS ネイティブバイナリでの音声モード修正

macOS ネイティブバイナリで、マイクのアクセス許可を付与したことがないターミナルで音声モードが無言で失敗する問題が修正されました。バイナリに `audio-input` entitlement が追加され、macOS が正しくマイク許可のダイアログを表示するようになります。

### `SessionEnd` フックのタイムアウト設定

`SessionEnd` フックが `hook.timeout` 設定に関わらず終了時に 1.5 秒でキルされてしまう問題が修正されました。環境変数 `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` でタイムアウトを設定できるようになり、セッション終了時に時間のかかるクリーンアップ処理を実行するフックも適切に完了できます。

```bash
export CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS=10000  # 10秒に設定
```

### マーケットプレイスプラグインの修正

- REPL 内での `/plugin install` がマーケットプレイスプラグイン（ローカルソース）で失敗する問題を修正
- マーケットプレイス更新時に git submodule が同期されない問題を修正（submodule 内のプラグインソースが更新後に壊れなくなります）

### 不明なスラッシュコマンドの入力消失を修正

引数付きの未知のスラッシュコマンドを実行した際、入力が無言で破棄されていた問題が修正されました。入力内容が警告として表示されるようになり、タイポなどによる意図しない入力消失が防止されます。

### Windows での RTL 文字レンダリング修正

Windows Terminal、conhost、および VS Code 統合ターミナルにおいて、ヘブライ語・アラビア語などの RTL（右から左）テキストが正しくレンダリングされない問題が修正されました。

### Windows での LSP サーバー修正

Windows 上で LSP（Language Server Protocol）サーバーが動作しない問題（ファイル URI の形式が不正だったことが原因）が修正されました。

### VSCode の Untitled セッション削除ボタンの修正

VSCode 拡張で Untitled セッションの削除ボタンが機能しなかった問題が修正されました。

## 技術的なポイント

- **メモリリーク対策**: ストリーミングジェネレーターの早期終了時にバッファが確実に解放されるよう修正。長時間稼働の Node.js プロセスで恩恵を受けられます
- **セキュリティ強化**: managed policy の `ask` ルールが適切に強制されるようになり、組織内でのポリシー管理の信頼性が向上
- **フックの柔軟性向上**: `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` 環境変数でセッション終了フックのタイムアウトを制御可能に
- **エージェント設定の互換性**: フルモデル ID がエージェント設定でも認識されるようになり、モデル指定の一貫性が確保
- **MCP の安定性**: OAuth 認証周りの複数の問題が修正され、Slack 等の OAuth 2.0 サーバーとの連携が改善
- **Windows サポート強化**: RTL テキスト表示と LSP サーバーの修正により、Windows 環境での使用体験が向上

## まとめ

v2.1.74 は新機能よりもバグ修正と安定性改善に重きを置いたリリースです。特に Node.js/npm コードパスのメモリリーク修正は、長期セッションや CI/CD 環境での利用において重要な改善です。また、`/context` コマンドの最適化提案機能により、コンテキスト管理の可視性と操作性が高まります。Windows ユーザーや MCP・LSP を活用している開発者にとっても恩恵の大きいアップデートといえます。速やかなアップデートをお勧めします。

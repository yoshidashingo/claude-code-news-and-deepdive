# Claude Code v2.1.72 リリース — 大規模バグ修正とワークフロー改善

## 概要

Claude Code v2.1.72 は、新機能の追加と大量のバグ修正を含む大型リリースです。SSH 環境でのファイル書き込み改善、`/plan` コマンドの強化、Worktree 管理の拡充など、日常的な開発ワークフローを向上させる変更が多数含まれています。また、bash コマンド解析のネイティブモジュール化により初期化速度の改善とメモリリーク修正も実現しています。

## 新機能

### `/copy` コマンドに `w` キーを追加

`/copy` 操作でフォーカスされた選択内容を、クリップボードを介さずに直接ファイルへ書き込めるようになりました。SSH 経由でリモート環境を操作している場合など、クリップボードが利用できない状況で特に便利です。

### `/plan` コマンドに説明引数を追加

`/plan` コマンドにオプションの説明引数が追加されました。

```
/plan fix the auth bug
```

このように記述することで、プランモードに入ると同時にその説明内容で即座に処理を開始できます。従来はプランモードに入ってから手動で指示する必要がありましたが、一行で完結するようになりました。

### `ExitWorktree` ツールの追加

`EnterWorktree` で入ったセッションを明示的に終了するための `ExitWorktree` ツールが追加されました。Worktree の分離セッション管理がより明確になります。

### `CLAUDE_CODE_DISABLE_CRON` 環境変数の追加

セッション途中でスケジュール済みの cron ジョブを即座に停止できる環境変数です。

```bash
export CLAUDE_CODE_DISABLE_CRON=1
```

長時間セッション中に意図しない定期処理が走るケースを防ぐために活用できます。

### bash 自動承認リストの拡充

`lsof`、`pgrep`、`tput`、`ss`、`fd`、`fdfind` が bash 自動承認リストに追加されました。これらは読み取り専用の一般的なコマンドであり、権限プロンプトの頻度が削減されます。

### Agent ツールの `model` パラメータ復活

Agent ツールで呼び出し単位でモデルを指定できる `model` パラメータが復活しました。用途に応じて異なるモデルを使い分けるユースケースで有効です。

### エフォートレベルの簡素化

エフォートレベルが `low / medium / high` の3段階に整理され（`max` は廃止）、新しいシンボル（○ ◐ ●）が採用されました。デフォルトに戻すには `/effort auto` を使用します。

### マーケットプレイス git URL の `.git` サフィックス省略に対応

Azure DevOps や AWS CodeCommit など、`.git` サフィックスなしの git URL 形式を使うサービスへの対応が追加されました。

## 改善点

### `/config` の操作性向上

- **Escape** キーで変更をキャンセル
- **Enter** キーで保存して閉じる
- **Space** キーで設定をトグル

### bash コマンド解析のネイティブモジュール化

bash コマンド解析がネイティブモジュールに切り替わりました。初期化速度の向上とメモリリークの解消が図られています。また、tree-sitter パーシングの改善により `find -exec`、変数代入、コマンド置換など多くのパターンで誤った権限プロンプトが抑制されます。npm でインストールした場合に tree-sitter が読み込まれなかった問題も修正されました。

### CLAUDE.md の HTML コメントを Claude の入力から除外

CLAUDE.md に記述した HTML コメント（`<!-- ... -->`）が自動挿入時に Claude へ渡されなくなりました。ただし Read ツールで明示的に読み込んだ場合はコメントも表示されます。これにより、CLAUDE.md に開発者向けの内部メモを書いても Claude の挙動に影響しません。

### プロンプトキャッシュの無効化修正（SDK `query()` 呼び出し）

SDK の `query()` 呼び出しでプロンプトキャッシュが正しく機能するようになり、入力トークンコストが最大 **12倍**削減される可能性があります。

### その他の改善

- 複数の並行セッション実行時に、上矢印キーの履歴が現在のセッションのメッセージを優先表示
- 音声入力のリポジトリ名・開発用語（regex, OAuth, JSON）への転写精度向上
- バンドルサイズを約 510 KB 削減
- 長時間セッションでの CPU 使用率改善

## バグ修正

### セッション・ワークフロー関連

- バックグラウンドタスクやフックが遅い場合に終了が遅くなる問題を修正
- `--continue` が `--compact` 後に最新ポイントから再開されない問題を修正
- `/clear` がバックグラウンドのエージェント・bash タスクを誤って終了させる問題を修正（フォアグラウンドタスクのみ対象に）
- `--effort` CLI フラグが起動時の設定書き込みでリセットされる問題を修正
- Ctrl+B でバックグラウンド化したクエリのトランスクリプトが失われる問題を修正

### Worktree 関連

- Task ツールの再開時に作業ディレクトリが正しく復元されない問題を修正
- バックグラウンドタスク通知に `worktreePath` と `worktreeBranch` が含まれない問題を修正

### プラグイン・フック関連

- スキルフックが1イベントあたり2回発火する問題を修正
- Windows の OneDrive フォルダでプラグインインストールが `EEXIST` エラーで失敗する問題を修正
- プロジェクトスコープのインストールが存在する場合にマーケットプレイスがユーザースコープインストールをブロックする問題を修正
- `CLAUDE_CODE_PLUGIN_CACHE_DIR` がリテラルの `~` ディレクトリを作成する問題を修正
- `hooks` 関連: `transcript_path` の誤ったディレクトリ参照、async フックが bash `read -r` で stdin を受け取れない問題など複数修正

### 権限・サンドボックス関連

- ワイルドカードルールがヒアドキュメントや引数なしコマンドにマッチしない問題を修正
- `sandbox.excludedCommands` が環境変数プレフィックス付きコマンドに失敗する問題を修正
- 並列ツール呼び出しで Read / WebFetch / Glob の失敗が兄弟ツールをキャンセルする問題を修正（Bash エラーのみカスケードするよう変更）

### その他のバグ修正

- U+2028 / U+2029 文字を含むファイルを Read した際のセッションクラッシュを修正
- チームエージェントがリーダーのモデルを継承しない問題を修正
- 音声モードの入力遅延・誤った「No speech detected」エラー・古いトランスクリプト再挿入を修正
- Bedrock API エラーを含むセッションの再開時クラッシュを修正

### VSCode 固有の修正・追加

- 統合ターミナルのスクロール速度をネイティブターミナルに合わせて修正
- 古いキーバインド設定での Shift+Enter が改行挿入ではなく送信になる問題を修正
- 入力ボーダーにエフォートレベルインジケータを追加
- `vscode://anthropic.claude-code/open` URI ハンドラを追加（`prompt` / `session` クエリパラメータでプログラム的にタブを開くことが可能）

## 技術的なポイント

- **bash 解析のネイティブ化**: tree-sitter を使ったネイティブモジュールへの移行により、初期化コストの削減とメモリリーク解消が実現。誤検知の権限プロンプト削減にも寄与
- **プロンプトキャッシュ修正**: SDK `query()` での最大 12 倍のトークンコスト削減は、大規模自動化シナリオで特に大きな恩恵をもたらす
- **CLAUDE.md HTML コメント除外**: `<!-- -->` 形式でドキュメント内に開発メモや無効化セクションを書いても Claude の挙動に影響しなくなり、CLAUDE.md の管理柔軟性が向上
- **並列ツール失敗の挙動変更**: Bash 以外のツール（Read / WebFetch / Glob）の失敗が兄弟ツールをキャンセルしなくなったため、並列処理の堅牢性が向上
- **ExitWorktree ツール**: EnterWorktree / ExitWorktree のペアで明示的なセッション境界管理が可能になり、Worktree を使った分離ワークフローが安全に構築できる

## まとめ

v2.1.72 は新機能よりもバグ修正と品質改善に重点を置いたリリースです。特にフック・プラグイン・Worktree・権限ルールにまたがる多数の修正は、複雑なワークフローを運用している開発者にとって安定性の大幅な向上につながります。SDK を使った自動化においてはプロンプトキャッシュ修正によるコスト削減効果も見逃せません。bash 解析のネイティブ化はパフォーマンス面での長期的な基盤強化として、今後のさらなる改善につながることが期待されます。

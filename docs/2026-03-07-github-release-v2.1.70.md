# Claude Code v2.1.70 リリース — 多数のバグ修正とVSCode拡張強化

## 概要

Claude Code v2.1.70 は 2026年3月6日にリリースされました。API エラー修正、Windows/WSL 環境の改善、パフォーマンス向上が主な内容です。加えて、VS Code 向けにセッション一覧表示・プラン閲覧・MCP 管理ダイアログという3つの新機能が追加されています。

## 新機能

### [VSCode] アクティビティバーへのセッション一覧追加

VS Code のアクティビティバーにスパークアイコンが追加され、すべての Claude Code セッションを一覧表示できるようになりました。各セッションはフルエディタとして開けるため、複数セッションの切り替えが格段に簡単になります。

### [VSCode] プランのマークダウンドキュメントビュー

プランをフルマークダウンドキュメントとして表示できるようになりました。コメントを追加してフィードバックを提供する機能も含まれており、プランの確認・レビューがより快適になります。

### [VSCode] ネイティブ MCP サーバー管理ダイアログ

チャットパネルで `/mcp` と入力することで、ネイティブの MCP サーバー管理ダイアログが起動します。サーバーの有効化・無効化、再接続、OAuth 認証の管理をターミナルを開かずに行えます。

## 改善点

### パフォーマンス改善

- **プロンプト入力の再レンダリングを約74%削減**: ターン中の UI 描画コストが大幅に軽減されました
- **起動時メモリを約426KB削減**: カスタム CA 証明書を使用していないユーザーが対象です
- **Remote Control の `/poll` レートを大幅削減**: 接続中の間欠ポーリングを1〜2秒から10分に1回へ変更し、サーバー負荷を約300分の1に削減。切断時の再接続はこれまで通り高速なポーリングが維持されます

### コンパクション改善

コンパクション時に画像を保持するよう改善され、プロンプトキャッシュの再利用が可能になりました。コンパクションが高速化・低コスト化されます。

### `/rename` コマンドの即時実行

`/rename` コマンドが Claude の処理中でも即座に動作するようになりました。以前はサイレントにキューに追加されていました。

## バグ修正

### API・モデル関連

- **`ANTHROPIC_BASE_URL` 使用時の API 400 エラー修正**: サードパーティゲートウェイ経由でツール検索を使う際に発生していたエラーを修正。プロキシエンドポイントを正しく検出し、`tool_reference` ブロックを無効化するようになりました
- **カスタム Bedrock 推論プロファイルでの `effort` パラメータエラー修正**: 標準的な Claude 命名パターンに合致しないモデル識別子で発生していた「`This model does not support the effort parameter`」エラーを修正しました
- **`ToolSearch` 直後の空レスポンス修正**: ツールスキーマがシステムプロンプト形式のタグとしてプロンプト末尾に展開されることで、モデルが早期停止していた問題を修正しました
- **MCP サーバー接続時のプロンプトキャッシュバスト修正**: `instructions` を持つ MCP サーバーが初回ターン以降に接続した際にプロンプトキャッシュが無効化される問題を修正しました

### 入力・インターフェース

- **遅い SSH 接続でのエンター動作修正**: Enter キーが送信ではなく改行を挿入してしまう問題を修正しました
- **マークダウンリンク内の `#NNN` 参照修正**: `#123` 形式の参照が現在のリポジトリのイシューではなく、リンク先 URL を正しく指すようになりました
- **「Model updated to Opus 4.6」通知の繰り返し修正**: `.claude/settings.json` にレガシーな Opus モデル文字列が設定されている場合に通知が繰り返し表示される問題を修正しました

### Windows/WSL

- **クリップボードの非 ASCII 文字化け修正**: CJK 文字や絵文字が Windows/WSL 上でクリップボードにコピーされると文字化けする問題を、PowerShell の `Set-Clipboard` を使用することで修正しました
- **VS Code 統合ターミナルからの起動時に余分なウィンドウが開く問題を修正**: Windows 上で VS Code 統合ターミナルから起動すると VS Code ウィンドウが余分に開いていた問題を修正しました

### 音声機能

- **Windows ネイティブバイナリでの音声モード修正**: 「`native audio module could not be loaded`」エラーで音声モードが利用できない問題を修正しました
- **セッション開始時のプッシュトゥトーク修正**: `voiceEnabled: true` を設定している場合にセッション開始直後にプッシュトゥトークが有効にならない問題を修正しました
- **マイクの無音検出エラーメッセージを改善**: マイクが無音を検出した場合と「音声が検出されなかった」場合のエラーメッセージを区別できるよう改善しました

### プラグイン・コマンド

- **`/plugin` でのインストール済み表示誤りを修正**: プラグインが正しくインストール済みとして表示されない問題を修正しました
- **起動時の「not found in marketplace」エラー修正**: マーケットプレイスインストール後に自動更新することで、起動時に「not found in marketplace」エラーが表示される問題を修正しました
- **古い git バージョンでの `/security-review` 修正**: `unknown option merge-base` エラーで `/security-review` コマンドが失敗する問題を修正しました
- **`/color` コマンドのリセット機能追加**: デフォルトカラーに戻す方法がなかった問題を修正。`/color default`、`/color gray`、`/color reset`、`/color none` のいずれでもデフォルトカラーに戻せるようになりました

### その他のバグ修正

- **`AskUserQuestion` プレビューのパフォーマンスリグレッション修正**: ノート入力でキー入力のたびにマークダウンレンダリングが再実行されていた問題を修正しました
- **機能フラグのディスクキャッシュ更新修正**: 起動時に読み込まれた機能フラグがセッションをまたいでディスクキャッシュを更新しない問題を修正しました
- **Claude Code Remote での `permissions.defaultMode` 設定修正**: Remote 環境では `acceptEdits` と `plan` 以外の値は無視されるようになりました
- **`--resume` 時のスキル一覧再注入を修正**: `--resume` のたびにスキル一覧が再注入されていた問題を修正し、1回のレジューム当たり約600トークンを節約できるようになりました
- **VS Code テレポートセッションでのマーカー表示修正**: テレポートセッション内でテレポートマーカーが表示されない問題を修正しました

## 技術的なポイント

- **プロキシ・ゲートウェイ対応の強化**: `ANTHROPIC_BASE_URL` でのサードパーティプロキシ利用時に `tool_reference` ブロックが自動的に無効化される。カスタムエンドポイントを使う組織環境での安定性が向上します
- **MCP 統合の安定化**: MCP サーバー接続タイミングに起因するプロンプトキャッシュバストを修正。MCP を活用したワークフローのコストが削減されます
- **Windows/WSL のファーストクラスサポート**: クリップボード、音声、VS Code 統合に関する複数の Windows 固有バグを一括修正。Windows ユーザーにとって大きな改善です
- **Remote Control のスケーラビリティ改善**: ポーリングレートを300分の1にしたことで、多数のクライアントが接続する大規模利用シナリオでのサーバー負荷が劇的に改善されます
- **VS Code 統合の深化**: アクティビティバー統合・プランビュー・MCP 管理ダイアログにより、VS Code を主要な作業環境とするユーザーにとって Claude Code がよりネイティブなツールになります

## まとめ

v2.1.70 は多岐にわたるバグ修正とパフォーマンス改善が中心のリリースです。特に API エラーへの対応強化、Windows/WSL 環境の安定化、そして VS Code 統合の大幅な機能追加が注目点です。プロキシ環境や Bedrock カスタムプロファイルを利用しているユーザー、Windows で Claude Code を使っているユーザーにとっては特に恩恵の大きいアップデートと言えます。VS Code 向けの新機能はターミナルを離れることなく Claude Code の主要機能を操作できる体験を提供しており、今後の IDE 統合の方向性を示しています。

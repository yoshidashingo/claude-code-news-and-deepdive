# Claude Code v2.1.69 リリース — 大規模なバグ修正とメモリ最適化、音声入力の多言語対応拡大

## 概要

Claude Code v2.1.69 は、多数のバグ修正とパフォーマンス改善を中心とした大規模リリースです。セキュリティ関連の重要な修正が複数含まれるほか、音声入力（Voice STT）が新たに10言語に対応し合計20言語をサポートします。また、長時間セッションにおけるメモリリークが複数箇所で修正され、安定性が大幅に向上しています。

## 新機能

### `/claude-api` スキルの追加

Claude API および Anthropic SDK を使ったアプリケーション開発を支援する `/claude-api` スキルが追加されました。Claude API を使ったコード実装や SDK の活用方法について、Claude Code 上で直接サポートを受けられます。

### Voice STT の多言語対応拡大（20言語に）

音声入力（Speech-to-Text）が新たに10言語に対応し、合計20言語をサポートするようになりました。追加された言語は以下の通りです。

- ロシア語、ポーランド語、トルコ語、オランダ語
- ウクライナ語、ギリシャ語、チェコ語、デンマーク語
- スウェーデン語、ノルウェー語

### Remote Control のセッション名カスタマイズ

`/remote-control` コマンドと `claude remote-control` に、オプションの名前引数が追加されました。

```bash
/remote-control My Project
claude remote-control --name "My Project"
```

これにより、`claude.ai/code` 上で表示されるセッションタイトルをカスタマイズできます。

### Bash モードの操作改善

- 空の bash プロンプト（`!`）で **Ctrl+U** を押すと bash モードを終了できるようになりました（従来は `Escape` と `Backspace` のみ対応）
- インタビュー形式の選択肢でテンキーが使用可能になりました（従来はQWERTY列の数字キーのみ対応）

### エフォートレベルの表示

ロゴおよびスピナーにエフォートレベル（例: "with low effort"）が表示されるようになり、現在のエフォート設定を一目で確認できます。

### `/reload-plugins` コマンド

プラグインの変更を有効にするために Claude Code を再起動する必要がなくなりました。`/reload-plugins` コマンドを実行するだけで、保留中のプラグイン変更を即座に反映できます。

### 新しい設定オプション

- **`sandbox.enableWeakerNetworkIsolation`**（macOS のみ）: `gh`、`gcloud`、`terraform` などの Go プログラムがカスタム MITM プロキシ使用時に TLS 証明書を検証できるようになります
- **`includeGitInstructions`** / **`CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS`** 環境変数: Claude のシステムプロンプトからビルトインのコミット・PR ワークフロー指示を削除できます
- **`oauth.authServerMetadataUrl`**: MCP サーバーが標準のディスカバリーに失敗した際のカスタム OAuth メタデータ探索 URL を指定可能
- **`pluginTrustMessage`**: 管理設定でプラグインの信頼警告に組織固有のコンテキストを追加可能

### フック・プラグイン機能の強化

- `InstructionsLoaded` フックイベントが追加され、`CLAUDE.md` や `.claude/rules/*.md` のロード時に処理をフックできます
- フックイベントに `agent_id`（サブエージェント）および `agent_type` フィールドが追加されました
- ステータスラインのフックコマンドに、`--worktree` セッション実行時のワークツリー情報（name、path、branch、元リポジトリディレクトリ）が含まれる `worktree` フィールドが追加されました
- プラグインソースタイプ `git-subdir` が追加され、git リポジトリ内のサブディレクトリをプラグインソースとして指定できます
- スキル内で自身のディレクトリを参照する `${CLAUDE_SKILL_DIR}` 変数が追加されました

### Team プランへのポリシー制限対応

Remote Control の制限などのポリシー制限取得が、従来の Enterprise ユーザーだけでなく Team プランの OAuth ユーザーにも対応しました。

### Sonnet 4.5 から Sonnet 4.6 への自動移行

Pro/Max/Team Premium の Sonnet 4.5 ユーザーは Sonnet 4.6 へ自動的に移行されます。

## 改善点

### パフォーマンス改善

- スピナーのアニメーションループ（50ms）をシェルから分離し、ターン中のレンダリングおよび CPU オーバーヘッドを削減
- React Compiler によるネイティブバイナリの UI レンダリングパフォーマンス向上
- `--worktree` 起動時の git サブプロセスを排除し、起動速度を改善
- macOS 起動時の冗長な設定ファイル再読み込みを排除
- MCP の `-p` 起動時に設定フェッチとローカル接続をパイプライン化し、並列処理を改善
- ファイルの存在確認でファイル内容の読み込みを回避（6箇所）し、ファイル操作のパフォーマンスを改善

### MCP バイナリコンテンツハンドリングの改善

MCP ツールが PDF、Office ドキュメント、音声ファイルを返す場合、生の base64 データを会話コンテキストに投入するのではなく、正しいファイル拡張子でディスクに保存するようになりました。`WebFetch` も同様にバイナリレスポンスを要約とともに保存します。

### メモリ使用量の削減

- Yoga WASM のプリロードを遅延化することでベースラインメモリを約16MB削減
- SDK および CCR セッションでの stream-json 出力によるメモリ使用量削減
- 大規模セッションの再開時のメモリ使用量を削減
- マルチエージェントタスクにおいてサブエージェントの最終レポートを簡潔化し、トークン使用量を削減

### その他の改善

- `/resume` ピッカーが最初のプロンプトではなく最新のプロンプトを表示するように変更
- `claude.ai` MCP コネクターの接続失敗時にサイレントにツールリストから消えるのではなく、通知が表示されるようになりました
- サンプルコマンドの提案が Haiku を呼び出す代わりに決定論的に生成されるようになりました
- コンパクション後の再開時に冒頭の要約が表示されなくなりました

## バグ修正

### セキュリティ関連の修正

- **ネストされたスキル探索の脆弱性修正**: `node_modules` などの `.gitignore` 対象ディレクトリからスキルが読み込まれる問題を修正
- **MCP サーバーの信頼ダイアログの修正**: 初回実行時にすべての `.mcp.json` サーバーが暗黙的に有効化されていた問題を修正。サーバーごとの承認ダイアログが正しく表示されるようになりました
- **シンボリックリンクバイパスの修正**: `acceptEdits` モードで、シンボリックリンクの親ディレクトリを経由してワーキングディレクトリ外にファイルを書き込める問題を修正
- **インタラクティブツールの自動許可の修正**: `AskUserQuestion` などのインタラクティブツールがスキルの `allowed-tools` にリストされている場合、権限プロンプトをバイパスして空の回答で自動実行されていた問題を修正
- **サンドボックスの非許可ドメインブロックの修正**: `allowManagedDomainsOnly` が有効な場合、非許可ドメインがユーザーに承認を求めるのではなく自動的にブロックされるようになりました

### 認証・モデル関連

- `--model claude-opus-4-0` および `--model claude-opus-4-1` が非推奨バージョンに解決されていた問題を修正
- トークンリフレッシュ時のプロファイルエンドポイント一時的な失敗で `.credentials.json` から `subscriptionType` が失われていた問題を修正（[#30185](https://github.com/anthropics/claude-code/issues/30185)）
- 複数の OAuth MCP サーバー使用時に macOS キーチェーンが破損していた問題を修正（`security -i` の stdin バッファオーバーフロー）

### 安定性・クラッシュ修正

- `claude remote-control` が npm インストール時に "bad option: --sdk-url" でクラッシュする問題を修正（[#28334](https://github.com/anthropics/claude-code/issues/28334)）
- タイムスタンプが不正な場合に `/stats` がクラッシュする問題を修正
- `--mcp-config` が破損ファイルを指す場合のハング修正
- Remote Control セッションでローカルスラッシュコマンド（`/voice`、`/cost`）実行時の Android アプリクラッシュを修正

### メモリリーク修正

- SDK/CCR セッションで会話メッセージが不要に保持されていたメモリリークを修正
- REPL レンダースコープが長いセッションで蓄積していたメモリリーク（約35MB / 1000ターン）を修正
- インプロセスのチームメートで親の全会話履歴がチームメートのライフタイム中ピン留めされていたメモリリークを修正
- インタラクティブモードでフックイベントが無制限に蓄積していたメモリリークを修正
- React Compiler の `memoCache` でメモリリークが発生していた問題を修正

### ファイル・ワークツリー関連

- Linux のサンドボックス Bash コマンド後にゴーストドットファイル（`.bashrc`、`HEAD` など）が未追跡ファイルとして表示される問題を修正
- Windows でのワークツリーファイルコピーを修正
- Windows でのグローバル `.claude` フォルダ検出を修正
- ネストされたワークツリーから実行した場合（`claude -w`）に CLAUDE.md、スラッシュコマンド、エージェントが重複していた問題を修正
- print モード（`claude -p`）で条件付き `.claude/rules/*.md` ファイルとネストされた CLAUDE.md が読み込まれない問題を修正

### UI・入力関連

- Ghostty over SSH で Shift+Enter が改行の代わりに `[27;2;13~` を出力していた問題を修正
- Claude が作業中にメッセージを送信すると Ctrl+S（スタッシュ）がクリアされていた問題を修正
- プランモードのフィードバック入力で複数行テキスト入力がサポートされていなかった問題を修正（バックスラッシュ+Enter および Shift+Enter で改行挿入が可能に）
- 入力ボックス上部の空白行にカーソルが移動しない問題を修正
- ドラフトテキストがある状態で Escape がターンを中断しない問題を修正（上矢印キーでキューに積まれたメッセージを引き戻せます）
- `ctrl+o`（トランスクリプトトグル）が長いセッションで数秒間フリーズする問題を修正
- ストリーミングエラー後の短いハングを修正
- 大量の未追跡バイナリファイルがある状態でコミットするとメモリが数GB単位で急増する問題を修正

### エージェント・ツール関連

- Agent ツールの `name` パラメータ経由でチームメートが誤ってネストされたチームメートを生成する問題を修正
- コンパクション中に `CLAUDE_CODE_MAX_OUTPUT_TOKENS` が無視されていた問題を修正
- SDK コンシューマー（Claude Code Remote Web UI、VSCode 拡張機能）で `/compact` の要約がユーザーバブルとして表示されていた問題を修正
- `TeammateIdle` と `TaskCompleted` フックが `{"continue": false, "stopReason": "..."}` による停止をサポートするように修正
- `WorktreeCreate` と `WorktreeRemove` プラグインフックがサイレントに無視されていた問題を修正

## VSCode 拡張機能の変更

- コンパクション結果が折りたたみ可能な "Compacted chat" カードとして表示されるようになりました
- 権限モードピッカーが管理設定の `permissions.disableBypassPermissionsMode` を尊重するようになりました
- チャットパネルで RTL テキスト（アラビア語、ヘブライ語、ペルシャ語）が逆方向に表示されていたリグレッション（v2.1.63 で発生）を修正

## 技術的なポイント

- **セキュリティ修正が多数含まれる重要リリース**: シンボリックリンクバイパス、スキルの gitignore 回避、MCP サーバーの信頼ダイアログのバイパスなど、セキュリティ上重要な修正が複数含まれています。早期のアップデートを推奨します
- **メモリ管理の大幅改善**: 長時間セッションにおける複数のメモリリークが修正され、REPL スコープで約35MB/1000ターンのメモリ削減、ベースラインで約16MB の削減が実現しました
- **`includeGitInstructions` 設定**: システムプロンプトに含まれるデフォルトの Git ワークフロー指示を無効化でき、カスタムの Git ワークフローを採用しているチームに有用です
- **`${CLAUDE_SKILL_DIR}` 変数**: スキルが自身のディレクトリを参照できるようになり、より可搬性の高いスキル開発が可能になりました
- **MCP バイナリコンテンツの改善**: PDF や Office ドキュメントなどのバイナリファイルが base64 として会話に埋め込まれなくなり、コンテキストウィンドウの節約とパフォーマンス向上につながります
- **Sonnet 4.5 から Sonnet 4.6 への自動移行**: Pro/Max/Team Premium ユーザーは自動的に最新モデルへ移行されます

## まとめ

v2.1.69 は機能追加よりもバグ修正と安定性向上に重点を置いたリリースです。特にセキュリティ修正（シンボリックリンクバイパス、スキル探索の脆弱性など）、長時間セッションにおける複数のメモリリーク修正、そして音声入力の20言語対応が主なハイライトです。

日常的に Claude Code を利用しているユーザーにとっては、メモリ使用量の改善やパフォーマンス向上により、長時間のセッションがより快適になることが期待されます。また、プラグイン・スキル開発者にとっても、`${CLAUDE_SKILL_DIR}` 変数や `InstructionsLoaded` フック、`git-subdir` プラグインソースなど、実用的な新機能が追加されています。セキュリティ修正を含む重要なリリースであるため、早期のアップデートを推奨します。

# Claude Code v2.1.73 リリース — modelOverrides 設定追加、多数のバグ修正と安定性向上

## 概要

Claude Code v2.1.73 では、カスタムプロバイダーモデル ID へのマッピングを可能にする `modelOverrides` 設定の追加と、SSL 証明書エラー時の改善されたガイダンスという 2 つの新機能が導入されました。また、フリーズや CPU ループ、デッドロック、セッション管理など多岐にわたる 16 件以上のバグ修正が含まれており、全体的な安定性が大幅に向上しています。さらに Bedrock・Vertex・Microsoft Foundry におけるデフォルト Opus モデルが Opus 4.6 に更新されました。

## 新機能

### `modelOverrides` 設定によるカスタムモデル ID のマッピング

モデルピッカーのエントリをカスタムプロバイダーのモデル ID にマッピングできる `modelOverrides` 設定が追加されました。これにより、AWS Bedrock の推論プロファイル ARN（Amazon Resource Name）などを指定して、特定のモデルバリアントを利用できるようになります。

例えば、Bedrock 上で特定の推論プロファイルを使用したい場合、設定ファイルに以下のように記述することで対応できます。

```json
{
  "modelOverrides": {
    "claude-opus-4-6": "arn:aws:bedrock:us-east-1::foundation-model/anthropic.claude-opus-4-6"
  }
}
```

企業環境や特殊なデプロイ構成で独自のモデル設定を持つチームにとって、柔軟性が大きく向上します。

### SSL 証明書エラー時のアクション可能なガイダンス

企業プロキシ環境などで SSL 証明書エラーが原因で OAuth ログインや接続チェックが失敗した際に、具体的な解決策を示すガイダンスが表示されるようになりました。特に `NODE_EXTRA_CA_CERTS` 環境変数の設定方法など、実用的な対処法が提示されます。

企業ネットワークで Claude Code を利用する際に発生しがちな「証明書エラー」問題に対し、ユーザーが自力で解決しやすくなります。

## 改善点

### Up キーによる割り込み後のプロンプト復元

Claude の応答を中断した後に上矢印キーを押すと、中断したプロンプトが復元されるとともに会話の巻き戻しが一度の操作で行われるようになりました。これにより、修正して再実行するワークフローがよりスムーズになります。

### `/effort` コマンドのリアルタイム動作

`/effort` コマンドが Claude の応答中にも機能するようになり、`/model` コマンドと同じ挙動に統一されました。

### その他の改善

- **IDE 検出の高速化**: 起動時の IDE 検出処理が高速化されました
- **macOS クリップボード画像貼り付けのパフォーマンス向上**: macOS での画像貼り付けが高速になりました
- **ボイスモードの安定性向上**: プッシュトゥトーク操作を素早く繰り返した際の一時的な接続障害を自動リトライするようになりました
- **Remote Control のスポーンモード選択プロンプト改善**: より詳細なコンテキストが表示されるようになりました
- **Bedrock・Vertex・Microsoft Foundry のデフォルト Opus モデルを Opus 4.6 に変更**: 従来の Opus 4.1 から最新の Opus 4.6 へ更新されました

## バグ修正

### 安定性・パフォーマンス関連

- **複雑な bash コマンドによるフリーズと 100% CPU ループの修正**: 権限プロンプトが複雑な bash コマンドをトリガーした際に発生するフリーズと CPU 使用率の問題を修正しました
- **スキルファイル変更時のデッドロック修正**: `.claude/skills/` ディレクトリに多数のスキルファイルがあるリポジトリで `git pull` を実行した際に発生していたデッドロックを修正しました
- **複数セッションでの Bash ツール出力消失の修正**: 同一プロジェクトディレクトリで複数の Claude Code セッションを実行した際に Bash ツールの出力が失われる問題を修正しました

### Bedrock・Vertex・Foundry 関連

- **サブエージェントのモデルダウングレード問題の修正**: `model: opus`/`sonnet`/`haiku` を指定したサブエージェントが Bedrock・Vertex・Microsoft Foundry 上で古いモデルバージョンにサイレントダウングレードされていた問題を修正しました

### セッション・フック関連

- **`/resume` で現在のセッションが表示される問題の修正**: セッションピッカーに現在のセッションが表示されてしまう問題を修正しました
- **SessionStart フックの二重発火修正**: `--resume` または `--continue` でセッションを再開した際に SessionStart フックが 2 回発火していた問題を修正しました
- **JSON 出力フックによる不要なシステムメッセージ注入の修正**: JSON 出力フックが毎ターンにモデルコンテキストへ no-op の system-reminder メッセージを注入していた問題を修正しました
- **バックグラウンド bash プロセスのクリーンアップ修正**: サブエージェントが起動したバックグラウンド bash プロセスがエージェント終了時にクリーンアップされない問題を修正しました

### コマンド・機能関連

- **`/ide` の `onInstall is not defined` クラッシュ修正**: 拡張機能の自動インストール時に発生していたクラッシュを修正しました
- **`/loop` の利用不可問題の修正**: Bedrock・Vertex・Foundry 環境やテレメトリ無効時に `/loop` が使用できなかった問題を修正しました
- **ボイスモードのセッション破損修正**: 低速な接続環境で新しい録音と重なった際にセッションが破損する問題を修正しました
- **`/heapdump` の Windows `EEXIST` エラー修正**: Desktop フォルダが既に存在する場合に発生していたエラーを修正しました

### プラットフォーム・環境関連

- **Linux サンドボックスの "ripgrep not found" エラー修正**: ネイティブビルドで ripgrep (rg) が見つからないとサンドボックスが起動しない問題を修正しました
- **Amazon Linux 2 および glibc 2.26 環境でのネイティブモジュール読み込み修正**: glibc 2.26 系のシステムでネイティブモジュールが読み込めなかった問題を修正しました
- **Remote Control 経由の画像受信時の API エラー修正**: `media_type: Field required` エラーが発生していた問題を修正しました
- **VSCode プロキシ・Bedrock/Vertex 環境での HTTP 400 エラー修正**: Claude 4.5 モデルを使用するプロキシ環境または Bedrock/Vertex 上の VSCode ユーザーで発生していた HTTP 400 エラーを修正しました

## 非推奨の変更

### `/output-style` コマンドの非推奨化

`/output-style` コマンドが非推奨となり、代わりに `/config` を使用することが推奨されます。出力スタイルはプロンプトキャッシングの効率を高めるため、セッション開始時に固定されるようになりました。

既存のワークフローで `/output-style` を使用している場合は、`/config` への移行を検討してください。

## 技術的なポイント

- **`modelOverrides` 設定**: Bedrock の推論プロファイルや Vertex のカスタムエンドポイントなど、プロバイダー固有のモデル ID を柔軟に指定できるようになりました。マルチプロバイダー環境での運用が大幅に改善されます
- **スキルファイルのデッドロック修正**: 大規模リポジトリで `.claude/skills/` ディレクトリを運用しているチームは、`git pull` 時のフリーズ問題が解消されます
- **サブエージェントのモデル指定**: `model: opus`/`sonnet`/`haiku` の短縮指定がクラウドプロバイダー上でも正しく最新モデルに解決されるようになりました
- **SessionStart フックの二重発火修正**: フック処理の冪等性に依存していたワークフローを見直す必要がなくなります
- **JSON 出力フックのノイズ排除**: フックが毎ターン不要なメッセージをコンテキストに追加していた問題が解消され、モデルの推論効率が改善されます
- **プロンプトキャッシング最適化**: `/output-style` をセッション固定にすることで、キャッシュヒット率の向上が期待できます
- **Linux ネイティブサポート強化**: Amazon Linux 2 を含む glibc 2.26 系の環境でもネイティブモジュールが動作するようになり、エンタープライズ Linux 環境での利用が広がります

## まとめ

v2.1.73 は新機能 2 件、改善 6 件、バグ修正 16 件以上を含む大規模なメンテナンスリリースです。特に複雑な bash コマンドによるフリーズや、スキルファイル変更時のデッドロックなど、実際の利用で遭遇しやすい安定性の問題が多数解消されています。

Bedrock・Vertex・Microsoft Foundry を利用しているユーザーにとっては、デフォルト Opus モデルの 4.6 への更新と、サブエージェントのモデル指定が正しく機能するようになった点が特に重要です。また `modelOverrides` 設定の追加により、企業環境でのカスタムデプロイ構成への対応がさらに柔軟になりました。

`/output-style` から `/config` への移行については、次のアップデートサイクルを見据えた計画的な対応を推奨します。

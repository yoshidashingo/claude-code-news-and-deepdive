# Claude Code v2.1.68: Opus 4.6 のデフォルトをミディアムエフォートに変更、ultrathink キーワードの復活

## 概要

Claude Code v2.1.68 では、Max および Team サブスクライバー向けに Opus 4.6 のデフォルトエフォートレベルがミディアムに変更されました。また、高負荷な思考を明示的に指示するための `ultrathink` キーワードが再導入され、さらに Opus 4 および Opus 4.1 が第一者 API から削除されました。

## 新機能

### `ultrathink` キーワードの復活

以前のバージョンで廃止されていた `ultrathink` キーワードが再導入されました。このキーワードをプロンプト内で使用すると、次のターンに限って**ハイエフォート（高努力）モード**が有効になります。

複雑なアーキテクチャ設計や難解なバグの調査など、特定のタスクでより徹底的な分析が必要な場合に活用できます。

```
ultrathink このコードのパフォーマンスボトルネックを詳細に分析してください
```

## 改善点

### Opus 4.6 のデフォルトエフォートをミディアムに変更

Max および Team サブスクライバー向けに、Opus 4.6 のデフォルトエフォートレベルが**ミディアム**に設定されました。

Anthropic によると、ミディアムエフォートはほとんどのタスクにおいてスピードと徹底性のバランスが取れた「スイートスポット」とのことです。タスクの性質に応じて、`/model` コマンドからいつでも変更可能です。

| エフォートレベル | 特徴 |
|---|---|
| Low | 高速、シンプルなタスク向け |
| **Medium（新デフォルト）** | スピードと徹底性のバランス |
| High | より詳細な分析（`ultrathink` で指定） |

## Breaking Changes

### Opus 4 および Opus 4.1 の削除

第一者 API（Anthropic の直接 API）から **Opus 4** および **Opus 4.1** が削除されました。これらのモデルをピン留めして使用していたユーザーは、自動的に **Opus 4.6** へ移行されます。

サードパーティ API（Amazon Bedrock、Google Vertex AI など）経由で利用しているユーザーへの影響については、それぞれのプラットフォームのドキュメントを確認してください。

## 技術的なポイント

- **エフォートレベルの柔軟な切り替え**: デフォルトのミディアムエフォートを基準としつつ、`ultrathink` キーワードでターン単位の高エフォートモードを指定できるため、コストとパフォーマンスのトレードオフを細かく制御できます
- **モデルの自動移行**: Opus 4 / 4.1 のピン留めユーザーは手動操作なしで Opus 4.6 へ移行されますが、モデルの挙動が変わる可能性があるため、重要なワークフローでの動作確認を推奨します
- **`/model` コマンドでの確認**: 現在使用中のモデルとエフォートレベルは `/model` コマンドで確認・変更できます

## まとめ

v2.1.68 は、Opus 4.6 の利用体験を最適化するリリースです。ミディアムエフォートをデフォルトにすることで日常的なタスクのレスポンス速度が向上し、`ultrathink` の復活により重要な場面での高精度な分析も維持されています。旧モデルからの自動移行についてはスムーズに行われますが、クリティカルなユースケースでは Opus 4.6 の動作を確認しておくとよいでしょう。

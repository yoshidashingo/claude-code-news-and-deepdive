# Claude Code v2.1.71 リリース — /loop コマンドやcronスケジューリング、多数のバグ修正

## 概要

Claude Code v2.1.71 がリリースされました。セッション内で繰り返し処理を自動化する `/loop` コマンドと cron スケジューリングツールが新たに追加され、音声入力のキーバインド変更も可能になりました。また、スタートアップのフリーズ、プラグイン管理、フォーク会話など多岐にわたるバグ修正も含まれています。

## 新機能

### `/loop` コマンドによる定期実行の自動化

新しく追加された `/loop` コマンドを使うと、プロンプトやスラッシュコマンドを一定間隔で繰り返し実行できます。

```
/loop 5m check the deploy
```

この例では、`check the deploy` というプロンプトを5分ごとに自動実行します。デプロイの監視やバックグラウンドでの定期チェックなど、継続的な作業を手動操作なしで行いたいユースケースに有効です。

### cron スケジューリングツール

セッション内での繰り返しプロンプトをサポートする cron スケジューリングツールが追加されました。`/loop` コマンドと連携し、より柔軟な定期実行の制御が可能です。

### 音声入力キーバインドのカスタマイズ（`voice:pushToTalk`）

`keybindings.json` に `voice:pushToTalk` キーバインドが追加され、音声アクティベーションキーを変更できるようになりました。デフォルトはスペースキーですが、`meta+k` のような修飾キー＋文字キーの組み合わせに変更することで、通常のタイピングへの干渉をゼロにできます。

```json
{
  "voice:pushToTalk": "meta+k"
}
```

## 改善点

### Bash 自動承認リストの拡充

以下のコマンドが Bash の自動承認リストに追加されました。

- `fmt`, `comm`, `cmp`, `numfmt`, `expr`, `test`, `printf`, `getconf`, `seq`, `tsort`, `pr`

これらのコマンドを使用する際に、都度承認プロンプトが表示されなくなります。テキスト処理や数値演算を伴うスクリプト処理がよりスムーズになります。

### スタートアップ時間の改善

ネイティブ画像プロセッサーのロードを初回使用時まで遅延させることで、起動時間が改善されました。

### ブリッジセッションの再接続速度向上

ラップトップがスリープから復帰した後のブリッジセッション再接続が、従来の最大10分から数秒以内に完了するよう改善されました。

### `/plugin uninstall` の挙動改善

プロジェクトスコープのプラグインをアンインストールする際、`.claude/settings.json` ではなく `.claude/settings.local.json` を変更するようになりました。これにより、アンインストール操作がチームメンバーの設定に影響を与えなくなります。

### `/debug` の機能拡張

`/debug` コマンドがセッション中にデバッグログのオン/オフをトグルできるようになりました。デバッグログはデフォルトでは書き出されなくなったため、必要なときだけ有効化できます。

### MCP サーバーの重複排除改善

プラグインが提供する MCP サーバーのうち、手動設定済みのサーバーと同一（コマンドまたは URL が同じ）のものは自動的にスキップされるようになりました。重複接続やツールセットの重複を防ぎ、`/plugin` メニューでスキップ内容を確認できます。

## バグ修正

### パフォーマンス・安定性

- **stdin フリーズの修正**: 長時間のセッションでキーストロークが処理されなくなる問題を修正しました。プロセスは稼働しているがキー入力を受け付けなくなるという症状が解消されます。
- **音声モード有効時のスタートアップフリーズ修正**: CoreAudio の初期化がメインスレッドをブロックすることによる5〜8秒の起動遅延を修正しました。システムのスリープ復帰後に発生していた問題です。
- **OAuth トークン一斉更新時の UI フリーズ修正**: 複数の claude.ai プロキシコネクターが期限切れ OAuth トークンを同時に更新する際に発生していた起動時 UI フリーズを修正しました。

### プラグイン・コネクター

- **複数インスタンス起動時のプラグイン消失を修正**: 複数の Claude Code インスタンスを実行すると、プラグインのインストール情報が失われる問題を修正しました。
- **OAuth トークン更新後のコネクター再接続失敗を修正**: claude.ai コネクターが OAuth トークン更新後に再接続できなくなる問題を修正しました。
- **MCP コネクター起動通知の修正**: org 設定のコネクター全てに通知が表示されていた問題を修正し、以前に接続していたコネクターのみに表示されるようになりました。
- **起動時の未認証コネクター通知ノイズを削除**: 未認証の org 登録済み claude.ai コネクターに関する不要な起動通知を削除しました。

### 会話・ファイル管理

- **`/fork` でのプランファイル共有問題を修正**: フォークされた会話が同一のプランファイルを共有してしまい、一方の編集が他方を上書きする問題を修正しました。
- **Read ツールの過大サイズ画像問題を修正**: 画像処理に失敗した際に過大サイズの画像がコンテキストに追加され、長い画像処理セッションで後続の操作が壊れる問題を修正しました。
- **バックグラウンドエージェント完了通知の修正**: 出力ファイルパスが通知に含まれていなかった問題を修正しました。コンテキスト圧縮後に親エージェントがエージェント結果を復元しやすくなります。

### コマンド・ワークフロー

- **heredoc コミットメッセージの誤権限プロンプトを修正**: heredoc を含む複合 bash コマンドで誤った権限プロンプトが表示されていた問題を修正しました。
- **`--print` の無限ハングを修正**: チームエージェントが設定されている場合に `--print` が永久にハングする問題を修正しました。終了ループが `in_process_teammate` タスクを待機しなくなりました。
- **`ToolSearch` 呼び出し後の余分な表示を修正**: `ToolSearch` を呼び出すたびに REPL に「❯ Tool loaded.」が表示されていた問題を修正しました。
- **Windows での mingw パス問題を修正**: モデルが mingw 形式のパスを使用した際に `cd <cwd> && git ...` の権限プロンプトが表示される問題を修正しました。
- **`/plugin marketplace add` の `@` 解析を修正**: `owner/repo@ref` 形式での `@` が正しく解析されず、以前は `#` のみが機能していた問題を修正しました。`strictKnownMarketplaces` 利用時の原因不明のエラーが解消されます。
- **`/permissions` タブの重複エントリを修正**: 同じディレクトリをスラッシュあり・なしで追加すると重複エントリが表示される問題を修正しました。
- **Chrome 拡張機能の自動検出スタック問題を修正**: ローカルに Chrome がない環境で実行した後、自動検出が「未インストール」で永久にスタックする問題を修正しました。
- **`/plugin marketplace update` のマージコンフリクトを修正**: マーケットプレイスがブランチ/タグ ref にピン止めされている際に `update` がマージコンフリクトで失敗する問題を修正しました。

## 技術的なポイント

- **`/loop` + cron によるセッション内自動化**: 定期的な監視タスクやデプロイ確認を自動化できる仕組みが整いました。エージェント的なワークフローに新たな選択肢が加わります。
- **音声入力の実用性向上**: `meta+k` 等の修飾キーコンビネーションをプッシュトゥトーク操作に割り当てることで、テキスト入力中に誤って音声が起動するリスクを排除できます。
- **プラグインの分離と安全性**: `.claude/settings.local.json` への変更分離により、プラグイン操作がチームの共有設定を汚染しないアーキテクチャが強化されました。
- **MCP サーバー管理の堅牢化**: 重複接続の自動スキップと可視化により、複雑なプラグイン環境でのデバッグが容易になります。
- **スタートアップ遅延の根本対処**: CoreAudio 初期化のブロッキング問題や画像プロセッサーの遅延ロードなど、起動時パフォーマンスに関わる複数の問題が修正されました。

## まとめ

v2.1.71 は、`/loop` コマンドと cron スケジューリングによるセッション内自動化という実用的な新機能を提供しつつ、スタートアップのフリーズ、プラグイン管理、フォーク会話のプランファイル競合など、日常的な利用で遭遇しやすい問題を幅広く修正したリリースです。特に複数インスタンスの同時起動やチーム開発環境での利用品質が向上しています。音声モードの使い勝手改善や Windows 環境での修正も含まれており、多様な利用環境に対応したアップデートとなっています。

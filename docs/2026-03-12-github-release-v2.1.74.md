# Claude Code v2.1.74 リリース — /context コマンド強化、メモリリーク修正、多数のバグ修正

## 概要

Claude Code v2.1.74 がリリースされました。`/context` コマンドに最適化提案機能が追加され、コンテキスト管理がより直感的になりました。また、Node.js/npm パスで発生していたメモリリークの修正をはじめ、MCP OAuth、プラグイン、LSP、音声モードなど幅広い領域の不具合が解消されています。VSCode 拡張機能の使い勝手の向上も含まれており、全体的な品質改善が図られたリリースです。

## 新機能

### `/context` コマンドへのアクション提案機能の追加

`/context` コマンドに、コンテキスト使用状況を分析して具体的な最適化ヒントを提示する機能が追加されました。以下のような問題を検出し、対処方法を提案します。

- **コンテキストを大量消費しているツールの特定**: どのツールがコンテキストを圧迫しているかを可視化
- **メモリの肥大化 (memory bloat) の検出**: 不要なメモリ内容が蓄積している場合に警告
- **容量警告と最適化ヒント**: コンテキスト上限に近づいた際に具体的な対策を提示

長時間のセッションでコンテキストが逼迫してきた場合に、単に「コンテキストが多い」と表示されるだけでなく、**何をどう改善すればよいか**が分かるようになりました。

### `autoMemoryDirectory` 設定の追加

自動メモリ (auto-memory) の保存先ディレクトリをカスタマイズできる `autoMemoryDirectory` 設定が追加されました。デフォルトのディレクトリではなく、プロジェクト固有のパスや共有ストレージへの配置が可能になります。

```json
{
  "autoMemoryDirectory": "/path/to/custom/memory/dir"
}
```

## バグ修正

### メモリリークの修正 (Node.js/npm)

ストリーミング API レスポンスのバッファが、ジェネレーターが途中で終了した際に解放されないケースがありました。これにより Node.js/npm パスで RSS (Resident Set Size) が際限なく増加するメモリリークが発生していましたが、今回修正されました。長時間稼働するセッションや大規模なプロジェクトでの安定性が向上します。

### ポリシー・設定関連

- **managed policy の `ask` ルールがバイパスされる問題を修正**: ユーザーの `allow` ルールや skill の `allowed-tools` によって、managed policy で設定された `ask` ルールが無効化されてしまうセキュリティ上の問題が修正されました。
- **agent frontmatter / `--agents` JSON での完全なモデル ID が無視される問題を修正**: `claude-opus-4-5` のような完全なモデル ID を agent の `model:` フィールドや `--agents` JSON config に指定しても無視されていた問題が解消されました。`--model` と同じモデル値を agent でも使用できるようになります。

### MCP OAuth 関連

- **コールバックポートが使用中の場合に認証がハングする問題を修正**: OAuth 認証のコールバックに使用するポートがすでに占有されている場合、認証処理が停止したままになる問題が修正されました。
- **リフレッシュトークン失効後に再認証が促されない問題を修正**: Slack のように HTTP 200 でエラーを返す OAuth サーバーにおいて、リフレッシュトークンが失効しても再認証を求めるプロンプトが表示されない問題が修正されました。

### 音声・ターミナル関連

- **macOS 音声モードがマイク権限なしで無音で失敗する問題を修正**: ターミナルへのマイク許可が一度も付与されていない環境では、音声モードが何も表示せずに失敗していました。macOS ネイティブバイナリに `audio-input` entitlement が追加され、macOS が適切にマイク許可のダイアログを表示するようになりました。
- **Windows 環境でのRTL テキストの描画不具合を修正**: ヘブライ語、アラビア語などの右から左に書く (RTL: Right-to-Left) テキストが、Windows Terminal、conhost、および VS Code 統合ターミナルで正しく描画されない問題が修正されました。

### プラグイン・セッション関連

- **REPL 内での `/plugin install` が失敗する問題を修正**: ローカルソースを持つマーケットプレイスプラグインを REPL 内から `/plugin install` でインストールしようとすると失敗していた問題が修正されました。
- **マーケットプレイス更新で git submodule が同期されない問題を修正**: プラグインソースが git submodule に存在する場合、マーケットプレイスのアップデート後に submodule が同期されず壊れてしまう問題が解消されました。
- **不明なスラッシュコマンドの入力が無音で消える問題を修正**: 引数付きの不明なスラッシュコマンドを実行すると、入力内容が何も表示されずに消えていました。現在は入力内容が警告として表示されるようになりました。
- **`SessionEnd` フックが 1.5 秒でタイムアウトする問題を修正**: `hook.timeout` の設定に関わらず、`SessionEnd` フックが終了時に強制的に 1.5 秒でキルされていた問題が修正されました。新しく追加された環境変数 `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` でタイムアウトを設定できます。

```bash
# SessionEnd フックのタイムアウトをミリ秒で指定 (例: 10秒)
export CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS=10000
```

### LSP・開発ツール関連

- **Windows での LSP サーバーが動作しない問題を修正**: ファイル URI の形式が不正であったため、Windows 環境で LSP (Language Server Protocol) サーバーが正常に起動しない問題が修正されました。
- **`--plugin-dir` でローカル開発版プラグインが優先されるよう変更**: `--plugin-dir` で指定したローカル開発コピーが、同名のマーケットプレイスにインストール済みのプラグインを上書きするように変更されました。ただし managed settings で force-enable されているプラグインはこの対象外です。

### VSCode 拡張機能

- **Untitled セッションの削除ボタンが機能しない問題を修正**: VSCode 拡張機能において、タイトルのないセッション (Untitled sessions) の削除ボタンが機能しない問題が修正されました。
- **統合ターミナルのスクロールホイール応答性の向上**: ターミナルアクセラレーションを考慮した改善により、VSCode 統合ターミナルでのスクロールホイールの反応が向上しました。

## 技術的なポイント

- **メモリ管理の改善**: ストリーミングレスポンスのバッファ解放漏れは、長時間稼働するエージェントや大規模プロジェクトでの安定性に直接影響するため、重要な修正です。
- **セキュリティ強化**: managed policy の `ask` ルールがユーザー設定でバイパスされていた問題は、セキュリティポリシーの適用において重大な抜け穴でした。企業や組織での利用において特に重要です。
- **`SessionEnd` フックのタイムアウト設定**: `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` 環境変数により、クリーンアップ処理などに時間のかかるフックを安全に実行できるようになりました。
- **マルチプラットフォーム対応の強化**: Windows の LSP 修正、RTL テキストの描画修正、macOS の音声モード修正など、プラットフォーム固有の問題が複数解消されています。
- **agent のモデル指定の統一**: `--model` と同じ形式でフルモデル ID を agent にも指定できるようになり、設定の一貫性が向上しました。

## まとめ

v2.1.74 は新機能こそ 2 件と少ないものの、メモリリークやセキュリティ上の問題を含む重要なバグ修正が多数含まれています。特にメモリリークの修正とポリシーのバイパス問題の解消は、本番環境や組織での利用において重大な改善です。また、Windows・macOS・RTL 言語サポートなどプラットフォーム横断的な品質向上も注目されます。`/context` コマンドのアクション提案機能は、コンテキスト管理に悩む開発者にとって実用的な新機能となっています。

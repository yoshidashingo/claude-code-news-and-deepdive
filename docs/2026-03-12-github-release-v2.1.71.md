# Claude Code v2.1.71 リリース — /loop コマンド、cron スケジューリング、多数のバグ修正

## 概要

Claude Code v2.1.71 では、定期的なプロンプト実行を可能にする `/loop` コマンドと cron スケジューリング機能が新たに追加されました。また、長時間セッションでのキー入力フリーズや音声モードでの起動遅延など、開発体験を阻害していた複数の重大なバグが修正されています。プラグインや MCP コネクタ周りの安定性も大幅に向上しました。

## 新機能

### `/loop` コマンド — 定期実行の自動化

新しく追加された `/loop` コマンドを使うと、プロンプトやスラッシュコマンドを指定した間隔で繰り返し実行できます。

```
/loop 5m check the deploy
/loop 10m /my-custom-command
```

デプロイ監視やステータス確認など、定期的にチェックしたいタスクを自動化するのに便利です。間隔は `5m`（5分）のように時間単位で指定します。

### cron スケジューリングツール

セッション内で繰り返しプロンプトをスケジュール実行できる cron ツールが追加されました。`/loop` コマンドと連携しており、より柔軟な定期タスク管理が可能になっています。

### 音声アクティベーションキーのリバインド対応

`keybindings.json` に `voice:pushToTalk` キーバインドが追加され、音声アクティベーションキーをカスタマイズできるようになりました。デフォルトはスペースキーですが、`meta+k` のような修飾キー付きの組み合わせも設定可能です。通常のタイピングと干渉しないキーバインドを選べるため、音声モードをより快適に利用できます。

```json
{
  "voice:pushToTalk": "meta+k"
}
```

### bash 自動承認リストへのコマンド追加

以下のコマンドが bash の自動承認リストに追加され、許可プロンプトなしに実行できるようになりました。

- `fmt`, `comm`, `cmp`, `numfmt`, `expr`, `test`, `printf`, `getconf`, `seq`, `tsort`, `pr`

テキスト処理や数値演算など、一般的な Unix コマンドが揃いました。

## 改善点

### 起動時間の短縮

ネイティブ画像プロセッサの読み込みを初回使用時まで遅延させることで、起動時間が改善されました。画像を扱わないセッションでは起動が速くなります。

### ブリッジセッション再接続の高速化

ラップトップがスリープから復帰した後のブリッジセッション再接続が、従来の最大 10 分待機から数秒以内に完了するよう改善されました。

### `/plugin uninstall` の挙動改善

プロジェクトスコープのプラグインをアンインストールする際、`.claude/settings.json` ではなく `.claude/settings.local.json` に無効化の設定が書き込まれるようになりました。これにより、チームメンバーの設定に影響を与えることなく、個人の環境でプラグインを管理できます。

### プラグイン提供 MCP サーバーの重複排除

プラグインが提供する MCP サーバーが手動設定済みのサーバーと重複する場合（同じコマンドや URL）、自動的にスキップされるようになりました。重複の抑制状況は `/plugin` メニューで確認できます。

### `/debug` コマンドの改善

`/debug` コマンドでデバッグロギングをセッション途中からオン・オフ切り替えられるようになりました。デバッグログはデフォルトでは書き込まれなくなっており、必要なときだけ有効化する運用が可能です。

## バグ修正

### セッション・操作性関連

- **長時間セッションでのキー入力フリーズ修正**: stdin がフリーズしてキーストロークが処理されなくなるが、プロセスは生き続けるという問題を修正しました。
- **音声モードの起動フリーズ修正**: 音声モード有効時にシステムスリープ復帰後、CoreAudio の初期化がメインスレッドをブロックして 5〜8 秒の起動遅延が発生していた問題を修正しました。
- **複数 OAuth トークン更新による起動 UI フリーズ修正**: 複数の claude.ai プロキシコネクタが同時に期限切れ OAuth トークンを更新しようとした際の起動 UI フリーズを修正しました。

### プラン・フォーク関連

- **`/fork` でのプランファイル共有問題を修正**: フォークした会話が同じプランファイルを参照していたため、一方のフォークでのプラン編集がもう一方を上書きしてしまう問題を修正しました。

### ツール・画像処理関連

- **Read ツールの過大画像問題を修正**: 画像処理が失敗した際に、処理できない大きな画像がコンテキストに追加され、長い画像セッションの後続ターンを壊していた問題を修正しました。
- **heredoc コミットメッセージの誤許可プロンプト修正**: heredoc を含む複合 bash コマンドで誤って許可プロンプトが表示されていた問題を修正しました。

### プラグイン・コネクタ関連

- **複数インスタンス時のプラグイン消失を修正**: 複数の Claude Code インスタンスを実行した際にプラグインのインストールが失われる問題を修正しました。
- **OAuth トークン更新後の claude.ai コネクタ再接続失敗を修正**: トークン更新後にコネクタが再接続できなくなる問題を修正しました。
- **MCP コネクタ起動通知の重複を修正**: 以前に接続していたコネクタのみに通知が表示されるよう修正しました（全組織設定コネクタに通知が出ていた問題）。
- **`/plugin marketplace update` のマージコンフリクト失敗を修正**: マーケットプレイスがブランチ/タグ ref にピン止めされている場合のマージコンフリクトエラーを修正しました。
- **`/plugin marketplace add owner/repo@ref` の `@` パース修正**: `@` が ref セパレータとして正しく認識されず、`#` しか機能していなかった問題を修正しました。

### その他

- **バックグラウンドエージェントの完了通知に出力ファイルパスを追加**: 親エージェントがコンテキスト圧縮後にエージェントの結果を復元しやすくなりました。
- **Bash ツールエラーメッセージの重複出力を修正**: コマンドが非ゼロ終了した際にエラーメッセージが重複していた問題を修正しました。
- **Chrome 拡張自動検出の永続的スタックを修正**: Chrome がインストールされていない環境で実行後、「未インストール」状態から抜け出せなくなる問題を修正しました。
- **`/permissions` Workspace タブの重複エントリを修正**: 末尾スラッシュの有無が異なる同一ディレクトリが重複登録される問題を修正しました。
- **チームエージェント設定時の `--print` ハングを修正**: 長期実行される `in_process_teammate` タスクを待ち続けて終了できなくなる問題を修正しました。
- **`ToolSearch` 後の "Tool loaded." 表示を修正**: `ToolSearch` を呼び出すたびに REPL に "❯ Tool loaded." と表示されていた問題を修正しました。
- **Windows での mingw スタイルパスによる `cd && git` プロンプトを修正**: モデルが mingw 形式のパスを使用した際に不要な許可プロンプトが表示されていた問題を修正しました。

## 技術的なポイント

- **`/loop` コマンドは Skill として実装**: `keybindings-help` や `simplify` と同様のスキルシステムを利用しており、`/loop 5m /foo` のように他のスラッシュコマンドをネストして呼び出せます。
- **`voice:pushToTalk` のキーバインド設定は `~/.claude/keybindings.json` で管理**: `meta+k` などの修飾キー付きバインドを設定することで、テキスト入力中に誤って音声モードが起動するタイピング干渉ゼロを実現できます。
- **プラグインスコープの分離強化**: `settings.local.json` を活用した個人設定とチーム設定の分離は、チーム開発における設定管理のベストプラクティスに沿っています。
- **MCP サーバー重複排除ロジック**: 同一コマンド・URL によるサーバー重複を自動検出し、接続の無駄を省く実装が追加されました。
- **CoreAudio ブロッキング問題の解決**: macOS 特有の CoreAudio 初期化をメインスレッド外に移すことで、音声モードの起動遅延が解消されました。

## まとめ

v2.1.71 は、定期実行自動化という実用的な新機能と、長期間にわたって開発者を悩ませてきたフリーズ・ハング問題の大規模な修正が特徴のリリースです。特に `/loop` コマンドはデプロイ監視や定期チェックを Claude Code 上で完結させる可能性を広げます。プラグインや MCP コネクタ周りの安定性向上も、チームでの利用環境を整える上で重要な改善です。起動時間や再接続速度の改善も加わり、全体的な開発体験が向上しています。

# Claude Code v2.1.70 リリース — Windows/WSL対応強化・VS Code機能拡張・パフォーマンス改善

## 概要

Claude Code v2.1.70 は、Windows/WSL 環境でのバグ修正を多数含む品質改善リリースです。API 連携・音声モード・クリップボード処理など幅広い問題が修正され、VS Code 向けには新しいセッション管理 UI や MCP サーバー管理ダイアログが追加されました。パフォーマンス面でも起動メモリ削減やプロンプト再レンダリングの大幅削減が実現されています。

## 新機能

### [VSCode] セッションマネージャーの追加

VS Code のアクティビティバーに Spark アイコンが追加され、現在起動中のすべての Claude Code セッションを一覧表示できるようになりました。各セッションをフルエディタとして開くことが可能で、複数セッションの管理が格段に容易になります。

### [VSCode] プランのマークダウンドキュメント表示

VS Code 内でプランの全文をマークダウンドキュメントとして表示できるようになりました。コメントを追加してフィードバックを提供する機能も備わっており、プランのレビューと修正がよりインタラクティブに行えます。

### [VSCode] ネイティブ MCP サーバー管理ダイアログ

チャットパネルで `/mcp` コマンドを使用すると、MCP（Model Context Protocol）サーバーを管理するネイティブダイアログが開くようになりました。ターミナルを切り替えることなく、サーバーの有効/無効切り替え・再接続・OAuth 認証の管理が可能です。

## 改善点

### コンパクションでの画像保持

コンパクション（会話履歴の圧縮）時にサマライザーリクエストへ画像を保持するよう改善されました。これによりプロンプトキャッシュの再利用が可能になり、コンパクションをより高速かつ低コストで実行できます。

### `/rename` コマンドの即時実行

これまで Claude が処理中の場合は `/rename` コマンドが暗黙的にキューに積まれていましたが、処理中でも即座に実行されるようになりました。

### Remote Control のポーリングレート最適化

Remote Control の `/poll` レートが接続中は 1〜2 秒から 10 分に 1 回に削減されました。サーバー負荷が約 300 倍軽減される一方、トランスポート切断時は即座に高速ポーリングへ復帰するため、再接続速度には影響しません。

### マイク無音時のエラーメッセージ改善

マイクが無音をキャプチャした場合のエラーメッセージが改善され、「音声が検出されなかった」場合と明確に区別できるようになりました。

## バグ修正

### API・モデル関連

- **`ANTHROPIC_BASE_URL` 使用時の API 400 エラー**: サードパーティゲートウェイ経由で API を使用する際、`tool_reference` ブロックがプロキシエンドポイントで 400 エラーを引き起こしていた問題を修正。ツール検索がプロキシエンドポイントを正しく検出し、`tool_reference` ブロックを無効化するようになりました。

- **カスタム Bedrock 推論プロファイルでの `effort` パラメータエラー**: 標準の Claude 命名パターンに一致しないモデル識別子（カスタム Bedrock 推論プロファイルなど）を使用した際に `API Error: 400 This model does not support the effort parameter` が発生していた問題を修正。

- **`ToolSearch` 直後のモデル空レスポンス**: サーバーがツールスキーマをシステムプロンプト風のタグとしてプロンプト末尾に展開していたため、モデルが早期に停止することがありました。この問題が修正されました。

- **MCP サーバー接続時のプロンプトキャッシュバスト**: `instructions` を持つ MCP サーバーが最初のターン後に接続した際にプロンプトキャッシュが無効化されていた問題を修正。

### UI・入力関連

- **低速 SSH 接続での Enter キー動作**: 遅い SSH 接続環境で Enter キーを押すと送信ではなく改行が挿入されていた問題を修正。

- **Windows/WSL でのクリップボード非 ASCII 文字破損**: CJK 文字や絵文字などの非 ASCII テキストがクリップボードにコピーする際に破損していた問題を修正。PowerShell の `Set-Clipboard` を使用することで解決されました。

- **Windows 起動時の VS Code ウィンドウ多重起動**: VS Code 統合ターミナルから起動した際に余分な VS Code ウィンドウが開いていた問題を修正。

- **`#NNN` を含むマークダウンリンクの誤参照**: マークダウンリンクに `#NNN`（issue/PR 番号）が含まれる場合、リンク先の URL ではなく現在のリポジトリを指してしまっていた問題を修正。

### 音声モード関連

- **Windows ネイティブバイナリでの音声モード起動失敗**: `native audio module could not be loaded` エラーで音声モードが使用できなかった問題を修正。

- **セッション開始時のプッシュトゥトーク無効**: 設定で `voiceEnabled: true` を指定していてもセッション開始時にプッシュトゥトークが有効化されていなかった問題を修正。

### プラグイン関連

- **プラグインのインストール状態誤表示**: `/plugin` コマンドでプラグインがインストール済みと不正確に表示されていた問題を修正。

- **起動直後の「マーケットプレイスに見つからない」エラー**: 新規インストール直後の起動時にプラグインが "not found in marketplace" エラーを表示していた問題を修正。マーケットプレイスインストール後に自動リフレッシュするようになりました。

### コマンド・設定関連

- **`/security-review` コマンドの `unknown option merge-base` エラー**: 古いバージョンの git を使用している環境で `/security-review` コマンドが失敗していた問題を修正。

- **`/color` コマンドのデフォルトリセット不可**: `/color` コマンドでデフォルトカラーに戻す方法がなかった問題を修正。`/color default`、`/color gray`、`/color reset`、`/color none` のいずれでもデフォルトに戻せるようになりました。

- **フィーチャーフラグキャッシュの古い値問題**: 早期起動時に読み込まれたフィーチャーフラグがディスクキャッシュを更新しないまま古い値が残り続けていた問題を修正。

- **Remote 環境での `permissions.defaultMode` 誤適用**: Claude Code Remote 環境で `acceptEdits` または `plan` 以外の `permissions.defaultMode` 値が適用されていた問題を修正。Remote 環境ではこれらの値は無視されるようになりました。

- **`--resume` 時のスキルリスト再注入**: `--resume` するたびにスキルリストが再注入されていた問題を修正。1 回の resume あたり約 600 トークンを節約できます。

- **VS Code テレポートセッションのマーカー未表示**: VS Code テレポートセッションでテレポートマーカーが描画されていなかった問題を修正。

- **Opus 4.6 モデル更新通知の繰り返し**: プロジェクトの `.claude/settings.json` にレガシーな Opus モデル文字列が固定されている場合に「Model updated to Opus 4.6」通知が繰り返し表示されていた問題を修正。

## 技術的なポイント

- **プロキシ環境での `tool_reference` ブロック制御**: サードパーティゲートウェイを `ANTHROPIC_BASE_URL` で使用する場合、ツール検索がエンドポイントをプロキシと判定して `tool_reference` ブロックを自動無効化するようになりました。独自の API ゲートウェイを構築・運用している場合は、この挙動を把握しておくことが重要です。

- **プロンプト再レンダリングの約 74% 削減**: ターン中のプロンプト入力再レンダリング回数が大幅に減少し、UI のレスポンシブ性が向上しました。

- **起動メモリの約 426KB 削減**: カスタム CA 証明書を使用していないユーザー向けに、起動時のメモリ使用量が削減されました。

- **Remote Control の `/poll` レート大幅削減**: 接続中のポーリングを 1〜2 秒から 10 分ごとに変更することで、サーバー側の負荷が約 300 倍軽減されます。大規模な Remote Control 利用環境では特に顕著な効果が期待できます。

- **Windows/WSL クリップボード処理の改善**: 非 ASCII 文字（日本語・中国語・韓国語・絵文字など）のクリップボード操作に PowerShell の `Set-Clipboard` を採用。Windows 環境での多言語対応が強化されました。

- **MCP サーバー管理の UI 統合**: `/mcp` コマンドによるネイティブ管理ダイアログにより、ターミナルを離れることなく MCP サーバーの操作が可能になりました。OAuth 認証フローもチャットパネル内で完結します。

## まとめ

v2.1.70 は新機能よりもバグ修正・品質改善に重点を置いたリリースです。特に Windows/WSL 環境のユーザーには多くの修正が適用されており、クリップボードの非 ASCII 文字破損や音声モードの問題が解消されます。VS Code ユーザー向けには、セッション管理 UI や MCP サーバー管理ダイアログといった実用的な機能が追加されました。また、プロンプト再レンダリングの削減や Remote Control のポーリングレート最適化など、パフォーマンス面での改善も着実に積み重ねられています。日本語を含む多言語環境で Claude Code を利用している開発者は、クリップボード修正の恩恵を直接受けられる重要なアップデートです。

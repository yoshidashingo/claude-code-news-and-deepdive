# Claude Code v2.1.56: VS Code クラッシュ問題の追加修正

## 概要

Claude Code v2.1.56 は、VS Code 拡張機能で発生していた `claude-vscode.editor.openLast` コマンドが見つからないというクラッシュの別の原因を修正したパッチリリースです。同様の問題に対する修正が以前のバージョンでも行われていましたが、今回は別の発生経路が特定され対処されました。

## バグ修正

### VS Code: "command 'claude-vscode.editor.openLast' not found" クラッシュの追加修正

今回修正されたのは、VS Code 拡張機能において `claude-vscode.editor.openLast` コマンドが見つからないというエラーが発生し、Claude Code がクラッシュする問題です。

このエラーは、Claude Code の VS Code 拡張機能がコマンドの登録前に実行を試みるタイミングの問題（レースコンディション）や、拡張機能のライフサイクルの特定フェーズでコマンドが未登録の状態でアクセスされることに起因していると考えられます。前回のリリースで一部の発生パターンは修正されていましたが、別の経路で同様のクラッシュが引き続き報告されていたため、今回さらなる対応が行われました。

このクラッシュが発生するとユーザーは Claude Code を正常に利用できなくなるため、早急な修正が必要とされていました。リリース後には多数のユーザーから肯定的な反応（👍、❤️、🚀 などのリアクション合計 22 件）が寄せられており、多くのユーザーが影響を受けていた問題であることがうかがえます。

## 技術的なポイント

- **対象環境**: VS Code 拡張機能を使用して Claude Code を利用しているユーザーが対象
- **問題の性質**: コマンド登録タイミングに関するバグで、特定の条件下でのみ発生するため再現が難しいケースが多い
- **段階的な修正**: 同一の症状に対して複数のリリースをまたいで修正が行われており、根本原因の特定に継続的な取り組みが必要なケースであることを示している
- **影響範囲**: 新機能の追加や破壊的変更（Breaking Changes）はなく、既存の動作に影響を与えない純粋なバグ修正

## まとめ

v2.1.56 は VS Code 環境での安定性向上に特化したパッチリリースです。`claude-vscode.editor.openLast` コマンドに関連するクラッシュが引き続き発生していたユーザーは、このバージョンへのアップデートで問題が解消される可能性があります。

VS Code で Claude Code を使用中にクラッシュが発生していた場合は、以下のコマンドでアップデートしてください。

```bash
npm update -g @anthropic-ai/claude-code
```

今後も VS Code 拡張機能の安定性改善が継続されることが期待されます。

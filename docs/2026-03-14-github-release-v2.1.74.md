# Claude Code v2.1.74 リリース — /context コマンド強化とメモリリーク修正など多数のバグ修正

## 概要

Claude Code v2.1.74 では、`/context` コマンドへの実用的なサジェスト機能の追加と `autoMemoryDirectory` 設定の新規追加という 2 つの新機能が導入されました。また、Node.js/npm コードパスでのメモリリーク、MCP OAuth 認証のハング、managed policy のバイパス、RTL テキストのレンダリングなど、多岐にわたるバグ修正が含まれています。特にセキュリティ・安定性・クロスプラットフォーム対応の改善が今回の主な焦点です。

## 新機能

### `/context` コマンドへの実用的なサジェスト追加

`/context` コマンドがアップグレードされ、コンテキストの使用状況を分析して具体的な最適化アドバイスを提示するようになりました。

- コンテキストを多く消費しているツールの特定
- メモリの肥大化（memory bloat）の検出
- コンテキスト容量の警告と具体的な最適化 tips の提示

長いセッションでコンテキストウィンドウが圧迫されがちな場面で、何が原因でどう対処すれば良いかを即座に把握できるようになります。

### `autoMemoryDirectory` 設定の追加

auto-memory（自動メモリ）の保存先ディレクトリをカスタマイズできる新しい設定 `autoMemoryDirectory` が追加されました。複数プロジェクトを扱う環境や、メモリファイルを特定の場所で一元管理したい場合に有用です。

```json
{
  "autoMemoryDirectory": "/path/to/custom/memory/dir"
}
```

## バグ修正

### メモリ・パフォーマンス

**ストリーミング API レスポンスバッファのメモリリーク修正**

Node.js/npm コードパスで、ストリーミング API レスポンスのジェネレーターが途中で終了した際にバッファが解放されないメモリリークが発生していました。これにより RSS（Resident Set Size）が際限なく増加する問題が起きていましたが、今回のリリースで修正されました。長時間稼働するセッションでの安定性が向上します。

### 認証・セキュリティ

**managed policy の `ask` ルールがバイパスされる問題の修正**

ユーザーの `allow` ルールや skill の `allowed-tools` 設定によって、managed policy（管理ポリシー）の `ask` ルールが上書きされてしまう脆弱性が修正されました。組織のポリシー管理を行う環境では特に重要な修正です。

**MCP OAuth 認証のハング修正**

コールバックポートがすでに使用中の場合、MCP OAuth 認証がハングアップする問題が修正されました。

**MCP OAuth リフレッシュトークン期限切れ後の再認証プロンプト修正**

HTTP 200 でエラーを返す OAuth サーバー（Slack など）において、リフレッシュトークンが期限切れになっても再認証プロンプトが表示されない問題が修正されました。

### モデル設定

**エージェントのフロントマターでフル model ID が無視される問題の修正**

エージェントのフロントマター `model:` フィールドや `--agents` JSON 設定で `claude-opus-4-5` などのフル model ID を指定しても、サイレントに無視されていた問題が修正されました。これらのフィールドで `--model` と同じ形式の model ID が使用できるようになりました。

### 音声・ターミナル

**macOS ネイティブバイナリでのボイスモード失敗の修正**

マイクの権限を一度も付与していなかったターミナルを使用している macOS ユーザーで、ボイスモードがサイレントに失敗する問題が修正されました。ネイティブバイナリに `audio-input` エンタイトルメントが追加され、macOS が適切に権限リクエストを表示するようになります。

**RTL テキスト（ヘブライ語・アラビア語など）のレンダリング修正**

Windows Terminal、conhost、VS Code 統合ターミナルで、ヘブライ語・アラビア語などの RTL（右から左）テキストが正しくレンダリングされない問題が修正されました。

**Windows での LSP サーバー動作不具合の修正**

ファイル URI が不正な形式になっていたことで、Windows 上で LSP（Language Server Protocol）サーバーが正常に動作しない問題が修正されました。

### フック・プラグイン

**`SessionEnd` フックのタイムアウト設定が無視される問題の修正**

`hook.timeout` 設定に関わらず、`SessionEnd` フックが終了時に 1.5 秒後に強制終了されていた問題が修正されました。環境変数 `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` でタイムアウトを設定できるようになりました。

```bash
export CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS=10000  # 10秒に設定する例
```

**REPL 内での `/plugin install` 失敗の修正**

ローカルソースを持つマーケットプレイスプラグインの REPL 内での `/plugin install` が失敗する問題が修正されました。

**マーケットプレイス更新時の git submodule 同期修正**

マーケットプレイスの更新で git submodule が同期されない問題が修正されました。submodule 内のプラグインソースが更新後に壊れることがなくなります。

### ユーザーインターフェース

**未知のスラッシュコマンドで引数が失われる問題の修正**

引数付きの未知のスラッシュコマンドを入力すると、入力内容がサイレントに破棄されていた問題が修正されました。現在は警告として入力内容が表示されるようになります。

**`--plugin-dir` のローカル開発コピーが優先されるよう変更**

`--plugin-dir` で指定したローカル開発コピーが、同名のインストール済みマーケットプレイスプラグインより優先されるように変更されました（managed settings で force-enabled なプラグインを除く）。プラグイン開発時のイテレーションが容易になります。

### VS Code 統合

- **Untitled セッションの削除ボタン修正**: Untitled セッションで削除ボタンが機能しない問題を修正
- **統合ターミナルのスクロール改善**: ターミナルに応じたアクセラレーションでスクロールホイールの応答性を向上

## 技術的なポイント

- **メモリリークの根本原因**: Node.js のジェネレーター早期終了時にストリーミングバッファが GC されない問題であり、長時間セッションの信頼性に直結する重要な修正
- **managed policy の優先順位**: ユーザー設定や skill 設定が組織ポリシーを上書きできないよう、ポリシー評価の順序が修正された。エンタープライズ利用での管理統制に影響
- **`CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS`**: 環境変数による SessionEnd フックタイムアウト制御が可能に。クリーンアップ処理が重い場合は適切な値を設定することを推奨
- **RTL テキスト対応**: Windows のターミナルエミュレーター複数環境でのレンダリング修正。国際化対応の向上
- **MCP OAuth の堅牢性向上**: ポート競合時のハングと HTTP 200 エラーレスポンス時のリフレッシュ失敗という 2 つのエッジケースを同時に対処

## まとめ

v2.1.74 は新機能の追加よりもバグ修正と安定性向上に重点を置いたリリースです。特にメモリリーク修正は長時間稼働する Node.js 環境で重要な改善であり、managed policy のバイパス修正はエンタープライズ利用においてセキュリティ上の重要な修正となります。また、`/context` コマンドの強化によりコンテキスト管理が対話的かつ実用的になり、Windows・macOS ともにプラットフォーム固有の問題が複数解消されています。プラグイン開発者にとっても `--plugin-dir` の優先順位変更は開発フローの改善につながるでしょう。

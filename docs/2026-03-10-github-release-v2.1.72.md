# Claude Code v2.1.72 リリース — 大規模バグ修正とUX改善、SDK トークンコスト最大12倍削減

## 概要

Claude Code v2.1.72 は 2026年3月10日にリリースされました。SSH 環境向けのファイル書き込み機能や `/plan` コマンドの強化など実用的な新機能が追加されたほか、bash コマンドパーサーのネイティブモジュール化による高速化、SDK の `query()` 呼び出しにおけるプロンプトキャッシュ修正によるトークンコスト最大 12 倍削減が目玉です。また、WorKtree、プラグイン、パーミッション判定など多岐にわたる不具合が修正されています。

## 新機能

### `/copy` コマンドへの `w` キー追加（SSH 環境対応）

`/copy` コマンドでフォーカス中の選択内容をクリップボードではなく直接ファイルへ書き込めるようになりました。SSH 接続環境ではクリップボードが使えないケースが多く、この機能により遠隔作業時のデータ転送が容易になります。

### `/plan` コマンドへの説明引数追加

プランモードへの移行と同時に作業内容を指定できるようになりました。

```
/plan fix the auth bug
```

従来はプランモードに入ってから内容を入力する必要がありましたが、1コマンドで即座にプランニングを開始できます。

### `ExitWorktree` ツールの追加

`EnterWorktree` で開始した worktree セッションを終了するための専用ツール `ExitWorktree` が追加されました。これにより worktree の開始・終了が対称的な操作で行えます。

### `CLAUDE_CODE_DISABLE_CRON` 環境変数

セッション中にスケジュールされた cron ジョブを即時停止できる環境変数が追加されました。長時間セッションで定期タスクを制御したい場合に役立ちます。

### `claude plugins` エイリアス

`claude plugin` コマンドの複数形エイリアス `claude plugins` が追加されました。コマンドを自然な英語として入力できるようになります。

### VSCode URI ハンドラー

`vscode://anthropic.claude-code/open` という URI ハンドラーが追加され、外部からプログラム的に Claude Code タブを開けるようになりました。`prompt` および `session` クエリパラメーターをオプションで指定できます。

```
vscode://anthropic.claude-code/open?prompt=fix+the+tests&session=abc123
```

## 改善点

### bash 自動承認許可リストの拡張

`lsof`、`pgrep`、`tput`、`ss`、`fd`、`fdfind` が読み取り専用コマンドとして自動承認リストに追加されました。これらは一般的な調査・診断コマンドであり、パーミッションプロンプトの表示頻度が削減されます。

### bash コマンドパーサーのネイティブモジュール化

bash コマンドのパースをネイティブモジュールに切り替えました。初期化速度の向上とメモリリークの解消が期待できます。さらに tree-sitter による構文解析が改善され、`find -exec`、変数代入、コマンド置換などのパターンで発生していた誤検知のパーミッションプロンプトが削減されました。

### エフォートレベルの簡略化

エフォートレベルが `low / medium / high / max` の4段階から `low / medium / high` の3段階に整理されました。表示シンボルも `○ ◐ ●` に統一され、永続的なアイコン表示から簡潔な通知表示に変更されています。デフォルトに戻すには `/effort auto` を使用します。

### `/config` UI の改善

- **Escape** キーで変更をキャンセル
- **Enter** キーで保存して閉じる
- **Space** キーでトグル設定を切り替え

直感的なキーバインドにより設定画面の操作性が向上しました。

### Agent ツールへの `model` パラメーター復元

Agent ツールで呼び出しごとにモデルを指定できる `model` パラメーターが復元されました。用途に応じて異なるモデルを使い分けるワークフローが再び可能になります。

### サードパーティプロキシゲートのバイパス

環境変数が設定されている場合、ツール検索がサードパーティプロキシゲートをバイパスするよう変更されました。なお `CLAUDE_CODE_PROXY_SUPPORTS_TOOL_REFERENCE` は廃止されています（後述の Breaking Changes 参照）。

### その他の改善

- マーケットプレイスの git URL で `.git` サフィックスなしのURL（Azure DevOps、AWS CodeCommit）に対応
- 複数の並行セッション実行時の上矢印履歴が現在のセッションのメッセージを優先表示
- ボイス入力でリポジトリ名や開発用語（regex、OAuth、JSON など）の転写精度が向上
- マーケットプレイスのクローン失敗時のエラーメッセージが詳細化
- バンドルサイズを約 510 KB 削減
- 長時間セッションでの CPU 使用率を改善

## バグ修正

### SDK の prompt cache 修正（トークンコスト最大12倍削減）

SDK の `query()` 呼び出しでプロンプトキャッシュが正しく機能していなかった問題を修正しました。この修正により入力トークンのコストが最大 12 倍削減される可能性があります。SDK を利用しているシステムでは特に重要な修正です。

### 並列ツール呼び出しの修正

`Read`、`WebFetch`、`Glob` ツールが失敗した際に並列実行中の兄弟ツール呼び出しがキャンセルされていた問題を修正しました。現在は bash エラーのみが連鎖するよう変更されています。

### Worktree 関連の修正

- Task ツールの再開時に作業ディレクトリが正しく復元されない問題を修正
- バックグラウンドタスク通知に `worktreePath` と `worktreeBranch` が含まれない問題を修正
- `/clear` がバックグラウンドのエージェント・bash タスクを終了させてしまう問題を修正（フォアグラウンドタスクのみが対象に）

### プラグイン関連の修正

- Windows の OneDrive フォルダーで `EEXIST` エラーが発生しインストールに失敗する問題を修正
- プロジェクトスコープのインストールが存在する場合にユーザースコープのインストールがブロックされる問題を修正
- `CLAUDE_CODE_PLUGIN_CACHE_DIR` がリテラルの `~` ディレクトリを作成してしまう問題を修正
- マーケットプレイス専用フィールドを含む `plugin.json` が読み込めない問題を修正

### パーミッションルールの修正

- ヒアドキュメント、埋め込み改行、引数なしのコマンドにワイルドカードルールが一致しない問題を修正
- env var プレフィックスを持つコマンドで `sandbox.excludedCommands` が動作しない問題を修正
- ネストされた CLI ツールで「常に許可」が過度に広いプレフィックスを提案する問題を修正
- 拒否ルールがすべてのコマンド形式に適用されない問題を修正

### ボイスモードの修正

- 入力遅延が断続的に発生する問題を修正
- プッシュトゥトーク解放後に「No speech detected」と誤検知される問題を修正
- 送信後に古いトランスクリプトがプロンプトに再表示される問題を修正

### hooks 関連の修正

- 再開・フォーク時に `transcript_path` が誤ったディレクトリを指す問題を修正
- 設定書き込み毎に `settings.json` からエージェントの `prompt` が削除される問題を修正
- `PostToolUse` のブロック理由が2回表示される問題を修正
- bash の `read -r` を使った非同期フックが stdin を受け取れない問題を修正
- バリデーションエラーメッセージのサンプルがバリデーションに失敗する問題を修正

### その他のバグ修正

- バックグラウンドタスクやフックの応答が遅い際にシャットダウンが遅延する問題を修正
- エージェントタスクの進捗が「Initializing…」のまま固まる問題を修正
- `--continue` が `--compact` 後に最新の時点から再開されない問題を修正
- フィードバックサーベイが長いセッションで頻繁に表示される問題を修正
- `--effort` CLI フラグが起動時の設定書き込みでリセットされる問題を修正
- `/clear` 後に Ctrl+B でのバックグラウンドクエリがトランスクリプトを失う、または新規会話を破損する問題を修正
- `/model` コマンドが Claude 動作中に実行すると結果を表示しない問題を修正
- プランモードのパーミッションプロンプトのテキスト入力中に数字キーがメニュー選択として扱われる問題を修正
- サンドボックスのファイル書き込み操作が誤ってプロンプトなしで許可される問題を修正
- `U+2028`/`U+2029` 文字を含むファイルの Read 時に Desktop/SDK がクラッシュする問題を修正
- `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` 設定時に終了時にターミナルタイトルがクリアされる問題を修正
- Bedrock API エラーを含むセッションの再開時にクラッシュする問題を修正
- Edit、Bash、Grep ツール入力で断続的に「expected boolean, received string」バリデーションエラーが発生する問題を修正
- 最初のメッセージに改行が含まれる会話からフォークした際にセッションタイトルが複数行になる問題を修正
- キュー内のメッセージに添付画像が表示されず、↑キーで編集時に画像が失われる問題を修正
- VSCode 統合ターミナルのスクロール速度がネイティブターミナルと一致しない問題を修正
- Shift+Enter が古いキーバインドを持つユーザーで改行挿入ではなく送信になる問題を修正

## Breaking Changes

### `CLAUDE_CODE_PROXY_SUPPORTS_TOOL_REFERENCE` の廃止

ツール検索のサードパーティプロキシゲートバイパスに使用していた環境変数 `CLAUDE_CODE_PROXY_SUPPORTS_TOOL_REFERENCE` が削除されました。新しい動作では環境変数の有無で自動的にバイパスが判断されます。この環境変数を利用していた場合は設定を見直してください。

### CLAUDE.md の HTML コメント非表示化

CLAUDE.md ファイルに記述された HTML コメント（`<!-- ... -->`）が、自動インジェクト時に Claude から見えなくなりました。Read ツールで明示的にファイルを読み込んだ場合は引き続き見えます。CLAUDE.md にコメントとして重要な指示を記載していた場合は、通常のテキストへの変換が必要です。

## 技術的なポイント

- **SDK コスト削減**: `query()` のプロンプトキャッシュ修正は、API を直接利用しているシステムにとって運用コストに直結する重要な修正です
- **bash パーサーのネイティブ化**: tree-sitter ベースのパースにより誤検知パーミッションプロンプトが大幅に削減され、開発体験が向上します
- **並列ツール呼び出しの安定化**: Read/Glob/WebFetch の失敗が並列処理全体をキャンセルしなくなり、エラー耐性が向上します
- **Worktree の信頼性向上**: `EnterWorktree` / `ExitWorktree` の対称設計と各種バグ修正により、マルチブランチ開発フローが安定します
- **CLAUDE.md の HTML コメント**: 自動インジェクト時に非表示になる仕様変更は、指示とメモを明確に分離する設計方針の表れです

## まとめ

v2.1.72 は新機能の数こそ控えめながら、SDK のトークンコスト最大 12 倍削減、bash パーサーの刷新、そして大量のバグ修正により、安定性と実用性が大きく向上したリリースです。特に SSH 環境での `/copy w` による直接ファイル書き込みや `/plan` への引数渡しは日常の開発フローで即座に役立ちます。また、Worktree・プラグイン・hooks まわりの修正が積み重なったことで、高度な自動化ワークフローの信頼性も高まりました。SDK を利用している開発者は prompt cache 修正によるコスト削減効果を特に注目してください。

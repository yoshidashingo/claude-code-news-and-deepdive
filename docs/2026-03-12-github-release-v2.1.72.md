# Claude Code v2.1.72 リリース — Worktree強化・Bash解析改善・大量バグ修正

## 概要

Claude Code v2.1.72 は新機能の追加から大規模なバグ修正まで、幅広い改善を含む大型リリースです。`ExitWorktree` ツールの追加により Worktree ワークフローが完結し、Bash コマンド解析のネイティブモジュール移行によってパフォーマンスと安全性が向上しました。また、40件を超えるバグが修正されており、特にパーミッション管理・音声入力・プラグインまわりの安定性が大きく改善されています。

## 新機能

### ExitWorktree ツール

`EnterWorktree` で開始したセッションを明示的に終了するための `ExitWorktree` ツールが追加されました。これにより Worktree の入退場が対称的に管理できるようになり、エージェントが Worktree セッションを自律的にクリーンアップできます。

### /copy コマンドへの `w` キー追加

`/copy` コマンドのフォーカス中の選択内容をクリップボードを経由せずファイルに直接書き込む `w` キーが追加されました。SSH 接続環境などクリップボードが利用できない状況で特に有用です。

### /plan コマンドの説明引数対応

`/plan` コマンドがオプションの説明引数を受け付けるようになりました。

```
/plan fix the auth bug
```

上記のように実行すると、プランモードに入りながらすぐに処理を開始します。毎回手動で説明を入力する手間が省けます。

### Agent ツールの `model` パラメータ復活

Agent ツールに `model` パラメータが復活し、ツール呼び出しごとにモデルを上書き指定できます。タスクに応じて高性能モデルと軽量モデルを使い分けるマルチエージェント構成が再び可能になりました。

### CLAUDE_CODE_DISABLE_CRON 環境変数

```bash
export CLAUDE_CODE_DISABLE_CRON=1
```

この環境変数を設定すると、セッション中にスケジュール済みの cron ジョブを即座に停止できます。デバッグやテスト中にバックグラウンドジョブを一時停止したい場面で役立ちます。

### VSCode URI ハンドラの追加

VSCode 拡張に `vscode://anthropic.claude-code/open` URI ハンドラが追加されました。外部ツールやスクリプトから新しい Claude Code タブをプログラム的に開けます。`prompt` および `session` クエリパラメータもサポートしています。

```
vscode://anthropic.claude-code/open?prompt=hello&session=abc123
```

### claude plugins エイリアス

`claude plugin` コマンドの別名として `claude plugins`（複数形）が追加されました。

## 改善点

### Bash コマンド解析のネイティブモジュール移行

Bash コマンドの解析処理がネイティブモジュールに切り替わり、初期化が高速化されるとともにメモリリークが解消されました。また、tree-sitter による解析が強化され、`find -exec` や変数代入、コマンド置換など、従来は不要な権限プロンプトが発生していたパターンに正しく対応するようになりました。**npm でインストールした場合に tree-sitter が読み込まれなかった問題**も合わせて修正されています。

### Bash 自動承認リストの拡張

以下のコマンドが自動承認リストに追加され、読み取り専用の一般的な操作での権限プロンプトが減少しました。

- `lsof` — 開いているファイルとネットワーク接続の確認
- `pgrep` — プロセス検索
- `tput` — ターミナル制御
- `ss` — ソケット統計
- `fd` / `fdfind` — 高速ファイル検索ツール

### Effort レベルの簡素化

Effort レベルから `max` が削除され、`low / medium / high`（○ ◐ ●）の3段階に統一されました。永続的なアイコン表示に代わり、短い通知で現在のレベルが示されます。デフォルトに戻すには `/effort auto` を使用します。

### /config UI の改善

- **Escape** キーで変更をキャンセル
- **Enter** キーで保存して閉じる
- **Space** キーで設定をトグル

### 音声入力の精度向上

リポジトリ名や一般的な開発用語（regex、OAuth、JSON など）の音声認識精度が向上しました。

### マーケットプレイス Git URL の対応拡大

Azure DevOps や AWS CodeCommit など、`.git` サフィックスなしの Git URL をマーケットプレイスがサポートするようになりました。

### CLAUDE.md の HTML コメント非表示化

CLAUDE.md の HTML コメント（`<!-- ... -->`）が自動インジェクト時に Claude から見えなくなりました。Read ツールで直接読み込んだ場合はこれまで通りコメントが見えます。CLAUDE.md 内に Claude に見せたくないメモを残したい場合に活用できます。

### バンドルサイズ削減

バンドルサイズが約 510 KB 削減されました。インストールや起動の高速化につながります。

### SDK プロンプトキャッシュの修正

SDK の `query()` 呼び出しにおけるプロンプトキャッシュの無効化バグが修正されました。この修正により**入力トークンコストが最大 12 倍削減**される場合があります。SDK を利用したアプリケーションでは特に大きな影響があります。

## バグ修正

### パーミッション・セキュリティ関連

- ワイルドカードルールが heredoc・改行埋め込み・引数なしのコマンドにマッチしない問題を修正
- `sandbox.excludedCommands` が環境変数プレフィックス付きコマンドで機能しない問題を修正
- "Always Allow" が今後マッチしないルールを保存してしまう問題を修正
- 否定（deny）ルールがすべてのコマンド形式に適用されない問題を修正
- 特定のファイル書き込み操作がプロンプトなしで許可されていた問題を修正
- `/tmp/claude/` などの許可ディレクトリへのリダイレクトが不要なプロンプトを出していた問題を修正

### Worktree 関連

- Task ツールの再開時に作業ディレクトリ（cwd）が復元されない問題を修正
- バックグラウンドタスク通知に `worktreePath` と `worktreeBranch` が含まれない問題を修正

### プラグイン関連

- Windows の OneDrive フォルダで `EEXIST` エラーによりインストールが失敗する問題を修正
- プロジェクトスコープのインストールが存在するときユーザースコープのインストールがブロックされる問題を修正
- `CLAUDE_CODE_PLUGIN_CACHE_DIR` がリテラルの `~` ディレクトリを作成する問題を修正
- マーケットプレイス専用フィールドを含む `plugin.json` が読み込み失敗する問題を修正
- `claude plugin validate` が `../` パス拒否時のエラーメッセージを改善

### 音声モード関連

- 入力の遅延（ラグ）を修正
- プッシュ・トゥ・トーク解放後の誤った「No speech detected」エラーを修正
- 送信後に古い文字起こしがプロンプトに再入力される問題を修正

### セッション・会話管理

- `--compact` 後に `--continue` が最新ポイントから再開しない問題を修正
- Ctrl+B のバックグラウンドクエリがトランスクリプトを失うまたは `/clear` 後に新しい会話を破損する問題を修正
- `/clear` がバックグラウンドのエージェント・Bash タスクを終了してしまう問題を修正（フォアグラウンドタスクのみ対象に変更）
- Bedrock API エラーを含むセッションを再開するとクラッシュする問題を修正
- 複数の並行セッション実行時に上矢印キーの履歴が現在のセッションのメッセージを優先して表示するよう改善
- Read ツールが U+2028 / U+2029 文字を含むファイルを返したときのデスクトップ・SDK クラッシュを修正

### その他

- Hooks のスキルから発火するイベントが1イベントにつき2回実行される問題を修正
- `--effort` CLI フラグが起動時の設定書き込みでリセットされる問題を修正
- エージェントタスクの進捗が「Initializing…」から動かない問題を修正
- バックグラウンドエージェントやタスクが実行中に Ctrl+C を2回押しても終了しない問題を修正
- チームエージェントがリーダーのモデルを継承するよう修正
- VSCode 統合ターミナルのスクロール速度がネイティブターミナルと一致しない問題を修正
- VSCode で古いキーバインドを使用しているユーザーの Shift+Enter が改行の代わりに送信してしまう問題を修正
- `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` が設定されているのに終了時にターミナルタイトルがクリアされる問題を修正
- 並列ツール呼び出しで Read / WebFetch / Glob のエラーが兄弟タスクをキャンセルしていた問題を修正（Bash のエラーのみがカスケードするように変更）

## 技術的なポイント

- **Bash 解析のネイティブ化**: tree-sitter ベースの解析により、セキュリティチェックの精度が向上し、誤検知による権限プロンプトが大幅に削減されました。npm インストール版での tree-sitter 読み込み失敗も修正済みです。
- **SDK キャッシュ修正の効果**: `query()` のプロンプトキャッシュ修正は最大 12x のトークン削減につながるため、Claude API を利用したアプリケーションのコスト最適化に直結します。
- **`ExitWorktree` ツールの追加**: `EnterWorktree` / `ExitWorktree` の対称設計により、エージェントが Worktree セッションのライフサイクルを自律管理できるようになりました。
- **CLAUDE.md コメント非表示**: `<!-- -->` コメントを使ったメタ情報の管理が可能になり、CLAUDE.md 内の人間向けメモをモデルに渡さない運用が容易になります。
- **`CLAUDE_CODE_DISABLE_CRON` 環境変数**: CI/CD や自動化環境でスケジュールジョブを制御するオプションが増え、実行環境に応じた細かな制御が可能です。
- **並列ツールのエラー伝播変更**: Read / WebFetch / Glob のエラーが兄弟タスクをキャンセルしなくなったため、並列ツール呼び出しの堅牢性が向上しました。

## まとめ

v2.1.72 は機能追加から安定性改善まで幅広い変更を含む大型リリースです。特に Bash コマンド解析のネイティブ化による誤検知削減と、SDK `query()` のプロンプトキャッシュ修正によるトークンコスト削減は、日常的な利用において目に見える恩恵をもたらします。40件を超えるバグ修正により、Worktree・プラグイン・音声入力・パーミッション管理などの各機能の信頼性が向上しました。`ExitWorktree` ツールや `model` パラメータの復活など、マルチエージェントワークフローを構築している開発者にとっても重要なアップデートが含まれています。

# Claude Code v2.1.69 リリース — 大規模バグ修正とメモリ改善、音声入力10言語追加

## 概要

Claude Code v2.1.69 は新機能の追加に加え、セキュリティ修正・メモリリーク解消・パフォーマンス改善を中心とした大規模なリリースです。Voice STT（音声入力）が新たに10言語に対応し合計20言語をサポート、`/reload-plugins` コマンドや `/claude-api` スキルなど開発体験を向上させる機能も多数追加されています。50件以上のバグ修正が含まれており、長時間セッションの安定性と応答性が大幅に向上しています。

## 新機能

### `/claude-api` スキル

Claude API および Anthropic SDK を使ったアプリケーション開発を支援する `/claude-api` スキルが追加されました。Anthropic SDK を活用するプロジェクトでこのスキルを呼び出すことで、API の利用方法やベストプラクティスについてのガイダンスをすぐに得られます。

### Voice STT — 10言語追加で計20言語対応

音声入力（STT: Speech-to-Text）が新たに以下の10言語に対応し、合計20言語をサポートするようになりました。

- ロシア語、ポーランド語、トルコ語、オランダ語、ウクライナ語
- ギリシャ語、チェコ語、デンマーク語、スウェーデン語、ノルウェー語

### Remote Control のカスタムセッション名

`/remote-control` および `claude remote-control` コマンドにオプションの name 引数が追加されました。

```bash
/remote-control My Project
claude remote-control --name "My Project"
```

`claude.ai/code` 上でセッションタイトルとして表示されるため、複数のセッションを同時に管理する場合に便利です。

### Bash モードでの Ctrl+U 対応

Bash プロンプト（`!`）が空の状態で `Ctrl+U` を押すと Bash モードを終了できるようになりました。これまでの `Escape` や `Backspace` と同様の動作です。

### エフォートレベルの表示

ロゴおよびスピナーにエフォートレベル（例: "with low effort"）が表示されるようになりました。現在どのエフォート設定で動作しているかが一目で確認できます。

### `/reload-plugins` コマンド

プラグインに変更を加えた際、Claude Code を再起動せずに `/reload-plugins` コマンドで変更を即時反映できるようになりました。プラグイン開発・デバッグの効率が向上します。

### 新しいフック・設定オプション

- **`InstructionsLoaded` フックイベント**: CLAUDE.md や `.claude/rules/*.md` がコンテキストに読み込まれたタイミングで発火します。
- **`includeGitInstructions` 設定** (`CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS` 環境変数): システムプロンプトから組み込みのコミット・PR ワークフロー指示を除去できます。
- **`sandbox.enableWeakerNetworkIsolation` 設定** (macOS のみ): カスタム MITM プロキシ使用時に `gh`、`gcloud`、`terraform` などの Go プログラムが TLS 証明書を検証できるようになります。
- **`${CLAUDE_SKILL_DIR}` 変数**: スキルが SKILL.md 内で自身のディレクトリを参照できるようになります。
- **フックイベントへの `agent_id` / `agent_type` 追加**: サブエージェントの識別が容易になります。
- **`worktree` フィールド**: `--worktree` セッション時にフックコマンドへ worktree の名前・パス・ブランチ情報を渡せます。
- **`git-subdir` プラグインソース**: Git リポジトリのサブディレクトリをプラグインソースとして指定できます。
- **`oauth.authServerMetadataUrl`**: MCP サーバーが標準の OAuth メタデータ検出に失敗した場合のカスタム URL を指定できます。

### Sonnet 4.5 → Sonnet 4.6 自動移行

Pro / Max / Team Premium プランの Sonnet 4.5 ユーザーは Sonnet 4.6 に自動移行されます。

## 改善点

### パフォーマンス

- スピナーのアニメーションループをシェルから分離し、CPU・描画オーバーヘッドを削減
- React Compiler によるネイティブバイナリの UI 描画パフォーマンス向上
- `--worktree` 起動時の git サブプロセスを削減
- macOS 起動時の設定ファイル再読み込みを最適化
- MCP `-p` 起動をパイプライン化しシーケンシャルバッチを並行処理に変更
- ファイル存在確認でのファイル内容読み込みを排除（6か所）

### メモリ使用量

- Yoga WASM の遅延プリロードによりベースラインメモリを約16MB削減
- SDK・CCR セッションのメモリフットプリント削減
- 大規模セッション再開時のメモリ使用量削減

### MCP バイナリコンテンツ処理

PDF、Office ドキュメント、音声ファイルなどを返すツールが、raw の base64 データをコンテキストに流し込む代わりに、適切なファイル拡張子でディスクに保存するようになりました。WebFetch も同様にバイナリレスポンスを保存します。

### その他

- マルチエージェントタスクでのサブエージェント最終レポートを簡潔化し、トークン使用量を削減
- `/resume` ピッカーが最初のプロンプトではなく最新のプロンプトを表示するよう変更
- コマンド例の提案を Haiku 呼び出しではなく決定的な生成に変更
- `--append-system-prompt-file` と `--system-prompt-file` がインタラクティブモードでも動作することをドキュメントに明記

## バグ修正

### セキュリティ関連

- **ネストしたスキル検出の脆弱性修正**: `node_modules` などの `.gitignore` 対象ディレクトリからスキルが読み込まれる問題を修正
- **シンボリックリンクバイパスの修正**: シンボリックリンク経由でファイルを書き込む際に作業ディレクトリを脱出できた問題を修正
- **トラストダイアログの修正**: `.mcp.json` の全サーバーが初回実行時にサイレントで有効化されていた問題を修正。サーバーごとの承認ダイアログが正常に表示されるようになりました。
- **インタラクティブツールの自動許可バイパス修正**: スキルの allowed-tools にリストされていると `AskUserQuestion` などのインタラクティブツールがサイレントで自動許可されていた問題を修正

### macOS 関連

- macOS キーチェーン: 複数の OAuth MCP サーバー使用時にキーチェーンが破損する問題を修正
- macOS の `security -i` stdin バッファオーバーフローによる OAuth メタデータ書き込み失敗を修正

### 認証・セッション管理

- トークンリフレッシュ時に `.credentials.json` の `subscriptionType` が失われる問題を修正（[#30185](https://github.com/anthropics/claude-code/issues/30185)）
- `--model claude-opus-4-0` / `claude-opus-4-1` が廃止済みバージョンに解決されていた問題を修正
- `claude remote-control` が npm インストール時に "bad option: --sdk-url" でクラッシュする問題を修正（[#28334](https://github.com/anthropics/claude-code/issues/28334)）

### メモリリーク修正

- 長時間 SDK/CCR セッションで会話メッセージが保持され続けるメモリリークを修正
- インタラクティブモードでフックイベントが無制限に蓄積するメモリリークを修正
- インプロセス teammates でペアレントの会話履歴がピン止めされ GC されないメモリリークを修正
- REPL レンダースコープが蓄積するメモリリーク（1000ターンで約35MB）を修正
- React Compiler の `memoCache` に古いメッセージ配列が蓄積するメモリリークを修正

### UI/UX

- プランモードフィードバック入力でマルチライン入力（`\`+Enter、Shift+Enter）が使えない問題を修正
- Ghostty over SSH で Shift+Enter が改行ではなく `[27;2;13~` を出力する問題を修正
- `Ctrl+S` のスタッシュがメッセージ送信時にクリアされる問題を修正
- `ctrl+o`（トランスクリプトトグル）が長いセッションで数秒フリーズする問題を修正
- `/stats` が不正なタイムスタンプを含むトランスクリプトでクラッシュする問題を修正
- Worktree がネストされたリポジトリ内で実行した際に CLAUDE.md やスラッシュコマンドが重複する問題を修正
- Linux でサンドボックス化された Bash コマンド後に `.bashrc`、`HEAD` などのゴーストドットファイルが未追跡ファイルとして現れる問題を修正
- 大きな未追跡バイナリファイルがある状態でコミットしようとした際のマルチGBメモリスパイクを修正
- ターミナルタイトルに `--agent` 使用時のエージェント名が表示されるよう改善
- VSCode: RTL テキスト（アラビア語、ヘブライ語、ペルシャ語）がチャットパネルで逆順にレンダリングされていた問題を修正（v2.1.63 リグレッション）

### VSCode 拡張機能

- コンパクション（会話圧縮）を折りたたみ可能な "Compacted chat" カードとして表示
- `permissions.disableBypassPermissionsMode` 設定を権限モードピッカーが適切に尊重するよう修正

## 技術的なポイント

- **セキュリティ優先の修正**: `node_modules` からのスキル読み込み、シンボリックリンクバイパス、MCP サーバーの自動信頼など複数のセキュリティ問題が同時修正されています。最新版へのアップデートを強く推奨します。
- **メモリ管理の抜本的改善**: 5件以上のメモリリークが修正され、ベースラインメモリも16MB削減。長時間・大規模セッションでの安定性が大幅に向上しています。
- **macOS キーチェーン修正は重要**: 複数の OAuth MCP サーバーを使用している macOS ユーザーは、繰り返し `/login` を求められる症状が解消されます。
- **`includeGitInstructions` 設定**: 独自の Git ワークフローを持つチームは、組み込み指示を無効化することでシステムプロンプトをクリーンに保てます。
- **フックシステムの拡充**: `InstructionsLoaded`、`agent_id`/`agent_type`、`worktree` フィールドの追加により、CI/CD パイプラインやカスタムオートメーションとの連携がより柔軟になります。
- **Sonnet 4.6 自動移行**: Pro/Max/Team Premium ユーザーは追加操作なしで最新モデルの恩恵を受けられます。

## まとめ

v2.1.69 は機能追加と品質改善の両面で充実したリリースです。特に複数のセキュリティ修正と大規模なメモリリーク解消は、本番環境や長時間のペアプログラミングセッションを行う開発者にとって重要な改善です。Voice STT の10言語追加やリモートコントロールのカスタム名設定など、グローバルな利用シーンへの対応も強化されています。プラグイン・スキル・フックのエコシステムも着実に成熟しており、Claude Code を中心とした自動化・カスタマイズの幅が広がっています。

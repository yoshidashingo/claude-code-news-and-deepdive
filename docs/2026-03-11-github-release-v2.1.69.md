# Claude Code v2.1.69 リリース — 大規模なバグ修正とメモリ最適化、多言語 Voice STT 対応

## 概要

Claude Code v2.1.69 は、60 件を超えるバグ修正とパフォーマンス改善を含む大規模なリリースです。新機能として `/claude-api` スキルの追加や Voice STT の 10 言語拡張などがある一方、セキュリティ脆弱性の修正や複数のメモリリーク対策が重点的に実施されています。長時間セッションの安定性と起動速度の向上が特に目立ちます。

## 新機能

### `/claude-api` スキル

Claude API や Anthropic SDK を使ったアプリケーション開発を支援する新しいスキル `/claude-api` が追加されました。コードが `anthropic` や `@anthropic-ai/sdk` をインポートしている場合、または Claude API の利用について質問した場合に自動でトリガーされます。

### Voice STT の多言語対応拡張

音声入力（STT）が新たに 10 言語に対応し、合計 20 言語をサポートするようになりました。追加された言語は以下のとおりです。

- ロシア語、ポーランド語、トルコ語、オランダ語、ウクライナ語
- ギリシャ語、チェコ語、デンマーク語、スウェーデン語、ノルウェー語

### Remote Control のカスタムセッション名

`/remote-control` および `claude remote-control` コマンドにオプションの `--name` 引数が追加されました。

```bash
/remote-control My Project
claude remote-control --name "My Project"
```

設定したセッション名は `claude.ai/code` 上で表示されます。

### プラグインの動的リロード

`/reload-plugins` コマンドにより、Claude Code を再起動せずに保留中のプラグイン変更を即座に有効化できるようになりました。開発中のプラグイン作業の効率が向上します。

### Git 組み込み指示の無効化

`includeGitInstructions` 設定（および環境変数 `CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS`）を追加しました。Claude のシステムプロンプトから、コミットや PR ワークフローに関する組み込みの指示を除去できます。独自のワークフロー指示を持つチームに有用です。

### その他の新機能

- **数値キーパッドサポート**: インタビュー形式の質問でテンキーからも選択肢を選べるようになりました
- **エフォートレベル表示**: ロゴとスピナーに現在のエフォート設定（例: "with low effort"）が表示されます
- **エージェント名のターミナルタイトル表示**: `claude --agent` 使用時にターミナルタイトルにエージェント名が表示されます
- **`sandbox.enableWeakerNetworkIsolation` 設定** (macOS のみ): カスタム MITM プロキシ使用時に `gh`、`gcloud`、`terraform` などの Go 製ツールが TLS 証明書を検証できるようにします
- **`${CLAUDE_SKILL_DIR}` 変数**: SKILL.md コンテンツ内でスキル自身のディレクトリを参照できます
- **`InstructionsLoaded` フックイベント**: CLAUDE.md や `.claude/rules/*.md` ファイルがコンテキストに読み込まれたときに発火します
- **フックイベントへの `agent_id` / `agent_type` 追加**: サブエージェントや `--agent` 使用時のフックイベントを識別しやすくなりました
- **`pluginTrustMessage` 設定**: マネージド設定でプラグイン信頼警告に組織固有のメッセージを追記できます
- **`pathPattern` 対応**: `strictKnownMarketplaces` で `hostPattern` に加え、正規表現によるファイル・ディレクトリマーケットプレイスのマッチングが可能になりました
- **`git-subdir` プラグインソース**: Git リポジトリ内のサブディレクトリをプラグインソースとして指定できます
- **`oauth.authServerMetadataUrl` 設定**: MCP サーバーの OAuth メタデータ検出に失敗した場合のカスタム URL を指定できます

## 改善点

### パフォーマンスとメモリ使用量

- **ベースラインメモリを約 16MB 削減**: Yoga WASM の遅延読み込みにより実現
- **スピナーのパフォーマンス改善**: 50ms アニメーションループをシェルから分離し、ターン中のレンダリングと CPU オーバーヘッドを削減
- **React Compiler による UI レンダリング高速化**: ネイティブバイナリでの UI レンダリングパフォーマンスが向上
- **`--worktree` 起動改善**: 起動パスから git サブプロセスを排除
- **macOS 起動改善**: マネージド設定解決時の冗長な設定ファイルリロードを排除
- **MCP `-p` 起動改善**: claude.ai 設定取得とローカル接続のパイプライン化
- **LSP ツールの改善**: ファイル全体を読み込まなくなりメモリ使用量を削減
- **セッションアップロードとメモリ同期の改善**: 大きなファイルをサイズ/バイナリチェック前にメモリに読み込まなくなりました

### ユーザーエクスペリエンス

- **MCP バイナリコンテンツの処理改善**: PDF、Office ドキュメント、音声ファイルを返すツールが、生の base64 をコンテキストに流し込む代わりに、正しい拡張子でデコードされたバイトをディスクに保存するようになりました
- **`/resume` ピッカーの改善**: セッションの最初のプロンプトではなく、最新のプロンプトを表示するようになりました
- **サブエージェント最終レポートの簡潔化**: マルチエージェントタスクのトークン使用量を削減
- **Sonnet 4.5 ユーザーの自動移行**: Pro/Max/Team Premium ユーザーは Sonnet 4.6 に自動移行されます

## バグ修正

### セキュリティ関連

- **ネストされたスキル検出の修正**: `node_modules` などの gitignore 対象ディレクトリからスキルが読み込まれる脆弱性を修正
- **シンボリックリンクバイパスの修正**: シンボリックリンクされた親ディレクトリを通じて `acceptEdits` モードで作業ディレクトリ外にファイルを書き込める問題を修正
- **サンドボックスのドメインブロック改善**: `allowManagedDomainsOnly` 有効時に非許可ドメインが自動ブロックされるようになりました（従来はユーザーに承認を求めていた）
- **インタラクティブツールの自動許可問題を修正**: スキルの allowed-tools にリストされた場合でも、`AskUserQuestion` 等のインタラクティブツールが空の回答で自動実行される問題を修正

### macOS 関連

- **macOS キーチェーン破損の修正**: 複数の OAuth MCP サーバー使用時に OAuth メタデータの大きなブロブが `security -i` の stdin バッファをオーバーフローし、古い認証情報が残って繰り返し `/login` を求められる問題を修正

### 認証・設定関連

- **`.credentials.json` の `subscriptionType` 消失の修正**: トークンリフレッシュ中にプロファイルエンドポイントが一時的に失敗した場合でも、サブスクリプションタイプが正しく保持されるようになりました
- **モデルバージョン解決の修正**: `--model claude-opus-4-0` や `--model claude-opus-4-1` が廃止版の Opus に解決される問題を修正
- **MCP 信頼ダイアログの修正**: 初回起動時にすべての `.mcp.json` サーバーが黙って有効化される問題を修正。サーバーごとの承認ダイアログが正しく表示されるようになりました
- **`--mcp-config` 破損ファイルのハング修正**: 破損したファイルを指定した場合のハングを修正

### UI・操作関連

- **Ctrl+U での bash モード終了**: 空の bash プロンプト（`!`）で Ctrl+U を使用して bash モードを終了できるようになりました（`escape` と `backspace` に合わせた動作）
- **プランモードのマルチライン入力対応**: バックスラッシュ+Enter と Shift+Enter で改行を挿入できるようになりました
- **Shift+Enter の修正**: Ghostty over SSH で `[27;2;13~` が出力される問題を修正
- **Ctrl+S スタッシュの修正**: Claude 作業中にメッセージを送信してもスタッシュが消去されなくなりました
- **`/stats` クラッシュの修正**: タイムスタンプが欠落または不正なエントリを含むトランスクリプトファイルでのクラッシュを修正
- **長いセッションでの Ctrl+O フリーズ修正**: 多くのファイル編集を含む長いセッションでトランスクリプトトグルが数秒フリーズする問題を修正
- **ターミナルフリッカーの修正**: スクロールバック境界でのアニメーション要素によるフリッカーを修正
- **macOS の UI フレームドロップ修正**: OAuth を使用する MCP サーバーでのフレームドロップを修正（v2.1.x からのリグレッション）

### メモリリーク関連

- **長期実行 SDK/CCR セッションのメモリリーク修正**: 会話メッセージが不必要に保持される問題を修正
- **React Compiler `memoCache` のメモリリーク修正**: 長いセッションで古いメッセージ配列のバージョンが蓄積される問題を修正
- **REPL レンダースコープのメモリリーク修正**: 1000 ターンで約 35MB 蓄積される問題を修正
- **インプロセスチームメイトのメモリ保持修正**: `/clear` やオートコンパクト後に親の会話履歴が GC されない問題を修正
- **フックイベントのメモリリーク修正**: インタラクティブモードで長いセッション中にフックイベントが無制限に蓄積される問題を修正

### その他のバグ修正

- **Linux のゴーストドットファイル修正**: サンドボックス化された Bash コマンド実行後に `.bashrc`、`HEAD` などが未追跡ファイルとして現れる問題を修正
- **Windows のワークツリーファイルコピー修正**
- **Windows のグローバル `.claude` フォルダ検出修正**
- **重複 CLAUDE.md・スラッシュコマンドの修正**: メインリポジトリ内にネストされたワークツリーから実行した場合の重複を修正
- **ワークツリーネストのプラグインフック修正**: `/plugin` 操作後に Stop/SessionEnd 等のフックが発火しない問題を修正
- **スキルの SKILL.md 読み込み修正**: コロンを含むスキル説明（例: "Triggers include: X, Y, Z"）が frontmatter から読み込まれない問題を修正
- **大きな未追跡バイナリファイルでのメモリスパイク修正**: GB 単位のメモリ使用増加を引き起こす問題を修正
- **Android アプリクラッシュの修正**: Remote Control セッションでローカルスラッシュコマンド実行時のクラッシュを修正

### SDK / VSCode 関連

- **[SDK] タスク作成の `activeForm` フィールド不要化**: スピナーがタスクのサブジェクトにフォールバックするようになりました
- **[VSCode] コンパクション表示の改善**: コンパクト化を折りたたみ可能な "Compacted chat" カードとして表示するようになりました
- **[VSCode] パーミッションモードピッカーの修正**: マネージド設定で `bypass` モードが `disable` に設定されている場合、ピッカーに表示されなくなりました
- **[VSCode] RTL テキストのレンダリング修正**: チャットパネルでアラビア語、ヘブライ語、ペルシャ語が逆方向にレンダリングされる問題を修正（v2.1.63 からのリグレッション）

## 技術的なポイント

- **セキュリティ修正が複数含まれており、アップデートを強く推奨**: gitignore ディレクトリからのスキル読み込み、シンボリックリンクバイパス、サンドボックスドメインブロックの各脆弱性が修正されています
- **メモリ使用量の大幅改善**: 複数のメモリリーク修正と遅延読み込みにより、長時間セッションでの安定性が向上
- **`includeGitInstructions: false` による Git 指示の無効化**: 独自のコミット・PRワークフローを持つ組織や、Claude のデフォルト動作をカスタマイズしたい開発者に有用
- **Voice STT の言語拡張**: 日本語は対象外ですが、欧州言語を中心に大幅に拡張されました
- **MCP バイナリコンテンツ処理の改善**: PDF や Office ドキュメントを扱う MCP ツールが実用的になりました
- **フックイベントの強化**: `InstructionsLoaded`、`agent_id`/`agent_type`、`worktree` フィールド追加により、より詳細な自動化ワークフローが構築できます

## まとめ

v2.1.69 は新機能の追加にとどまらず、セキュリティと安定性の改善に重点を置いた包括的なリリースです。特に複数のメモリリーク修正とセキュリティ脆弱性への対応は、長時間・大規模な利用環境において重要な改善となっています。`/claude-api` スキルや `includeGitInstructions` 設定など、開発ワークフローをよりきめ細かくカスタマイズできる機能も充実しており、日常的に Claude Code を活用している開発者にとって早めのアップデートが推奨されます。

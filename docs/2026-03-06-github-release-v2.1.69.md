# Claude Code v2.1.69 リリース — 音声対応言語拡張、多数のメモリ改善、セキュリティ修正

## 概要

Claude Code v2.1.69 は、音声 STT（音声認識）の対応言語を 10 言語から 20 言語に倍増させ、長時間セッションで顕在化していた複数のメモリリークを修正した大型アップデートです。セキュリティ面ではスキル検出やシンボリックリンク経由のエスケープなどの脆弱性が修正され、プラグインシステムや MCP 周りの改善も多数含まれています。

## 新機能

### `/claude-api` スキルの追加

Claude API および Anthropic SDK を用いたアプリケーション開発を支援する `/claude-api` スキルが追加されました。SDK を使ったアプリ構築時に呼び出すことで、コード生成や実装例の提案を受けられます。

### 音声 STT の対応言語が 20 言語に拡大

Voice STT（音声テキスト変換）に 10 言語が追加され、合計 20 言語に対応しました。新たに追加された言語は以下のとおりです。

- ロシア語、ポーランド語、トルコ語、オランダ語、ウクライナ語
- ギリシャ語、チェコ語、デンマーク語、スウェーデン語、ノルウェー語

### キーボード操作の改善

- 空の bash プロンプト（`!`）で **Ctrl+U** を押すと bash モードを終了できるようになりました（`Escape` や `Backspace` と同じ動作）
- Claude のインタビュー質問でテンキー（数値キーパッド）を使って選択肢を選べるようになりました

### `/remote-control` へのカスタム名引数の追加

`/remote-control` コマンドおよび `claude remote-control` に任意の名前引数を指定できるようになりました。

```bash
/remote-control My Project
claude remote-control --name "My Project"
```

これにより claude.ai/code 上でカスタムセッションタイトルが表示されます。

### 努力レベル（Effort Level）の表示

ロゴとスピナーに現在の effort 設定が表示されるようになりました（例: "with low effort"）。どの effort 設定が有効かを一目で確認できます。

### 新しい設定オプション

| 設定 | 説明 |
|---|---|
| `sandbox.enableWeakerNetworkIsolation` (macOS のみ) | カスタム MITM プロキシ使用時に `gh`、`gcloud`、`terraform` などの Go プログラムが TLS 証明書を検証できるようにする |
| `includeGitInstructions` / `CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS` | Claude のシステムプロンプトからビルトインのコミット・PR ワークフロー指示を除外する |
| `pluginTrustMessage` | プラグインインストール前の信頼警告に組織固有のコンテキストを追記する |
| `oauth.authServerMetadataUrl` | MCP サーバーで標準ディスカバリが失敗した場合のカスタム OAuth メタデータ URL を指定する |

### プラグイン・スキル関連の追加

- **`/reload-plugins`** コマンド: 再起動なしでプラグインの変更を即時反映
- **`${CLAUDE_SKILL_DIR}` 変数**: スキルが自身のディレクトリを SKILL.md 内で参照可能に
- **`InstructionsLoaded` フックイベント**: CLAUDE.md や `.claude/rules/*.md` の読み込み時に発火
- **`git-subdir` プラグインソースタイプ**: git リポジトリ内のサブディレクトリをプラグインソースとして指定可能
- フックイベントに **`agent_id`**（サブエージェント識別）と **`agent_type`** フィールドを追加
- `--worktree` セッションのフックに **`worktree`** フィールドを追加（名前、パス、ブランチ、元リポジトリのディレクトリ情報）

## 改善点

### MCP バイナリコンテンツ処理の改善

PDF、Office ドキュメント、音声ファイルを返すツールが、base64 をそのまま会話コンテキストに流し込む代わりに、正しいファイル拡張子でデコードしてディスクに保存するようになりました。WebFetch も同様にバイナリレスポンスをサマリーとともに保存します。

### パフォーマンス・起動速度の改善

- スピナーのアニメーションループ（50ms）をシェルから切り離し、ターン中のレンダリングと CPU 負荷を削減
- React Compiler による UI レンダリングパフォーマンスの向上
- Yoga WASM のプリロードを遅延化することでベースラインメモリを約 **16MB 削減**
- macOS 起動時のキーチェーン不要ルックアップを省略、設定ファイルの重複リロードを排除
- `-p` モードでの MCP 起動をパイプライン化し、逐次バッチ処理から並行処理へ変更
- 大量のスキル・プラグインがインストールされている場合の起動遅延を改善

### Sonnet 4.5 → 4.6 の自動移行

Pro/Max/Team Premium ユーザーの Sonnet 4.5 が自動的に Sonnet 4.6 に移行されます。

### その他

- `/resume` ピッカーが最初のプロンプトではなく**最新のプロンプト**を表示するように変更
- `--append-system-prompt-file` および `--system-prompt-file` がインタラクティブモードでも動作することをドキュメントに明記
- claude.ai MCP コネクタの失敗がサイレントではなく通知として表示されるように改善
- LSP ツールのレンダリングとメモリコンテキスト構築でファイル全体を読み込まないよう改善

## バグ修正

### セキュリティ修正

- **ネストされたスキル検出の脆弱性修正**: `node_modules` などの gitignore 対象ディレクトリからスキルが読み込まれる問題を修正
- **信頼ダイアログの修正**: `.mcp.json` の全サーバーが初回起動時に無条件で有効化されていた問題を修正。サーバーごとの承認ダイアログが正しく表示されるようになりました
- **シンボリックリンク経由のエスケープ修正**: シンボリックリンクされた親ディレクトリ経由でファイルを書き込む際に作業ディレクトリ外にエスケープできた問題を修正（`acceptEdits` モード）
- **`allowManagedDomainsOnly` の動作修正**: 許可されていないドメインがユーザーへのプロンプトなしで自動ブロックされるようになりました

### メモリリークの修正

長時間セッションでの複数のメモリリークが修正されました。

- 長時間の SDK/CCR セッションで会話メッセージが不要に保持されていた問題
- in-process teammates での親会話履歴のピン留めによるメモリリーク（`/clear` やオートコンパクト後も GC されなかった）
- React Compiler の `memoCache` に古いメッセージ配列が蓄積される問題
- REPL レンダリングスコープが長時間セッションで蓄積される問題（1000 ターンで約 **35MB**）
- インタラクティブモードでフックイベントが無制限に蓄積される問題

### macOS / OAuth 関連の修正

- **macOS キーチェーン破損の修正**: 複数の OAuth MCP サーバー使用時に大きな OAuth メタデータ blob が `security -i` の stdin バッファをオーバーフローし、古い認証情報が残り続ける問題を修正
- **`subscriptionType` の消失問題**: トークンリフレッシュ中にプロファイルエンドポイントが一時的に失敗した際に "Claude Pro"/"Claude Max" が "Claude API" と表示される問題を修正（#30185）
- OAuth MCP サーバー使用時の macOS UI フレームドロップを修正（v2.1.x からのリグレッション）

### UI・入力関連の修正

- Ghostty over SSH で Shift+Enter が改行の代わりに `[27;2;13~` を出力する問題を修正
- プランモードのフィードバック入力で複数行テキスト入力が正しく機能しない問題を修正（`\`+Enter および Shift+Enter が改行を挿入するように）
- Ctrl+O（トランスクリプト切り替え）が長いセッションで数秒間フリーズする問題を修正
- Escape キーが入力ボックスにテキストがある状態で実行中のターンを中断しない問題を修正
- `/stats` がタイムスタンプ不正エントリでクラッシュする問題を修正

### ワークツリー・プラグイン関連の修正

- メインリポジトリ内にネストされたワークツリー（`claude -w`）での CLAUDE.md、スラッシュコマンド、エージェント、ルールの重複を修正
- プラグインの Stop/SessionEnd 等のフックが `/plugin` 操作後に発火しない問題を修正
- `WorktreeCreate` と `WorktreeRemove` プラグインフックが無視される問題を修正

### Windows 関連の修正

- ワークツリーのファイルコピーを修正
- グローバル `.claude` フォルダの検出を修正
- Git Bash で CMD スタイルの `2>nul` リダイレクトを使用した際に実際の `nul` ファイルが作成される問題を修正

### VSCode 拡張の修正

- RTL テキスト（アラビア語、ヘブライ語、ペルシャ語）がチャットパネルで逆順に表示される問題を修正（v2.1.63 からのリグレッション）
- パーミッションモードピッカーが `permissions.disableBypassPermissionsMode` を尊重するように修正

## 技術的なポイント

- **セキュリティ強化**: スキル検出、MCP 信頼ダイアログ、シンボリックリンクエスケープなど、複数のセキュリティ問題が一括修正されました。プラグインや MCP サーバーを多用している環境では早期のアップデートを推奨します。
- **メモリ管理の抜本的改善**: 長時間セッションでの複数のメモリリークが修正され、特に 1000 ターン以上の長期セッションでのメモリ使用量が大幅に改善されます。CCR（Claude Code Remote）や SDK セッションでも効果が期待できます。
- **`includeGitInstructions` 設定**: デフォルトのコミット・PR ワークフロー指示をシステムプロンプトから除外できるようになりました。独自のワークフロー指示を持つチームでトークン節約や制御に活用できます。
- **`sandbox.enableWeakerNetworkIsolation`**: 企業環境で MITM プロキシを使用している場合に `gh` や `terraform` などのツールが TLS 検証を通過できるようにする設定です。セキュリティとの兼ね合いを考慮した上で活用してください。
- **Sonnet 4.5 → 4.6 の自動移行**: Pro/Max/Team Premium ユーザーは明示的な操作なしに最新モデルへ移行されます。既存のワークフローへの影響がないか確認することを推奨します。
- **MCP バイナリコンテンツ**: PDF や Office ドキュメントをツール経由で扱う場合の処理が改善され、コンテキスト汚染が防がれます。

## まとめ

v2.1.69 は新機能よりも品質改善に重点を置いたリリースです。長時間セッションでのメモリ安定性、複数のセキュリティ脆弱性の修正、macOS/Windows/VSCode 全プラットフォームにわたるバグ修正が含まれています。特にメモリリークの修正は、重量級タスクを長時間実行するユーザーや CCR セッションを活用している企業ユーザーに大きなメリットをもたらします。音声入力の多言語対応拡大はグローバルな利用シーンの拡大にも貢献しており、今後の Claude Code のさらなる普及に向けた布石とも言えます。

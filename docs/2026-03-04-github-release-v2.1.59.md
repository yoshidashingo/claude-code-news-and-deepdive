# Claude Code v2.1.59 リリース — Auto-Memory と /copy コマンドの追加

## 概要

Claude Code v2.1.59 では、Claude が有用なコンテキストを自動的に記憶する **Auto-Memory** 機能と、コードブロックを個別にコピーできる **`/copy` コマンド**が新たに追加されました。また、複合 Bash コマンドの権限管理やマルチエージェントセッションのメモリ効率も改善されています。コミュニティから 63 件以上のポジティブなリアクションを受けた注目アップデートです。

## 新機能

### Auto-Memory によるコンテキストの自動保存

Claude が会話の中で有用なコンテキスト（プロジェクトの慣習、ユーザーの好み、重要な決定事項など）を自動的に判断し、メモリに保存するようになりました。これにより、セッションをまたいでも過去の情報を引き継いだ作業が可能になります。

保存されたメモリは `/memory` コマンドで管理できます。不要なメモリの削除や内容の確認が行えるため、Claude に記憶させる情報をユーザー自身でコントロールできます。

```
/memory
```

### `/copy` コマンドによるコードブロックの選択コピー

レスポンスにコードブロックが含まれる場合、新しい `/copy` コマンドでインタラクティブなピッカーが表示されます。このピッカーから以下の操作が可能です。

- **個別のコードブロックを選択してコピー**: 複数のコードブロックが含まれるレスポンスから、必要なブロックだけを取り出せます
- **レスポンス全体をコピー**: 従来通り、レスポンス全文をクリップボードに送ることもできます

複数のコマンドやスクリプト例が並ぶ回答から必要な部分だけを素早く取得できるため、作業効率が向上します。

## 改善点

### 複合 Bash コマンドの権限プレフィックス管理の改善

`&&` で複数のコマンドを繋いだ複合 Bash コマンド（例: `cd /tmp && git fetch && git push`）に対する "always allow"（常に許可）の処理が改善されました。

従来はコマンド全体を一つの文字列として扱い、プレフィックスマッチングが機能しないケースがありましたが、今回の改善でサブコマンドごとにスマートなプレフィックスを計算するようになりました。これにより、複合コマンドに対しても適切な権限管理ができるようになっています。

**改善前の例:**
```
cd /tmp && git fetch && git push
# 全体を1つのコマンドとして扱い、個別の許可設定が難しかった
```

**改善後:**
```
cd /tmp && git fetch && git push
# サブコマンド (cd, git fetch, git push) ごとにプレフィックスを評価
```

### タスクリストの表示順序の改善

短いタスクリスト（TodoWrite で管理するタスクリストなど）の並び順が改善され、より自然で使いやすい表示になりました。

### マルチエージェントセッションのメモリ使用量の最適化

マルチエージェントセッションにおいて、完了したサブエージェントのタスク状態を解放することでメモリ使用量が削減されました。複数のサブエージェントを並列実行する大規模なタスクでも、安定して動作するようになっています。

## バグ修正

### MCP OAuth トークンリフレッシュの競合状態を修正

複数の Claude Code インスタンスを同時に実行している場合に発生していた、MCP (Model Context Protocol) の OAuth トークンリフレッシュ時の競合状態（race condition）が修正されました。複数ウィンドウや並列セッションで Claude Code を使用している開発者は、この修正によりトークン認証の安定性が向上します。

### 作業ディレクトリが削除された場合のエラーメッセージを修正

シェルコマンドの実行中に作業ディレクトリが削除されていた場合、エラーメッセージが表示されない問題が修正されました。これにより、問題の原因を素早く特定できるようになります。

## 技術的なポイント

- **Auto-Memory の管理**: `/memory` コマンドで定期的にメモリの内容を確認・整理することで、Claude に提供するコンテキストの品質を維持できます
- **`/copy` の活用場面**: 長いコードレビューや複数のスクリプト例を含む回答から、必要なコードだけを効率よく取得できます
- **複合コマンドの権限設定**: `&&` で繋いだコマンドに対する "always allow" がサブコマンド単位で機能するため、よく使う複合コマンドをより細かく管理できます
- **マルチエージェントの安定性**: サブエージェントのメモリ解放により、並列エージェントを多用するワークフローのリソース消費が抑制されます
- **MCP 利用者向け**: 複数インスタンスを同時実行している場合の OAuth 認証が安定するため、MCP を活用した開発環境で恩恵を受けられます

## まとめ

v2.1.59 は、Claude Code の日常的な使い勝手を大きく向上させるアップデートです。Auto-Memory によるコンテキスト保持はセッションをまたいだ継続的な作業を支援し、`/copy` コマンドはコードの取り出しを直感的にします。また、複合コマンドの権限管理改善やマルチエージェントのメモリ最適化は、より複雑な開発ワークフローへの対応力を高めるものです。これらの改善により、Claude Code はより「使いやすく、賢い」ツールへと進化しています。

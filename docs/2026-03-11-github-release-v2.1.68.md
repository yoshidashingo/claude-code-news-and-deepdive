# Claude Code v2.1.68 リリース: Opus 4.6 のデフォルト effort 変更と「ultrathink」の復活

## 概要

Claude Code v2.1.68 では、Opus 4.6 モデルの推論 effort (処理深度) のデフォルト値が Max・Team サブスクライバー向けに medium に変更されました。また、一度廃止されていた `ultrathink` キーワードが再導入され、必要なときだけ高 effort モードに切り替えられるようになっています。加えて、ファーストパーティ API から Opus 4 および Opus 4.1 が削除され、これらをピン留めしていたユーザーは自動的に Opus 4.6 へ移行されます。

## 新機能

### Opus 4.6 のデフォルト effort が medium に

Max および Team プランのサブスクライバーに対して、Opus 4.6 モデルの推論 effort がデフォルトで **medium** に設定されました。

medium effort はスピードと処理の丁寧さのバランスが取れた設定であり、大多数のタスクで十分な品質を提供します。設定は `/model` コマンドからいつでも変更可能です。

```
/model
```

### 「ultrathink」キーワードの再導入

以前のバージョンで削除されていた `ultrathink` キーワードが復活しました。このキーワードをプロンプト中に含めると、**次のターンのみ** high effort モードが有効になります。複雑な設計の議論やデバッグなど、特に深い思考を必要とする場面で活用できます。

```
ultrathink このアーキテクチャの問題点を全て洗い出してください
```

## 改善点・変更点

### Opus 4 / 4.1 のファーストパーティ API からの削除

ファーストパーティ API 上の Claude Code で利用できるモデルから、**Opus 4** および **Opus 4.1** が削除されました。これらのモデルをピン留めしていたユーザーは、自動的に **Opus 4.6** へ移行されます。手動での設定変更は不要です。

## 技術的なポイント

- **effort の段階**: Claude Code における推論 effort は low / medium / high の 3 段階が存在し、モデルの思考深度とレイテンシのトレードオフを制御します
- **ultrathink の適用スコープ**: `ultrathink` は宣言したターンにのみ適用されます。継続的に高 effort を使いたい場合は `/model` からデフォルト設定を変更してください
- **自動モデル移行**: Opus 4 / 4.1 からのマイグレーションはシームレスに行われますが、モデル固有の挙動に依存したプロンプトがある場合は動作確認を推奨します
- **対象プラン**: medium effort デフォルト化は Max および Team サブスクライバーが対象です。他のプランでの挙動については `/model` コマンドで確認してください

## まとめ

v2.1.68 は、Opus 4.6 の日常的な利用体験を最適化することに焦点を当てたリリースです。medium effort をデフォルトとすることで応答速度と品質のバランスが改善され、`ultrathink` の復活によってユーザーは必要に応じて high effort に切り替えられる柔軟性を得ました。また、旧モデル (Opus 4 / 4.1) の廃止により、モデルラインナップが Opus 4.6 に一本化されます。今後は Opus 4.6 を軸に各種機能の改善が進むことが期待されます。

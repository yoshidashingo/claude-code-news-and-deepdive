# Claude Code v2.1.70 リリース — 大規模バグ修正とVS Code機能強化

## 概要

Claude Code v2.1.70 は、20件以上のバグ修正と複数のパフォーマンス改善を含む大型メンテナンスリリースです。Windows/WSL環境でのクリップボード文字化け修正やAPI互換性の向上など、実運用で遭遇しやすい問題が広く解消されています。また、VS Code向けにセッション管理UIやネイティブMCPサーバー管理ダイアログなど、開発体験を向上させる新機能も追加されました。

## 新機能

### [VS Code] セッションマネージャーの追加

VS Code のアクティビティバーにスパークアイコンが追加され、すべての Claude Code セッションを一覧表示できるようになりました。セッションはフルエディターとして開くことができ、複数セッションの管理がより直感的になります。

### [VS Code] プラン表示のMarkdownドキュメントビュー

VS Code 上でプランをフルMarkdownドキュメントとして表示できるようになりました。コメントを追加してフィードバックを提供する機能も含まれており、AI とのプランニング作業が一層スムーズになります。

### [VS Code] ネイティブMCPサーバー管理ダイアログ

チャットパネルで `/mcp` コマンドを実行すると、ネイティブのMCPサーバー管理ダイアログが開くようになりました。サーバーの有効化・無効化、再接続、OAuth認証の管理をターミナルを切り替えることなく行えます。

## 改善点

- **`/rename` コマンドの改善**: Claude が処理中でも `/rename` コマンドが即座に動作するようになりました。以前は処理完了まで暗黙的にキューイングされていました。
- **プロンプト入力のレンダリング最適化**: ターン中のプロンプト入力再レンダリング回数が約74%削減され、入力の応答性が向上しました。
- **起動メモリの削減**: カスタムCA証明書を使用していないユーザーで起動時のメモリ使用量が約426KB削減されました。
- **コンパクション最適化**: コンパクション（要約処理）時にサマライザーリクエスト内の画像が保持されるようになり、プロンプトキャッシュの再利用が可能になりました。処理速度とコストの両面で改善が期待できます。
- **Remote Control のポーリング削減**: 接続中の `/poll` 間隔が1〜2秒から10分に延長され、サーバー負荷が約300倍削減されました。切断時の再接続には引き続き高速ポーリングが使用されます。
- **マイクエラーメッセージの改善**: 無音キャプチャと「音声未検出」エラーを区別するメッセージに改善されました。

## バグ修正

### API・モデル関連

- **`ANTHROPIC_BASE_URL` 使用時のAPI 400エラー修正**: サードパーティゲートウェイ経由でツール検索を使用するとエラーが発生していた問題を修正。プロキシエンドポイントを正しく検出し、`tool_reference` ブロックを無効化するようになりました。
- **Bedrockカスタム推論プロファイルのエラー修正**: `claude-*` の命名規則に合致しないカスタムモデル識別子を使用した場合に「API Error: 400 This model does not support the effort parameter」が発生していた問題を修正しました。
- **ToolSearch後の空レスポンス修正**: `ToolSearch` 実行直後にモデルが空のレスポンスを返すことがある問題を修正。サーバーがシステムプロンプト形式のタグでツールスキーマをレンダリングしていたことでモデルが早期停止していました。

### キャッシュ・設定関連

- **MCPサーバー接続後のプロンプトキャッシュバスト修正**: `instructions` を持つMCPサーバーが最初のターン後に接続した際にキャッシュが不正に無効化される問題を修正しました。
- **フィーチャーフラグのキャッシュ修正**: 起動初期に読み込まれるフィーチャーフラグがセッション間でディスクキャッシュを更新せず、古い値が残り続けていた問題を修正しました。
- **Remote環境での `permissions.defaultMode` 修正**: Claude Code Remote環境で `acceptEdits` または `plan` 以外の `permissions.defaultMode` 値が適用されてしまっていた問題を修正。現在は無視されます。
- **`--resume` 時のスキルリスト再注入修正**: `--resume` のたびにスキルリストが再注入されていた問題を修正し、1回のリジュームあたり約600トークンを節約できるようになりました。

### Windows・WSL関連

- **クリップボードの非ASCII文字化け修正**: Windows/WSL環境で日本語（CJK）や絵文字などの非ASCII文字がクリップボードにコピーされると文字化けしていた問題を修正。PowerShell の `Set-Clipboard` を使用するよう変更されました。
- **VS Code起動時の余分なウィンドウ修正**: Windows の VS Code 統合ターミナルから起動した際に余分なウィンドウが開く問題を修正しました。
- **Windows ネイティブバイナリでの音声モード修正**: 「native audio module could not be loaded」エラーが発生していた問題を修正しました。

### UI・コマンド関連

- **SSHでのEnterキー動作修正**: 低速なSSH接続でEnterキーを押すと送信ではなく改行が挿入されていた問題を修正しました。
- **Markdownリンクの `#NNN` 参照修正**: `#NNN` 形式のフラグメントを含むMarkdownリンクが現在のリポジトリのIssueへのリンクとして誤解釈される問題を修正しました。
- **「Model updated to Opus 4.6」通知の重複修正**: `.claude/settings.json` にレガシーのOpusモデル文字列が設定されている場合に通知が繰り返し表示される問題を修正しました。
- **プラグイン表示の修正**: `/plugin` でプラグインが誤ってインストール済みと表示される問題、およびマーケットプレイスインストール直後に「not found in marketplace」エラーが表示される問題を修正しました。
- **`/security-review` コマンドの修正**: 古いgitバージョンで `unknown option merge-base` エラーが発生していた問題を修正しました。
- **`/color` コマンドのリセット修正**: デフォルトカラーへのリセット手段がなかった問題を修正。`/color default`、`/color gray`、`/color reset`、`/color none` のいずれでもデフォルトカラーに戻せるようになりました。
- **`AskUserQuestion` パフォーマンス修正**: ノート入力フィールドへのキーストロークのたびにMarkdownレンダリングが再実行されていたパフォーマンスリグレッションを修正しました。
- **プッシュトゥトーク修正**: 設定で `voiceEnabled: true` を設定しているにもかかわらず、セッション開始時にプッシュトゥトークが有効化されない問題を修正しました。
- **VS Codeテレポートマーカー修正**: VS Codeのテレポートセッションでテレポートマーカーが表示されない問題を修正しました。

## 技術的なポイント

- **`ANTHROPIC_BASE_URL` + ツール検索の互換性**: サードパーティAPIゲートウェイ（OpenAI互換プロキシなど）を利用している場合、`tool_reference` ブロックがサポートされずAPIエラーが起きるケースがありました。本修正でプロキシ検出が自動化され、追加設定なしで動作するようになります。
- **Bedrockカスタムプロファイル対応**: AWS Bedrock で独自の推論プロファイルを使用している場合、`effort` パラメーターに関するエラーが発生することがありました。標準のClaude命名パターンに合致しない場合は自動的に対象外とする処理が追加されました。
- **Windows/WSLのクリップボード対応**: CLIツールでのクリップボード操作はエンコーディング問題が発生しやすく、日本語などのマルチバイト文字を多用するユーザーには特に恩恵が大きい修正です。
- **Remote Controlの負荷軽減**: ポーリング間隔を1〜2秒から10分に変更したことで、多数のユーザーが接続するサーバーサイドの負荷が劇的に削減されます。切断検知は引き続き高速に行われるため、UXへの影響はありません。
- **トークン節約**: `--resume` 時のスキルリスト再注入を抑制することで、長期セッションでのコスト削減が期待できます。

## まとめ

v2.1.70 は新機能よりも安定性と品質の向上に重点を置いたリリースです。特に Windows/WSL ユーザー、Bedrock カスタム推論プロファイルユーザー、サードパーティAPIゲートウェイ利用者にとって重要な修正が含まれています。VS Code との統合もさらに深まり、セッション管理やMCPサーバー管理がターミナル不要で行えるようになりました。日常的に Claude Code を使用している開発者であれば、早めのアップデートをお勧めします。

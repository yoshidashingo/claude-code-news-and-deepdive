# Claude Code v2.1.68 リリース: Opus 4.6 のデフォルト努力レベル変更と ultrathink キーワード復活

## 概要

Claude Code v2.1.68 では、Max および Team サブスクライバー向けに Opus 4.6 のデフォルト努力レベル（effort level）が medium（中程度）に設定されました。また、高い努力レベルを一時的に有効化できる `ultrathink` キーワードが再導入されています。さらに、旧モデルである Opus 4 と Opus 4.1 がファーストパーティ API から削除され、これらを固定していたユーザーは自動的に Opus 4.6 へ移行されます。

## 新機能

### `ultrathink` キーワードの再導入

以前一時的に廃止されていた `ultrathink` キーワードが復活しました。プロンプトに `ultrathink` と記述することで、**次のターンだけ**高い努力レベル（high effort）が適用されます。

```
ultrathink このコードのアーキテクチャ全体を見直して最適化案を提案してください
```

通常は medium effort で十分なタスクでも、特に複雑な問題や深い推論が必要な場面では `ultrathink` を活用することで、より徹底的な応答が得られます。

## 改善点

### Opus 4.6 のデフォルト努力レベルを medium に変更

Max および Team サブスクライバーにおいて、Opus 4.6 のデフォルト努力レベルが **medium**（中程度）に変更されました。公式の説明によると、medium effort はスピードと徹底度のバランスが取れた「スイートスポット」として位置づけられており、ほとんどのタスクにおいて最適なパフォーマンスを発揮するとされています。

努力レベルはいつでも `/model` コマンドから変更できます。高負荷なタスクには `ultrathink` キーワードを使うか、`/model` から設定を変更してください。

## Breaking Changes

### Opus 4 / Opus 4.1 の削除

ファーストパーティ API において、Opus 4 および Opus 4.1 が Claude Code から削除されました。これらのモデルを明示的に固定（pin）していたユーザーは、**自動的に Opus 4.6 へ移行**されます。

移行後の動作確認として、以下の点に注意してください:

- `/model` コマンドで現在使用中のモデルを確認できます
- Opus 4.6 は Opus 4.1 に比べて性能が向上しており、基本的には移行後も問題なく利用できます
- 独自スクリプトや設定ファイルでモデル名を直接指定している場合は、`claude-opus-4-6` または最新の推奨モデル ID に更新してください

## 技術的なポイント

- **デフォルト effort が medium に**: Opus 4.6 は以前より高い effort がデフォルトだった可能性があり、medium への変更はレスポンス速度と処理コストのバランスを改善するための調整です
- **`ultrathink` は1ターン限定**: `ultrathink` を指定しても次のターン以降は medium effort に戻ります。連続して高努力レベルを維持したい場合は `/model` コマンドで設定変更が必要です
- **モデル自動移行**: Opus 4 / 4.1 のユーザーは自動移行されますが、API 経由で Claude Code を利用しているケースでは設定ファイルやコードのモデル ID 指定の更新が必要になる場合があります
- **Max・Team 向けの変更**: 今回の effort level のデフォルト変更は Max および Team サブスクライバーが対象です。他のプランでの挙動については公式ドキュメントを確認してください

## まとめ

v2.1.68 は Opus 4.6 の利用体験を最適化するアップデートです。medium effort をデフォルトとすることで、日常的なタスクでの応答速度とコストを改善しつつ、`ultrathink` キーワードで必要な場面には高い推論能力を発揮できる柔軟な設計となっています。また、旧モデルの整理による Opus 4.6 への統一は、今後のモデルサポートをシンプルにする方向性を示しています。旧モデルを明示的に指定していたユーザーは設定の見直しをお勧めします。

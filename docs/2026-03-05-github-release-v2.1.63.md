# Claude Code v2.1.63 リリース: HTTP Hooks、メモリリーク修正、新スラッシュコマンドなど

## 概要

Claude Code v2.1.63 では、HTTP Hooks の追加や `/simplify`・`/batch` スラッシュコマンドの追加など、複数の新機能が導入されました。また、長期セッションにおける多数のメモリリーク・リスナーリークが修正されており、安定性と信頼性が大きく向上したリリースです。

## 新機能

### `/simplify` と `/batch` スラッシュコマンドの追加

2つの新しいビルトインスラッシュコマンドが追加されました。

- **`/simplify`**: 変更されたコードについて再利用性・品質・効率を確認し、改善点を修正します
- **`/batch`**: バッチ処理に関連するコマンドです

これらはバンドル済みのスラッシュコマンドとして提供されており、即座に利用可能です。

### HTTP Hooks の追加

従来のシェルコマンドを実行する Hooks に加えて、**HTTP Hooks** が新たにサポートされました。HTTP Hooks では、JSON を URL に POST し、JSON レスポンスを受け取ることができます。これにより、外部の Web サービスやマイクロサービスと Claude Code のイベントを連携させることが容易になります。

```json
{
  "type": "http",
  "url": "https://example.com/hooks/claude-event",
  "method": "POST"
}
```

### `/model` コマンドの改善

`/model` コマンドのスラッシュコマンドメニューに、現在アクティブなモデルが表示されるようになりました。どのモデルを使用しているかを確認する際に便利です。

### Git Worktree 間でのプロジェクト設定と Auto Memory の共有

同一リポジトリの git worktree 間で、プロジェクト設定と auto memory が共有されるようになりました。複数の worktree を使って並行して開発している場合でも、設定を一元管理できます。

### `ENABLE_CLAUDEAI_MCP_SERVERS=false` 環境変数の追加

claude.ai の MCP サーバーを自動的に利用可能にする機能をオプトアウトするための環境変数 `ENABLE_CLAUDEAI_MCP_SERVERS=false` が追加されました。MCP サーバーを制限したい環境での利用に役立ちます。

### MCP OAuth 認証のフォールバック改善

MCP OAuth 認証時に、自動的な localhost リダイレクトが機能しない場合、コールバック URL を手動でペーストして認証を完了できるようになりました。ネットワーク環境によって自動リダイレクトが利用できない場合でも認証が完了できます。

### `/copy` コマンドに「常に全レスポンスをコピー」オプションを追加

`/copy` ピッカーに「Always copy full response」オプションが追加されました。このオプションを選択すると、以降の `/copy` コマンドはコードブロックピッカーをスキップして、レスポンス全体を直接コピーします。

### VSCode: セッションの名前変更・削除アクション

VSCode 拡張機能のセッションリストに、セッションのリネームと削除のアクションが追加されました。セッション管理がより柔軟になります。

## バグ修正

### スラッシュコマンド出力の表示修正

`/cost` などのローカルスラッシュコマンドの出力が、ユーザー送信メッセージとして表示されていた問題が修正されました。システムメッセージとして正しく表示されるようになります。

### メモリリーク・リスナーリークの大規模修正

本リリースでは、特に長期セッションにおける多数のメモリリーク・リスナーリークが修正されています。

**リスナーリーク修正:**
- ブリッジポーリングループのリスナーリーク
- MCP OAuth フロークリーンアップのリスナーリーク
- 自動承認時のインタラクティブパーミッションハンドラーのリスナーリーク
- WebSocket トランスポート再接続時のリスナーリーク

**メモリリーク修正:**
- Hooks 設定メニューのナビゲーション時のメモリリーク
- bash コマンドプレフィックスキャッシュのメモリリーク
- MCP ツール/リソースキャッシュのサーバー再接続時のリーク
- IDE ホスト IP 検出キャッシュが異なるポート間で結果を誤共有する問題
- git ルート検出キャッシュの無制限成長によるメモリリーク（長期セッションで特に深刻）
- JSON パースキャッシュの無制限成長によるメモリリーク
- 長期実行チームメイトが AppState にメッセージを保持し続ける問題（会話のコンパクション後も残存）
- MCP サーバーフェッチキャッシュが切断時にクリアされないリーク（頻繁に再接続するサーバーで問題が拡大）

**その他の修正:**
- ファイルカウントキャッシュが glob の無視パターンを考慮しない問題
- VSCode: リモートセッションが会話履歴に表示されない問題
- REPL ブリッジで初期接続フラッシュ中に新規メッセージと履歴メッセージが混在することによるメッセージ順序の競合状態
- `/clear` コマンドがキャッシュされたスキルをリセットしない問題（古いスキル内容が新しい会話に残存する問題）

## 改善点

### サブエージェントを使った長期セッションのメモリ使用量改善

サブエージェントを使用した長期セッションにおいて、コンテキストコンパクション時に重いプログレスメッセージペイロードを除去することでメモリ使用量が改善されました。

## 技術的なポイント

- **HTTP Hooks** の追加により、外部 Web サービスとの連携がシェルスクリプト不要で実現可能になりました。CI/CD パイプラインや Webhook エンドポイントとの統合が容易です
- **git worktree サポート**の強化により、複数ブランチを並行開発するワークフローでも Claude Code の設定を一元管理できます
- **多数のメモリリーク修正**は、長時間稼働させる利用シナリオ（チームメイト機能の常時起動など）での安定性に直結します。特に MCP サーバーを頻繁に再接続する環境では顕著な改善が期待できます
- **`ENABLE_CLAUDEAI_MCP_SERVERS=false`** は、セキュリティポリシーや環境制約により外部 MCP サーバーへの接続を制限したい企業・チームに有用です
- `/clear` のスキルキャッシュリセット修正により、スラッシュコマンドのカスタマイズを行っているユーザーがより信頼性高く新しい会話を開始できます

## まとめ

v2.1.63 は、新機能の追加と安定性向上の両面で充実したリリースです。特に HTTP Hooks の追加は Hooks の活用範囲を大きく広げるものであり、外部サービスとのインテグレーションに取り組む開発者にとって注目の機能です。また、長期セッションでの多数のメモリリーク修正は、Claude Code を継続的に使用するチームや、サブエージェント・MCP サーバーを多用するワークフローにおいて、パフォーマンスと安定性の改善をもたらします。

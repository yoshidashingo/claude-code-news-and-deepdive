# Claude Code v2.1.69 リリース — 大規模バグ修正・メモリ最適化・新機能多数

## 概要

Claude Code v2.1.69 は、多数のメモリリーク修正・パフォーマンス改善に加え、セキュリティ関連の重要なバグ修正を含む大型リリースです。新機能としては Voice STT の10言語追加、`/reload-plugins` コマンド、`InstructionsLoaded` フックイベントなど開発者向けの機能拡張も充実しています。また、Pro/Max/Team Premium ユーザーの Sonnet 4.5 から Sonnet 4.6 への自動移行も実施されます。

## 新機能

### `/claude-api` スキルの追加

Claude API と Anthropic SDK を使ったアプリケーション構築を支援する専用スキル `/claude-api` が追加されました。SDK を使った開発ワークフローを始めやすくなります。

### Voice STT の言語サポート拡充

音声入力（Speech-to-Text）の対応言語が 10 から 20 へ倍増しました。新たに追加された言語は以下の通りです。

- ロシア語、ポーランド語、トルコ語、オランダ語、ウクライナ語
- ギリシャ語、チェコ語、デンマーク語、スウェーデン語、ノルウェー語

### リモートコントロールのカスタムセッションタイトル

`/remote-control` と `claude remote-control` コマンドに任意の `name` 引数が追加されました。

```bash
/remote-control My Project
claude remote-control --name "My Project"
```

`claude.ai/code` 上に表示されるセッションタイトルをカスタマイズでき、複数セッションの管理が容易になります。

### UI の使いやすさ向上

- **Bash モード終了の改善**: 空の bash プロンプト（`!`）で `Ctrl+U` を押すと bash モードを終了できるようになりました（`Escape` と `Backspace` と同様の動作）。
- **数値キーパッド対応**: Claude の質問への回答選択で、QWERTY 上部の数字キーに加えてテンキーも利用可能になりました。
- **努力レベルの可視化**: ロゴとスピナーに現在のエフォートレベル（例：「低い努力で」）が表示されるようになり、設定状態を確認しやすくなりました。
- **エージェント名表示**: `claude --agent` 使用時にターミナルタイトルにエージェント名が表示されます。

### `/reload-plugins` コマンド

プラグインの変更を Claude Code を再起動せずに即時反映できる `/reload-plugins` コマンドが追加されました。プラグイン開発時の反復作業が効率化されます。

### スキル・プラグイン開発向け強化

- **`${CLAUDE_SKILL_DIR}` 変数**: スキルの SKILL.md 内から自身のディレクトリパスを参照できます。
- **`InstructionsLoaded` フックイベント**: CLAUDE.md や `.claude/rules/*.md` ファイルがコンテキストに読み込まれた際に発火するフックイベントが追加されました。
- **`git-subdir` プラグインソースタイプ**: git リポジトリ内の特定のサブディレクトリをプラグインソースとして指定できます。
- **フックイベントへの `agent_id`・`agent_type` 追加**: サブエージェントと `--agent` 使用時の識別情報がフックイベントに含まれるようになりました。
- **`worktree` フィールドのフック追加**: `--worktree` セッション実行時にワークツリーの名前・パス・ブランチ・元リポジトリ情報がフックコマンドに提供されます。

### 設定オプションの追加

- **`includeGitInstructions`** (および `CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS` 環境変数): Claude のシステムプロンプトから組み込みのコミット・PR ワークフロー指示を除外できます。
- **`sandbox.enableWeakerNetworkIsolation`** (macOS のみ): カスタム MITM プロキシ使用時に `gh`、`gcloud`、`terraform` などの Go プログラムが TLS 証明書を検証できるようになります。
- **`pluginTrustMessage`**: マネージド設定でプラグインのインストール前信頼警告に組織固有のコンテキストを追加できます。

## 改善点

### パフォーマンス・起動速度

- **スピナー**: 50ms アニメーションループを shell から分離し、レンダリングと CPU オーバーヘッドを削減。
- **React Compiler による UI 高速化**: ネイティブバイナリでのレンダリングパフォーマンスが向上。
- **ワークツリー起動**: 起動パスから git サブプロセスを削除して高速化。
- **macOS 起動**: マネージド設定解決時の冗長な settings-file リロードを削除。
- **MCP `-p` 起動**: claude.ai の設定取得をローカル接続と並列パイプライン化し、sequential batching から concurrency pool に変更。
- **ベースラインメモリ**: Yoga WASM プリロードの遅延化により約 16MB 削減。

### メモリ使用量の改善

- **SDK/CCR セッション**: stream-json 出力でメモリフットプリントを削減。
- **大規模セッション再開**: 圧縮された会話履歴を含む大規模セッション再開時のメモリ削減。
- **マルチエージェントタスク**: より簡潔なサブエージェント最終レポートによりトークン使用量を削減。

### MCP バイナリコンテンツの改善

PDF、Office ドキュメント、音声ファイルを返す MCP ツールが、raw base64 を会話コンテキストに直接展開する代わりに、適切なファイル拡張子でディスクに保存するようになりました。WebFetch もバイナリレスポンスをサマリーとともに保存します。

### その他

- **`/resume` ピッカー**: 最初のプロンプトではなく最近のプロンプトを表示し、セッション識別が容易に。
- **例示コマンド提案**: Haiku 呼び出しから決定論的な生成に変更し、不要な API コールを削減。
- **Sonnet 4.6 自動移行**: Pro/Max/Team Premium の Sonnet 4.5 ユーザーが Sonnet 4.6 に自動移行されます。
- **\[VSCode\]** 圧縮チャットを折り畳み可能な「Compacted chat」カードとして表示。

## バグ修正

### セキュリティ関連（重要）

- **ネストされたスキル検出の脆弱性修正**: `node_modules` など gitignore 対象ディレクトリからスキルが誤って読み込まれる問題を修正。
- **信頼ダイアログの修正**: 初回実行時に `.mcp.json` の全サーバーがサイレントに有効化されていた問題を修正。サーバー単位の承認ダイアログが正しく表示されるようになりました。
- **シンボリックリンクバイパスの修正**: symlinked 親ディレクトリ経由で新規ファイルを書き込んだ場合に `acceptEdits` モードで作業ディレクトリをエスケープできた問題を修正。
- **管理ドメインポリシーの強制**: `allowManagedDomainsOnly` 有効時、許可されていないドメインへのアクセスが自動的にブロックされるようになりました（以前はバイパス可能な承認ダイアログが表示されていた）。
- **スキルの allowed-tools による権限回避の修正**: `AskUserQuestion` などのインタラクティブツールがスキルの allowed-tools に列挙されていた場合、permission prompt をスキップして空の回答で実行されていた問題を修正。

### クラッシュ・ハング修正

- `claude remote-control` が npm インストール時に "bad option: --sdk-url" でクラッシュする問題 (#28334)
- `--model claude-opus-4-0` / `--model claude-opus-4-1` が非推奨バージョンに解決されていた問題
- `/stats` コマンドが不正なタイムスタンプを含むトランスクリプトでクラッシュする問題
- `--mcp-config` が破損ファイルを指す際のハング
- ストリーミングエラー後の長いセッションでの一時的なハング（トランスクリプトの全書き換えではなく truncate に変更）

### 認証・認証情報

- **macOS キーチェーン破損**: 複数の OAuth MCP サーバー使用時に大きな OAuth メタデータが `security -i` の stdin バッファをオーバーフローし、古い認証情報が残留して繰り返し `/login` を求められる問題を修正。
- **サブスクリプション型の消失**: トークンリフレッシュ時に profile endpoint が一時的に失敗した場合、`.credentials.json` の `subscriptionType` が失われ「Claude Pro」ではなく「Claude API」と表示される問題を修正 (#30185)。

### メモリリーク

v2.1.69 では複数のメモリリークが一括修正されています。

- **in-process teammates**: 親の会話履歴全体がチームメイトのライフタイム中ピン留めされ、`/clear` や auto-compact 後も GC されない問題。
- **React Compiler `memoCache`**: 古いメッセージ配列のバージョンが蓄積する問題。
- **REPL render スコープ**: 1000 ターンあたり約 35MB 蓄積していた問題。
- **フックイベント**: interactive mode でのフックイベントが無制限に蓄積する問題。
- **SDK/CCR 長時間セッション**: 会話メッセージが不必要に保持される問題。

### UI・入力

- **Shift+Enter**: Ghostty over SSH で改行の代わりに `[27;2;13~` が印字される問題を修正。
- **Escape キー**: 入力ボックスに下書きテキストがある場合に実行中の turn を中断できない問題を修正。
- **計画モードフィードバック**: 複数行テキスト入力に未対応だった問題（`Backslash+Enter` と `Shift+Enter` で改行挿入に対応）。
- **Ctrl+O フリーズ**: 大量のファイル編集を含む長いセッションでトランスクリプト切り替え時に数秒フリーズする問題。
- **Stash クリア**: Claude が動作中にメッセージを送信すると `Ctrl+S` の stash がクリアされる問題。
- **\[VSCode\]** RTL テキスト（アラビア語・ヘブライ語・ペルシャ語）がチャットパネルで反転表示されていた問題（v2.1.63 からの regression）。

### ファイルシステム・OS

- **Ghost dotfiles**: Linux の sandbox bash コマンド実行後に `.bashrc`・`HEAD` などが untracked files として表示される問題。
- **大規模バイナリファイルのメモリスパイク**: untracked の大きなバイナリファイルが存在する場合のコミット時に数 GB のメモリスパイクが発生する問題。
- **Windows**: ワークツリーファイルコピーとグローバル `.claude` フォルダ検出の問題。

### プラグイン・スキル・ワークツリー

- **ワークツリー重複**: メインリポジトリ内にネストされたワークツリー（`claude -w`）で CLAUDE.md・slash commands・agents・rules が重複する問題。
- **プラグインフック**: `/plugin` 操作後に Stop/SessionEnd 等のフックが発火しない問題。
- **同名フックコマンドの衝突**: 2 つのプラグインが同じ `${CLAUDE_PLUGIN_ROOT}/...` コマンドテンプレートを使用した場合にフックがサイレントにドロップされる問題。
- **スキル定義の colon 問題**: SKILL.md frontmatter のスキル説明にコロンが含まれる場合に読み込みに失敗する問題。
- **`description:` なしスキル**: frontmatter に `description:` フィールドがないプロジェクトスキルが利用可能スキルリストに表示されない問題。

## 技術的なポイント

- **セキュリティの多層改善**: スキルの gitignore バイパス、シンボリックリンクエスケープ、信頼ダイアログのサイレント承認という 3 つのセキュリティ問題が同時修正されました。サンドボックス環境の信頼性が向上しています。
- **メモリ管理の大規模改善**: React Compiler のメモ化リーク、REPL スコープ蓄積、フックイベント蓄積など、長時間セッションで顕在化するメモリリークが複数修正されました。長時間使用するユーザーほど恩恵を受けられます。
- **`InstructionsLoaded` フックイベント**: CLAUDE.md の読み込みをフックできるようになったことで、プロジェクトルール適用の監査やカスタム初期化処理の実装が可能になります。
- **`includeGitInstructions` 設定**: 組み込みの git ワークフロー指示をシステムプロンプトから除外できるため、独自のコミット規約を持つチームが Claude の挙動をより細かく制御できます。
- **MCP バイナリコンテンツの改善**: PDF や音声ファイルが raw base64 としてコンテキストに流れ込まなくなったことで、コンテキストウィンドウの無駄遣いを防げます。
- **Sonnet 4.6 への自動移行**: Pro/Max/Team Premium ユーザーは手動変更なしに最新モデルへ移行されます。

## まとめ

v2.1.69 はセキュリティ修正・メモリリーク解消・パフォーマンス改善を中心に据えた大型メンテナンスリリースです。特にメモリ周りの修正は多岐にわたり、長時間セッションや大規模マルチエージェントタスクを行うユーザーには実感できる改善が期待されます。同時に、スキル・プラグイン・フックといった拡張機能の信頼性も大幅に向上しており、Claude Code をカスタマイズして業務フローに組み込んでいる開発者にとっても重要なアップデートです。Voice STT の言語拡充により日本語以外の多言語環境での利用も広がり、グローバルなユーザーにとってもより使いやすいバージョンとなっています。

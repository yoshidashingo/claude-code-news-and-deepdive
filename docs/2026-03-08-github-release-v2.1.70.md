# Claude Code v2.1.70 リリース — VS Code 統合強化と大量バグ修正

## 概要

Claude Code v2.1.70 は、20 件以上のバグ修正と複数のパフォーマンス改善、そして VS Code 向けの新機能 3 件を含む大型アップデートです。Windows / WSL 環境での動作品質が大きく改善されたほか、Remote Control のポーリング頻度削減によりサーバー負荷が約 300 分の 1 に抑えられています。VS Code 拡張機能ではセッション管理 UI・プラン表示・MCP 管理ダイアログが追加されました。

## 新機能

### VS Code セッションマネージャー

VS Code のアクティビティバーにスパーク（⚡）アイコンが追加され、すべての Claude Code セッションを一覧表示できるようになりました。各セッションはフルエディターとして開くことができ、複数セッションの切り替えが格段に便利になります。

### VS Code プランビューア

Claude が生成するプラン（計画）を VS Code 上でフル Markdown ドキュメントとして表示できるようになりました。コメントを追加してフィードバックを提供できるため、エージェントとのやり取りがより直感的になります。

### VS Code ネイティブ MCP 管理ダイアログ

チャットパネルで `/mcp` コマンドを実行すると、MCP（Model Context Protocol）サーバーの有効化・無効化・再接続・OAuth 認証管理をターミナルを開かずに行えるネイティブダイアログが起動します。MCP サーバーの日常的な管理操作がグラフィカルに行えるようになりました。

## 改善点

### Remote Control ポーリング頻度の大幅削減

接続中の `/poll` レートが従来の 1〜2 秒から **10 分に 1 回**へと変更されました。これによりサーバー負荷が約 **300 分の 1** に削減されます。なお、接続が切れた場合は即座に高頻度ポーリングへ切り替わるため、再接続性能には影響しません。

### プロンプト入力の再レンダリング削減

ターン実行中のプロンプト入力の再レンダリング回数が約 **74% 削減**されました。入力フィールドの応答性が向上し、特に長いプロンプトを編集する際の体験が改善されます。

### コンパクション（要約）の最適化

コンパクション処理でサマライザーリクエストに画像が保持されるようになりました。プロンプトキャッシュが再利用可能になり、コンパクションが**より高速かつ低コスト**で実行されます。

### その他の改善

- `/rename` コマンドが Claude の処理中でも即時実行できるようになりました（従来はサイレントにキューイングされていました）
- スタートアップ時のメモリ使用量がカスタム CA 証明書を使用しないユーザーで約 **426KB 削減**されました
- マイクが無音を検出した場合と「音声が検出されない」場合のエラーメッセージが区別されるようになりました

## バグ修正

### API・モデル関連

- **`ANTHROPIC_BASE_URL` 使用時の API 400 エラー**: サードパーティゲートウェイ経由でのアクセス時に、`tool_reference` ブロックを誤って送信していた問題を修正。プロキシエンドポイントを正しく検出して自動的に無効化します。
- **effort パラメーターエラー**: カスタム Bedrock 推論プロファイルや標準的な Claude 命名パターンに合致しないモデル識別子を使用した際に発生する `API Error: 400 This model does not support the effort parameter` エラーを修正しました。
- **`ToolSearch` 後の空レスポンス**: サーバーがツールスキーマをシステムプロンプト形式のタグでプロンプト末尾にレンダリングするため、モデルが早期に停止していた問題を修正しました。

### キャッシュ・パフォーマンス

- **MCP `instructions` によるキャッシュバスト**: 最初のターン後に `instructions` を持つ MCP サーバーが接続された場合にプロンプトキャッシュが無効化されていた問題を修正しました。
- **フィーチャーフラグのキャッシュ**: 起動初期に読み込まれるフィーチャーフラグのディスクキャッシュが更新されず、古い値がセッション間で持続していた問題を修正しました。
- **`--resume` 時のスキルリスト再注入**: `--resume` のたびにスキルリストが再注入されていた問題を修正。**1 回のレジュームあたり約 600 トークンの節約**になります。

### Windows / WSL 固有

- **クリップボードの非 ASCII 文字化け**: Windows/WSL 環境で CJK 文字や絵文字がクリップボード経由でコピーした際に化けていた問題を修正。PowerShell の `Set-Clipboard` を使用することで解決しました。
- **VS Code 起動時の余分なウィンドウ**: Windows の VS Code 統合ターミナルから起動した場合に複数の VS Code ウィンドウが開いていた問題を修正しました。
- **Windows ネイティブバイナリでの音声モード**: `native audio module could not be loaded` エラーで音声モードが動作しなかった問題を修正しました。

### 音声・UI

- **push-to-talk の起動時不活性化**: `voiceEnabled: true` を設定しているにもかかわらず、セッション開始時に push-to-talk が有効にならなかった問題を修正しました。
- **SSH 接続時の Enter キー動作**: 低速な SSH 接続環境で Enter キーが送信ではなく改行を挿入していた問題を修正しました。
- **`AskUserQuestion` ダイアログのパフォーマンス**: ノート入力のキーストロークごとに Markdown レンダリングが再実行されていた性能劣化を修正しました。

### その他のバグ修正

- **Markdown リンクの `#NNN` 参照**: リンク内の `#NNN` が現在のリポジトリの Issue/PR へのリンクとして誤解釈されていた問題を修正しました。
- **「Model updated to Opus 4.6」通知の繰り返し**: `.claude/settings.json` に旧 Opus モデル文字列が固定されている場合に通知が繰り返し表示されていた問題を修正しました。
- **プラグイン表示の誤り**: `/plugin` コマンドでプラグインが誤ってインストール済みと表示される問題、およびマーケットプレイスインストール後に「not found」エラーが出る問題を修正しました。
- **`/security-review` コマンドのエラー**: 古いバージョンの git で `unknown option merge-base` エラーが発生していた問題を修正しました。
- **`/color` コマンドのデフォルト復元**: デフォルトカラーへ戻す方法がなかった問題を修正。`/color default`・`/color gray`・`/color reset`・`/color none` のいずれかで元のカラーに戻せるようになりました。
- **Remote 環境での `permissions.defaultMode`**: `acceptEdits` と `plan` 以外の値が Claude Code Remote 環境でも適用されていた問題を修正し、Remote では無視されるようになりました。
- **VS Code テレポートマーカー**: VS Code テレポートセッションでマーカーが表示されなかった問題を修正しました。

## 技術的なポイント

- **サードパーティ API ゲートウェイとの互換性向上**: `ANTHROPIC_BASE_URL` を使用したプロキシ構成でのツール呼び出しが正しく機能するようになりました。Azure OpenAI 互換エンドポイント等を利用している場合も安心して使えます。
- **Bedrock カスタムモデルのサポート改善**: 標準の `claude-*` 命名規則に合致しないカスタム推論プロファイルでも effort パラメーターエラーなく動作します。
- **Remote Control の効率化**: ポーリングレートを大幅削減したことで、大規模展開時のサーバーコストが実質的に削減されます。
- **Windows ネイティブ対応の強化**: クリップボード・音声・VS Code 統合の 3 点を修正し、Windows ネイティブバイナリの品質が大きく向上しました。
- **MCP 管理の GUI 化**: `/mcp` コマンドによるダイアログで OAuth 認証も含めた MCP サーバー管理がターミナル不要で行えます。エンタープライズ環境での MCP 導入ハードルが下がります。

## まとめ

v2.1.70 は新機能よりも品質改善に重点を置いたリリースです。特に Windows / WSL 環境での使用感、API ゲートウェイとの互換性、パフォーマンスの 3 点が大きく改善されました。VS Code 向けには、セッション管理・プランビューア・MCP 管理ダイアログという実用的な 3 機能が追加され、IDE からの操作効率が向上しています。Remote Control のポーリング削減は小さな変更に見えますが、チームや CI 環境での利用においてインフラコストに直結する重要な改善です。今後のリリースでも安定性とパフォーマンスの継続的な改善が期待されます。

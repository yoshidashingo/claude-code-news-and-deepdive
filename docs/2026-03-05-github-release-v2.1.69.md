# Claude Code v2.1.69 リリース — セキュリティ修正・メモリ最適化・多言語音声入力対応など大量アップデート

## 概要

Claude Code v2.1.69 は、セキュリティ修正・大規模なメモリリーク解消・パフォーマンス改善を中心に、新機能追加とバグ修正を多数含む大型リリースです。音声入力 (Voice STT) が 10 言語追加されて計 20 言語に対応し、プラグイン・フック・ワークツリー周りの信頼性が大幅に向上しています。

## 新機能

### `/claude-api` スキルの追加

Claude API および Anthropic SDK を使ったアプリケーション開発を支援する組み込みスキル `/claude-api` が追加されました。SDK の使い方やサンプルコードを Claude が直接サポートしてくれます。

### Voice STT — 10 言語追加で計 20 言語対応

音声入力 (Speech-to-Text) に以下の 10 言語が新たに追加されました。

- ロシア語、ポーランド語、トルコ語、オランダ語、ウクライナ語
- ギリシャ語、チェコ語、デンマーク語、スウェーデン語、ノルウェー語

従来の 10 言語と合わせて計 20 言語に対応しています。

### リモートコントロールのカスタムセッションタイトル

`/remote-control` コマンドおよび `claude remote-control` に任意の名前引数が追加されました。

```bash
# セッションタイトルを指定する
claude remote-control --name "My Project"
# または
/remote-control My Project
```

指定したタイトルが `claude.ai/code` のセッション一覧に表示されます。

### エフォートレベルの表示

ロゴとスピナーに現在のエフォート設定（例: `with low effort`）が表示されるようになりました。どのエフォートレベルで動作しているかを一目で確認できます。

### `includeGitInstructions` 設定

Claude のシステムプロンプトに組み込まれている Git コミット・PR ワークフローの指示を無効化できる設定が追加されました。

```bash
# 環境変数で無効化
CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS=1 claude
```

独自のコミットフローを定義しているチームで活用できます。

### `/reload-plugins` コマンド

Claude Code を再起動せずにプラグインの変更を反映できる `/reload-plugins` コマンドが追加されました。プラグイン開発の反復サイクルが短縮されます。

### フック・プラグインの強化

- `InstructionsLoaded` フックイベント: CLAUDE.md や `.claude/rules/*.md` がコンテキストにロードされたタイミングで発火
- フックイベントに `agent_id`（サブエージェント）・`agent_type` フィールドを追加
- ワークツリーセッションでは `worktree` フィールド（名前、パス、ブランチ、元リポジトリのディレクトリ）をステータスラインフックに追加
- プラグインソースタイプに `git-subdir`（git リポジトリ内のサブディレクトリ指定）を追加
- `${CLAUDE_SKILL_DIR}` 変数: SKILL.md 内でスキル自身のディレクトリを参照可能

### その他の新設定

- `sandbox.enableWeakerNetworkIsolation`（macOS 限定）: カスタム MITM プロキシ使用時に `gh`、`gcloud`、`terraform` などの Go 製ツールが TLS 証明書を検証できるよう許可
- `oauth.authServerMetadataUrl`: MCP サーバー向けのカスタム OAuth メタデータ発見 URL を指定可能
- `pluginTrustMessage`: プラグインのインストール時に表示されるトラスト警告に組織固有のメッセージを追加
- `pathPattern`（`strictKnownMarketplaces`）: ファイル・ディレクトリのマーケットプレイスソースを正規表現でマッチング

### UI の操作性改善

- テンキー（Numeric Keypad）でインタビュー選択肢を選べるように（従来は QWERTY 上部の数字行のみ対応）
- 空の bash プロンプト（`!`）で Ctrl+U を押すと bash モードを終了（`Escape` や `Backspace` と同様の動作）
- `claude --agent` 使用時にターミナルタイトルにエージェント名を表示

## バグ修正

### セキュリティ関連

- **ネストされたスキル探索の脆弱性修正**: `node_modules` など `.gitignore` で除外されたディレクトリからスキルが読み込まれていた問題を修正
- **シンボリックリンク経由のエスケープ修正**: `acceptEdits` モードでシンボリックリンク経由の新規ファイル書き込みがワーキングディレクトリ外に漏洩する問題を修正
- **サンドボックスドメインバイパスの修正**: `allowManagedDomainsOnly` が有効な場合に非許可ドメインが自動ブロックされず確認プロンプトが表示されていた問題を修正
- **トラストダイアログの修正**: 初回実行時に `.mcp.json` の全サーバーが暗黙的に有効化されていた問題を修正し、サーバーごとの承認ダイアログが正しく表示されるように

### 認証・クレデンシャル

- **macOS キーチェーン破損の修正**: 複数の OAuth MCP サーバー使用時に大きな OAuth メタデータが stdin バッファをオーバーフローし、繰り返し `/login` を要求する問題を修正
- **`.credentials.json` の `subscriptionType` 消失の修正**: プロファイルエンドポイントの一時的な失敗でサブスクリプション種別が "Claude API" にリセットされる問題を修正（[#30185](https://github.com/anthropics/claude-code/issues/30185)）

### ファイル・ワークツリー操作

- **Linux のゴーストドットファイル修正**: サンドボックス化された Bash コマンド実行後に `.bashrc`、`HEAD` などが未追跡ファイルとして表示されていた問題を修正
- **ネストされたワークツリーでの重複修正**: メインリポジトリ内にネストされたワークツリーで CLAUDE.md、スラッシュコマンド、エージェント、ルールが重複していた問題を修正
- **Windows でのワークツリーファイルコピー修正**
- **Windows でのグローバル `.claude` フォルダ検出修正**

### 会話・セッション管理

- **コンパクション後のエラー修正**: ツールバッチ途中で中断されたセッション再開時の API 400 エラーを修正
- **孤立したツール結果の修正**: 孤立した `tool_result` ブロックで始まる会話の再開時に発生していた "unexpected tool_use_id" エラーを修正
- **ネストされたチームメイトの修正**: Agent ツールの `name` パラメータ経由で誤ってネストされたチームメイトが生成されていた問題を修正
- **`CLAUDE_CODE_MAX_OUTPUT_TOKENS` がコンパクション中に無視される問題を修正**
- **`/clear` の完全クリア**: セッションキャッシュが完全に消去されるよう修正

### UI・操作性

- **Ghostty over SSH での Shift+Enter**: `[27;2;13~` が出力される代わりに正しく改行が挿入されるよう修正
- **Ctrl+O のフリーズ修正**: 長いセッションでトランスクリプトトグルが数秒固まる問題を修正
- **プランモードの複数行入力**: フィードバック入力ボックスでバックスラッシュ+Enter・Shift+Enter による改行が可能に
- **`/stats` クラッシュ修正**: タイムスタンプが欠損・不正なエントリを含むトランスクリプトでクラッシュしていた問題を修正
- **Escape でのターン中断修正**: 入力ボックスにドラフトテキストがある場合に Escape がターン中断として機能しない問題を修正

### プラグイン・フックシステム

- **フック非発火の修正**: `/plugin` 操作後に Stop/SessionEnd などのフックが発火しない問題を修正
- **重複フックの修正**: 2 つのプラグインが同じコマンドテンプレートを使用している場合にフックがドロップされる問題を修正
- **インタラクティブツールの自動許可修正**: スキルの許可ツールリストに `AskUserQuestion` などが含まれている場合に空の回答で自動許可されていた問題を修正
- **`WorktreeCreate` / `WorktreeRemove` フックの無視修正**

### モデル・SDK 関連

- **Opus モデル解決の修正**: `--model claude-opus-4-0` / `--model claude-opus-4-1` が非推奨バージョンに解決されていた問題を修正
- **`remote-control` クラッシュ修正**: npm インストール時の "bad option: --sdk-url" エラーを修正（[#28334](https://github.com/anthropics/claude-code/issues/28334)）
- **[VSCode] RTL テキスト修正**: アラビア語・ヘブライ語・ペルシャ語のテキストが逆順に表示されていたリグレッション（v2.1.63 以降）を修正

## 改善点

### パフォーマンス

- **スピナーの最適化**: 50ms アニメーションループを外部シェルから分離し、ターン中の描画・CPU オーバーヘッドを軽減
- **起動速度の改善**:
  - `--worktree` 起動パスからの git サブプロセスを削除
  - エンタープライズ・チームユーザー向けの不要なキーチェーンルックアップを削除
  - Yoga WASM のプリロードを遅延化（ベースラインメモリ約 16MB 削減）
- **MCP 起動の改善**: claude.ai 設定取得をローカル接続とパイプライン化し、逐次バッチ処理の代わりに並行プールを使用
- **ファイル操作の改善**: ファイルの存在確認のためにコンテンツを読み込まないよう修正（6 箇所）

### メモリ使用量の大幅削減

- React Compiler の `memoCache` に古いメッセージ配列バージョンが蓄積するメモリリークを修正
- REPL のレンダリングスコープが長いセッションで蓄積するメモリリーク（1000 ターンで約 35MB）を修正
- インプロセスのチームメイトで親の会話履歴が GC されずに残るメモリリークを修正
- フックイベントが長いセッションで無限に蓄積するメモリリークを修正
- SDK・CCR セッションのベースラインメモリを削減（stream-json 出力の最適化）
- 大きなセッションの再開時のメモリ削減

### バイナリコンテンツ処理の改善

MCP ツールが PDF・Office ドキュメント・音声ファイルを返す場合、生の base64 データを会話コンテキストに流し込む代わりに、正しい拡張子でディスクに保存するようになりました。`WebFetch` も同様にバイナリレスポンスをサマリーとともに保存します。

### ユーザー体験の変更

- **Sonnet 4.5 → Sonnet 4.6 の自動移行**: Pro/Max/Team Premium ユーザーは自動的に Sonnet 4.6 に移行
- **`/resume` ピッカーの改善**: 最初のプロンプトではなく最新のプロンプトを表示
- **コンパクション後の再開**: 不要な要約まとめをスキップして会話を継続
- **[VSCode] コンパクション表示**: 折りたたみ可能な "Compacted chat" カードとして表示

## 技術的なポイント

- **セキュリティ修正が複数**: スキル探索のパストラバーサル、シンボリックリンクエスケープ、サンドボックスドメインバイパス、OAuth サーバーの暗黙的信頼など、複数のセキュリティ問題が修正されており、アップデートを優先すべきリリースです
- **メモリリーク修正の規模**: React の `memoCache`、REPL スコープ、チームメイト履歴、フックイベントなど複数の独立したメモリリークが同時に修正されており、長時間セッションでの安定性が大幅に向上します
- **`includeGitInstructions` 設定**: システムプロンプトに組み込まれた Git 指示を無効化する機能は、独自のワークフローやルール（`.claude/rules/` 等）で Git 操作を完全に制御したいチームにとって重要な設定です
- **`InstructionsLoaded` フック**: CLAUDE.md や rules ファイルのロードタイミングを捕捉できるため、ルール適用の検証や動的な設定変更に活用できます
- **MCP バイナリコンテンツの改善**: PDF・音声などを扱う MCP ツールとの統合が実用的になり、マルチモーダルなワークフローの構築が容易になります
- **`worktree` フィールドのフック追加**: ワークツリー環境を意識したフック処理（プロジェクト別の通知、ログ分離など）が実装しやすくなります

## まとめ

v2.1.69 は機能追加よりも**品質・安定性・セキュリティの強化**に重点を置いたリリースです。特に複数のセキュリティ修正とメモリリーク解消は、本番環境や長時間セッションでの運用に直接影響する重要な改善です。

音声入力の 20 言語対応や `includeGitInstructions` 設定など開発ワークフローを柔軟にする機能も充実しており、チーム・エンタープライズ向けのポリシー管理機能も着実に拡充されています。セキュリティ修正を含むため、早期のアップデートが推奨されます。

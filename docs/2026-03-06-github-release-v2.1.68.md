# Claude Code v2.1.68: Opus 4.6 のデフォルト設定変更と ultrathink キーワードの復活

## 概要

Claude Code v2.1.68 では、Max および Team プランのユーザー向けに Opus 4.6 のデフォルト推論努力レベルが「medium」に変更されました。また、高努力モードを一時的に有効化する `ultrathink` キーワードが復活し、旧モデル（Opus 4 / Opus 4.1）がファーストパーティ API から削除されました。

## 新機能

### ultrathink キーワードの復活

以前に廃止されていた `ultrathink` キーワードが再び利用可能になりました。プロンプトに `ultrathink` と記述することで、**次のターンのみ**高努力（high effort）モードで推論を行わせることができます。

特定の複雑なタスクに対してだけ高努力モードを使いたい場面で便利です。例えば、難解なアーキテクチャ設計の検討や、複雑なバグの原因分析などに活用できます。

```
ultrathink この設計のトレードオフを詳細に分析してください。
```

## 改善点

### Opus 4.6 のデフォルト努力レベルが medium に

Max および Team プランのユーザーに対して、Opus 4.6 のデフォルト推論努力レベルが **medium** になりました。

medium effort は「速度」と「徹底的な推論」のバランスが取れた設定であり、ほとんどのタスクに適しているとされています。デフォルトのまま使い続けても品質面での問題はなく、必要に応じて `/model` コマンドでいつでも変更できます。

- **low effort**: 速度優先、簡単なタスク向け
- **medium effort (新デフォルト)**: バランス重視、大多数のタスクに最適
- **high effort**: 精度優先、複雑な推論が必要なタスク向け（`ultrathink` で一時的に有効化）

## Breaking Changes

### Opus 4 および Opus 4.1 の廃止

ファーストパーティ API 経由の Claude Code から **Opus 4** および **Opus 4.1** が削除されました。これらのモデルをピン留めしていたユーザーは、自動的に **Opus 4.6** へ移行されます。

手動での対応は不要ですが、モデルの挙動が変わる可能性があるため、移行後の動作確認を推奨します。現在使用しているモデルは `/model` コマンドで確認できます。

## 技術的なポイント

- `ultrathink` キーワードはターン単位で有効になるため、常時高努力モードにするよりもコスト効率が良い
- medium effort はほとんどのユースケースをカバーするよう設計されており、デフォルト変更によってレスポンス速度の向上が期待できる
- Opus 4 / 4.1 から Opus 4.6 への自動移行は、ユーザー側の操作なしに行われる
- モデルや努力レベルの変更は `/model` コマンドからいつでも実行可能

## まとめ

v2.1.68 は、Opus 4.6 を Claude Code の標準として確立するリリースです。medium effort をデフォルトにすることで、大多数のユーザーに速度と品質のバランスが取れた体験を提供しつつ、`ultrathink` キーワードで必要なときだけ高精度な推論を呼び出せる柔軟性も確保されました。旧モデルの廃止による影響は自動移行で最小化されていますが、移行後の動作確認はしておくと安心です。

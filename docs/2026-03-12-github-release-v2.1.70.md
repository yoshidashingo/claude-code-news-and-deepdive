# Claude Code v2.1.70 リリース — 大量のバグ修正とVS Code新機能

## 概要

Claude Code v2.1.70 は、API接続・Windows環境・音声機能を中心とした多数のバグ修正と、パフォーマンス改善を含むリリースです。VS Code 向けには、セッション管理・プラン表示・MCP サーバー管理の3つの新機能が追加されました。特にリモートコントロールのポーリング頻度が大幅に削減されており、サーバー負荷の観点でも重要な改善となっています。

## 新機能

### VS Code: セッション一覧のSparkアイコン

VS Code のアクティビティバーに Spark アイコンが追加されました。このアイコンをクリックすると、現在アクティブなすべての Claude Code セッションが一覧表示され、各セッションをフルエディタとして開くことができます。複数のセッションを並行して管理しているユーザーにとって、切り替え操作が格段に改善されます。

### VS Code: プランのマークダウンドキュメント表示

VS Code 上でプランをフルマークダウンドキュメントとして閲覧できるようになりました。またプランに対してコメントを追加してフィードバックを提供する機能も追加されています。テキストベースのチャットパネル内での確認よりも、構造化されたプランの全体像を把握しやすくなります。

### VS Code: ネイティブMCPサーバー管理ダイアログ

チャットパネルで `/mcp` コマンドを実行すると、MCP（Model Context Protocol）サーバーの管理ダイアログが開くようになりました。このダイアログからサーバーの有効/無効切り替え・再接続・OAuth 認証管理が行えます。これまでターミナルを切り替えて操作する必要があった MCP の管理が、VS Code 内で完結するようになります。

## 改善点

### パフォーマンス大幅改善

- **プロンプト入力の再レンダリング削減（約74%）**: ターン中の不要な UI 再描画が大幅に減少し、入力のレスポンスが向上します。
- **リモートコントロールのポーリング間隔を最適化**: `/poll` の頻度が接続中は 1〜2秒から10分に変更されました。サーバー負荷が約300倍削減される大きな改善です。なお、切断検知時は即座に高頻度ポーリングに戻るため、再接続速度には影響ありません。
- **起動時メモリ削減（約426KB）**: カスタム CA 証明書を使用していないユーザーの起動時メモリ使用量が削減されました。
- **コンパクション時に画像を保持**: サマライザーリクエストで画像が保持されるようになり、プロンプトキャッシュの再利用が可能になりました。より高速かつ低コストなコンパクションが実現します。

### その他の改善

- マイクがサイレンス（無音）を検出した際のエラーメッセージが改善され、「音声が検出されない」との区別が明確になりました。
- `/rename` コマンドが Claude の処理中でも即時実行できるようになりました（以前はサイレントにキューイングされていました）。

## バグ修正

### API・接続関連

- **`ANTHROPIC_BASE_URL` 使用時の API 400 エラーを修正**: サードパーティゲートウェイ経由でAPIを使用している場合、`tool_reference` ブロックが誤って送信されていました。プロキシエンドポイントを正しく検出し、該当ブロックを無効化するよう修正されました。
- **カスタム Bedrock プロファイル使用時の `effort parameter` エラーを修正**: カスタム Bedrock 推論プロファイルや、標準的な Claude 命名パターンに一致しないモデル識別子を使用している場合に発生していた `API Error: 400 This model does not support the effort parameter` が修正されました。
- **`ToolSearch` 直後の空レスポンスを修正**: `ToolSearch` 実行直後にモデルが空のレスポンスを返す問題が修正されました。サーバーがツールスキーマをシステムプロンプトスタイルのタグとしてプロンプト末尾に挿入することで、モデルが早期停止してしまうケースへの対処です。
- **MCP サーバー接続時のプロンプトキャッシュバストを修正**: `instructions` を持つ MCP サーバーが最初のターン後に接続した場合、プロンプトキャッシュが無効化される問題が修正されました。

### 入力・SSH関連

- **遅い SSH 接続でのEnterキー問題を修正**: 低速な SSH 接続環境で Enter キーを押した際に、送信ではなく改行が挿入されてしまう問題が修正されました。

### Windows/WSL関連

- **クリップボードでの非 ASCII 文字破損を修正**: Windows/WSL 環境で CJK 文字や絵文字をクリップボードにコピーすると文字化けする問題が修正されました。PowerShell の `Set-Clipboard` を使用することで正しく処理されます。日本語ユーザーにとって特に重要な修正です。
- **VS Code 起動時の余分なウィンドウ生成を修正**: VS Code の統合ターミナルから起動した際に、余分な VS Code ウィンドウが開いてしまう問題が修正されました。
- **音声モードの起動失敗を修正**: Windows ネイティブバイナリで `native audio module could not be loaded` エラーにより音声モードが使用できなかった問題が修正されました。
- **プッシュトゥトーク が動作しない問題を修正**: `voiceEnabled: true` を設定していても、セッション開始時にプッシュトゥトークが有効にならない問題が修正されました。

### UI・コマンド関連

- **マークダウンリンクの `#NNN` 参照先を修正**: マークダウンリンク内に `#NNN` 形式の参照が含まれている場合、リンク先 URL ではなく現在のリポジトリの Issue を指してしまう問題が修正されました。
- **「Model updated to Opus 4.6」通知の繰り返し表示を修正**: プロジェクトの `.claude/settings.json` に旧 Opus モデル文字列が固定されている場合に、モデル更新通知が繰り返し表示される問題が修正されました。
- **プラグインの表示状態を修正**: プラグインが誤って「インストール済み」と表示される問題、およびマーケットプレイスインストール後の初回起動時に「見つかりません」エラーが表示される問題が修正されました。後者はインストール後に自動リフレッシュを行うことで対処しています。
- **`/security-review` コマンドを修正**: 古い git バージョンで `unknown option merge-base` エラーにより `/security-review` が実行できなかった問題が修正されました。
- **`/color` コマンドにリセット機能を追加**: `/color` コマンドでデフォルトカラーに戻す方法がなかった問題が修正されました。`/color default`、`/color gray`、`/color reset`、`/color none` のいずれでもデフォルトカラーに戻せるようになりました。
- **`AskUserQuestion` プレビューダイアログのパフォーマンス回帰を修正**: notes 入力フィールドへのキーストロークのたびにマークダウンレンダリングが再実行されていたパフォーマンス問題が修正されました。
- **起動時の feature flags キャッシュ問題を修正**: 起動初期段階で読み込まれた feature flags がディスクキャッシュを更新しないため、古い値がセッションをまたいで維持される問題が修正されました。
- **Remote 環境での `permissions.defaultMode` 設定を修正**: Claude Code Remote 環境では `acceptEdits` と `plan` 以外の `permissions.defaultMode` 値が正しく無視されるようになりました。
- **`--resume` 時のスキルリスト再挿入を修正**: `--resume` のたびにスキルリストが再挿入されていた問題が修正され、1回の resume あたり約600トークンが節約されます。
- **VS Code テレポートセッションのマーカー表示を修正**: VS Code テレポートセッションでテレポートマーカーが正しくレンダリングされない問題が修正されました。

## 技術的なポイント

- **サードパーティ API ゲートウェイ利用者は恩恵大**: `ANTHROPIC_BASE_URL` を使ったプロキシ経由の接続で発生していた 400 エラーが修正されたため、Azure OpenAI 互換エンドポイントや自社ゲートウェイを使用している環境での安定性が向上します。
- **Windows 環境の品質向上**: クリップボード・音声・VS Code 起動・WSL など Windows 固有の複数の問題がまとめて修正されました。Windows ユーザーはアップデートを強くお勧めします。特に日本語等の非 ASCII 文字を扱うユーザーにはクリップボード修正が重要です。
- **Remote Control のポーリング削減はスケーラビリティに貢献**: 接続中のポーリング間隔が1〜2秒から10分になることで、サーバー負荷が約300倍削減されます。Claude Code Remote を大規模に利用している組織にとって大きな改善です。
- **`--resume` のトークン節約**: スキルリストの再挿入が防がれたことで、resume 操作ごとに約600トークンが節約されます。頻繁に resume を使うワークフローではコスト削減に直結します。
- **VS Code 統合の強化**: セッション管理・プラン閲覧・MCP サーバー管理がすべて VS Code 内で完結するようになりました。IDE を離れることなく Claude Code の主要機能を操作できる環境が整いつつあります。

## まとめ

v2.1.70 は新機能よりもバグ修正と安定性向上に重点を置いたリリースです。特に Windows ユーザー、サードパーティ API ゲートウェイ利用者、MCP を活用している開発者にとって、実際の作業上の問題が多数解消されています。またリモートコントロールのポーリング削減やプロンプト入力レンダリングの最適化など、日常的な操作感にも影響するパフォーマンス改善が含まれています。VS Code 向けの新機能は、IDE 中心のワークフローをさらに強化する方向性を示しており、今後のアップデートでも継続的な拡充が期待されます。

# Claude Code v2.1.70 リリース — 大規模バグ修正とVS Code統合の強化

## 概要

Claude Code v2.1.70 は、API互換性・クリップボード・音声モード・プラグイン管理など幅広い領域にまたがる20件以上のバグ修正を含むリリースです。パフォーマンス改善（プロンプト再レンダリング約74%削減）や VS Code 向けの新機能（セッション一覧・プラン表示・MCP管理ダイアログ）も追加されています。

## 新機能

### VS Code 統合の大幅強化

VS Code ユーザー向けに3つの新機能が追加されました。

**セッション一覧（スパークアイコン）**

VS Code のアクティビティバーにスパークアイコンが追加され、現在起動中のすべての Claude Code セッションを一覧表示できるようになりました。セッションはフルエディタとして開くことができ、マルチセッションの切り替えが格段に便利になります。

**プランのMarkdownドキュメントビュー**

プランをVS Code 内でフルMarkdownドキュメントとして表示できるようになりました。コメントを追加してフィードバックを送ることも可能です。

**ネイティブ MCP サーバー管理ダイアログ**

チャットパネルで `/mcp` コマンドを実行すると、ネイティブのMCPサーバー管理ダイアログが開きます。サーバーの有効化・無効化・再接続・OAuth認証をターミナルを切り替えることなく操作できます。

## 改善点

### パフォーマンス改善

- **プロンプト入力の再レンダリングを約74%削減**: タイピング中の描画コストが大幅に下がり、長い入力でも快適に操作できます
- **起動時メモリを約426KB削減**: カスタムCA証明書を使用していないユーザーが対象です
- **Remote Control の `/poll` レートを約300倍削減**: 接続中のポーリング間隔を1〜2秒から10分に変更。サーバー負荷が大幅に減少します。接続断からの再接続は影響を受けず、即座にポーリングが再開されます

### UX・操作性の向上

- **`/rename` コマンドの即時反映**: Claude が処理中でも `/rename` が使えるようになりました。以前はサイレントにキューに入れられ、処理完了後まで待つ必要がありました
- **マイクエラーメッセージの改善**: 無音キャプチャと「音声が検出されなかった」ケースを明確に区別するエラーメッセージが表示されるようになりました
- **コンパクションの改善**: サマライザーリクエストで画像が保持されるようになり、プロンプトキャッシュの再利用によるコンパクションの高速化・低コスト化が実現します

## バグ修正

### API・モデル互換性

- **サードパーティゲートウェイでの API 400 エラー修正**: `ANTHROPIC_BASE_URL` にプロキシエンドポイントを設定している場合、ツール検索がプロキシを正しく検出して `tool_reference` ブロックを無効化するようになりました
- **カスタム Bedrock 推論プロファイルでの 400 エラー修正**: `API Error: 400 This model does not support the effort parameter` が、標準的なClaudeの命名パターンに一致しないモデル識別子（カスタム Bedrock 推論プロファイルなど）で発生していた問題を修正しました
- **ToolSearch 直後の空レスポンス修正**: サーバーがツールスキーマをシステムプロンプト形式のタグとしてプロンプト末尾にレンダリングすることで、モデルが早期停止していた問題を修正しました

### プロンプトキャッシュ・MCP

- **MCPサーバー接続時のキャッシュバスト修正**: `instructions` を持つ MCP サーバーが初回ターン後に接続した際にプロンプトキャッシュがバストされていた問題を修正しました
- **`--resume` 時のスキル注入修正**: セッション再開のたびにスキルリストが再注入されていた問題を修正。1回の再開あたり約600トークンの節約になります

### Windows・SSH環境

- **CJK・絵文字のクリップボード文字化け修正**: Windows/WSL 環境でクリップボードへのコピー時に非ASCII文字（CJK文字や絵文字）が文字化けしていた問題を、PowerShell の `Set-Clipboard` に切り替えることで修正しました
- **Windows 起動時の余分な VS Code ウィンドウ修正**: VS Code 統合ターミナルから起動した際に余分なウィンドウが開いていた問題を修正しました
- **SSH経由でのEnterキー動作修正**: 低速SSH接続でEnterキーが送信ではなく改行を挿入していた問題を修正しました

### 音声モード

- **Windows ネイティブバイナリでの音声モード修正**: `native audio module could not be loaded` エラーで音声モードが起動できなかった問題を修正しました
- **起動時のプッシュトゥトーク修正**: `voiceEnabled: true` を設定に記述していても、セッション開始時にプッシュトゥトークが有効にならなかった問題を修正しました

### UI・コマンド

- **Markdownリンクの `#NNN` 参照修正**: `#NNN` 形式のフラグメントを含むMarkdownリンクが、リンク先ではなく現在のリポジトリのIssueに誤って紐付けられていた問題を修正しました
- **「Model updated to Opus 4.6」通知の重複修正**: `.claude/settings.json` にレガシーなOpusモデル文字列が設定されている場合に、同じ通知が繰り返し表示されていた問題を修正しました
- **プラグイン管理の修正**: `/plugin` でプラグインが誤って「インストール済み」と表示される問題と、起動直後に「マーケットプレイスで見つかりません」と表示される問題をそれぞれ修正しました
- **`/security-review` の git バージョン互換性修正**: 古い git バージョンで `unknown option merge-base` エラーが発生していた問題を修正しました
- **`/color` コマンドのリセット対応**: デフォルトカラーに戻す手段がなかった問題を修正。`/color default`・`/color gray`・`/color reset`・`/color none` のいずれでもデフォルトに戻せるようになりました
- **`AskUserQuestion` ダイアログのパフォーマンス修正**: ノート入力欄のキー入力ごとにMarkdownが再レンダリングされていたパフォーマンス問題を修正しました

### その他

- **機能フラグのキャッシュ修正**: 起動直後に読み込まれた機能フラグのディスクキャッシュが更新されず、古い値がセッションをまたいで引き継がれていた問題を修正しました
- **Remote 環境での権限設定修正**: `permissions.defaultMode` に `acceptEdits` と `plan` 以外の値が設定されている場合、Claude Code Remote 環境でも誤って適用されていた問題を修正。現在は無視されるようになりました
- **VS Code テレポートマーカーの修正**: テレポートされたセッションでテレポートマーカーが表示されなかった問題を修正しました

## 技術的なポイント

- **プロキシ対応の強化**: `ANTHROPIC_BASE_URL` でサードパーティゲートウェイを利用しているユーザーは、`tool_reference` ブロック関連の 400 エラーが自動的に解消されます
- **Bedrock カスタムプロファイル対応**: 標準の Claude 命名規則に従わないモデル ID を使用している場合も effort パラメータのエラーが発生しなくなりました
- **Windows 環境の品質向上**: クリップボード・音声モード・起動動作など、Windows 固有の問題が集中的に修正されており、Windows/WSL ユーザーにとって大きな改善となります
- **VS Code 統合のネイティブ化**: MCP 管理やプラン表示が VS Code の UI に統合され、ターミナルを行き来する必要が減りました
- **Remote Control の効率化**: ポーリング間隔の大幅延長により、Claude Code を Remote 環境で常時接続して使うワークフローでのサーバー負荷が劇的に改善されます

## まとめ

v2.1.70 は新機能よりもバグ修正と安定性向上に重点を置いたリリースです。特に Windows・WSL 環境でのクリップボード・音声モード・起動動作の修正は、非ASCII言語（日本語・中国語・韓国語など）のユーザーやWindows環境のユーザーにとって直接的な恩恵があります。

VS Code 向けの3つの新機能（セッション一覧・プランビュー・MCP管理ダイアログ）は、VS Code を主な開発環境として使用しているユーザーのワークフローを大きく改善します。また、プロンプト再レンダリングの74%削減やRemote Controlのポーリング最適化など、体感パフォーマンスに影響する改善も含まれており、日常的な使い勝手の向上も期待できます。

# Claude Code v2.1.74 リリース — /context コマンド強化とメモリリーク修正

## 概要

Claude Code v2.1.74 がリリースされました。本リリースでは `/context` コマンドへの実用的な最適化提案機能の追加や、`autoMemoryDirectory` 設定の導入といった新機能のほか、Node.js パスでのメモリリーク修正、MCP OAuth 認証の不具合修正、RTL テキストのレンダリング修正など、多岐にわたるバグ修正が含まれています。

## 新機能

### `/context` コマンドへのアクション可能な提案

`/context` コマンドに、コンテキスト最適化のための具体的な提案機能が追加されました。コンテキストを大量消費しているツール、メモリの肥大化、容量警告を自動的に検出し、それぞれに対する最適化のヒントを提示します。

コンテキスト管理に悩んでいる開発者にとって特に役立つ改善です。長時間のセッションや大規模プロジェクトで「何がコンテキストを圧迫しているのか」を把握しやすくなります。

### `autoMemoryDirectory` 設定の追加

自動メモリの保存先ディレクトリをカスタマイズできる `autoMemoryDirectory` 設定が追加されました。プロジェクトやチームの構成に合わせてメモリファイルの格納場所を指定できるようになります。

## バグ修正

### メモリリークの修正（重要）

**Node.js/npm パスでのメモリリーク**が修正されました。ストリーミング API レスポンスのバッファが、ジェネレーターが途中で終了された際に解放されていなかった問題です。これにより RSS（Resident Set Size）が際限なく増加する現象が発生していました。長時間稼働する環境やバックグラウンド処理を多用する場合に顕著な影響があったため、今回の修正は重要度が高いといえます。

### ポリシー・設定関連

- **マネージドポリシーのバイパス問題を修正**: マネージドポリシーの `ask` ルールが、ユーザーの `allow` ルールやスキルの `allowed-tools` によって上書きされてしまうセキュリティ上の問題を修正しました。

- **エージェントでのフル Model ID が無視される問題を修正**: `claude-opus-4-5` のようなフル形式の Model ID が、エージェントの frontmatter `model:` フィールドや `--agents` JSON 設定で黙って無視されていた問題を修正しました。エージェントが `--model` と同じ形式のモデル値を受け付けるようになりました。

### MCP OAuth 認証関連

- **コールバックポートが使用中の場合にハングする問題を修正**: MCP OAuth 認証で、コールバックポートがすでに使用中の場合に処理がハングしていた問題を修正しました。

- **リフレッシュトークン期限切れ後の再認証プロンプトが表示されない問題を修正**: HTTP 200 でエラーを返す OAuth サーバー（Slack など）において、リフレッシュトークンが期限切れになっても再認証を促すプロンプトが表示されない問題を修正しました。

### 音声・ターミナル関連

- **macOS でのボイスモード失敗を修正**: ターミナルにマイクの権限を一度も付与していないユーザーで、macOS ネイティブバイナリのボイスモードがサイレントに失敗する問題を修正しました。バイナリに `audio-input` エンタイトルメントが追加され、macOS が適切にマイクアクセスの許可を求めるようになります。

- **RTL テキストのレンダリング修正**: ヘブライ語・アラビア語などの右横書き（RTL）テキストが Windows Terminal、conhost、VS Code 統合ターミナルで正しくレンダリングされない問題を修正しました。

- **Windows での LSP サーバー問題を修正**: ファイル URI が不正な形式になっていたことで、Windows 上で LSP サーバーが動作しない問題を修正しました。

### フック・プラグイン関連

- **`SessionEnd` フックのタイムアウトを設定可能に**: `SessionEnd` フックが `hook.timeout` 設定に関わらず終了時に 1.5 秒後に強制終了されていた問題を修正しました。`CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` 環境変数でタイムアウト時間を設定できるようになりました。

- **REPL 内での `/plugin install` 失敗を修正**: ローカルソースを持つマーケットプレイスプラグインの REPL 内インストールが失敗する問題を修正しました。

- **マーケットプレイス更新時の git サブモジュール同期を修正**: プラグインのアップデート時に git サブモジュールが同期されず、サブモジュール内のプラグインソースが壊れる問題を修正しました。

- **不明なスラッシュコマンドへの入力が消える問題を修正**: 引数付きの不明なスラッシュコマンドを実行した際、入力内容が黙って破棄されていた問題を修正しました。入力内容が警告として表示されるようになります。

- **`--plugin-dir` の動作変更**: `--plugin-dir` で指定したローカル開発用コピーが、同名のインストール済みマーケットプレイスプラグインを上書きするようになりました（マネージド設定で強制有効化されている場合を除く）。プラグイン開発時のワークフローが改善されます。

### VS Code 統合

- **Untitled セッションの削除ボタンが機能しない問題を修正**
- **統合ターミナルのスクロールホイール応答性を改善**: ターミナル対応の加速処理により、スクロール操作がより快適になりました。

## 技術的なポイント

- **メモリリーク対策**: ストリーミングレスポンスを多用するワークロードでは RSS の肥大化が問題になりやすいため、今回の修正は長期稼働環境での安定性向上に直結します。
- **ポリシー管理の堅牢化**: マネージドポリシーが `allow` ルールに上書きされる問題の修正は、企業環境でのセキュリティポリシー適用に重要です。
- **`CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS`**: 後処理が重いフックを使用している場合、この環境変数でタイムアウトを延長することで予期せぬ中断を防げます。
- **エージェント設定のモデル指定**: フル Model ID（例: `claude-opus-4-5`）がエージェント設定で正しく認識されるようになったため、エージェントファイルでのモデル設定が柔軟になります。
- **Windows での LSP 修正**: Windows ユーザーは LSP を利用した言語サーバー機能（補完・定義ジャンプなど）が正常に動作するようになります。

## まとめ

v2.1.74 は新機能よりもバグ修正が中心のリリースですが、メモリリークやポリシーバイパスといった重要度の高い問題が解決されています。特に Node.js パスでのメモリリーク修正は、長時間稼働環境での安定性に大きく貢献します。また、Windows・macOS それぞれのプラットフォーム固有の問題も多数修正されており、クロスプラットフォームでの品質向上が図られています。`/context` コマンドの最適化提案機能は、大規模プロジェクトでのコンテキスト管理を改善するための実用的な機能として注目です。

# Claude Code v2.1.71 リリース — /loop コマンドとcronスケジューリング、多数のバグ修正

## 概要

Claude Code v2.1.71 では、定期的なプロンプト実行を自動化する `/loop` コマンドとcronスケジューリングツールが新たに追加されました。また、音声モードやプラグイン、フォーク会話など多岐にわたる分野で 20 件を超えるバグ修正が行われており、全体的な安定性と起動パフォーマンスが大きく向上しています。

## 新機能

### `/loop` コマンド — 定期実行の自動化

セッション内でプロンプトやスラッシュコマンドを一定間隔で繰り返し実行できる `/loop` コマンドが追加されました。

```
/loop 5m check the deploy
```

上記の例では、`check the deploy` というプロンプトを 5 分ごとに自動実行します。デプロイ監視やステータスチェックなど、定期的な確認作業を Claude Code に委ねたい場合に活用できます。

### cron スケジューリングツール

セッション内での定期プロンプト実行を支援するcronスケジューリングツールも追加されました。`/loop` コマンドのバックエンドとして機能し、より細かいスケジュール制御が可能になっています。

### 音声起動キーのリバインド対応

`keybindings.json` に `voice:pushToTalk` キーバインドが追加され、音声起動キーをカスタマイズできるようになりました。デフォルトはスペースキーですが、`meta+k` のようなモディファイアキーとアルファベットの組み合わせに変更することで、通常の入力と干渉しない設定が可能です。

```json
{
  "voice:pushToTalk": "meta+k"
}
```

### Bash 自動承認リストへの追加

以下のコマンドが Bash の自動承認（auto-approval）リストに追加されました:

- `fmt`, `comm`, `cmp`, `numfmt`, `expr`, `test`, `printf`, `getconf`, `seq`, `tsort`, `pr`

テキスト処理や数値演算、ファイル比較など、日常的なシェル操作での承認プロンプト頻度が減少します。

## 改善点

### 起動時間の短縮

ネイティブ画像プロセッサの読み込みを初回使用時まで遅延させることで、起動時間が改善されました。画像処理を使用しないセッションでは特に効果が大きくなります。

### ラップトップウェイクアップ後の再接続改善

スリープからの復帰後、ブリッジセッションの再接続が最大 10 分かかっていた問題が解消され、数秒以内に完了するようになりました。

### `/plugin uninstall` の挙動改善

プロジェクトスコープのプラグインをアンインストールする際、`.claude/settings.json` ではなく `.claude/settings.local.json` に無効化設定が書き込まれるようになりました。これにより、チームメンバーの設定に影響を与えることなくプラグインを管理できます。

### プラグイン提供 MCP サーバーの重複排除

手動設定済みのサーバーと同一のコマンド/URL を持つプラグイン提供 MCP サーバーが自動的にスキップされるようになりました。重複接続やツールセットの重複を防ぎ、`/plugin` メニューで抑制状況を確認できます。

### `/debug` コマンドの改善

デバッグログがデフォルトで書き込まれなくなり、`/debug` コマンドでセッション途中からデバッグロギングをトグルできるようになりました。

## バグ修正

### セッション・入力関連

- **長時間セッションでの stdin フリーズ修正**: キー入力が処理されなくなるがプロセスは生き続けるという問題を修正しました。
- **音声モードでの起動フリーズ修正**: システムウェイクアップ後に CoreAudio の初期化がメインスレッドをブロックすることで発生していた 5〜8 秒の起動フリーズを修正しました。
- **claude.ai プロキシコネクタの同時 OAuth トークン更新による起動 UI フリーズを修正**: 複数のコネクタが同時にトークンを更新する際の競合状態を解消しました。

### フォーク・プラン関連

- **`/fork` でのプランファイル共有問題を修正**: フォーク元とフォーク先が同じプランファイルを参照し、一方の編集がもう一方を上書きしてしまう問題を修正しました。

### Read ツール・画像処理

- **オーバーサイズ画像がコンテキストに残る問題を修正**: 画像処理に失敗した場合に巨大な画像データがコンテキストに積み重なり、長い画像多用セッションで後続のターンが壊れる問題を修正しました。

### Bash・権限関連

- **ヒアドキュメント付きコマンドでの誤った権限プロンプトを修正**: ヒアドキュメントを含む複合 Bash コマンド（コミットメッセージなど）で不必要な権限確認が表示されていた問題を修正しました。
- **Windows での mingw スタイルパスに対する git コマンドプロンプト問題を修正**: モデルが mingw スタイルのパスを使用した場合に `cd <cwd> && git ...` のプロンプトが出ていた問題を修正しました。

### プラグイン関連

- **複数インスタンス実行時のプラグインインストール消失を修正**
- **`/plugin marketplace update` でのマージコンフリクトを修正**: マーケットプレイスがブランチ/タグ ref にピン留めされている場合の問題を修正しました。
- **`/plugin marketplace add owner/repo@ref` の `@` パース問題を修正**: 従来は `#` しかref区切り文字として動作せず、`@` を使うと `strictKnownMarketplaces` との組み合わせで診断困難なエラーが発生していました。

### claude.ai コネクタ関連

- **OAuth トークン更新後の再接続失敗を修正**
- **MCP コネクタの起動通知重複を修正**: 組織設定済みコネクタすべてに通知が出ていた問題を修正し、以前に接続済みのコネクタのみに制限しました。
- **未認証の組織登録済みコネクタの起動ノイズを削減**

### その他のバグ修正

- **バックグラウンドエージェント完了通知での出力ファイルパス欠落を修正**: 親エージェントがコンテキスト圧縮後にエージェント結果を復元しにくい問題を解消しました。
- **Bash ツールのエラーメッセージ重複を修正**: コマンドがゼロ以外のステータスで終了した場合の出力重複を修正しました。
- **Chrome 拡張機能の自動検出が「未インストール」で固まる問題を修正**
- **`/permissions` Workspace タブの重複エントリを修正**: 末尾スラッシュの有無で同じディレクトリが二重登録される問題を修正しました。
- **`--print` がチームエージェント設定時に無限ハングする問題を修正**: 長命な `in_process_teammate` タスクを待ち続けていた終了ループを修正しました。
- **`ToolSearch` 呼び出しのたびに「❯ Tool loaded.」が REPL に表示される問題を修正**

## 技術的なポイント

- **`/loop` + cron スケジューリング**: 定期監視・定期チェックのユースケースを Claude Code 単体で完結できるようになりました。CI/CD の待機確認やサービスの死活監視など、繰り返しタスクの自動化に活用できます。
- **`voice:pushToTalk` のカスタマイズ**: `keybindings.json` でモディファイア+アルファベットキーに変更することで、テキスト入力中の誤発話リスクをほぼゼロにできます。
- **起動パフォーマンスの多面的改善**: 画像プロセッサの遅延ロード、CoreAudio ブロッキング修正、OAuth 競合解消の 3 点が組み合わさり、特に音声モードユーザーの体験が大幅に向上します。
- **プラグイン管理の信頼性向上**: マルチインスタンス時のインストール消失、MCP サーバー重複、アンインストール時のチーム設定汚染という 3 つの課題が一度に解消されました。
- **`/fork` プランファイル分離**: フォーク会話を並行探索のワークフローとして安心して使えるようになりました。

## まとめ

v2.1.71 は `/loop` コマンドによる定期実行自動化という実用的な新機能に加え、音声モード・プラグイン・フォーク会話・Bash 自動承認など幅広い分野での安定性向上が特徴的なリリースです。特に起動時間の改善とスリープ復帰後の再接続高速化は、日常的に Claude Code を使う開発者にとって即座に体感できる改善といえます。プラグインエコシステムの信頼性向上も着実に進んでおり、チームでの利用シーンでの使いやすさが増しています。

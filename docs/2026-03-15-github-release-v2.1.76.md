# Claude Code v2.1.76 リリース — MCP Elicitation 対応、PostCompact フック、/effort コマンドなど多数の新機能と修正

## 概要

Claude Code v2.1.76 は、MCP（Model Context Protocol）サーバーがタスク実行中に構造化入力をリクエストできる **Elicitation（エリシテーション）** 機能を中心に、多くの新機能と改善が含まれるリリースです。新しいフック機構、CLI フラグの追加、大規模モノレポ向けの worktree 最適化など開発者の生産性に直結する変更が多数盛り込まれています。また、deferred tools のスキーマ消失バグをはじめとする 20 件超のバグ修正も含まれます。

## 新機能

### MCP Elicitation サポート

MCP サーバーが対話型ダイアログ経由でタスクの途中に構造化入力をリクエストできるようになりました。フォームフィールドまたはブラウザ URL を通じてユーザーに入力を促す仕組みです。

これに合わせて、新しいフックも 2 種類追加されました。

- **`Elicitation` フック**: MCP サーバーからのエリシテーションリクエストをインターセプトする
- **`ElicitationResult` フック**: エリシテーションの応答を MCP サーバーへ返す前にオーバーライドする

これにより、MCP サーバーと Claude Code のインタラクションをより細かく制御できるようになります。

### `-n` / `--name` CLI フラグ

```bash
claude --name "My Feature Branch Session"
```

セッション起動時に表示名を設定できる `-n` / `--name <name>` フラグが追加されました。複数のセッションを並行して管理する際に識別しやすくなります。

### `worktree.sparsePaths` 設定（大規模モノレポ向け）

`claude --worktree` を使用する際に、git sparse-checkout を活用して必要なディレクトリのみをチェックアウトする設定が追加されました。

```json
{
  "worktree": {
    "sparsePaths": ["packages/my-service", "shared/utils"]
  }
}
```

数千のパッケージを含む大規模モノレポでは、チェックアウト対象を絞ることでパフォーマンスが大幅に向上します。

### `PostCompact` フック

会話コンパクション（コンテキスト圧縮）が完了した後に発火する新しいフックです。コンパクション後に特定の処理を実行したい場合に利用できます。

### `/effort` スラッシュコマンド

```
/effort high
/effort low
```

モデルの努力レベルを対話的に設定できる `/effort` スラッシュコマンドが追加されました。タスクの複雑さに応じて適切なレベルを選択することで、レスポンス速度とコスト・品質のバランスを調整できます。

### セッション品質調査（エンタープライズ向け）

エンタープライズ管理者は `feedbackSurveyRate` 設定を通じて、セッション品質調査のサンプルレートを構成できるようになりました。

## 改善点

### `--worktree` 起動パフォーマンスの向上

`--worktree` モードの起動時、git refs を直接読み込むようになり、リモートブランチがすでにローカルに存在する場合は冗長な `git fetch` をスキップします。大規模リポジトリでの起動が顕著に速くなります。

### Background Agent の改善

バックグラウンド Agent を強制停止した場合でも、それまでの部分的な処理結果が会話コンテキストに保持されるようになりました。エラーや中断が発生しても、途中経過を失わずに済みます。

### モデルフォールバック通知の改善

モデルフォールバック（代替モデルへの切り替え）通知が、verbose モード設定に関わらず常に表示されるようになりました。また、モデル名が人間が読みやすい形式で表示されます。

### ブロッククォートの可読性向上

ダークターミナルテーマ使用時、ブロッククォートが暗いテキストではなく左バー付きのイタリック体で表示されるようになりました。

### Stale Worktree の自動クリーンアップ

並列実行が途中で中断された場合に残留する worktree が自動的にクリーンアップされるようになりました。手動での後片付けが不要になります。

### Remote Control セッションタイトルの改善

Remote Control のセッションタイトルが、汎用的な "Interactive session" ではなく最初のプロンプト内容から自動生成されるようになりました。

### `--plugin-dir` の変更

`--plugin-dir` はサブコマンドとの互換性のため、1 つのパスのみ受け入れるようになりました。複数のプラグインディレクトリを指定する場合は、フラグを繰り返し使用してください。

```bash
# 変更前（複数パスをカンマ区切りで指定）
claude --plugin-dir /path/a,/path/b

# 変更後（フラグを繰り返し）
claude --plugin-dir /path/a --plugin-dir /path/b
```

## バグ修正

### Deferred Tools のスキーマ消失（重要）

`ToolSearch` 経由でロードされた deferred tools が会話コンパクション後に入力スキーマを失い、配列や数値パラメータが型エラーで拒否される問題が修正されました。この不具合は本リリースノートの冒頭に記載されているとおり、コンパクションを経た長い会話で顕著に影響していたものです。

### その他の主要な修正

- **スラッシュコマンド "Unknown skill"**: スラッシュコマンドが "Unknown skill" と表示される問題を修正
- **プランモードの再承認**: プランがすでに承認済みであるにも関わらず、再承認を求めてしまう問題を修正
- **音声モードのキープレス**: パーミッションダイアログやプランエディタが開いている間にキー入力が無視される問題を修正
- **Windows での `/voice`**: npm 経由でインストールした場合に Windows で `/voice` が動作しない問題を修正
- **誤った "Context limit reached"**: 1M コンテキストのセッションでスキルを使用した際に誤ってコンテキスト制限エラーが表示される問題を修正
- **Auto-Compaction の無限リトライ**: 連続してコンパクション失敗した場合に無限にリトライし続ける問題を修正。サーキットブレーカーにより 3 回の試行後に停止するようになりました
- **tmux over SSH のクリップボード**: tmux を使って SSH 接続している環境でクリップボードへのコピーが機能しない問題を修正
- **`Bash(cmd:*)` パーミッションルール**: 引用符付き引数に `#` が含まれる場合、ルールがマッチしない問題を修正
- **Remote Control の複数問題**: アイドル状態のセッションが突然終了する、急速なメッセージ送信時にバッチ処理されない、JWT 更新後に stale な作業アイテムが再配信される問題を修正
- **Bridge セッションの回復**: 長時間の WebSocket 切断後にセッションが復帰できない問題を修正
- **[VSCode] gitignore のカンマ**: カンマを含む gitignore パターンが `@-mention` ファイルピッカーでファイルタイプ全体を除外してしまう問題を修正

## 技術的なポイント

- **MCP Elicitation** は MCP の新しい標準機能であり、サーバー側から能動的にユーザー入力をリクエストできる双方向インタラクションを実現します。`Elicitation` / `ElicitationResult` フックと組み合わせることで、エンタープライズ環境での承認フローや入力バリデーションの自動化が可能になります
- **`worktree.sparsePaths`** は git の `sparse-checkout` 機能を活用しており、チェックアウト対象を宣言的に管理できます。CI/CD パイプラインや大規模モノレポ環境での `--worktree` 利用に特に有効です
- **Auto-Compaction のサーキットブレーカー** は、障害発生時に無限ループを防ぐための重要な安定性改善です。特に長時間稼働するエージェント処理での信頼性が向上します
- **`--plugin-dir` の変更**はサブコマンドサポートのための破壊的変更（仕様変更）であり、複数のプラグインディレクトリを使用しているユーザーは設定の更新が必要です
- **Deferred tools のスキーマ修正**は、ToolSearch を多用する複雑なエージェントワークフローで信頼性に直結する修正です

## まとめ

v2.1.76 は MCP Elicitation という新しいインタラクションモデルの導入と、多数の安定性改善が特徴のリリースです。特に大規模モノレポ環境での worktree 機能強化、deferred tools の信頼性修正、auto-compaction のサーキットブレーカー導入は、本番環境での長時間エージェント運用においても重要な改善といえます。`--plugin-dir` の仕様変更に注意しながら、アップデートを検討してください。

# Claude Code v2.1.59 リリース — 自動メモリ機能と /copy コマンドを追加

## 概要

Claude Code v2.1.59 では、Claude が会話から有用なコンテキストを自動的に保存する **auto-memory 機能**が導入されました。また、コードブロックを効率的にコピーできる **`/copy` コマンド**の追加や、複合 bash コマンドの権限管理の改善など、開発者の日常的なワークフローを向上させる複数のアップデートが含まれています。

## 新機能

### 自動メモリ機能（auto-memory）

Claude が会話の中から有用なコンテキストを自動的に検出し、永続的なメモリとして保存するようになりました。これにより、ユーザーが繰り返し伝える必要があった情報（プロジェクト固有の規約、好みの設定など）を Claude が自動的に記憶できます。

保存されたメモリは `/memory` コマンドで確認・管理できます。

```
/memory
```

自動保存される内容の例：
- プロジェクトで使用するコーディング規約や命名規則
- よく使うライブラリやツールの設定
- ユーザーが明示的に伝えた作業上の好み

### `/copy` コマンドの追加

レスポンス内にコードブロックが含まれている場合に、インタラクティブなピッカーを表示する `/copy` コマンドが追加されました。複数のコードブロックが存在する場合でも、特定のブロックだけを選択してコピーしたり、レスポンス全体をまとめてコピーしたりすることが可能です。

```
/copy
```

ピッカーでは以下を選択できます：
- レスポンス内の個別のコードブロック（言語やブロック番号で識別）
- レスポンス全体

長いレスポンスに複数のコードスニペットが含まれる場面で特に便利な機能です。

## 改善点

### 複合 bash コマンドのプレフィックス提案の改善

「常に許可（always allow）」の設定において、`&&` で連結された複合 bash コマンドのプレフィックス提案がより精細になりました。

**改善前の動作：**
```bash
cd /tmp && git fetch && git push
```
上記のようなコマンド全体が1つのエントリとして扱われていました。

**改善後の動作：**
各サブコマンド（`cd /tmp`、`git fetch`、`git push`）に対して個別のスマートなプレフィックスが計算されます。これにより、「git push を含むコマンドは常に許可」といった、より適切な粒度での権限管理が可能になります。

### 短いタスクリストの順序改善

タスクリストが短い場合の表示順序が改善されました。ユーザーがタスクの進捗を把握しやすくなります。

### マルチエージェントセッションのメモリ使用量改善

マルチエージェントセッションにおいて、完了したサブエージェントのタスク状態を解放するようになりました。長時間にわたる複雑なエージェントセッションでのメモリ消費を抑制し、安定性が向上します。

## バグ修正

### MCP OAuth トークン更新の競合条件を修正

複数の Claude Code インスタンスを同時に起動している場合、MCP（Model Context Protocol）の OAuth トークンリフレッシュ処理で競合条件（race condition）が発生することがありました。この問題が修正され、複数インスタンスの並列使用がより安定して動作するようになりました。

### ワーキングディレクトリ削除時のエラーメッセージ改善

シェルコマンドの実行中にワーキングディレクトリが削除されていた場合、エラーメッセージが不明瞭でした。今回の修正により、明確なエラーメッセージが表示されるようになり、問題の原因を素早く特定できます。

## 技術的なポイント

- **auto-memory の活用**: 自動メモリ機能はプロジェクト固有の情報を Claude に学習させる強力な手段です。`/memory` コマンドで保存内容を定期的に確認し、不要な情報を整理することを推奨します
- **`/copy` によるコードの取り扱い**: 複数のコードブロックを含むレスポンスで個別にコピーが可能になり、コードの貼り付けミスが減少します
- **複合コマンドの権限粒度**: `&&` チェーンを多用する開発者には、より適切な「always allow」設定が適用されるようになります。セキュリティと利便性のバランスが向上します
- **マルチエージェントのメモリ管理**: 大規模なエージェントタスクを実行する場合、完了済みサブエージェントの状態解放によりリソース効率が改善されます
- **MCP の安定性向上**: 複数ターミナルや複数プロジェクトで Claude Code を並行して使用するケースでの信頼性が高まります

## まとめ

v2.1.59 は、**自動メモリ機能**という大きな新機能を中心に、日常的な開発体験を細かく改善するアップデートです。特に auto-memory は、Claude Code をプロジェクトに長期的に活用しているユーザーにとって、繰り返しのコンテキスト共有を削減できる便利な機能です。

`/copy` コマンドの追加や複合コマンドの権限改善も、ターミナル作業の効率化に直結します。MCP を活用している開発者にとっても、OAuth 周りの競合条件修正は安定性向上として歓迎されるでしょう。

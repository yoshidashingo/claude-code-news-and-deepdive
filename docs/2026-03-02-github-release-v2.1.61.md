# Claude Code v2.1.61: Windows における設定ファイル破損バグの修正

## 概要

Claude Code v2.1.61 は、Windows 環境において設定ファイルが破損する問題を修正したパッチリリースです。並行書き込み（concurrent writes）が発生した際にコンフィグファイルが壊れるという重大な不具合が解消されました。機能追加はなく、Windows ユーザーの安定性向上に特化したリリースです。

## バグ修正

### Windows での並行書き込みによる設定ファイル破損の修正

今回のリリースで修正された問題は、**Windows 環境で複数の書き込み処理が同時に発生した際に設定ファイル（config file）が破損する**というものです。

#### 問題の背景

Claude Code は動作中にさまざまな設定を config ファイルに保存します。複数のプロセスやスレッドが同時に同一ファイルへ書き込もうとすると、書き込み内容が混在・上書きされてファイルの内容が不正な状態になる「レースコンディション（race condition）」が発生する可能性があります。

Windows のファイルシステムは macOS や Linux と異なるファイルロックの挙動を持つため、クロスプラットフォームアプリケーションではこのような OS 固有の問題が生じやすい傾向があります。Unix 系 OS では一般的にアトミックな `rename` 操作が利用可能ですが、Windows では同様の手法が直接使えないケースがあり、排他制御の実装に追加の工夫が必要です。

#### 影響範囲

この問題は **Windows ユーザーのみ** に影響します。以下のような症状が発生していた可能性があります。

- Claude Code の起動時にエラーが表示される
- 設定内容が突然リセットされる
- config ファイルの内容が JSON として解析できない不正な形式になる

#### 修正内容

本リリースでは、Windows 環境における config ファイルへの書き込み処理に適切な排他制御が実装され、並行書き込みが発生してもファイルが正常に保持されるようになりました。

## 技術的なポイント

- **対象 OS**: Windows のみ（macOS・Linux には影響なし）
- **問題の種類**: レースコンディションによるファイル破損（data corruption）
- **修正の性質**: ファイル書き込み時の排他制御（ロック機構）の改善
- **ユーザーへの影響**: 設定の消失や起動エラーが解消される
- **マイグレーション**: 特別な移行作業は不要。アップデートするだけで問題が解消される

## まとめ

v2.1.61 は Windows ユーザー向けの重要な安定性修正を含むパッチリリースです。並行書き込みによる config ファイル破損という再現性のある不具合が解消されたことで、Windows 環境での Claude Code の信頼性が向上します。

Windows 上で Claude Code を使用していて、設定が突然リセットされたり起動時にエラーが発生したりする問題を経験している方は、速やかに v2.1.61 以降へアップデートすることをお勧めします。

# Claude Code v2.1.73 リリース — modelOverrides 設定追加、多数のバグ修正と改善

## 概要

Claude Code v2.1.73 がリリースされました。カスタムプロバイダーのモデル ID をマッピングできる `modelOverrides` 設定の追加、SSL 証明書エラー時の案内改善など新機能が加わっています。また、パーミッションプロンプトによる CPU 高負荷や各種デッドロック、サブエージェントのモデルバージョン問題など、多数のバグが修正されています。Bedrock / Vertex / Microsoft Foundry 上でのデフォルト Opus モデルが Opus 4.6 に変更された点も注目です。

## 新機能

### `modelOverrides` 設定

モデルピッカーのエントリをカスタムプロバイダーのモデル ID にマッピングできる `modelOverrides` 設定が追加されました。例えば Bedrock の推論プロファイル ARN（Amazon Resource Name）を指定するといったユースケースに対応します。

```json
{
  "modelOverrides": {
    "claude-opus-4-6": "arn:aws:bedrock:us-east-1::foundation-model/..."
  }
}
```

エンタープライズ環境で特定の推論エンドポイントやカスタムデプロイを使用している場合に便利な機能です。

### SSL 証明書エラー時のアクションガイダンス

企業プロキシなど SSL 証明書の検証に問題が生じる環境で OAuth ログインや接続チェックが失敗した際、具体的な解決策（`NODE_EXTRA_CA_CERTS` 環境変数の設定など）が案内されるようになりました。これにより、企業ネットワーク内で Claude Code を使い始める際のトラブルシューティングが容易になります。

## 改善点

- **Up 矢印キーの挙動改善**: Claude の応答を中断した後に Up 矢印キーを押すと、中断したプロンプトが復元されると同時に会話が 1 ステップで巻き戻されるようになりました。
- **IDE 検出速度の向上**: 起動時の IDE 検出が高速化されました。
- **macOS クリップボード画像貼り付けのパフォーマンス改善**: macOS 上での画像貼り付け操作が高速になりました。
- **`/effort` コマンドの改善**: Claude が応答中でも `/effort` が使用できるようになりました。`/model` コマンドと同様の動作に統一されています。
- **ボイスモードの接続安定性向上**: 素早い push-to-talk 操作中に一時的な接続障害が発生した場合、自動リトライするようになりました。
- **Remote Control スポーンモード選択プロンプトの改善**: コンテキスト情報が充実し、より分かりやすい選択画面になりました。
- **デフォルト Opus モデルの更新**: Bedrock・Vertex・Microsoft Foundry 上でのデフォルト Opus モデルが **Opus 4.1 から Opus 4.6** に変更されました。

## バグ修正

### パフォーマンス・安定性

- **パーミッションプロンプトによるフリーズと CPU 高負荷の修正**: 複雑な bash コマンドに対してパーミッションプロンプトが表示される際に発生していたフリーズや CPU 100% ループを修正しました。
- **スキルファイル大量変更時のデッドロック修正**: `git pull` などで大量のスキルファイルが一度に変更された場合（`.claude/skills/` ディレクトリが大規模なリポジトリなど）に Claude Code がフリーズするデッドロックを修正しました。
- **同一プロジェクトでの複数セッション時の出力消失を修正**: 同じプロジェクトディレクトリで複数の Claude Code セッションを実行した際に Bash ツールの出力が失われる問題を修正しました。

### サブエージェント関連

- **モデルバージョンのサイレントダウングレード修正**: `model: opus` / `sonnet` / `haiku` を指定したサブエージェントが Bedrock・Vertex・Microsoft Foundry 上で古いモデルバージョンにサイレントダウングレードされていた問題を修正しました。
- **バックグラウンドプロセスのクリーンアップ修正**: サブエージェントが起動したバックグラウンド bash プロセスが、エージェント終了後も残り続ける問題を修正しました。

### コマンド・UI 関連

- **`/resume` コマンドの修正**: セッションピッカーに現在のセッションが表示されていた問題を修正しました。
- **`/ide` コマンドのクラッシュ修正**: 拡張機能の自動インストール時に `onInstall is not defined` エラーでクラッシュする問題を修正しました。
- **`/loop` コマンドの可用性修正**: Bedrock・Vertex・Foundry 環境やテレメトリ無効時に `/loop` が使用できなかった問題を修正しました。

### フック関連

- **SessionStart フックの二重起動修正**: `--resume` または `--continue` でセッションを再開した際に SessionStart フックが 2 回起動していた問題を修正しました。
- **JSON 出力フックの不要メッセージ注入修正**: JSON 出力フックが毎ターン no-op の `system-reminder` メッセージをモデルのコンテキストに注入していた問題を修正しました。

### プラットフォーム固有

- **ボイスモードのセッション破損修正**: 低速な接続と新しい録音が重なった際にボイスモードのセッションが破損する問題を修正しました。
- **Linux サンドボックスの起動エラー修正**: ネイティブビルドで "ripgrep (rg) not found" エラーにより Linux サンドボックスが起動できない問題を修正しました。
- **Amazon Linux 2 / glibc 2.26 対応の修正**: Amazon Linux 2 やその他の glibc 2.26 システムで Linux ネイティブモジュールが読み込めない問題を修正しました。
- **Remote Control 画像受信エラーの修正**: Remote Control 経由で画像を受信した際の "media_type: Field required" API エラーを修正しました。
- **Windows `/heapdump` エラー修正**: Desktop フォルダが既に存在する Windows 環境で `/heapdump` が `EEXIST` エラーで失敗する問題を修正しました。
- **VSCode プロキシ・Bedrock/Vertex 環境の HTTP 400 エラー修正**: プロキシ環境や Bedrock/Vertex 上で Claude 4.5 モデルを使用している VSCode ユーザーに発生していた HTTP 400 エラーを修正しました。

## 非推奨化

- **`/output-style` コマンドの非推奨化**: `/output-style` コマンドが非推奨となりました。今後は `/config` を使用してください。出力スタイルはプロンプトキャッシングの効率化のため、セッション開始時に固定されるようになりました。

## 技術的なポイント

- **`modelOverrides` によるエンタープライズ対応強化**: Bedrock の推論プロファイルや各クラウドプロバイダーの独自モデル ID を柔軟にマッピングできるようになり、エンタープライズ環境でのカスタマイズ性が大幅に向上しました。
- **サブエージェントのモデル指定信頼性向上**: `model: opus` などの省略形指定が各プロバイダーで正確に最新モデルにマッピングされるようになりました。マルチエージェントワークフローを組んでいる場合は恩恵を受けられます。
- **デフォルト Opus が 4.6 に統一**: Bedrock・Vertex・Foundry ユーザーは設定変更なしで Opus 4.6 の性能を享受できます。ただし既存ワークフローへの影響がないか確認することを推奨します。
- **フックの信頼性改善**: SessionStart の二重起動修正と JSON 出力フックの不要メッセージ注入修正により、自動化パイプラインやカスタムフックを活用している場合の動作が安定します。
- **Linux ネイティブビルドの互換性改善**: Amazon Linux 2 や glibc 2.26 システムへの対応改善により、AWS 環境での利用がしやすくなりました。
- **`/output-style` の非推奨化とプロンプトキャッシング**: 出力スタイルをセッション開始時に固定することで、プロンプトキャッシングの効率が向上します。`/config` への移行を早めに行うことを推奨します。

## まとめ

v2.1.73 は新機能 2 件、改善 7 件、バグ修正 15 件以上という充実したリリースです。特に企業環境での利便性向上（`modelOverrides`、SSL エラーガイダンス、Amazon Linux 2 対応）と、サブエージェントやフックの信頼性改善が目立ちます。パーミッションプロンプトによる CPU 高負荷など日常的に遭遇しやすいバグも修正されており、全ユーザーにとってアップデートを推奨できるリリースといえます。また、`/output-style` の非推奨化により `/config` への統一が進んでおり、今後の設定管理の変化にも注目です。

# Claude Code v2.1.72 リリース — 大量のバグ修正と開発体験の大幅改善

## 概要

Claude Code v2.1.72 は、新機能の追加と多数のバグ修正を含む大型リリースです。`/plan` コマンドへの説明引数追加や `ExitWorktree` ツールの導入など、ワークフローを改善する機能が複数加わりました。また、bash のパーミッション誤検知の削減やプロンプトキャッシュの修正（入力トークンコスト最大 12 倍削減）など、パフォーマンスと信頼性の向上も顕著です。

## 新機能

### `/plan` コマンドへの説明引数追加

`/plan` コマンドに説明を直接渡すことで、プランモードに入りながら即座に作業を開始できるようになりました。

```
/plan fix the auth bug
```

従来は `/plan` で起動後にプロンプトを入力する必要がありましたが、これによりワンステップで計画立案を開始できます。

### SSH 環境向け `/copy` の `w` キー追加

`/copy` コマンドのインタラクティブ画面で `w` キーを押すと、フォーカスした選択内容をクリップボードを経由せずに直接ファイルへ書き出せます。SSH 接続環境など、クリップボードが使えない状況で特に有効です。

### `ExitWorktree` ツールの追加

`EnterWorktree` で開始したワークツリーセッションを明示的に終了するための `ExitWorktree` ツールが追加されました。ワークツリーを使った隔離作業の制御がより柔軟になります。

### `CLAUDE_CODE_DISABLE_CRON` 環境変数

セッション中にスケジュール済み cron ジョブを即座に停止できる環境変数が追加されました。長時間セッションで不要なバックグラウンド処理を止めたい場合に使用できます。

```bash
export CLAUDE_CODE_DISABLE_CRON=1
```

### Agent ツールの `model` パラメータ復元

`Agent` ツールに `model` パラメータが復元され、呼び出しごとにモデルを指定できるようになりました。タスクの性質に応じて最適なモデルを選択する高度な活用が可能です。

### bash 自動承認許可リストの拡張

`lsof`、`pgrep`、`tput`、`ss`、`fd`、`fdfind` が許可リストに追加されました。これらは読み取り専用の一般的な操作で、不要な権限確認ダイアログが削減されます。

### マーケットプレイス git URL の `.git` サフィックスなし対応

Azure DevOps や AWS CodeCommit のように `.git` サフィックスを付けない URL 形式を使用する git ホスティングサービスに対応しました。

### 努力レベルの簡素化

努力レベルが `low / medium / high` の 3 段階に整理され（`max` は廃止）、新しいシンボル `○ ◐ ●` で視覚的に確認できるようになりました。`/effort auto` でデフォルト設定にリセットできます。

## 改善点

### bash パーミッション誤検知の大幅削減

tree-sitter によるパース処理が改善され、`find -exec`、変数割り当て、コマンド置換など多くのパターンで不要な権限確認が表示されなくなりました。また、npm でインストールした場合に tree-sitter がロードされない問題も修正されました。

bash コマンドパーサー自体もネイティブモジュールに切り替わり、初期化速度の向上とメモリリークの解消が実現しています。

### CLAUDE.md HTML コメントの非表示化

CLAUDE.md を自動注入する際、HTML コメント（`<!-- ... -->`）が Claude に見えないようになりました。コメントはあくまで人間向けの注記として活用でき、`Read` ツールで直接読み込む場合には引き続き表示されます。

### `/config` UI の改善

- `Escape` キーで変更をキャンセル
- `Enter` キーで保存して閉じる
- `Space` キーで設定をトグル

### 複数セッション実行時の履歴改善

上矢印キーで履歴を遡る際、複数の並行セッションが動いている場合でも現在のセッションのメッセージが優先表示されるようになりました。

### 音声入力の認識精度向上

リポジトリ名や開発用語（regex、OAuth、JSON など）の音声認識精度が改善されました。

### バンドルサイズ削減

約 510 KB のバンドルサイズ削減が行われました。インストール・起動のオーバーヘッドが軽減されます。

### `claude plugins` エイリアス追加

`claude plugin` コマンドの複数形エイリアス `claude plugins` が追加されました。

## バグ修正

### パフォーマンス・安定性

- **遅い終了の修正**: バックグラウンドタスクやフックの応答が遅い場合に終了が詰まる問題を修正
- **プロンプトキャッシュ修正**: SDK `query()` 呼び出しでのプロンプトキャッシュ無効化を修正。入力トークンコストが最大 12 倍削減される場合があります
- **CPU 使用率の改善**: 長時間セッションでの CPU 使用率を改善
- **セッションクラッシュ修正**: Bedrock API エラーを含むセッションを再開した際のクラッシュを修正
- **U+2028/U+2029 文字によるクラッシュ修正**: Desktop/SDK で Read がこれらの特殊文字を含むファイルを返した際のクラッシュを修正

### ワークフロー・操作

- **エージェントタスク進捗表示**: 「初期化中…」のまま止まる問題を修正
- **`--continue` の修正**: `--compact` 後に最新ポイントから再開できない問題を修正
- **`/clear` の修正**: バックグラウンドのエージェント/bash タスクが誤って削除される問題を修正（フォアグラウンドのタスクのみ対象に）
- **ワークツリー隔離の修正**: `Task` ツールの再開時に作業ディレクトリが復元されない問題、およびバックグラウンドタスク通知に `worktreePath` と `worktreeBranch` が含まれない問題を修正
- **`/model` の修正**: Claude が処理中に `/model` を実行しても結果が表示されない問題を修正
- **Ctrl+C 二回押しの修正**: バックグラウンドにエージェントやタスクがある場合に終了できない問題を修正

### hooks

- **スキルフック二重発火の修正**: hooks 有効なスキルをモデルが呼び出した際にフックが 2 回発火する問題を修正
- **`transcript_path` の修正**: 再開・フォークしたセッションで誤ったディレクトリを指していた問題を修正
- **async フックの修正**: bash の `read -r` で stdin を受け取れない問題を修正
- **`--effort` フラグの修正**: 起動時の設定書き込みでリセットされる問題を修正

### プラグイン

- Windows の OneDrive フォルダで `EEXIST` エラーによりインストールが失敗する問題を修正
- プロジェクトスコープのインストールが存在する場合にマーケットプレイスがユーザースコープインストールをブロックする問題を修正
- `CLAUDE_CODE_PLUGIN_CACHE_DIR` が `~` をリテラルディレクトリとして作成する問題を修正
- `plugin.json` にマーケットプレイス専用フィールドがある場合に読み込みに失敗する問題を修正

### 音声モード

- 入力ラグの解消
- push-to-talk 解放後の誤「音声なし」エラーを修正
- 古いトランスクリプトが送信後にプロンプトを再充填する問題を修正

### パーミッション・サンドボックス

- ファイル書き込みが確認なしに許可される問題を修正
- `/tmp/claude/` など許可リスト済みディレクトリへのリダイレクトが不要なプロンプトを表示する問題を修正
- ワイルドカードルールが heredoc や埋め込み改行、引数なしコマンドにマッチしない問題を修正
- deny ルールがすべてのコマンド形式に適用されない問題を修正

### VSCode 拡張

- 統合ターミナルのスクロール速度をネイティブターミナルと一致するよう修正
- 古いキーバインディング設定ユーザーで `Shift+Enter` が改行でなく送信してしまう問題を修正
- 努力レベルインジケーターを入力ボーダーに追加
- `vscode://anthropic.claude-code/open` URI ハンドラを追加。`prompt` および `session` クエリパラメータに対応しており、プログラムから新しい Claude Code タブを開けます

## 技術的なポイント

- **プロンプトキャッシュ修正の効果**: SDK `query()` 呼び出しで最大 12 倍の入力トークンコスト削減が期待できます。大量の API 呼び出しを行う自動化ワークフローでは大幅なコスト削減につながる可能性があります
- **bash パーサーのネイティブ化**: tree-sitter ベースのネイティブモジュールへの切り替えにより、初期化速度向上とメモリリーク解消が実現。長時間・大量操作のセッションで恩恵を受けやすい改善です
- **CLAUDE.md HTML コメントの活用**: `<!-- -->` 形式のコメントを CLAUDE.md に記述することで、Claude には見せずに人間向けの注記・メモを残せます。チーム運用での CLAUDE.md 管理に有用です
- **`ExitWorktree` ツール**: `EnterWorktree` / `ExitWorktree` のペアが揃ったことで、エージェントによるワークツリーの分離実行がより確実に制御できるようになりました
- **環境変数 `CLAUDE_CODE_DISABLE_CRON`**: CI/CD 環境や一時的なデバッグ実行で cron を無効化したい場合に、コード変更なしで制御できます
- **VSCode URI ハンドラ**: `vscode://anthropic.claude-code/open?prompt=...&session=...` を使って外部ツールやスクリプトから Claude Code タブをプログラム制御で開くことが可能になりました

## まとめ

v2.1.72 は新機能と大量のバグ修正が混在する充実したリリースです。特に注目すべきはプロンプトキャッシュ修正によるトークンコスト削減で、SDK を使った自動化ワークフローに直接的な効果があります。bash パーミッション誤検知の削減やバンドルサイズ縮小など、日常の使い勝手を地道に向上させる改善も多く含まれています。ワークツリー周りの修正や hooks の安定性向上により、複雑なエージェント構成を使っているチームにとっても重要なアップデートと言えます。

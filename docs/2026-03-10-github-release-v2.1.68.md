# Claude Code v2.1.68: Opus 4.6 のデフォルト effort 変更と ultrathink キーワード復活

## 概要

Claude Code v2.1.68 では、Opus 4.6 の思考努力度（effort）がデフォルトで **medium** に設定されるようになりました。また、一時的に廃止されていた `ultrathink` キーワードが復活し、必要なときだけ高い推論深度を利用できるようになっています。さらに、旧世代モデルである Opus 4 と Opus 4.1 がファーストパーティ API から削除されました。

## 新機能

### ultrathink キーワードの復活

一度削除されていた `ultrathink` キーワードが再導入されました。このキーワードをプロンプトに含めると、**次のターンだけ high effort モード**で動作します。

```
ultrathink この設計を徹底的にレビューして
```

通常は medium effort で軽快に作業しながら、複雑な問題に直面した場合にのみ `ultrathink` を使って深い推論を引き出せます。これにより、速度と精度のトレードオフをユーザーが細かくコントロールできるようになります。

## 改善点

### Opus 4.6 のデフォルト effort を medium に変更

Max および Team プランのサブスクライバーを対象に、Opus 4.6 のデフォルト effort が **medium** に変更されました。

| effort レベル | 特徴 |
|---|---|
| low | 最速、単純なタスク向け |
| **medium（新デフォルト）** | 速度と精度のバランスが取れた最適解 |
| high | 最も深い推論、複雑なタスク向け |

公式には「ほとんどのタスクで medium がスイートスポット」と説明されており、これまでの high effort デフォルトに比べてレスポンスが速くなることが期待されます。effort レベルはいつでも `/model` コマンドから変更できます。

## Breaking Changes

### Opus 4 / Opus 4.1 の廃止

ファーストパーティ API において、**Opus 4** および **Opus 4.1** が Claude Code から削除されました。これらのモデルをピン留めして使用していたユーザーは、**自動的に Opus 4.6 へ移行**されます。

手動での対応は不要ですが、以下の点に注意してください。

- `ANTHROPIC_MODEL` 環境変数や設定ファイルで旧モデル名を明示指定している場合は、`claude-opus-4-6` に更新することを推奨します
- Opus 4.6 はデフォルトで medium effort となるため、以前の high effort 相当の挙動が必要な場合は `/model` コマンドで effort を調整するか、`ultrathink` キーワードを活用してください

## 技術的なポイント

- **デフォルト effort の変更**により、Max/Team ユーザーは設定変更なしにより速いレスポンスを得られるようになります
- **ultrathink キーワード**は「ターン単位」の制御であり、セッション全体ではなく1回のリクエストにのみ high effort を適用します。必要な場面に絞った使用が効果的です
- **Opus 4 / 4.1 の削除**はモデルラインナップの整理を目的としており、Opus 4.6 への一本化が進んでいます
- effort レベルの変更は `/model` コマンドからインタラクティブに行えるため、タスクの性質に応じた柔軟な使い分けが可能です

## まとめ

v2.1.68 は、Opus 4.6 のデフォルト動作を「速度と精度のバランス重視」に調整した実用的なアップデートです。`ultrathink` の復活により、重要な場面では高精度な推論を選択肢として確保しつつ、日常的な作業では快適なレスポンス速度を享受できるようになりました。Opus 4 系モデルを使用していたユーザーは Opus 4.6 へ自動移行されるため、基本的に手動対応は不要ですが、設定ファイルのモデル名を見直すことをお勧めします。

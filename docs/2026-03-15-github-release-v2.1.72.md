# Claude Code v2.1.72 リリース — SSH対応のファイル書き込み、ExitWorktreeツール、大規模バグ修正

## 概要

Claude Code v2.1.72 は、新機能・UX改善・パフォーマンス向上・多数のバグ修正を含む大型リリースです。SSH 環境でのクリップボード問題を回避するファイル直接書き込み機能や、ワークツリーセッションを終了する `ExitWorktree` ツールが追加されました。また、bash パーミッションプロンプトの誤検知削減や SDK の prompt cache 修正により、日常的な開発体験が大きく向上しています。

## 新機能

### `/copy` の `w` キー — SSH環境でのクリップボード問題を解消

`/copy` コマンドで選択中のコンテンツを表示しているとき、`w` キーを押すことでクリップボードを経由せずファイルに直接書き込めるようになりました。SSH 接続先のリモートサーバーではクリップボード共有が機能しないケースが多く、この機能はそのような環境で特に有用です。

### `/plan` にオプション説明文を追加

`/plan` コマンドにオプションの説明引数が追加されました。

```
/plan fix the auth bug
```

このように実行すると、プランモードに入ると同時に指定した説明でプランニングが即座に開始されます。これまで `/plan` と入力した後に改めて指示を入力する必要がありましたが、1ステップで起動できるようになりました。

### `ExitWorktree` ツール

`EnterWorktree` ツールで開始したワークツリーセッションを明示的に終了するための `ExitWorktree` ツールが追加されました。ワークツリーの分離環境を使うワークフローが完結するようになります。

### `CLAUDE_CODE_DISABLE_CRON` 環境変数

セッション途中でスケジュール済みの cron ジョブを即座に停止する環境変数 `CLAUDE_CODE_DISABLE_CRON` が追加されました。長時間セッション中に定期タスクを一時的に無効化したい場合に便利です。

```bash
export CLAUDE_CODE_DISABLE_CRON=1
```

### エフォートレベルの簡略化

エフォートレベルが `low / medium / high / max` の4段階から `low / medium / high` の3段階に整理されました。各レベルのシンボルも新しく `○ ◐ ●` に変更され、常時表示のアイコンではなく簡潔な通知として表示されるようになっています。デフォルト設定に戻すには `/effort auto` を使用します。

### `claude plugins` エイリアス

`claude plugin` コマンドの複数形エイリアスとして `claude plugins` が追加されました。直感的にコマンドを入力できるようになります。

### VSCode: `vscode://anthropic.claude-code/open` URI ハンドラー

VSCode 拡張に URI ハンドラーが追加されました。外部ツールやスクリプトから Claude Code タブをプログラム的に開けるようになります。

```
vscode://anthropic.claude-code/open?prompt=<プロンプト>&session=<セッションID>
```

`prompt` と `session` はオプションのクエリパラメーターです。

## 改善点

### bash 自動承認リストの拡充

`lsof`、`pgrep`、`tput`、`ss`、`fd`、`fdfind` が bash の自動承認許可リストに追加されました。これらは読み取り専用の一般的な操作であり、毎回パーミッションプロンプトが表示されていた問題が解消されます。

### bash パーミッションプロンプトの誤検知大幅削減

tree-sitter による bash コマンドのパース処理が改善され、`find -exec`、変数代入、コマンド置換など、これまで誤ってパーミッションプロンプトを発生させていたパターンが正しく処理されるようになりました。また、npm インストール版で tree-sitter がロードされない問題も修正されています。

### Agent ツールの `model` パラメーター復活

Agent ツールに `model` パラメーターが復元され、呼び出しごとにモデルを個別指定できるようになりました。チームエージェント構成などで特定のエージェントに異なるモデルを使用したい場合に活用できます。

### `/config` の操作性向上

`/config` コマンドのキーバインドが改善されました。

- **Escape**: 変更をキャンセル
- **Enter**: 変更を保存して閉じる
- **Space**: 設定のトグル

### 上矢印キーの履歴表示改善

複数の並行セッションを実行している場合、上矢印キーによる履歴表示が現在のセッションのメッセージを優先して表示するようになりました。

### バンドルサイズ削減とCPU使用率改善

バンドルサイズが約 510 KB 削減されました。また、長時間セッションでの CPU 使用率も改善されています。

### Azure DevOps / AWS CodeCommit の git URL 対応

`.git` サフィックスなしのマーケットプレイス git URL に対応しました。Azure DevOps や AWS CodeCommit のリポジトリ URL をそのまま使用できます。

### CLAUDE.md HTMLコメントの非表示化

CLAUDE.md の HTML コメント（`<!-- ... -->`）が自動インジェクト時に Claude から隠されるようになりました。ガイドラインのメモや内部的な注釈を Claude に見せずに管理できます。なお、`Read` ツールで直接読み込んだ場合はコメントが表示されます。

## バグ修正

### パフォーマンス・安定性

- **SDK `query()` の prompt cache 修正**: prompt cache の無効化が修正され、入力トークンコストを最大 12 倍削減できるようになりました
- **バックグラウンドタスク終了時の遅延修正**: バックグラウンドタスクやフックのレスポンスが遅い場合にセッション終了が詰まる問題を修正
- **長時間セッションの CPU 使用率改善**

### エージェント・ワークツリー

- **エージェントタスク進捗の「Initializing…」フリーズ修正**
- **チームエージェントがリーダーのモデルを継承するよう修正**
- **ワークツリー分離の問題修正**: `Task` ツールの再開時に cwd が復元されない問題、バックグラウンドタスク通知に `worktreePath` と `worktreeBranch` が含まれない問題を修正

### `/clear` と バックグラウンドタスク

- `/clear` がバックグラウンドエージェント・bash タスクを誤って終了させていた問題を修正。フォアグラウンドタスクのみクリアされるようになりました
- `--continue` が `--compact` 後に最新地点から再開できない問題を修正
- Ctrl+B のバックグラウンドクエリがトランスクリプトを失う、または `/clear` 後に新しい会話を破損させる問題を修正
- ダブル Ctrl+C でバックグラウンドエージェント・タスクが実行中の場合に終了できない問題を修正

### 並列ツール呼び出し

`Read` / `WebFetch` / `Glob` ツールのいずれかが失敗した場合に、並列実行中の兄弟ツール呼び出しがキャンセルされていた問題を修正しました。現在は `Bash` のエラーのみがカスケードするようになっています。

### パーミッションルールのマッチング

- ワイルドカードルールが heredoc、埋め込み改行、引数なしのコマンドにマッチしない問題を修正
- `sandbox.excludedCommands` が環境変数プレフィックスで失敗する問題を修正
- "Always Allow" が再度マッチしないルールを保存する問題を修正
- 否定ルールがすべてのコマンド形式に適用されない問題を修正

### プラグイン

- Windows の OneDrive フォルダーで `EEXIST` エラーによりインストールが失敗する問題を修正
- プロジェクトスコープのインストールが存在する場合にマーケットプレイスがユーザースコープのインストールをブロックする問題を修正
- `CLAUDE_CODE_PLUGIN_CACHE_DIR` が `~` をリテラルディレクトリとして作成する問題を修正
- marketplace のみのフィールドを持つ `plugin.json` が読み込めない問題を修正

### フック

- `transcript_path` が再開・フォークされたセッションで誤ったディレクトリを指す問題を修正
- `agent` の `prompt` が設定書き込みのたびに `settings.json` から削除される問題を修正
- PostToolUse ブロック理由が二重表示される問題を修正
- 非同期フックが bash `read -r` で stdin を受け取れない問題を修正

### VSCode 拡張

- 統合ターミナルのスクロール速度がネイティブターミナルと一致しない問題を修正
- 古いキーバインド設定を持つユーザーで Shift+Enter が改行挿入ではなく送信してしまう問題を修正
- エフォートレベルインジケーターを入力ボーダーに追加

### その他

- 音声モードの入力遅延、プッシュトゥトーク解放後の誤「No speech detected」エラー、送信後にプロンプトに古いトランスクリプトが再入力される問題を修正
- U+2028 / U+2029 文字を含むファイルを `Read` ツールで読み込んだ際に Desktop / SDK がクラッシュする問題を修正
- `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` 設定時に終了時にターミナルタイトルがクリアされる問題を修正
- Bedrock API エラーが含まれるセッションを再開するとクラッシュする問題を修正
- フィードバックサーベイが長時間セッションで頻繁に表示される問題を修正

## 技術的なポイント

- **SDK の prompt cache 最大 12 倍コスト削減**: `query()` 呼び出しの prompt cache 無効化バグの修正は、コストと速度の両面で大きな影響があります。SDK を使った自動化ワークフローでは即座に恩恵を受けられます
- **tree-sitter による bash パース改善**: ネイティブモジュールへの切り替えにより、初期化速度向上とメモリリークが解消されました。誤検知の減少でインタラクティブな開発体験も改善されます
- **`CLAUDE_CODE_DISABLE_CRON` / `CLAUDE_CODE_PROXY_SUPPORTS_TOOL_REFERENCE` の変更**: `CLAUDE_CODE_PROXY_SUPPORTS_TOOL_REFERENCE` 環境変数は削除されました。ツール検索がサードパーティプロキシのゲートをバイパスする動作は環境変数の設定で制御されるよう変更されています
- **CLAUDE.md HTMLコメントの非表示化**: ドキュメント管理の観点から重要な変更です。Claude に見せたくない内部メモを HTML コメントとして記述できるようになります
- **並列ツール呼び出しの修正**: `Read` / `WebFetch` / `Glob` の失敗が他ツールをキャンセルしなくなったことで、並列実行の信頼性が向上しました
- **バンドルサイズ 510 KB 削減**: 起動時間の短縮に貢献します

## まとめ

v2.1.72 は非常に広範な変更を含むリリースです。SSH 環境でのクリップボード問題への対応、ワークツリーワークフローの完結化、bash パーミッションプロンプトの誤検知削減など、日常的な開発作業の摩擦を減らす改善が揃っています。特に SDK を使った自動化ワークフローでは、prompt cache の修正によって最大 12 倍のトークンコスト削減が期待でき、実用的なインパクトが大きいと言えます。多数のバグ修正によって全体的な安定性も向上しており、アップデートを強くお勧めします。

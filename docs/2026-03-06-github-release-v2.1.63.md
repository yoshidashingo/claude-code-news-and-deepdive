# Claude Code v2.1.63 リリース: HTTP Hooks、メモリリーク修正、新スラッシュコマンド追加

## 概要

Claude Code v2.1.63 では、HTTP Hooks の追加や `/simplify`・`/batch` スラッシュコマンドの新規導入など、複数の新機能が追加されました。また、長期セッションで問題となっていた多数のメモリリーク・リスナーリークが集中的に修正されており、安定性が大幅に向上しています。git worktree 間でのプロジェクト設定共有など、チーム開発に役立つ改善も含まれています。

## 新機能

### `/simplify` と `/batch` スラッシュコマンドの追加

新たに 2 つのバンドルスラッシュコマンドが追加されました。

- **`/simplify`**: コードのシンプル化・リファクタリングを支援するコマンドです。
- **`/batch`**: 複数の操作をまとめて実行するためのコマンドです。

### HTTP Hooks のサポート

これまでシェルコマンドのみ対応していた Hooks に、**HTTP Hooks** が追加されました。JSON を指定の URL に POST し、レスポンスとして JSON を受け取ることができます。シェルスクリプトを介さずに外部サービスや API と直接連携できるため、CI/CD パイプラインや Webhook との統合が容易になります。

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "http",
            "url": "https://example.com/hooks/pre-tool",
            "timeout": 5000
          }
        ]
      }
    ]
  }
}
```

### MCP OAuth 認証の手動 URL 貼り付けフォールバック

MCP OAuth 認証時に、localhost への自動リダイレクトが失敗した場合でも、コールバック URL を手動で貼り付けることで認証を完了できるフォールバック手段が追加されました。ネットワーク環境によって自動リダイレクトが動作しないケースに対応できます。

### `ENABLE_CLAUDEAI_MCP_SERVERS=false` 環境変数

claude.ai の MCP サーバーを利用可能にする機能をオプトアウトするための環境変数が追加されました。セキュリティポリシーや運用上の理由で claude.ai の MCP サーバーを無効化したい場合に使用できます。

```bash
ENABLE_CLAUDEAI_MCP_SERVERS=false claude
```

### `/copy` コマンドに「常にフルレスポンスをコピー」オプションを追加

`/copy` コマンドのピッカーに **"Always copy full response"** オプションが追加されました。選択すると、以後の `/copy` コマンドはコードブロックピッカーをスキップしてレスポンス全体を直接コピーするようになります。

### VSCode: セッションの名前変更・削除アクション

VSCode 拡張機能のセッション一覧に、セッションの名前変更と削除のアクションが追加されました。複数のセッションを管理する際の利便性が向上します。

## 改善点

### git worktree 間でプロジェクト設定と auto memory を共有

同一リポジトリの git worktree 間で、**プロジェクト設定（Project configs）** と **auto memory** が共有されるようになりました。複数の worktree を使って並行作業する際も、設定や記憶を統一して管理できます。

### `/model` コマンドに現在のモデルを表示

`/model` コマンドのメニューに、現在アクティブなモデルが表示されるようになりました。どのモデルを使用しているかを素早く確認できます。

### subagent を使った長期セッションのメモリ使用量改善

コンテキスト圧縮（context compaction）時に、重いプログレスメッセージのペイロードを除去するようになり、subagent を使った長期セッションでのメモリ使用量が改善されました。

## バグ修正

### スラッシュコマンド出力の表示修正

`/cost` などのローカルスラッシュコマンドの出力が、ユーザー送信メッセージとして表示されていた問題が修正されました。正しくシステムメッセージとして表示されるようになります。

### `/clear` でキャッシュされたスキルがリセットされない問題の修正

`/clear` コマンドを実行してもキャッシュされたスキルがリセットされず、古いスキルの内容が新しい会話に引き継がれてしまう問題が修正されました。

### VSCode: リモートセッションが会話履歴に表示されない問題の修正

VSCode でリモートセッションが会話履歴に表示されない問題が修正されました。

### ファイルカウントキャッシュが glob ignore パターンを無視する問題の修正

ファイルカウントキャッシュが `.gitignore` などの glob ignore パターンを無視していた問題が修正されました。

### IDE ホスト IP 検出キャッシュがポートをまたいで共有される問題の修正

IDE ホストの IP 検出キャッシュが、異なるポート間で誤って結果を共有していた問題が修正されました。

### REPL ブリッジのレース条件修正

REPL ブリッジにおいて、初回接続のフラッシュ中に新しいメッセージと履歴メッセージが混在し、メッセージの順序が崩れるレース条件が修正されました。

### 多数のメモリリーク・リスナーリーク修正

長期間使用した際に問題となっていた以下のメモリリーク・リスナーリークが修正されました:

- ブリッジポーリングループのリスナーリーク
- MCP OAuth フロークリーンアップのリスナーリーク
- 自動承認時のインタラクティブパーミッションハンドラーのリスナーリーク
- Bash コマンドプレフィックスキャッシュのメモリリーク
- MCP ツール/リソースキャッシュの再接続時リーク
- WebSocket トランスポート再接続時のリスナーリーク
- git ルート検出キャッシュの無制限増大
- JSON パースキャッシュの無制限増大
- 長期実行中の teammate が AppState に全メッセージを保持し続ける問題
- MCP サーバーフェッチキャッシュが切断時にクリアされない問題
- フックス設定メニューナビゲート時のメモリリーク

## 技術的なポイント

- **HTTP Hooks** の導入により、シェルスクリプトを経由せずに外部 API や Webhook と直接統合できるようになりました。セキュリティ面でもシェルインジェクションリスクを低減できます。
- **メモリリーク修正の集中対応**: 今回のリリースでは、長期セッションや頻繁な再接続を伴う使用シナリオで顕在化していたリーク問題が網羅的に修正されており、本番環境での安定運用に直結する改善です。
- **git worktree サポート強化**: 複数ブランチを並行して作業する開発スタイルにおいて、設定の一貫性が保たれるようになりました。
- **MCP OAuth フォールバック**: ネットワーク制限がある環境（プロキシ経由など）でも MCP 認証が完了できるようになり、エンタープライズ環境での利用が容易になります。
- `ENABLE_CLAUDEAI_MCP_SERVERS=false` は、CI 環境やセキュリティ要件の厳しい環境でクラウドサービスへの接続を制限したい場合に有用です。

## まとめ

v2.1.63 は新機能の追加と品質向上の両面で充実したリリースです。特に HTTP Hooks は Hooks 機能の大きな拡張であり、外部サービスとの統合パターンが広がります。また、多数のメモリリーク修正は長時間・高負荷での使用安定性を高めるもので、CI/CD 環境や長期セッションを多用する開発者にとって特に恩恵が大きいでしょう。git worktree の改善により、大規模プロジェクトでのマルチブランチ開発ワークフローも一層スムーズになります。

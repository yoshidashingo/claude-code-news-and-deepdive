# Claude Code v2.1.71 リリース — /loop コマンド、Cron スケジューリング、音声キーバインド改善など

## 概要

Claude Code v2.1.71 では、定期的なタスクを自動化する `/loop` コマンドと Cron スケジューリングツールが新たに追加されました。また、音声入力のキーバインドをカスタマイズできる `voice:pushToTalk` 設定が追加され、長時間セッションや起動時のフリーズに関する多数のバグ修正が行われています。プラグイン管理や MCP コネクタ周りの安定性も大きく向上しています。

## 新機能

### `/loop` コマンドによる繰り返し実行

新たに追加された `/loop` コマンドを使うと、指定した間隔でプロンプトやスラッシュコマンドを自動的に繰り返し実行できます。

```
/loop 5m check the deploy
```

上記の例では、5分ごとにデプロイの状態確認を行います。CI/CD パイプラインの監視や定期的なステータスチェックなど、ポーリングが必要な作業を Claude Code 内で完結させられるようになります。

### Cron スケジューリングツール

セッション内での定期的なプロンプト実行を支援する Cron スケジューリングツールが追加されました。`/loop` コマンドはこの仕組みの上に構築されており、より柔軟なスケジュール管理が可能です。

### `voice:pushToTalk` キーバインドのカスタマイズ

音声入力（Voice Mode）のアクティベーションキーを `keybindings.json` でリバインドできるようになりました。デフォルトはスペースキーですが、`meta+k` のような修飾子＋文字のコンボに変更することで、通常のタイピング中に誤って音声入力が起動する干渉をゼロにできます。

```json
{
  "voice:pushToTalk": "meta+k"
}
```

### Bash 自動承認許可リストの拡張

以下のコマンドが Bash 自動承認の許可リストに追加されました。頻繁に使用するユーティリティコマンドで毎回承認を求められることがなくなります。

- `fmt`, `comm`, `cmp`, `numfmt`, `expr`, `test`, `printf`, `getconf`, `seq`, `tsort`, `pr`

## 改善点

### 起動時間の短縮

ネイティブ画像プロセッサの読み込みが初回使用時まで遅延されるようになり、起動時間が改善されました。

### ブリッジセッション再接続の高速化

ラップトップがスリープから復帰した後のブリッジセッション再接続が、従来の最大 10 分待機から数秒以内に完了するようになりました。

### プラグイン管理の改善

`/plugin uninstall` がプロジェクトスコープのプラグインを `.claude/settings.local.json` で無効化するようになりました（従来は `.claude/settings.json` を変更していた）。これにより、チームメンバーの設定に影響を与えずにプラグインを無効化できます。

また、プラグインが提供する MCP サーバーが手動設定済みのサーバー（同一コマンド/URL）と重複する場合、自動的にスキップされるようになりました。重複の抑制状況は `/plugin` メニューで確認できます。

### `/debug` によるセッション中のデバッグ切り替え

デバッグログがデフォルトで出力されなくなった代わりに、`/debug` コマンドでセッション中にデバッグロギングをオン/オフできるようになりました。

## バグ修正

### セッションフリーズ関連

- **長時間セッションでの stdin フリーズ**: キーストロークが処理されなくなるのに、プロセスは生き続けるという問題が修正されました。
- **Voice Mode 有効時の起動フリーズ**: システムスリープ復帰後に CoreAudio 初期化がメインスレッドをブロックし、5〜8 秒の起動フリーズが発生していた問題が修正されました。
- **OAuth トークン同時更新による起動 UI フリーズ**: 複数の Claude Code インスタンスが claude.ai プロキシコネクタの期限切れトークンを同時にリフレッシュしようとした際に UI がフリーズする問題が修正されました。

### `/fork` のプランファイル共有問題

`/fork` で作成したフォーク会話が同一のプランファイルを共有していたため、一方のフォークでの編集がもう一方に上書きされる問題が修正されました。

### Read ツールの過大な画像コンテキスト

画像処理が失敗した際に Read ツールが過サイズの画像をコンテキストに追加してしまい、画像が多いセッションの後続ターンが壊れる問題が修正されました。

### Bash の誤った権限プロンプト

Heredoc を含む Bash コマンド（例: Heredoc スタイルのコミットメッセージを使った `git commit`）で、誤って権限確認プロンプトが表示される false positive が修正されました。

### プラグイン・コネクタ関連

- 複数の Claude Code インスタンスを並行して起動した際にプラグインのインストールが消失する問題を修正
- OAuth トークン更新後に claude.ai コネクタが再接続に失敗する問題を修正
- 組織設定の claude.ai MCP コネクタ起動通知が、全コネクタに対して表示されていた問題を修正（以前接続済みのものだけに限定）

### バックグラウンドエージェントの通知

バックグラウンドエージェント完了通知に出力ファイルパスが含まれていなかった問題が修正されました。コンテキスト圧縮後に親エージェントがエージェント結果を回収しにくかった問題が解消されます。

### その他のバグ修正

- Bash ツールのエラーメッセージで出力が重複していた問題を修正
- Chrome 拡張機能の自動検出が「未インストール」のまま永続的にスタックする問題を修正
- `/plugin marketplace update` でマージコンフリクトが発生する問題を修正
- `/plugin marketplace add owner/repo@ref` で `@` の解析が正しく動作しなかった問題を修正（従来は `#` のみ有効だった）
- `/permissions` の Workspace タブで、末尾スラッシュの有無が異なる同一ディレクトリが重複表示される問題を修正
- チームエージェント設定時に `--print` が無限ハングする問題を修正（`in_process_teammate` タスクの待機ループを修正）
- `ToolSearch` 呼び出し後に REPL で「❯ Tool loaded.」が毎回表示される問題を修正
- Windows 上で mingw スタイルのパスを使用した場合に `cd <cwd> && git ...` の権限確認が表示される問題を修正
- 未認証の組織登録 claude.ai コネクタに対する起動通知のノイズを削減

## 技術的なポイント

- **`/loop` と Cron**: セッション内で時間駆動の自動化が可能になり、外部スクリプトなしでポーリングや監視タスクを実現できます。
- **音声キーバインドの分離**: `voice:pushToTalk` を修飾子キーに割り当てることで、コーディング中の誤起動を防止できます。
- **プラグインスコープの改善**: `settings.local.json` への書き込みにより、個人の設定変更がリポジトリにコミットされなくなり、チーム開発での安全性が向上します。
- **MCP サーバー重複排除**: 同一コマンドや URL で複数の接続が張られることを防ぐことで、ツールセットの重複によるトークン消費の無駄を削減できます。
- **ブリッジ再接続の改善**: スリープ復帰後の再接続ロジック見直しにより、作業の中断時間が大幅に短縮されます。

## まとめ

v2.1.71 は、自動化・繰り返しタスクのサポートと安定性の両面で大きな前進をしたリリースです。`/loop` コマンドと Cron スケジューリングにより、これまで外部ツールが必要だったポーリングや定期チェックを Claude Code 内で完結させられるようになります。また、長時間セッションや Voice Mode 使用時のフリーズ問題、プラグイン管理の不具合など、日常的な使用で遭遇しやすい問題が多数修正されており、全体的な信頼性が向上しています。プラグインのスコープ管理とMCPサーバーの重複排除の改善は、チームでの利用においても特に歓迎されるでしょう。

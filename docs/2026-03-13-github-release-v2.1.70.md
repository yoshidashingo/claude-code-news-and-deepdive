# Claude Code v2.1.70 リリース — 多数のバグ修正とVS Code統合の大幅強化

## 概要

Claude Code v2.1.70 は、API エラー・Windows 環境・音声機能・プラグインなど幅広い領域にわたる 20 件超のバグ修正を中心としたリリースです。パフォーマンス面ではプロンプト入力の再レンダリングを約 74% 削減するなど顕著な改善が図られています。また VS Code 統合においては、セッション一覧アイコン・プラン Markdown ビュー・ネイティブ MCP サーバー管理ダイアログという 3 つの新機能が追加されました。

## 新機能

### VS Code アクティビティバーへのセッション一覧追加

VS Code のアクティビティバーに Claude Code 専用のスパークアイコンが追加されました。クリックするとアクティブなすべての Claude Code セッションが一覧表示され、各セッションをフルエディタとして開くことができます。ターミナルを切り替えながらセッションを探す手間が不要になり、マルチセッション運用が大幅に快適になります。

### プラン用 Markdown ドキュメントビュー（VS Code）

VS Code 上でプランのフル Markdown ドキュメントビューが利用可能になりました。プランに対してコメントを追加してフィードバックを提供する機能も含まれており、チームでのレビューや承認フローに組み込みやすくなっています。

### ネイティブ MCP サーバー管理ダイアログ（VS Code）

チャットパネルで `/mcp` コマンドを実行すると、MCP サーバーの有効化・無効化・再接続・OAuth 認証管理がダイアログ上で完結するようになりました。これまでターミナルで設定ファイルを直接編集する必要があった操作が、GUI で直感的に行えます。

## 改善点

### パフォーマンス改善

- **プロンプト入力の再レンダリングを約 74% 削減**: ターン中の入力フォームの不要な再描画が大幅に抑制され、入力レスポンスが向上します。
- **起動時メモリを約 426KB 削減**: カスタム CA 証明書を使用していないユーザー向けに、起動時のメモリフットプリントが削減されました。
- **Remote Control `/poll` レートを 300 倍削減**: 接続中のポーリング間隔が 1〜2 秒から 10 分に延長され、サーバー負荷が大幅に低減します。切断検知時は即時に高頻度ポーリングへ復帰するため、再接続速度への影響はありません。
- **コンパクション時の画像プロンプトキャッシュ保持**: サマライザーリクエストに画像を含めるようにすることでプロンプトキャッシュが再利用され、コンパクションの速度とコストが改善されます。

### その他の改善

- **`/rename` コマンドを処理中でも実行可能に**: 以前はキューに積まれて処理が終わるまで待たされていましたが、Claude が処理中でも即時にリネームが反映されるようになりました。
- **マイクの無音キャプチャのエラーメッセージ改善**: 「no speech detected」と区別した分かりやすいエラーメッセージが表示されます。

## バグ修正

### API・モデル関連

- **`ANTHROPIC_BASE_URL` 使用時の API 400 エラー修正**: サードパーティゲートウェイ経由でアクセスしている場合に `tool_reference` ブロックが原因で発生していたエラーを解消しました。ツール検索がプロキシエンドポイントを正しく検出し、自動的に無効化します。
- **カスタム Bedrock 推論プロファイルでの `effort` パラメーターエラー修正**: `This model does not support the effort parameter` エラーが、標準の Claude 命名規則に合わないカスタムモデル識別子で発生していた問題を修正しました。
- **`ToolSearch` 直後の空レスポンス修正**: サーバーがツールスキーマをシステムプロンプトスタイルのタグとしてプロンプト末尾に追加していたため、モデルが早期停止してしまう問題を修正しました。
- **MCP サーバー接続後のプロンプトキャッシュバスト修正**: `instructions` を持つ MCP サーバーが最初のターン後に接続した際にキャッシュが無効化されていた問題を修正しました。

### Windows / WSL 環境

- **クリップボードでの非 ASCII 文字破損修正**: Windows/WSL 環境において CJK 文字や絵文字をクリップボードにコピーすると文字化けしていた問題を修正しました。PowerShell の `Set-Clipboard` を使用することで正しく処理されます。
- **VS Code 統合ターミナルからの起動時に余分なウィンドウが開く問題を修正**: Windows の VS Code 統合ターミナルから Claude Code を起動すると、不要な VS Code ウィンドウが追加で開いてしまう問題が解消されました。
- **Windows ネイティブバイナリでの音声モードエラー修正**: `native audio module could not be loaded` というエラーで音声機能が使えなかった問題を修正しました。
- **`voiceEnabled: true` 設定時のプッシュトゥトーク起動失敗修正**: セッション開始時に設定が反映されず、プッシュトゥトークが有効にならない問題を修正しました。

### 入力・表示

- **低速 SSH 接続での Enter キー動作修正**: 低速な SSH 接続環境で Enter キーを押すと改行が挿入されてしまい、送信できない問題を修正しました。
- **`#NNN` を含むマークダウンリンクの誤ったリンク先修正**: マークダウン中の `#NNN` 形式のリンクが外部 URL ではなく現在のリポジトリの Issue として解釈されていた問題を修正しました。
- **Opus 4.6 の「Model updated」通知の重複表示修正**: プロジェクトの `.claude/settings.json` にレガシーな Opus モデル文字列がピン留めされている場合に、通知が繰り返し表示される問題を修正しました。
- **`/color` コマンドのリセット手段を追加**: 以前はデフォルトカラーに戻す方法がありませんでしたが、`/color default`・`/color gray`・`/color reset`・`/color none` のいずれかでリセットできるようになりました。
- **テレポートマーカーが VS Code テレポートセッションで表示されない問題を修正**

### プラグイン

- **`/plugin` でのインストール状態が不正確に表示される問題を修正**
- **マーケットプレイスからインストール直後に「not found in marketplace」エラーが出る問題を修正**: インストール後に自動リフレッシュが行われるようになりました。

### その他

- **`/security-review` が古い git バージョンで失敗する問題を修正**: `unknown option merge-base` エラーが出る問題を修正しました。
- **`--resume` 時のスキルリスト再注入を修正**: 毎回スキルリストが再注入されて約 600 トークンが無駄に消費されていた問題を修正しました。
- **`permissions.defaultMode` がリモート環境で誤って適用される問題を修正**: `acceptEdits` または `plan` 以外の値は Claude Code Remote 環境では無視されるようになりました。
- **`AskUserQuestion` プレビューダイアログのパフォーマンス回帰を修正**: ノート入力のキーストロークごとにマークダウンレンダリングが走っていたパフォーマンス問題を修正しました。
- **起動時のフィーチャーフラグキャッシュが更新されない問題を修正**: 古い値がセッションをまたいで残り続けていた問題を解消しました。

## 技術的なポイント

- **プロキシ環境でのツール検索の改善**: `ANTHROPIC_BASE_URL` を使ったサードパーティゲートウェイでは `tool_reference` ブロックがサポートされないケースがあります。今回の修正でプロキシエンドポイントを自動検出して適切にフォールバックするため、独自のゲートウェイを経由している環境での安定性が向上します。
- **MCP とプロンプトキャッシュの相互作用**: MCP サーバーの `instructions` フィールドがプロンプトキャッシュのキーに影響することが明らかになりました。接続タイミングによるキャッシュバストを防ぐことで、コストと速度の両面での恩恵が期待できます。
- **Remote Control のポーリング最適化**: 接続中のポーリング頻度を大幅に下げつつ、切断時には即座に高頻度ポーリングへ切り替えるアダプティブな方式を採用しています。マルチエージェント・CI 環境での運用コスト削減に直結します。
- **Windows でのクリップボード処理**: CJK や絵文字など非 ASCII 文字のクリップボード問題は、PowerShell の `Set-Clipboard` コマンドレットへの切り替えで解決されました。WSL 環境を使う日本語ユーザーには特に重要な修正です。

## まとめ

v2.1.70 は新機能よりもバグ修正と安定性強化に注力したリリースです。特に Windows/WSL 環境でのクリップボード問題・音声機能の修正は日本語ユーザーにとって重要な改善です。また VS Code 統合における MCP サーバー管理ダイアログの追加は、MCP を活用したワークフロー構築をより手軽にするものです。パフォーマンス面でもプロンプト入力の再レンダリング削減・Remote Control のポーリング最適化など実用的な改善が含まれており、日常的な利用体験の向上が見込まれます。

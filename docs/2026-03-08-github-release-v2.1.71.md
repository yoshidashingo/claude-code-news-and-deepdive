# Claude Code v2.1.71 リリース — /loop コマンドやCronスケジューリング、多数のバグ修正

## 概要

Claude Code v2.1.71 がリリースされました。今回のリリースでは、定期的なプロンプト実行を可能にする `/loop` コマンドと Cron スケジューリング機能が新たに追加されました。また、起動時のフリーズやセッション凍結など長年ユーザーを悩ませてきたパフォーマンス問題が複数修正されており、特に Voice モードユーザーや claude.ai コネクタ利用者に大きな恩恵があります。

## 新機能

### `/loop` コマンド — 定期実行を手軽に

最も注目すべき新機能が `/loop` コマンドです。指定した間隔でプロンプトやスラッシュコマンドを繰り返し実行できます。

```
/loop 5m check the deploy
```

上記の例では、5分ごとにデプロイの状態確認を自動的に行います。CI/CD パイプラインの監視や、長時間タスクの定期チェックなど、繰り返しの確認作業を自動化する用途に活用できます。

### Cron スケジューリングツール

セッション内での定期的なプロンプト実行を管理する Cron スケジューリングツールも追加されました。`/loop` コマンドと組み合わせることで、より柔軟なスケジュール管理が可能になります。

### Voice モードのキーバインディング改善

`keybindings.json` から `voice:pushToTalk` キーをカスタマイズできるようになりました（デフォルトはスペースキー）。`meta+k` のような修飾キーと文字の組み合わせに対応しており、通常のタイピング中にキーが誤って反応する問題を防げます。

```json
{
  "voice:pushToTalk": "meta+k"
}
```

### Bash 自動承認ホワイトリストの拡張

以下の標準 Unix コマンドが Bash 自動承認ホワイトリストに追加されました。

- `fmt`, `comm`, `cmp`, `numfmt`, `expr`
- `test`, `printf`, `getconf`, `seq`, `tsort`, `pr`

これらのコマンドを使用する際に毎回権限プロンプトが表示されなくなり、スクリプト処理やデータ変換タスクがよりスムーズに行えます。

## バグ修正

### 起動・パフォーマンス関連

**起動フリーズの修正（Voice モード）**
Voice モードが有効な環境で、システムのスリープ復帰後に CoreAudio の初期化がメインスレッドをブロックし、5〜8 秒の起動フリーズが発生していた問題が修正されました。ネイティブ画像プロセッサの読み込みを初回使用時まで遅延させることで、全体的な起動時間も短縮されています。

**起動 UI フリーズの修正**
複数の claude.ai プロキシコネクタが同時に期限切れの OAuth トークンをリフレッシュしようとした際に UI が凍結する問題が修正されました。

**stdin 凍結の修正**
長時間のセッションでキーストロークが処理されなくなるにもかかわらずプロセスが動き続ける問題が修正されました。長時間作業するユーザーにとって重要な修正です。

### セッション・フォーク管理

**`/fork` の競合問題の修正**
`/fork` コマンドでフォークした会話が同一のプランファイルを共有してしまい、一方のフォークでの編集がもう一方に上書きされる問題が修正されました。フォーク機能を活用する場合は v2.1.71 以降へのアップデートを推奨します。

**複数インスタンスでのプラグインロスト修正**
複数の Claude Code インスタンスを同時に起動している場合に、プラグインのインストール情報が失われる問題が修正されました。

### 接続性・コネクタ関連

**claude.ai コネクタの再接続改善**
OAuth トークンリフレッシュ後にコネクタが再接続できなくなる問題が修正されました。また、ブリッジセッションの再接続時間がラップトップのスリープ復帰後に最大 10 分かかっていたのが、数秒で完了するよう大幅に改善されました。

**MCP コネクタの通知ノイズ削減**
起動時に org 設定の全コネクタに対して通知が表示されていた問題が修正され、以前に接続実績のあるコネクタのみに絞られるようになりました。未認証の org 登録 claude.ai コネクタの起動通知ノイズも削除されています。

### ツール・コマンド関連

**Read ツールの画像処理修正**
画像処理に失敗した際に、サイズ超過の画像がコンテキストに残り続けて以降のターンが壊れる問題が修正されました。画像を多用するセッションでの安定性が向上します。

**Bash コマンドの誤権限プロンプト修正**
heredoc 形式のコミットメッセージを含む複合 Bash コマンドに対して誤って権限プロンプトが表示されていた問題が修正されました。git コミット時のワークフローがよりスムーズになります。

**Windows での git コマンド修正**
Windows 環境でモデルが mingw スタイルのパスを使用した場合に `cd <cwd> && git ...` の実行確認プロンプトが不要に表示されていた問題が修正されました。

**背景エージェント完了通知の修正**
完了通知に出力ファイルパスが含まれておらず、コンテキスト圧縮後に親エージェントが結果を回復できない問題が修正されました。

### プラグイン・マーケットプレイス

**`/plugin marketplace add` の `@` 解析修正**
`owner/repo@ref` 形式でのプラグイン追加が正しく動作していなかった問題が修正されました（従来は `#` のみが動作していました）。

**`/plugin marketplace update` のマージコンフリクト修正**
マーケットプレイスがブランチやタグ ref に固定されている場合に `update` コマンドがマージコンフリクトで失敗する問題が修正されました。

**`/plugin uninstall` の改善**
プロジェクトスコープのプラグインを無効化する際、`.claude/settings.json` ではなく `.claude/settings.local.json` に変更が書き込まれるようになりました。チームメンバーの設定に影響を与えずにプラグインを管理できます。

**MCP サーバーの重複排除改善**
プラグインが提供する MCP サーバーが手動設定済みのサーバーと重複する場合（同一コマンド/URL）、自動的にスキップされるようになりました。抑制されたサーバーは `/plugin` メニューで確認できます。

### その他の細かな修正

- **`/permissions` タブ**: 末尾スラッシュの有無による同一ディレクトリの重複エントリを修正
- **`--print` フラグ**: チームエージェント設定時に無限にハングする問題を修正（`in_process_teammate` タスクを終了ループで待機しなくなりました）
- **REPL 通知**: `ToolSearch` 呼び出し後に「Tool loaded.」が重複表示される問題を修正
- **`/debug` コマンド**: セッション途中でのデバッグログの ON/OFF 切り替えに対応
- **Chrome 拡張自動検出**: Chrome 未インストール環境で「not installed」状態に永続的に固定される問題を修正
- **Bash エラーメッセージ**: ゼロ以外のステータスで終了した際の重複出力を修正

## 技術的なポイント

- **`/loop` と Cron**: 定期実行のユースケースとして、デプロイ監視・ログ監視・定期的なコードレビューなどが考えられます。インターバルは `5m`（分）形式で指定します
- **Voice モードの CoreAudio 問題**: macOS のスリープ復帰後に CoreAudio が非同期初期化を完了する前にメインスレッドが呼び出していたことが原因で、非同期初期化パターンへの変更で解決されています
- **プラグイン設定のスコープ分離**: `.claude/settings.json`（チーム共有）と `.claude/settings.local.json`（個人ローカル）の使い分けが徹底され、チーム開発環境での設定管理が改善されています
- **ブリッジセッション再接続**: 10 分から数秒への大幅な改善は、ノートパソコンでの開発ワークフローに直接影響します
- **Bash ホワイトリスト拡張**: テキスト処理・数値計算系のコマンド群が追加されており、シェルスクリプトを多用するタスクでの承認フローが簡略化されます

## まとめ

v2.1.71 は新機能と安定性改善の両面で充実したリリースです。`/loop` コマンドによる定期実行機能は、監視・自動化タスクにおける新しいユースケースを開拓するものです。一方で、起動フリーズ・stdin 凍結・`/fork` の競合といった実運用上の問題点も多数解消されており、長時間使用や複数インスタンス運用の安定性が向上しています。

特に Voice モードユーザーと claude.ai コネクタを利用しているチームは、起動時間の短縮と接続安定性の大幅な改善を実感できるでしょう。プラグイン管理の改善（settings.local.json への書き込み・MCP サーバーの重複排除）は、チーム開発環境における Claude Code 運用をよりクリーンに保つための重要な一歩です。

# Claude Code v2.1.72 リリース — ワークツリー強化・effort 簡素化・大量バグ修正

## 概要

Claude Code v2.1.72 は、ワークツリー管理の強化や `/plan` コマンドの利便性向上など多数の新機能に加え、音声モード・プラグイン・権限ルールを含む幅広い領域での大量バグ修正を含む大型アップデートです。bash パースをネイティブモジュールへ切り替えることで初期化速度の向上とメモリリークの解消も実現しています。また SDK `query()` のプロンプトキャッシュ修正により、入力トークンコストが最大 12 倍削減されるケースもあります。

## 新機能

### `ExitWorktree` ツールの追加

`EnterWorktree` で開始した worktree（作業ツリー）セッションを明示的に終了するための `ExitWorktree` ツールが追加されました。これにより、worktree の開始・終了をプログラム的に対称な操作として扱えるようになります。

### `/copy` コマンドに `w` キーを追加

`/copy` コマンドでフォーカスされている選択内容を、クリップボードを経由せずに直接ファイルへ書き込む `w` キーが追加されました。SSH 接続中など、クリップボードが利用できない環境で特に役立ちます。

### `/plan` コマンドへの説明引数の追加

`/plan` コマンドにオプションの説明引数が追加され、プランモードへの移行と同時にタスクの概要を渡せるようになりました。

```
/plan fix the auth bug
```

このように実行することで、プランモードに入りながら即座に指定した内容でプランニングを開始できます。

### `claude plugins` エイリアスの追加

`claude plugin` コマンドの エイリアスとして `claude plugins`（複数形）が追加されました。どちらのコマンドも同じ動作をします。

### `CLAUDE_CODE_DISABLE_CRON` 環境変数の追加

新しい環境変数 `CLAUDE_CODE_DISABLE_CRON` を設定することで、セッション中にスケジュール済みの cron ジョブを即座に停止できます。長期実行セッションでのジョブ管理に役立ちます。

### Agent ツールへの `model` パラメータの復活

Agent ツールに `model` パラメータが再び利用可能になりました。これにより、Agent ツールの呼び出しごとに使用するモデルをオーバーライドできます。チームエージェントがリーダーのモデルを継承するよう修正された変更も含まれています。

### VSCode: `vscode://anthropic.claude-code/open` URI ハンドラの追加

VSCode 拡張に URI ハンドラが追加されました。`vscode://anthropic.claude-code/open` を呼び出すことで、新しい Claude Code タブをプログラムから開けます。`prompt` および `session` クエリパラメータもサポートしています。

```
vscode://anthropic.claude-code/open?prompt=Hello&session=abc123
```

## 改善点

### effort レベルの簡素化

effort レベルが `low / medium / high / max` の 4 段階から `low / medium / high` の 3 段階に簡素化されました。表示シンボルも新しくなっています。

| レベル | シンボル |
|--------|--------|
| low    | ○      |
| medium | ◐      |
| high   | ●      |

また、effort レベルは永続的なアイコン表示から簡潔な通知形式に変わりました。デフォルト（auto）に戻すには `/effort auto` を実行します。

### bash 自動承認許可リストの拡充

`lsof`、`pgrep`、`tput`、`ss`、`fd`、`fdfind` が bash の自動承認許可リストに追加されました。これらは一般的な読み取り専用操作に使われるコマンドであり、不要な権限プロンプトの表示が減ります。

### `/config` の操作性向上

`/config` コマンドのキー操作が改善されました。

- **Escape**: 変更をキャンセル
- **Enter**: 変更を保存して閉じる
- **Space**: 設定をトグル

### 上矢印による履歴表示の改善

複数の並行セッションを実行中に上矢印キーで履歴を遡る際、現在のセッションのメッセージが優先して表示されるようになりました。

### 音声入力の認識精度向上

音声入力における文字起こしの精度が改善され、リポジトリ名や開発でよく使う用語（regex、OAuth、JSON など）がより正確に認識されるようになりました。

### bash パースのネイティブモジュール切り替え

bash コマンドのパースがネイティブモジュールへ切り替えられました。これにより、初期化速度が向上し、以前存在していたメモリリークも解消されています。また、tree-sitter パースの強化により、`find -exec`・変数代入・コマンド置換などに起因する誤った権限プロンプトが削減されました。npm インストール版で tree-sitter が読み込まれなかった問題も修正されています。

### CLAUDE.md HTML コメントの非表示化

`CLAUDE.md` 内の HTML コメント（`<!-- ... -->`）が Claude に自動注入される際に非表示になりました。ただし、Read ツールで明示的に読み込んだ場合にはコメントは引き続き見えます。これにより、CLAUDE.md 内に内部メモやメタ情報を HTML コメントとして記載しておくことができます。

### プロキシゲートのバイパス改善

ツール検索において、環境変数が設定されている場合にサードパーティプロキシゲートをバイパスするよう変更されました。なお、以前使用されていた `CLAUDE_CODE_PROXY_SUPPORTS_TOOL_REFERENCE` 環境変数は削除されています。

### バンドルサイズの削減

バンドルサイズが約 510 KB 削減されました。

## バグ修正

### SDK・パフォーマンス関連

- **SDK `query()` のプロンプトキャッシュ**: 無効化の不具合を修正し、入力トークンコストを最大 12 倍削減
- **長期セッションの CPU 利用率**: 長時間セッションでの CPU 使用率を改善
- **低速な終了処理**: バックグラウンドタスクやフックの応答が遅い場合の終了遅延を修正

### セッション・会話管理

- **`--continue` フラグ**: `--compact` 後に最新の地点から正しく再開できない問題を修正
- **`/clear` コマンド**: バックグラウンドのエージェント・bash タスクが誤って削除される問題を修正（フォアグラウンドタスクのみが対象に）
- **Ctrl+B のバックグラウンドクエリ**: `/clear` 後に transcript が失われたり新規会話が破損する問題を修正
- **ダブル Ctrl+C**: バックグラウンドのエージェント・タスク実行中に終了できない問題を修正
- **Escape キー**: クエリのキャンセル後に Escape キーが反応しなくなる問題を修正

### ワークツリー関連

- **Task ツールの再開**: 作業ディレクトリが正しく復元されない問題を修正
- **バックグラウンドタスク通知**: `worktreePath` と `worktreeBranch` が欠落する問題を修正

### 音声モード関連

- 入力の遅延
- プッシュトゥトークを離した後の誤った「音声が検出されません」エラー
- 送信後に古い transcript が再表示される問題

### プラグイン関連

- Windows の OneDrive フォルダで `EEXIST` エラーによりインストールが失敗する問題
- プロジェクトスコープのインストールが存在する場合にマーケットプレイスがユーザースコープのインストールをブロックする問題
- `CLAUDE_CODE_PLUGIN_CACHE_DIR` がリテラルの `~` ディレクトリを作成する問題
- マーケットプレイス専用フィールドを含む `plugin.json` が読み込めない問題

### 権限・セキュリティ関連

- **ワイルドカードルール**: heredocs・埋め込み改行・引数なしのコマンドにマッチしない問題を修正
- **`sandbox.excludedCommands`**: 環境変数プレフィックス付きのコマンドで失敗する問題を修正
- **"Always Allow"**: 再びマッチしない可能性のある権限ルールが保存される問題を修正
- **deny ルール**: 全コマンド形式に適用されない問題を修正
- **ファイル書き込み操作**: プロンプトなしで誤って許可されるケースを修正
- **`/tmp/claude/` への出力リダイレクト**: 不要なプロンプトが表示される問題を修正

### フック関連

- **`transcript_path`**: 再開・フォークされたセッションで誤ったディレクトリを指す問題を修正
- **エージェントの `prompt`**: 設定書き込みのたびに `settings.json` から削除される問題を修正
- **PostToolUse のブロック理由**: 2 回表示される問題を修正
- **非同期フック**: bash `read -r` で stdin を受け取れない問題を修正
- **スキルフック**: モデルがスキルを呼び出した場合にフックが 2 回発火する問題を修正

### その他のバグ修正

- **エージェントのタスク進捗**: 「Initializing…」から進まない問題を修正
- **`/model` コマンド**: Claude が動作中に実行すると結果が表示されない問題を修正
- **プランモードの権限プロンプト**: テキスト入力中に数字キーがメニュー選択として扱われる問題を修正
- **セッションクラッシュ**: Read が U+2028/U+2029 文字を含むファイルを返した場合の Desktop/SDK でのクラッシュを修正
- **並行ツール呼び出し**: Read・WebFetch・Glob の失敗が兄弟ツール呼び出しをキャンセルしていた問題を修正（Bash のエラーのみカスケードするよう変更）
- **Bash データ URL 出力の画像**: 過剰なサイズや切り詰め問題を修正
- **Bedrock API エラー**: エラーを含むセッションを再開した際のクラッシュを修正
- **ツール入力のバリデーションエラー**: Edit・Bash・Grep で断続的に発生する「expected boolean, received string」エラーを修正
- **セッションタイトル**: 最初のメッセージが複数行の場合にタイトルが複数行になる問題を修正
- **キューイングされたメッセージ**: 添付画像が表示されない、↑ 編集時に画像が失われる問題を修正
- **ターミナルタイトル**: `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` 設定時でも終了時にタイトルがクリアされる問題を修正
- **フィードバックサーベイ**: 長時間セッションで過剰に表示される問題を修正
- **`--effort` CLI フラグ**: 起動時の設定書き込みでリセットされる問題を修正
- **マーケットプレイス対応**: `.git` サフィックスなしの git URL に対応（Azure DevOps、AWS CodeCommit）

### VSCode 固有の修正

- **統合ターミナルのスクロール速度**: ネイティブターミナルと一致するよう修正
- **Shift+Enter**: 旧キーバインディングを使用するユーザーで改行の代わりに送信が実行される問題を修正
- **effort レベルインジケータ**: 入力ボーダーに表示を追加

## 技術的なポイント

- **プロキシ環境変数の変更**: `CLAUDE_CODE_PROXY_SUPPORTS_TOOL_REFERENCE` が削除されました。プロキシ経由でツール検索を行っている場合は、新しい環境変数への移行が必要です
- **SDK の `query()` コストが大幅改善**: プロンプトキャッシュの修正により、`query()` を多用するアプリケーションでは入力トークンコストが最大 12 倍削減されます。SDK ベースのシステムでは特に恩恵が大きいです
- **`ExitWorktree` によるワークツリー管理の対称化**: `EnterWorktree` / `ExitWorktree` のペアが揃ったことで、worktree を使ったフロー制御がよりロバストになります
- **bash 権限プロンプトの誤検知削減**: tree-sitter パースの強化で、`find -exec` や変数代入など多くのパターンで不要なプロンプトが表示されなくなります。日常的な開発作業の中断が大幅に減少します
- **`/plan` のインライン起動**: `/plan fix the auth bug` のように引数を渡すことで、プランモードへの遷移と内容入力を一操作で完結できます。スクリプトや自動化シナリオでも活用できます
- **CLAUDE.md コメントの活用**: HTML コメント（`<!-- ... -->`）を使った内部メモやメタ情報が Claude に漏れなくなりました。チームでの CLAUDE.md 管理に役立つ改善です
- **並行ツール失敗の挙動変更**: Read・WebFetch・Glob のエラーが兄弟ツール呼び出しをキャンセルしなくなりました。Bash のみがエラーをカスケードします。並行ツール呼び出しを活用するワークフローで信頼性が向上します

## まとめ

v2.1.72 は、ワークツリーのサポート強化・effort レベルの整理・bash パースのネイティブ化といった機能改善に加え、音声モード・プラグイン・フック・権限ルールを中心とした非常に多数のバグ修正を含む充実したリリースです。特に SDK `query()` のプロンプトキャッシュ修正は、Claude Code を組み込んだアプリケーションの運用コスト削減に直結するため、SDK を利用している開発者にはアップデートの優先度が高いと言えます。また、bash コマンドの権限プロンプト誤検知の削減と許可リストの拡充により、日常的な開発体験も一段とスムーズになることが期待されます。

# Claude Code v2.1.70 リリース — 大規模バグ修正とVS Code統合の強化

## 概要

Claude Code v2.1.70 は、20件を超えるバグ修正と複数の改善を含む大型メンテナンスリリースです。Windows/WSL 環境での安定性向上、API 互換性問題の解消、VS Code 統合機能の拡充が主な変更点です。特に VS Code ユーザー向けに、セッション管理・プラン表示・MCP サーバー管理の3つの新機能が追加されました。

## 新機能

### [VS Code] セッション管理パネル

VS Code のアクティビティバーにスパークアイコンが追加され、すべての Claude Code セッションを一覧表示できるようになりました。各セッションはフルエディタとして開くことができ、複数セッションの並行管理が容易になります。

### [VS Code] プランのマークダウンドキュメントビュー

プラン（計画）の内容を VS Code 上でフルマークダウンドキュメントとして表示できるようになりました。コメントを追加してフィードバックを提供する機能も搭載されており、チームでの計画レビューがより直感的に行えます。

### [VS Code] ネイティブ MCP サーバー管理ダイアログ

チャットパネルで `/mcp` コマンドを実行することで、MCP（Model Context Protocol）サーバーの有効化・無効化、再接続、OAuth 認証の管理がターミナルを開くことなく行えるようになりました。

## 改善点

- **音声エラーメッセージの改善**: マイクが無音をキャプチャした場合と「音声が検出されない」場合のエラーメッセージを区別するよう改善されました
- **コンパクション時のプロンプトキャッシュ活用**: コンパクション処理でサマライザーリクエストに画像を保持するようになり、プロンプトキャッシュの再利用によって高速かつ低コストなコンパクションが可能になりました
- **`/rename` コマンドの即時実行**: Claude が処理中でも `/rename` コマンドが即時実行されるようになりました（以前はサイレントにキューに入れられていました）
- **プロンプト入力の再レンダリング削減**: ターン中のプロンプト入力の再レンダリング回数が約 74% 削減されました
- **起動時メモリ削減**: カスタム CA 証明書を使用していないユーザーの起動時メモリが約 426KB 削減されました
- **Remote Control ポーリング頻度の最適化**: 接続中の `/poll` レートが 1〜2 秒ごとから 10 分に 1 回に削減され、サーバー負荷が約 300 分の 1 になりました。トランスポート切断時の再接続は引き続き高速ポーリングで対応します

## バグ修正

### API・モデル互換性

- **サードパーティゲートウェイでの API 400 エラー修正**: `ANTHROPIC_BASE_URL` にサードパーティゲートウェイを指定した際に発生していた 400 エラーを修正しました。ツール検索がプロキシエンドポイントを正しく検出し、`tool_reference` ブロックを無効化するようになりました
- **カスタム Bedrock 推論プロファイルでの effort パラメータエラー修正**: 標準的な Claude 命名パターンに一致しないカスタム Bedrock 推論プロファイルやモデル識別子使用時に発生していた `API Error: 400 This model does not support the effort parameter` を修正しました
- **`ToolSearch` 直後の空レスポンス修正**: `ToolSearch` 実行直後にモデルが空レスポンスを返す問題を修正しました。サーバーがシステムプロンプト形式のタグを付けてツールスキーマをレンダリングしていたためにモデルが早期停止していました

### プロンプトキャッシュ・セッション

- **MCP サーバー接続時のプロンプトキャッシュ無効化修正**: `instructions` を持つ MCP サーバーが初回ターン後に接続した際に発生していたプロンプトキャッシュのバスト（無効化）を修正しました
- **`--resume` 時のスキルリスト再挿入修正**: `--resume` のたびにスキルリストが再挿入されていた問題を修正しました。1 回のリジューム当たり約 600 トークンの節約になります
- **機能フラグのディスクキャッシュ更新修正**: 起動初期に読み込まれた機能フラグがセッション間でディスクキャッシュを更新せず古い値が持続していた問題を修正しました

### Windows / WSL

- **クリップボードの非 ASCII 文字化け修正**: Windows/WSL 環境で CJK 文字や絵文字などの非 ASCII テキストがクリップボードにコピーする際に文字化けしていた問題を修正しました。PowerShell の `Set-Clipboard` を使用するよう変更されています
- **VS Code 統合ターミナルからの起動時に余分なウィンドウが開く問題修正**: Windows 上で VS Code の統合ターミナルから起動した際に余分な VS Code ウィンドウが開いていた問題を修正しました
- **Windows ネイティブバイナリでの音声モード修正**: `native audio module could not be loaded` エラーで音声モードが失敗していた問題を修正しました

### 入力・UI

- **低速 SSH 接続での Enter キー動作修正**: 低速な SSH 接続でタイピング中に Enter キーが送信ではなく改行を挿入していた問題を修正しました
- **プッシュ・トゥ・トーク起動修正**: 設定で `voiceEnabled: true` を指定した場合にセッション開始時にプッシュ・トゥ・トークが有効にならない問題を修正しました
- **マークダウンリンクの `#NNN` 参照修正**: `#NNN` 形式のリンクが現在のリポジトリではなく正しい URL を指すよう修正しました
- **`AskUserQuestion` プレビューのパフォーマンス修正**: ノート入力欄でキー入力のたびにマークダウンレンダリングが再実行されていたパフォーマンスリグレッションを修正しました
- **テレポートマーカーのレンダリング修正**: VS Code のテレポートセッションでテレポートマーカーが表示されない問題を修正しました

### プラグイン・コマンド

- **プラグインインストール状態の誤表示修正**: `/plugin` コマンドでプラグインが正確にインストール済みとして表示されない問題を修正しました
- **起動時のプラグインマーケットプレース「not found」エラー修正**: マーケットプレースのインストール後に自動更新することで、新規起動時の「not found in marketplace」エラーを修正しました
- **`/security-review` コマンドの git 互換性修正**: 古い git バージョンで `unknown option merge-base` エラーが発生していた問題を修正しました
- **`/color` コマンドのリセット機能追加**: デフォルトカラーに戻す方法がなかった問題を修正しました。`/color default`、`/color gray`、`/color reset`、`/color none` でデフォルトに戻せるようになりました

### 通知・設定

- **「Model updated to Opus 4.6」通知の繰り返し修正**: `.claude/settings.json` にレガシーな Opus モデル文字列が固定されている場合に通知が繰り返し表示されていた問題を修正しました
- **Remote 環境での `permissions.defaultMode` 修正**: Claude Code Remote 環境で `acceptEdits` および `plan` 以外の `permissions.defaultMode` 設定値が適用されていた問題を修正し、それらは無視されるようになりました

## 技術的なポイント

- **Windows/WSL ユーザーは即時アップデートを推奨**: クリップボードの CJK・絵文字文字化け、音声モード、VS Code 多重起動など、Windows 固有の問題が複数修正されています
- **サードパーティ API ゲートウェイ利用者への恩恵**: `ANTHROPIC_BASE_URL` を使用している環境での 400 エラーが解消され、プロキシ経由での利用が安定します
- **カスタム Bedrock 推論プロファイル利用者への恩恵**: 標準外のモデル識別子での effort パラメータエラーが解消されます
- **パフォーマンス改善の積み重ね**: プロンプト入力の再レンダリング 74% 削減、起動メモリ 426KB 削減、Remote Control ポーリング 300 分の 1 と、複数の最適化が行われています
- **VS Code 統合の大幅強化**: セッション管理、プランビュー、MCP 管理の 3 機能が追加され、VS Code ユーザーはターミナルとエディタを行き来する頻度が大幅に減ります
- **`--resume` のトークン節約**: スキルリストの再挿入を防ぐことで、1 リジューム当たり約 600 トークンの節約になります

## まとめ

v2.1.70 は新機能よりもバグ修正と安定性向上を重視したリリースです。特に Windows/WSL 環境の品質が大きく改善されており、該当ユーザーはアップデートの恩恵を最も受けられます。また、VS Code 向けの 3 つの新機能（セッション管理、プランビュー、MCP 管理）は、エディタ統合の完成度をさらに高めるものです。パフォーマンス面でも複数の最適化が施されており、日常的な利用体験の向上が期待できます。今後のリリースでもこの方向での改善が続くことが予想されます。

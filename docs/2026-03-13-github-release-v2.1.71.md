# Claude Code v2.1.71 リリース — /loop コマンドと cron スケジューリングの追加、多数のバグ修正

## 概要

Claude Code v2.1.71 では、プロンプトやスラッシュコマンドを指定間隔で繰り返し実行できる `/loop` コマンドと cron スケジューリングツールが新たに追加されました。また、長期セッションでの入力フリーズや起動時のパフォーマンス問題など、開発体験に影響していた多数のバグが修正されています。音声機能のキーバインド設定や Bash 自動承認リストの拡充など、使い勝手を向上させる改善も含まれています。

## 新機能

### `/loop` コマンド — 定期実行の自動化

セッション内でプロンプトやスラッシュコマンドを指定した間隔で繰り返し実行できる `/loop` コマンドが追加されました。

```
/loop 5m check the deploy
```

上記の例では、5分ごとにデプロイ状態の確認を自動的に実行します。CI/CD パイプラインの監視や、定期的なステータスチェックなど、繰り返し実行が必要なタスクに活用できます。

### cron スケジューリングツール

セッション内で定期実行プロンプトを管理するための cron スケジューリングツールが追加されました。これにより、`/loop` コマンドと合わせて、より柔軟なスケジュール管理が可能になります。

### 音声機能のキーバインド設定

音声の Push-to-Talk 機能に `voice:pushToTalk` キーバインドが追加され、`keybindings.json` から自由にキー設定を変更できるようになりました。デフォルトはスペースキーですが、`meta+k` のような修飾キー+文字の組み合わせも利用可能で、通常のテキスト入力との干渉がありません。

```json
// keybindings.json の設定例
{
  "voice:pushToTalk": "meta+k"
}
```

### Bash 自動承認リストの拡充

Bash ツールの自動承認コマンドに以下が追加されました:
`fmt`, `comm`, `cmp`, `numfmt`, `expr`, `test`, `printf`, `getconf`, `seq`, `tsort`, `pr`

テキスト処理や数値計算など、日常的なシェルコマンドの権限プロンプトが不要になり、ワークフローがよりスムーズになります。

## 改善点

### 起動・接続パフォーマンスの向上

- ネイティブ画像プロセッサの読み込みを初回使用時まで遅延させることで、起動時間が短縮されました
- ラップトップのスリープ復帰後のブリッジセッション再接続が、従来の最大 10 分待機から数秒で完了するようになりました

### プラグイン管理の改善

`/plugin uninstall` コマンドがプロジェクトスコープドプラグインを `.claude/settings.local.json` に記録するよう変更されました。これにより、チームメンバーが共有する `.claude/settings.json` を変更せずにプラグインを無効化できます。

また、プラグインが提供する MCP サーバーの重複排除が改善されました。手動で設定済みのサーバーと同一のコマンド/URL を持つプラグイン提供サーバーは自動的にスキップされ、重複接続とツールセットの重複が防止されます。

### デバッグログのオンデマンド切り替え

`/debug` コマンドがセッション中にデバッグログのオン/オフをトグルできるように更新されました。デバッグログはデフォルトで無効になっており、必要な時だけ有効化できます。

## バグ修正

### パフォーマンス・フリーズ問題

- **長期セッションの入力フリーズ**: キーストロークが処理されなくなりつつもプロセスが生き続けるバグを修正しました
- **音声モードの起動フリーズ（5〜8 秒）**: システムスリープからの復帰後に CoreAudio の初期化がメインスレッドをブロックしていた問題を修正しました
- **OAuth トークン更新時の UI フリーズ**: 多数の claude.ai プロキシコネクタが同時に期限切れトークンを更新する際に発生していたフリーズを修正しました

### `/fork` コマンドのプランファイル共有問題

`/fork` でフォークした会話が同じプランファイルを共有してしまい、一方のフォークでの編集がもう一方に上書きされるバグが修正されました。フォークした会話はそれぞれ独立したプランファイルを持つようになります。

### Read ツールの画像処理

画像処理に失敗した際に過大サイズの画像がコンテキストに追加されてしまい、画像を多用する長いセッションで後続のターンが壊れる問題が修正されました。

### Bash コマンドの誤権限プロンプト

heredoc を使用したコミットメッセージを含む複合 Bash コマンドに対して誤った権限プロンプトが表示されていた問題が修正されました。

### claude.ai コネクタ関連

- OAuth トークン更新後にコネクタが再接続に失敗する問題を修正しました
- MCP コネクタのスタートアップ通知が、全ての組織設定コネクタではなく以前に接続したコネクタのみに表示されるようになりました
- 複数の Claude Code インスタンスを同時実行した際にプラグインインストールが失われる問題を修正しました

### その他のバグ修正

- **バックグラウンドエージェント**: 完了通知に出力ファイルパスが含まれない問題を修正。コンテキスト圧縮後のエージェント結果回収が容易になりました
- **Bash エラーメッセージ**: コマンドが非ゼロステータスで終了した際のエラーメッセージ重複出力を修正しました
- **Chrome 拡張機能の自動検出**: ローカルに Chrome がないマシンで実行した後、"not installed" 状態に永続的にスタックする問題を修正しました
- **`/plugin marketplace update`**: marketplace がブランチ/タグ参照で固定されている場合のマージ競合エラーを修正しました
- **`/plugin marketplace add owner/repo@ref`**: `@` を参照セパレータとして正しく解析できなかった問題（以前は `#` のみ対応）を修正しました
- **`/permissions` ワークスペースタブ**: 同じディレクトリがトレーリングスラッシュあり/なしで追加された際の重複エントリを修正しました
- **`--print` オプション**: チームエージェントが設定されている場合に無限にハングする問題を修正しました
- **REPL の ToolSearch**: 毎回「❯ Tool loaded.」が表示される冗長な出力を修正しました
- **Windows の mingw スタイルパス**: モデルが mingw スタイルパスを使用した際に `cd <cwd> && git ...` の確認プロンプトが表示される問題を修正しました
- **未認証コネクタの通知**: 認証されていない組織登録 claude.ai コネクタのスタートアップ通知を削除しました

## 技術的なポイント

- **`/loop` と cron ツール**: 繰り返し実行タスクの自動化が可能になり、CI/CD 監視や定期ヘルスチェックのユースケースに対応します
- **音声キーバインドのカスタマイズ**: `keybindings.json` で `voice:pushToTalk` キーを自由に設定できるため、既存のキーバインドとの競合を避けやすくなりました
- **画像プロセッサの遅延読み込み**: 起動時のコスト削減に貢献しており、画像を使わないワークフローでは特に恩恵が大きい改善です
- **`/fork` のプランファイル分離**: 並行して複数のアプローチを試す際に、フォーク間でのプラン干渉がなくなりました
- **プラグインの `settings.local.json` 化**: チーム開発における個人設定の分離がより適切に行われるようになります。`settings.json` をバージョン管理している場合でも、個人のプラグイン変更がコミットに混入しません

## まとめ

v2.1.71 は新機能と品質改善のバランスが取れたリリースです。特に `/loop` コマンドと cron スケジューリングは、反復的な監視タスクや自動化ワークフローを構築するうえで有用な機能です。また、長期セッションでの入力フリーズや起動時のパフォーマンス問題など、日常的な開発体験に影響していた複数のバグが一度に解消されており、全体的な安定性が向上しています。プラグイン管理やコネクタ周りの修正も多く含まれており、チーム環境での利用がより信頼性の高いものになりました。

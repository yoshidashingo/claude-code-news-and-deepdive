# Claude Code v2.1.63 リリース: HTTP フック追加と大規模なメモリリーク修正

## 概要

Claude Code v2.1.63 がリリースされました。今回のリリースでは `/simplify` や `/batch` などの新しいバンドル済みスラッシュコマンドの追加、HTTP フックという新しいフック機能の導入が目玉です。また、長期実行セッションに影響していた複数のメモリリークが一挙に修正され、安定性が大きく向上しています。

## 新機能

### `/simplify` と `/batch` スラッシュコマンド

新たに `/simplify` と `/batch` の2つのスラッシュコマンドがバンドル済みコマンドとして追加されました。これまでユーザーが自前で定義する必要があったワークフローを、すぐに利用できるようになりました。

### HTTP フック

これまでのフック機能はシェルコマンドの実行のみに対応していましたが、新たに **HTTP フック**が追加されました。任意の URL に JSON データを POST し、JSON レスポンスを受け取ることができます。これにより、外部の Web サービスや自作 API と Claude Code のイベントを連携させるワークフローが構築しやすくなります。

設定例のイメージ（フック設定ファイル内）:

```json
{
  "type": "http",
  "url": "https://example.com/webhook",
  "events": ["tool_use"]
}
```

### `/model` コマンドの改善

`/model` コマンドのスラッシュコマンドメニューに、現在アクティブなモデル名が表示されるようになりました。複数のモデルを使い分けている場合に、現在どのモデルが有効かを素早く確認できます。

### MCP OAuth 認証の手動 URL ペースト対応

MCP (Model Context Protocol) の OAuth 認証フローで、自動的なローカルホストへのリダイレクトが失敗した場合に、コールバック URL を手動でペーストして認証を完了できるフォールバック機能が追加されました。ネットワーク環境の制約でリダイレクトが機能しない場合でも認証を完了できます。

### `/copy` コマンドの「常にフル応答をコピー」オプション

`/copy` コマンドのピッカーに「Always copy full response（常にフル応答をコピー）」オプションが追加されました。このオプションを選択すると、以降の `/copy` コマンドではコードブロックの選択ステップをスキップして、応答全体を直接クリップボードにコピーします。

### VSCode: セッションのリネーム・削除

VSCode 拡張機能のセッション一覧から、セッションのリネームと削除が行えるようになりました。セッション管理の利便性が向上しています。

### `ENABLE_CLAUDEAI_MCP_SERVERS=false` 環境変数

`ENABLE_CLAUDEAI_MCP_SERVERS=false` を設定することで、claude.ai の MCP サーバーを利用可能にする機能をオプトアウトできるようになりました。セキュリティポリシーや環境要件に応じて制御が可能です。

## 改善点

### git worktree 間でのプロジェクト設定・自動メモリの共有

同じリポジトリの git worktree 間で、プロジェクト設定と自動メモリが共有されるようになりました。複数の worktree を使って並行作業している場合でも、設定やメモリが一貫して利用できます。

### サブエージェント使用時のコンテキストコンパクション改善

サブエージェントを使用する長期セッションにおいて、コンテキストコンパクション時にプログレスメッセージの重いペイロードをストリップするようになりました。メモリ使用量の削減につながります。

## バグ修正

### UI 表示の修正

- ローカルスラッシュコマンド（`/cost` など）の出力が、システムメッセージではなくユーザー送信メッセージとして表示されていた問題を修正しました。
- VSCode でリモートセッションが会話履歴に表示されない問題を修正しました。
- `/clear` コマンドがキャッシュ済みスキルをリセットせず、古いスキル内容が新しい会話に残り続ける問題を修正しました。

### メモリリークの大規模修正

今回のリリースでは、長期実行セッションに影響する多数のメモリリークが修正されました:

- **ブリッジポーリングループ**: リスナーリークを修正
- **MCP OAuth フロー**: クリーンアップ時のリスナーリークを修正
- **インタラクティブパーミッションハンドラー**: 自動承認時のリスナーリークを修正
- **Bash コマンドプレフィックスキャッシュ**: メモリリークを修正
- **MCP ツール/リソースキャッシュ**: サーバー再接続時のキャッシュリークを修正
- **IDE ホスト IP 検出キャッシュ**: ポート間で結果を誤って共有していた問題を修正
- **WebSocket リスナー**: トランスポート再接続時のリークを修正
- **git root 検出キャッシュ**: 長期実行セッションで無制限に増加するメモリリークを修正
- **JSON 解析キャッシュ**: 長期セッションで無制限に増加するメモリリークを修正
- **長期実行中のチームメイト**: AppState にすべてのメッセージを保持し続けていたメモリリークを修正
- **MCP サーバーのフェッチキャッシュ**: 切断時にキャッシュがクリアされず、頻繁に再接続するサーバーでメモリ使用量が増加し続ける問題を修正
- **ファイルカウントキャッシュ**: glob の無視パターンが考慮されていなかった問題を修正

### 競合状態の修正

REPL ブリッジにおける競合状態を修正しました。初期接続のフラッシュ中に新しいメッセージが履歴メッセージと混在してサーバーに届き、メッセージの順序に問題が生じていたケースが解消されました。

## 技術的なポイント

- **HTTP フック**は既存のシェルコマンド形式のフックと並行して利用でき、外部システムとの統合をより柔軟に行えます
- 今回の修正で対象となったメモリリークの多くは**長期実行セッション**や**頻繁な再接続**が発生する環境で顕在化していたものです。長時間 Claude Code を使い続けるユーザーほど恩恵が大きいリリースです
- **git worktree サポートの強化**により、並行開発ブランチ間でコンテキストを維持しやすくなりました
- MCP サーバーとの接続安定性に関するキャッシュ・リスナー管理が全体的に見直されています

## まとめ

v2.1.63 は新機能の追加と並んで、長期運用時の安定性に直結するメモリリーク修正が多数含まれたリリースです。特に HTTP フックの追加は、Claude Code を既存の開発インフラや CI/CD パイプラインと連携させる際の選択肢を大きく広げます。長時間セッションの安定性向上とあわせて、積極的なアップデートを推奨します。

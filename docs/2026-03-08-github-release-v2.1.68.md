# Claude Code v2.1.68: Opus 4.6 のデフォルト努力レベルと ultrathink の復活

## 概要

Claude Code v2.1.68 では、Max および Team サブスクライバー向けに Opus 4.6 のデフォルト努力レベルが「medium（中）」に設定されました。また、一度廃止されていた `ultrathink` キーワードが復活し、必要なときにハイエフォートモードを使えるようになりました。さらに、ファーストパーティ API から Opus 4 と Opus 4.1 が削除され、これらを固定していたユーザーは自動的に Opus 4.6 へ移行されます。

## 新機能

### Opus 4.6 のデフォルト努力レベルを medium に設定

Max および Team サブスクライバーを対象に、Opus 4.6 のデフォルト努力レベルが **medium（中）** になりました。これはスピードと精度のバランスが取れたモードとして位置づけられており、ほとんどのタスクに適しています。

設定は `/model` コマンドでいつでも変更できます。用途に応じて低・中・高の努力レベルを使い分けることで、レスポンス速度とコスト効率を最適化できます。

### `ultrathink` キーワードの復活

以前廃止されていた `ultrathink` キーワードが再導入されました。プロンプトの中で `ultrathink` を使用すると、**次のターンのみ高努力（high effort）モード**が有効になります。

複雑なアーキテクチャ設計や難易度の高いデバッグなど、特に深い思考が必要な場面で活用できます。デフォルトは medium のまま維持されるため、必要なときだけ高努力モードを一時的に切り替えられる柔軟な設計になっています。

```
# 使用例
ultrathink このシステムのアーキテクチャを詳しく設計してください
```

## 改善点

### Opus 4 / Opus 4.1 の廃止と自動移行

ファーストパーティ API において Opus 4 および Opus 4.1 が Claude Code から削除されました。これらのモデルを固定していたユーザーは、**自動的に Opus 4.6 へ移行**されます。手動での設定変更は不要です。

Opus 4.6 は従来の Opus 4 系に比べてパフォーマンスが向上しており、今回のアップデートで全ユーザーが最新モデルの恩恵を受けられるようになります。

## 技術的なポイント

- **デフォルト努力レベルの変更対象**: Max サブスクライバーおよび Team サブスクライバーのみ。無料プランや Pro プランへの影響は記載なし
- **`ultrathink` の適用範囲**: 次のターン（1回のリクエスト）のみ高努力モードが有効。永続的な設定変更ではない
- **モデル移行の自動化**: Opus 4 / 4.1 を固定していたユーザーへの通知・手動対応は不要。Claude Code 側で自動的に Opus 4.6 へ切り替わる
- **努力レベルのカスタマイズ**: `/model` コマンドでいつでも変更可能。タスクの性質に合わせてレベルを選択することで、コストとレスポンス速度のトレードオフを制御できる

## まとめ

v2.1.68 は、Opus 4.6 の使い勝手を向上させる実用的なアップデートです。デフォルトを medium に設定することで、多くのユースケースで余分なコストをかけずに応答速度を改善します。一方、`ultrathink` の復活により、複雑なタスクへの対応力も確保されました。Opus 4 系の廃止は後方互換性に影響しますが、自動移行によってユーザーへの負担は最小限に抑えられています。今後も Opus 4.6 を軸とした機能拡張が続くと見られ、努力レベル制御の仕組みがさらに洗練されていくことが期待されます。
